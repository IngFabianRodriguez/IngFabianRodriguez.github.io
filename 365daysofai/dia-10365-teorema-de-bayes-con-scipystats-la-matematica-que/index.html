Día 10: Teorema de Bayes con scipy.stats — La Matemática que Hace que tu IA Actualice sus Creencias (y No Quede como un Loro)

🎯 El Problema Real

Listo, parce. Imagínate que estás en el equipo de fraude de un banco. Tu modelo dice: "Este usuario tiene 80% de probabilidad de ser legítimo". Pero ¿qué pasa cuando ese mismo usuario hace una transferencia a las 3 AM a un país raro? ¿Tu modelo sigue diciendo 80%? Si la respuesta es "sí", entonces tu IA es un loro que repite lo mismo sin importar la evidencia nueva.

Eso es exactamente lo que el Teorema de Bayes soluciona: actualizar probabilidades cuando llega información nueva. No es filosofía de café; es matemática de producción. Y cuando digo "producción", hablo de sistemas de recomendación, detección de anomalías, filtros de spam y hasta diagnósticos médicos.

📐 La Matemática que No Te Enseñaron Bien (Pero Deberían)

Teorema de Bayes: El Update Matemático

La fórmula que hace que tu modelo "piense":

$$P(H \mid E) = \frac{P(E \mid H) \cdot P(H)}{P(E)}$$

Donde:

  • $P(H | E)$: Probabilidad posterior — Lo que crees después de ver la evidencia $E$.
  • $P(E | H)$: Verosimilitud (Likelihood) — Qué tan probable es ver $E$ si $H$ es verdad.
  • $P(H)$: Probabilidad a priori (Prior) — Lo que creías antes de ver $E$.
  • $P(E)$: Evidencia marginal — Normalización para que todo sume 1.

Analogía de Oficina

Imagínate que tu compañero de equipo siempre llega tarde el lunes (tu prior es que hoy llegará tarde). Pero ves que hoy sacó carro (evidencia $E$). Si sabes que cuando saca carro llega temprano el 90% de las veces $(P(E | llegó temprano))$, entonces actualizas tu creencia: hoy probablemente llegue temprano.

Eso es Bayes. Simple, ¿no? Pero en producción, la cosa se pone seria.

🧠 Por Qué Esto Importa en ML/AI

1. Clasificación Bayesiana (Naive Bayes)

Spam vs. No Spam. Cada palabra es evidencia. Bayes actualiza la probabilidad de que el correo sea basura.

2. Inferencia Bayesiana

No das una sola predicción; das una distribución de probabilidades. Es como decir: "No estoy 100% seguro, pero estas son las probabilidades de cada escenario".

3. A/B Testing y Bandits

¿El botón rojo convierte mejor que el azul? Bayes actualiza tus creencias con cada clic. Thompson Sampling usa esto a full.

4. Sistemas de Recomendación

Netflix: "Este usuario vio Breaking Bad (evidencia). ¿Qué probabilidad hay de que le guste Better Call Saul?". Bayes actualiza.

Ejemplo 1: Diagnóstico Médico Bayesiano

Escenario: Prueba de una enfermedad rara (prevalencia 0.1%). La prueba tiene 95% de sensibilidad y 90% de especificidad. Si sale positiva, ¿cuál es la probabilidad real de tener la enfermedad?

"""
Bayesian Medical Test Analysis con scipy.stats.
Calcula probabilidades posteriores para diagnósticos médicos.

Autor: Hector Fabian Rodriguez
Stack: scipy.stats, numpy, dataclasses
"""

from dataclasses import dataclass, field
from typing import Protocol, TypedDict
import numpy as np
from scipy import stats
import logging

# Configuración de logging
logging.basicConfig(level=
logging.INFO
)
logger = logging.getLogger(__name__)


class BayesianTest(Protocol):
    """Protocol para tests bayesianos. Permite duck typing."""
    def calculate_posterior(self, prior: float, likelihood: float, evidence: float) -> float:
        ...


class TestMetrics(TypedDict):
    """Métricas de un test médico. TypedDict para validación estática."""
    sensitivity: float  # P(Test+ | Enfermedad+)
    specificity: float  # P(Test- | Enfermedad-)
    prevalence: float   # P(Enfermedad+) — Prior


@dataclass(frozen=True, slots=True)  # Inmutable y eficiente en memoria
class BayesianDiagnostic:
    """
    Diagnóstico bayesiano para pruebas médicas.
    
    Args:
        sensitivity: Tasa de verdaderos positivos [0, 1]
        specificity: Tasa de verdaderos negativos [0, 1]
        prevalence: Prevalencia de la enfermedad [0, 1]
    
    Raises:
        ValueError: Si las probabilidades están fuera de rango
    
    Example:
        >>> diag = BayesianDiagnostic(sensitivity=0.95, specificity=0.90, prevalence=0.001)
        >>> diag.posterior_positive()
        0.009405684754521964  # Solo 0.94% de probabilidad real
    """
    sensitivity: float
    specificity: float
    prevalence: float
    
    # Validación post-inicialización (dataclass no tiene __init__)
    def __post_init__(self) -> None:
        if not (0 <= self.sensitivity <= 1):
            raise ValueError(f"Sensitivity debe estar en [0,1], recibido: {self.sensitivity}")
        if not (0 <= self.specificity <= 1):
            raise ValueError(f"Specificity debe estar en [0,1], recibido: {self.specificity}")
        if not (0 <= self.prevalence <= 1):
            raise ValueError(f"Prevalence debe estar en [0,1], recibido: {self.prevalence}")
    
    def posterior_positive(self) -> float:
        """
        Calcula P(Enfermedad+ | Test+) usando Bayes.
        
        Derivación matemática:
        P(E+ | T+) = P(T+ | E+) * P(E+) / P(T+)
        
        Donde:
        - P(T+ | E+) = sensitivity
        - P(E+) = prevalence
        - P(T+) = P(T+ | E+)*P(E+) + P(T+ | E-)*P(E-)
        
        Returns:
            Probabilidad posterior de tener la enfermedad
        """
        # Prior
        p_disease = self.prevalence
        p_no_disease = 1 - self.prevalence
        
        # Likelihood
        p_pos_given_disease = self.sensitivity  # Verdadero positivo
        p_pos_given_no_disease = 1 - self.specificity  # Falso positivo
        
        # Evidencia marginal (Ley de probabilidad total)
        p_positive = (p_pos_given_disease * p_disease + 
                     p_pos_given_no_disease * p_no_disease)
        
        # Teorema de Bayes
        posterior = (p_pos_given_disease * p_disease) / p_positive
        
        
logger.info
(f"Prior: {p_disease:.6f}, Posterior: {posterior:.6f}")
        return posterior
    
    def posterior_negative(self) -> float:
        """
        Calcula P(Enfermedad+ | Test-) — Probabilidad de estar enfermo si el test sale negativo.
        
        Importante: Un test negativo NO garantiza ausencia de enfermedad.
        
        Returns:
            Probabilidad posterior de tener la enfermedad con test negativo
        """
        p_disease = self.prevalence
        p_no_disease = 1 - self.prevalence
        
        # P(Test- | Enfermedad+) = 1 - sensitivity (Falso negativo)
        p_neg_given_disease = 1 - self.sensitivity
        # P(Test- | Enfermedad-) = specificity (Verdadero negativo)
        p_neg_given_no_disease = self.specificity
        
        p_negative = (p_neg_given_disease * p_disease + 
                     p_neg_given_no_disease * p_no_disease)
        
        posterior = (p_neg_given_disease * p_disease) / p_negative
        
        
logger.info
(f"Posterior con test negativo: {posterior:.6f}")
        return posterior

# EJEMPLO DE USO: PRUEBA DE ENFERMEDAD RARA
# ============================================================================

if __name__ == "__main__":
    # Configuración del test
    test = BayesianDiagnostic(
        sensitivity=0.95,   # 95% de verdaderos positivos
        specificity=0.90,   # 90% de verdaderos negativos
        prevalence=0.001    # 0.1% de la población tiene la enfermedad
    )
    
    print("=" * 70)
    print("🏥 ANÁLISIS BAYESIANO DE PRUEBA MÉDICA")
    print("=" * 70)
    
    # Probabilidad con test positivo
    p_disease_positive = test.posterior_positive()
    print(f"\n📊 Test POSITIVO:")
    print(f"   Probabilidad real de enfermedad: {p_disease_positive:.4%}")
    print(f"   ⚠️  Solo 1 de cada {int(1/p_disease_positive)} es verdadero positivo")
    
    # Probabilidad con test negativo
    p_disease_negative = test.posterior_negative()
    print(f"\n📊 Test NEGATIVO:")
    print(f"   Probabilidad de tener la enfermedad: {p_disease_negative:.6%}")
    print(f"   ✅ El test negativo es muy confiable en este caso")
    
    print("\n" + "=" * 70)
    print("💡 LECCIÓN:")
    print("   Con enfermedades raras, un test positivo NO es diagnóstico.")
    print("   Siempre necesitas una segunda prueba (actualización bayesiana).")
    print("=" * 70)

¿Qué Dolor Evita Este Código?

  1. Falsos positivos en producción: Imagina que tu sistema de fraude marca 1000 transacciones como sospechosas. Sin Bayes, investigas las 1000. Con Bayes, sabes que solo 10 son reales. Ahorras tiempo, dinero y cordura.
  2. Inmutabilidad con frozen=True: Una vez creado el objeto, nadie puede cambiar los parámetros por error. Eso evita bugs raros en sistemas concurrentes.
  3. __slots__: Reduce memoria. En un sistema con millones de objetos, esto importa.
  4. Logging: En producción, no debuggeas con print(). Usas logs que puedes filtrar y agregar.

🔥 Ejemplo 2: Filtro de Spam Bayesiano (Naive Bayes)

Vamos a implementar un clasificador de spam simple pero robusto.

"""
Naive Bayes Spam Filter con scipy.stats y Laplace Smoothing.
Clasificador probabilístico para detección de spam.

Autor: Hector Fabian Rodriguez
Stack: scipy.stats, numpy, collections
"""

from collections import defaultdict, Counter
from typing import List, Dict, Tuple, Set
from dataclasses import dataclass, field
import numpy as np
from scipy import stats
import logging

logger = logging.getLogger(__name__)


@dataclass
class NaiveBayesClassifier:
    """
    Clasificador Naive Bayes multinomial para texto.
    
    Asume independencia condicional entre features (palabras).
    Usa Laplace smoothing para evitar probabilidades cero.
    
    Attributes:
        alpha: Parámetro de Laplace smoothing (default=1.0)
        class_priors: P(Class) — Distribución a priori de clases
        feature_probs: P(Feature | Class) — Verosimilitud de cada palabra
        vocab: Set de todas las palabras vistas en entrenamiento
    """
    alpha: float = 1.0  # Laplace smoothing
    class_priors: Dict[str, float] = field(default_factory=dict)
    feature_probs: Dict[str, Dict[str, float]] = field(default_factory=lambda: defaultdict(dict))
    vocab: Set[str] = field(default_factory=set)
    
    def __post_init__(self) -> None:
        if self.alpha <= 0:
            raise ValueError(f"Alpha debe ser > 0, recibido: {self.alpha}")
    
    def fit(self, X: List[List[str]], y: List[str]) -> None:
        """
        Entrena el clasificador Naive Bayes.
        
        Args:
            X: Lista de documentos (cada documento es lista de palabras)
            y: Lista de etiquetas (ej: ['spam', 'ham', 'spam', ...])
        
        Complejidad: O(N * M) donde N = docs, M = palabras promedio
        """
        if len(X) != len(y):
            raise ValueError(f"X y y deben tener la misma longitud: {len(X)} != {len(y)}")
        
        # Contadores
        class_counts = Counter(y)
        feature_counts: Dict[str, Counter] = defaultdict(Counter)
        
        # Construir vocabulario y contar features
        for doc, label in zip(X, y):
            self.vocab.update(doc)
            feature_counts[label].update(doc)
        
        total_docs = len(y)
        vocab_size = len(self.vocab)
        
        # Calcular priors: P(Class)
        for cls, count in class_counts.items():
            self.class_priors[cls] = count / total_docs
        
        # Calcular likelihoods: P(Word | Class) con Laplace smoothing
        for cls in class_counts:
            total_words_in_class = sum(feature_counts[cls].values())
            
            for word in self.vocab:
                # Laplace smoothing: (count + alpha) / (total + alpha * vocab_size)
                word_count = feature_counts[cls].get(word, 0)
                self.feature_probs[cls][word] = (
                    (word_count + self.alpha) / 
                    (total_words_in_class + self.alpha * vocab_size)
                )
        
        
logger.info
(f"Modelo entrenado: {len(self.vocab)} palabras, {len(class_counts)} clases")
    
    def predict_proba(self, doc: List[str]) -> Dict[str, float]:
        """
        Calcula P(Class | Document) para cada clase.
        
        Usa log-probabilidades para evitar underflow:
        log P(C | D) ∝ log P(C) + Σ log P(w_i | C)
        
        Args:
            doc: Documento como lista de palabras
        
        Returns:
            Dict con probabilidades posteriores por clase
        """
        log_probs = {}
        
        for cls, prior in self.class_priors.items():
            # Log-prior
            log_prob = np.log(prior)
            
            # Sumar log-likelihoods de cada palabra
            for word in doc:
                if word in self.vocab:
                    log_prob += np.log(self.feature_probs[cls][word])
                # Si la palabra no está en vocab, la ignoramos (asumimos prob uniforme)
            
            log_probs[cls] = log_prob
        
        # Convertir log-probs a probabilidades normalizadas
        # Usamos log-sum-exp trick para estabilidad numérica
        max_log_prob = max(log_probs.values())
        exp_probs = {cls: np.exp(lp - max_log_prob) for cls, lp in log_probs.items()}
        total = sum(exp_probs.values())
        
        return {cls: prob / total for cls, prob in exp_probs.items()}
    
    def predict(self, doc: List[str]) -> str:
        """
        Predice la clase más probable para un documento.
        
        Args:
            doc: Documento como lista de palabras
        
        Returns:
            Etiqueta de la clase predicha
        """
        probs = self.predict_proba(doc)
        return max(probs, key=probs.get)  # type: ignore


# ============================================================================
# EJEMPLO DE USO: CLASIFICADOR DE SPAM
# ============================================================================

if __name__ == "__main__":
    # Dataset de ejemplo (en producción, esto vendría de una DB)
    X_train = [
        ["win", "money", "prize", "click", "now"],
        ["meeting", "schedule", "tomorrow", "project"],
        ["free", "money", "win", "click"],
        ["report", "deadline", "submit", "project"],
        ["congratulations", "prize", "claim", "now"],
        ["budget", "meeting", "schedule", "deadline"]
    ]
    
    y_train = ["spam", "ham", "spam", "ham", "spam", "ham"]
    
    # Entrenar clasificador
    clf = NaiveBayesClassifier(alpha=1.0)
    
clf.fit
(X_train, y_train)
    
    print("=" * 70)
    print("📧 CLASIFICADOR NAIVE BAYES PARA SPAM")
    print("=" * 70)
    
    # Probar con nuevos documentos
    test_docs = [
        ["win", "free", "money"],
        ["meeting", "project", "deadline"],
        ["prize", "click", "now"]
    ]
    
    for i, doc in enumerate(test_docs, 1):
        probs = clf.predict_proba(doc)
        prediction = clf.predict(doc)
        
        print(f"\n📄 Documento {i}: {' '.join(doc)}")
        print(f"   Predicción: {prediction.upper()}")
        print(f"   Probabilidades:")
        for cls, prob in probs.items():
            print(f"      - {cls}: {prob:.4%}")
    
    print("\n" + "=" * 70)
    print("💡 LECCIÓN:")
    print("   Naive Bayes asume independencia (ingenuo), pero funciona.")
    print("   En producción, Gmail usa versiones sofisticadas de esto.")
    print("=" * 70)

¿Por Qué Log-Probabilidades?

Cuando multiplicas muchas probabilidades pequeñas (ej: $0.001 \times 0.002 \times 0.0003$), terminas con números tan pequeños que Python los redondea a cero (underflow).

Solución: Usar logaritmos.

$$\log(a \times b) = \log(a) + \log(b)$$

En vez de multiplicar, sumas. Y al final conviertes de vuelta con np.exp().

🧪 Ejemplo 3: Actualización Bayesiana Iterativa con scipy.stats

Ahora sí, vamos a usar scipy.stats para distribuciones conjugadas. Esto es inferencia bayesiana real.

"""
Bayesian Inference con distribuciones conjugadas (Beta-Binomial).
Actualización iterativa de creencias con evidencia nueva.

Autor: Hector Fabian Rodriguez
Stack: scipy.stats, matplotlib
"""

from typing import Tuple
import numpy as np
from scipy import stats
import matplotlib.pyplot as plt

# Configurar matplotlib para gráficos bonitos

plt.style
.use('seaborn-v0_8-darkgrid')


class BayesianCoinFlip:
    """
    Inferencia bayesiana para estimar la probabilidad de una moneda.
    
    Usa distribución Beta como prior y likelihood Binomial.
    Beta es conjugada de Binomial, entonces posterior es también Beta.
    
    Matemática:
    - Prior: Beta(α, β)
    - Likelihood: Binomial(n, k)
    - Posterior: Beta(α + k, β + n - k)
    
    Attributes:
        alpha: Parámetro α de la distribución Beta (prior)
        beta: Parámetro β de la distribución Beta (prior)
        observations: Lista de (éxitos, total) observados
    """
    
    __slots__ = ('alpha', 'beta', 'observations')  # Optimización de memoria
    
    def __init__(self, alpha: float = 1.0, beta: float = 1.0):
        """
        Inicializa con prior uniforme Beta(1, 1) por defecto.
        
        Args:
            alpha: Parámetro α > 0 (éxitos + 1)
            beta: Parámetro β > 0 (fallos + 1)
        """
        if alpha <= 0 or beta <= 0:
            raise ValueError(f"α y β deben ser > 0: α={alpha}, β={beta}")
        
        self.alpha = alpha
        self.beta = beta
        self.observations: list[Tuple[int, int]] = []
    
    def update(self, successes: int, trials: int) -> None:
        """
        Actualiza el posterior con nueva evidencia.
        
        Teorema de Bayes para conjugadas:
        Beta(α, β) + Binomial(n, k) → Beta(α + k, β + n - k)
        
        Args:
            successes: Número de éxitos observados
            trials: Número total de intentos
        
        Raises:
            ValueError: Si successes > trials o valores negativos
        """
        if successes < 0 or trials < 0:
            raise ValueError(f"Valores no pueden ser negativos: k={successes}, n={trials}")
        if successes > trials:
            raise ValueError(f"Éxitos no pueden exceder trials: {successes} > {trials}")
        
        failures = trials - successes
        self.alpha += successes
        self.beta += failures
        self.observations.append((successes, trials))
        
        
logger.info
(f"Update: α={self.alpha}, β={self.beta}")
    
    def posterior_distribution(self) -> stats.beta:
        """
        Retorna la distribución posterior actual como objeto scipy.stats.beta.
        
        Returns:
            Distribución Beta con parámetros actualizados
        """
        return stats.beta(self.alpha, self.beta)
    
    def mean(self) -> float:
        """
        Media (esperanza) del posterior: E[θ] = α / (α + β)
        
        Returns:
            Estimación puntual de la probabilidad
        """
        return self.alpha / (self.alpha + self.beta)
    
    def credible_interval(self, confidence: float = 0.95) -> Tuple[float, float]:
        """
        Intervalo de credibilidad (análogo bayesiano al intervalo de confianza).
        
        Args:
            confidence: Nivel de confianza (default: 95%)
        
        Returns:
            Tupla (lower, upper) con límites del intervalo
        """
        dist = self.posterior_distribution()
        alpha_level = (1 - confidence) / 2
        return dist.ppf(alpha_level), dist.ppf(1 - alpha_level)
    
    def plot_evolution(self, save_path: str | None = None) -> None:
        """
        Grafica la evolución del posterior con cada actualización.
        
        Args:
            save_path: Ruta para guardar el gráfico (opcional)
        """
        if not self.observations:
            raise ValueError("No hay observaciones para graficar")
        
        fig, ax = plt.subplots(figsize=(12, 6))
        x = np.linspace(0, 1, 1000)
        
        # Prior inicial
        prior = stats.beta(1.0, 1.0)
        ax.plot(x, prior.pdf(x), 'k--', alpha=0.5, label='Prior: Beta(1,1)', linewidth=2)
        
        # Evolución del posterior
        alpha_temp, beta_temp = 1.0, 1.0
        for i, (k, n) in enumerate(self.observations, 1):
            alpha_temp += k
            beta_temp += (n - k)
            dist = stats.beta(alpha_temp, beta_temp)
            
            color = 
plt.cm
.viridis(i / len(self.observations))
            ax.plot(x, dist.pdf(x), color=color, 
                   label=f'Obs {i}: {k}/{n} (α={alpha_temp:.1f}, β={beta_temp:.1f})',
                   linewidth=2)
        
        ax.set_xlabel('θ (Probabilidad de Éxito)', fontsize=12)
        ax.set_ylabel('Densidad', fontsize=12)
        ax.set_title('Evolución Bayesiana del Posterior', fontsize=14, fontweight='bold')
        ax.legend(loc='best', fontsize=10)
        ax.grid(True, alpha=0.3)
        
        if save_path:
            plt.savefig(save_path, dpi=300, bbox_inches='tight')
        
        plt.tight_layout()
        
plt.show
()


# ============================================================================
# EJEMPLO DE USO: ESTIMAR PROBABILIDAD DE UNA MONEDA
# ============================================================================

if __name__ == "__main__":
    print("=" * 70)
    print("🪙 INFERENCIA BAYESIANA: ESTIMACIÓN DE MONEDA SESGADA")
    print("=" * 70)
    
    # Inicializar con prior uniforme (no sabemos nada)
    coin = BayesianCoinFlip(alpha=1.0, beta=1.0)
    
    # Experimentos iterativos
    experiments = [
        (6, 10),    # 6 caras de 10 lanzamientos
        (55, 100),  # 55 caras de 100 lanzamientos
        (12, 20),   # 12 caras de 20 lanzamientos
    ]
    
    print("\n📊 Prior: Beta(1, 1) — Uniforme (no sesgo conocido)")
    print(f"   Media inicial: {coin.mean():.4f}")
    
    for i, (k, n) in enumerate(experiments, 1):
        coin.update(k, n)
        mean = coin.mean()
        lower, upper = coin.credible_interval(0.95)
        
        print(f"\n🎲 Experimento {i}: {k} éxitos de {n} intentos")
        print(f"   Posterior: Beta(α={coin.alpha}, β={coin.beta})")
        print(f"   Media (estimación): {mean:.4f}")
        print(f"   IC 95%: [{lower:.4f}, {upper:.4f}]")
    
    print("\n" + "=" * 70)
    print("💡 LECCIÓN:")
    print("   Con más evidencia, el intervalo se achica (más certeza).")
    print("   Esto es inferencia bayesiana: actualización continua.")
    print("=" * 70)
    
    # Graficar evolución
    coin.plot_evolution()

¿Qué Dolor Evita Este Código?

  1. No necesitas infinitos datos: Con 10 observaciones ya tienes una estimación razonable. En Machine Learning frecuentista, necesitas miles de datos.
  2. Incertidumbre cuantificada: No solo dices "la probabilidad es 0.58". Dices "es 0.58 ± 0.05 con 95% de confianza". Eso es crítico en producción.
  3. Actualización online: Cada vez que llega un dato nuevo, actualizas. No reentrenar desde cero.

📊 Cuándo Usar Bayes en Producción

Caso de Uso Por Qué Bayes
Filtros de Spam Naive Bayes es rápido y funciona con pocos datos.
A/B Testing Thompson Sampling (bayesiano) converge más rápido que ε-greedy.
Diagnósticos Médicos Actualizar probabilidades con cada síntoma/prueba.
Sistemas de Recomendación Bayesian Personalized Ranking (BPR) para ranking probabilístico.
Detección de Anomalías Bayesian Online Changepoint Detection para detectar cambios en streams de datos.

1. Olvidar Laplace Smoothing

# ❌ MAL: Probabilidad cero explota todo
if word not in vocab:
    prob = 0  # BOOM: log(0) = -∞

# ✅ BIEN: Laplace smoothing
prob = (count + alpha) / (total + alpha * vocab_size)

2. Multiplicar Probabilidades sin Log

# ❌ MAL: Underflow
prob = 0.001 * 0.002 * 0.0003  # → 0.0 en Python

# ✅ BIEN: Sumar logs
log_prob = np.log(0.001) + np.log(0.002) + np.log(0.0003)
prob = np.exp(log_prob)

3. Prior Incorrecto

Si tu prior es $Beta(1, 1)$ (uniforme), estás diciendo "no sé nada". Pero si sabes que una moneda es aproximadamente justa, usa $Beta(50, 50)$ (prior informativo).
