Día 40/365: Pruning (Poda de Árboles) - La Diferencia Entre Memorizar Ruido y Generalizar

¿Qué más, parcero? Llegamos al Día 40 del #365DaysOfAI y hoy vamos a hablar de algo que el 80% de los científicos de datos ignoran hasta que sus árboles de decisión les explotan en producción: Pruning (poda).

No, no es "ese parámetro raro que sklearn tiene por defecto". Es la diferencia entre un modelo que memoriza cada maldito ruido del training set y uno que realmente generaliza. Después de 12 años viendo modelos con 99.9% de accuracy en train y 60% en test, puedo decirte que el problema casi SIEMPRE es un árbol sin podar.

El dolor real: tus árboles crecen hasta tener hojas con un solo sample, memorizando outliers y ruido. Cuando llega data nueva, el modelo colapsa. Y todos se preguntan "¿por qué falló si en train andaba perfecto?" Esa vaina se llama overfitting, y la poda es tu salvavidas.

Dejá el café, acomodate, y preparate para entender la matemática detrás de ccp_alpha (Cost-Complexity Pruning alpha) y cómo usar este parámetro correctamente para que tus árboles no se conviertan en bosques de memorización.

🎯 ¿Qué Carajos es Pruning y Por Qué Debería Importarte?

Mirá, pruning es como cuando estás escribiendo código y tenés que decidir si cada if-else extra realmente agrega valor o solo está haciendo el código más complejo y frágil. Un árbol sin podar es como ese archivo de 5000 líneas con 47 niveles de anidación: técnicamente funciona en ese caso específico, pero es imposible de mantener y falla con cualquier input nuevo.

Definición técnica (sin rodeos):

Pruning (poda) es el proceso de reducir la complejidad de un árbol de decisión eliminando sub-árboles que no aportan poder predictivo significativo, balanceando el trade-off entre accuracy en training y capacidad de generalización. Matemáticamente, buscamos minimizar la función de costo-complejidad:

$$R_\alpha(T) = R(T) + \alpha |T|$$

donde R(T) es el error de clasificación del árbol $T$, $|T|$ es el número de nodos terminales (hojas), y $\alpha \geq 0$ es el parámetro de penalización por complejidad (el famoso ccp_alpha en sklearn).

Tipos principales:

  • Pre-Pruning (Early Stopping): Detenés el crecimiento del árbol durante el entrenamiento basándote en criterios como max_depth, min_samples_split, min_samples_leaf. Es rápido pero puede ser demasiado conservador y perder splits importantes.
  • Post-Pruning (Cost-Complexity Pruning): Dejás que el árbol crezca completamente y luego lo podás hacia atrás, eliminando ramas basándote en el costo-complejidad. Es más costoso computacionalmente pero genera mejores resultados porque ve el panorama completo antes de decidir qué cortar.

CUIDADO: Mucha gente confunde usar max_depth=5 (pre-pruning arbitrario) con una poda óptima basada en validación cruzada del ccp_alpha (post-pruning sistemático). El primero es un hyperparámetro que adivinás, el segundo es una decisión basada en evidencia empírica del trade-off bias-variance.

📐 La Fórmula de Cost-Complexity Pruning

Acá es donde la mayoría se pierde porque no entienden qué carajo está optimizando ccp_alpha. Esta es LA ecuación que controla toda la magia del pruning en sklearn:

$$R_\alpha(T) = R(T) + \alpha |T|$$

Esta ecuación parece simple pero tiene TODO el poder de controlar el trade-off entre performance y complejidad. La idea es encontrar el sub-árbol $T \subseteq T_{\text{max}}$ que minimiza esta función de costo-complejidad. Cuando $\alpha = 0$, no hay penalización por complejidad y obtenés el árbol completo (overfitting). Cuando $alpha \to \infty$, la penalización es tan alta que terminás con un solo nodo (underfitting).

Vamos a desmenuzarla símbolo por símbolo porque acá es donde el 90% de los científicos de datos se quedan con "es un hyperparámetro más" sin entender la física detrás.

Descomposición Completa: Entendiendo Cada Símbolo

$R_\alpha(T)$ (Cost-Complexity Measure) - El Costo Total que Queremos Minimizar

R_α(T) = función objetivo a minimizar
       = costo del modelo considerando error + complejidad

Analogía: Es como el costo total de mantener un sistema en producción. No solo importa cuántos bugs tiene (error), sino también cuántos microservicios tenés desplegados (complejidad). Más servicios = más costo operacional, incluso si cada uno funciona bien.

En Machine Learning:

  • Para $\alpha$ pequeño: priorizás performance, aceptás complejidad
  • Para $\alpha$ grande: priorizás simplicidad, aceptás algo de error

Dimensionalidad: $R_\alpha(T)$ es un escalar que representa el costo total del árbol $T$ bajo el régimen de penalización $\alpha$.

Diferencia CRÍTICA: $R_\alpha(T)$ NO es el error en test set. Es una función de costo durante el entrenamiento que balancea impureza en train con complejidad estructural.

Ejemplo numérico concreto:

import numpy as np

# Simulemos dos árboles diferentes
# Árbol complejo: bajo error, alta complejidad
R_T_complex = 0.05  # error muy bajo
num_leaves_complex = 127  # muchas hojas

# Árbol simple: error moderado, baja complejidad  
R_T_simple = 0.15  # error más alto
num_leaves_simple = 8  # pocas hojas

# Con \alpha pequeño (favorece complejidad)
\alpha_small = 0.001
cost_complex_small_alpha = R_T_complex + \alpha_small * num_leaves_complex
cost_simple_small_alpha = R_T_simple + \alpha_small * num_leaves_simple

print(f"\alpha = {\alpha_small}")
print(f"  Árbol complejo: {cost_complex_small_alpha:.4f}")  # 0.177
print(f"  Árbol simple: {cost_simple_small_alpha:.4f}")     # 0.158
print(f"  Ganador: {'Complejo' if cost_complex_small_alpha < cost_simple_small_alpha else 'Simple'}")

# Con \alpha grande (penaliza complejidad)
\alpha_large = 0.01
cost_complex_large_alpha = R_T_complex + \alpha_large * num_leaves_complex
cost_simple_large_alpha = R_T_simple + \alpha_large * num_leaves_simple

print(f"\n\alpha = {\alpha_large}")
print(f"  Árbol complejo: {cost_complex_large_alpha:.4f}")  # 1.320
print(f"  Árbol simple: {cost_simple_large_alpha:.4f}")     # 0.230
print(f"  Ganador: {'Complejo' if cost_complex_large_alpha < cost_simple_large_alpha else 'Simple'}")

Salida:

  Árbol simple: 0.1580
  Ganador: Simple

\alpha = 0.01
  Árbol complejo: 1.3200

$R(T)$ (Resubstitution Error) - El Error en Training Data

R(T) = error del árbol T en los datos de entrenamiento
     = suma ponderada de impureza en todas las hojas

Analogía: Es como medir cuántos bugs encontrás cuando corrés tu test suite completo. No te dice nada sobre bugs que vas a encontrar en producción, pero sí te da una baseline de qué tan bien el modelo ajusta los datos que vio.

En Machine Learning:

  • Para clasificación: típicamente la tasa de misclassification o Gini impurity
  • Para regresión: típicamente MSE o MAE

Dimensionalidad: $R(T)$ es un escalar en el rango $[0, +\infty)$. Para clasificación normalizada, suele estar en $[0, 1]$.

Diferencia CRÍTICA: $R(T)$ decrece monótonamente a medida que el árbol crece. Un árbol más grande SIEMPRE tiene menor o igual $R(T)$ que uno más pequeño en los mismos datos de entrenamiento. Por eso necesitamos la penalización por complejidad.

Ejemplo numérico concreto:

from sklearn.datasets import make_classification
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import accuracy_score

# Dataset de ejemplo
X, y = make_classification(n_samples=200, n_features=4, n_informative=3,
                          n_redundant=1, random_state=42)

# Árbol sin límite (overfitting garantizado)
tree_overfit = DecisionTreeClassifier(random_state=42)
tree_overfit.fit(X, y)

# Árbol limitado
tree_limited = DecisionTreeClassifier(max_depth=3, random_state=42)
tree_limited.fit(X, y)

# R(T) aproximado como 1 - accuracy en train
R_T_overfit = 1 - accuracy_score(y, tree_overfit.predict(X))
R_T_limited = 1 - accuracy_score(y, tree_limited.predict(X))

print(f"R(T) árbol sin límite: {R_T_overfit:.4f}")  # ~0.0 (perfecto en train)
print(f"R(T) árbol limitado: {R_T_limited:.4f}")    # ~0.12 (error en train)
print(f"Hojas árbol sin límite: {tree_overfit.get_n_leaves()}")
print(f"Hojas árbol limitado: {tree_limited.get_n_leaves()}")

\alpha (Complexity Parameter) - El Dial que Controlás

α = parámetro de penalización por complejidad
  = ccp_alpha en sklearn
  = cuánto "cobrás" por cada hoja adicional

Analogía: Es como el precio por CPU/memoria en tu cloud provider. Si el precio es bajo ($\alpha$ pequeño), vas a agregar instancias como loco porque son baratas. Si el precio es alto ($\alpha$ grande), vas a optimizar cada línea de código antes de agregar otra instancia.

En Machine Learning:

  • Para $\alpha = 0$: no hay penalización, crecimiento máximo del árbol
  • Para $\alpha > 0$: cada hoja adicional debe "justificar" su existencia reduciendo el error al menos en $\alpha$
  • Para $\alpha \to \infty$: solo queda la raíz (modelo trivial)

Dimensionalidad: $\alpha \in [0, +\infty)$ es un escalar que controla el balance bias-variance. Valores típicos en práctica: $[0.0001, 0.1]$.

Diferencia CRÍTICA: ccp_alpha NO es lo mismo que min_impurity_decrease. El primero es global (afecta el árbol completo después de crecer), el segundo es local (afecta decisiones de split durante el crecimiento).

Ejemplo numérico concreto:

from sklearn.tree import DecisionTreeClassifier

# Probamos diferentes alphas
alphas = [0.0, 0.01, 0.05, 0.1]

for \alpha in alphas:
    tree = DecisionTreeClassifier(ccp_alpha=\alpha, random_state=42)
    tree.fit(X, y)
    
    train_acc = accuracy_score(y, tree.predict(X))
    n_leaves = tree.get_n_leaves()
    
    print(f"ccp_alpha={\alpha:5.2f} | Hojas: {n_leaves:3d} | Train Acc: {train_acc:.3f}")

Salida típica:

ccp_alpha= 0.00 | Hojas: 109 | Train Acc: 1.000
ccp_alpha= 0.01 | Hojas:   7 | Train Acc: 0.885
ccp_alpha= 0.05 | Hojas:   3 | Train Acc: 0.805
ccp_alpha= 0.10 | Hojas:   1 | Train Acc: 0.530

$|T|$ (Number of Terminal Nodes) - Conteo de Hojas

|T| = número de nodos terminales (hojas) del árbol T
    = complejidad estructural
    = parámetros efectivos del modelo

Analogía: Es como contar cuántas funciones distintas tenés en tu codebase. Más funciones = más superficie de ataque para bugs, más mantenimiento, más complejidad. No importa que cada función sea perfecta individualmente.

En Machine Learning:

  • Para árboles binarios completos de profundidad $d$: $|T| = 2^d$
  • Para árboles desbalanceados: $|T|$ puede ser mucho menor
  • Cada hoja representa una región de decisión distinta en el espacio de features

Dimensionalidad: $|T| \in \mathbb{N}^+$ es un entero positivo. Para un dataset de $n$ samples, $|T| \leq n$ (un sample por hoja en el caso extremo).

Diferencia CRÍTICA: $|T|$ cuenta HOJAS, no nodos totales. Un árbol con 3 splits tiene 4 hojas. Esta es la métrica relevante porque cada hoja es una predicción distinta.

Ejemplo numérico concreto:

from sklearn.tree import DecisionTreeClassifier

tree = DecisionTreeClassifier(max_depth=3, random_state=42)
tree.fit(X, y)

n_nodes = tree.tree_.node_count
n_leaves = tree.get_n_leaves()
max_depth = tree.get_depth()

print(f"Nodos totales: {n_nodes}")
print(f"Hojas |T|: {n_leaves}")
print(f"Profundidad: {max_depth}")
print(f"Nodos internos: {n_nodes - n_leaves}")

# Relación: para árbol binario perfecto
# n_nodes = 2^(depth+1) - 1
# n_leaves = 2^depth
expected_leaves_perfect = 2**max_depth
print(f"\nHojas en árbol perfecto de depth {max_depth}: {expected_leaves_perfect}")
print(f"Nuestro árbol tiene: {n_leaves} (puede ser desbalanceado)")

📐 Cómo sklearn Encuentra el Valor Óptimo de \alpha

El algoritmo de Cost-Complexity Pruning (CART) no prueba todos los valores posibles de $\alpha$. En cambio, usa un método inteligente basado en la propiedad de que existe una secuencia finita de sub-árboles óptimos.

El Algoritmo de Minimal Cost-Complexity Pruning

Paso 1: Definir el Effective alpha de un Nodo

Para cada nodo interno $t$ en el árbol completo $T_{\text{max}}$, calculamos:

$$\alpha_{\text{eff}}(t) = \frac{R(t) - R(T_t)}{|T_t| - 1}$$

donde:

  • $R(t)$ es el error si colapsamos el sub-árbol en $t$ a una hoja
  • $R(T_t)$ es el error del sub-árbol actual con raíz en $t$
  • $|T_t|$ es el número de hojas del sub-árbol $T_t$

Explicación: $\alpha_{\text{eff}}(t)$ representa el "precio por hoja" al cual es indiferente mantener el sub-árbol $T_t$ o colapsarlo a una sola hoja. Si $\alpha > \alpha_{\text{eff}}(t)$, conviene colapsar.

Paso 2: Identificar el Nodo con Menor alpha Efectivo

En cada iteración, encontramos el nodo $t^*$ con el menor $\alpha_{\text{eff}}$:

$$t^* = \underset{t \in \text{internal nodes}}{\arg\min} \alpha_{\text{eff}}(t)$$

Paso 3: Colapsar y Repetir

Colapsamos el sub-árbol en t^ a una hoja, registramos $\alpha_{\text{eff}}(t^)$, y repetimos hasta llegar a la raíz. Esto genera una secuencia de árboles $T_0, T_1, \ldots, T_k$ y sus correspondientes $\alpha_0 < \alpha_1 < \ldots < \alpha_k$.

from sklearn.tree import DecisionTreeClassifier
import numpy as np

# Entrenar árbol completo
clf = DecisionTreeClassifier(random_state=42)
clf.fit(X, y)

# sklearn calcula esta secuencia automáticamente
path = clf.cost_complexity_pruning_path(X, y)
ccp_alphas = path.ccp_alphas
impurities = path.impurities

print("Secuencia de alphas efectivos (primeros 10):")
for i, (\alpha, impurity) in enumerate(zip(ccp_alphas[:10], impurities[:10])):
    print(f"  α_{i} = {\alpha:.6f}, impureza = {impurity:.4f}")

Salida típica:

Secuencia de alphas efectivos (primeros 10):
  α_0 = 0.000000, impureza = 0.0000
  α_1 = 0.000500, impureza = 0.0050
  α_2 = 0.001000, impureza = 0.0150
  ...

📐 Selección Óptima de ccp_alpha por Validación Cruzada

La secuencia de alphas nos da los candidatos, pero necesitamos validación cruzada para elegir el óptimo. El proceso:

Paso 1: Generar Secuencia de alphas

from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import cross_val_score
import numpy as np

# Árbol base para obtener alphas
clf = DecisionTreeClassifier(random_state=42)
path = clf.cost_complexity_pruning_path(X_train, y_train)
ccp_alphas = path.ccp_alphas[:-1]  # Eliminamos el último (árbol trivial)

Paso 2: Evaluar Cada alpha con Cross-Validation

Para cada $\alpha_i$ en la secuencia:

$$\text{Score}_{\text{CV}}(\alpha_i) = \frac{1}{K} \sum_{k=1}^{K} \text{Accuracy}(T_{\alpha_i}, D_{\text{val}}^{(k)})$$

train_scores = []
test_scores = []

for alpha in ccp_alphas:
    clf = DecisionTreeClassifier(ccp_alpha=alpha, random_state=42)
    # Cross-validation en train set
    cv_scores = cross_val_score(clf, X_train, y_train, cv=5, 
                                scoring='accuracy')
    train_scores.append(cv_scores.mean())

Paso 3: Seleccionar alpha Óptimo

$$\alpha^* = \underset{\alpha_i}{\arg\max} \text{Score}_{\text{CV}}(\alpha_i)$$

optimal_idx = np.argmax(test_scores)
optimal_alpha = ccp_alphas[optimal_idx]
print(f"\alpha óptimo: {optimal_alpha:.6f}")

🚨 Errores Comunes (y Cómo No Cagarla)

Error 1: Usar ccp_alpha Arbitrario Sin Validación

El problema:

# ❌ PROHIBIDO: Inventarse un \alpha sin evidencia
tree = DecisionTreeClassifier(ccp_alpha=0.01, random_state=42)
tree.fit(X_train, y_train)

print(f"Accuracy en test: {accuracy_score(y_test, tree.predict(X_test)):.3f}")
# "Oye, 0.01 se ve bien, ¿no?" 🤡

Por qué falla:

El valor óptimo de ccp_alpha depende COMPLETAMENTE de tu dataset: escala de features, número de samples, nivel de ruido, y distribución de clases. Un $\alpha = 0.01$ puede ser perfecto para un problema y desastroso para otro. Estás adivinando un hyperparámetro crítico.

Matemáticamente, el problema es que estás minimizando:

$$R_\alpha(T) = R(T) + \alpha |T|$$

pero sin saber qué valor de $\alpha$ balancea correctamente bias-variance PARA TU DATA. Es como configurar el learning rate de un optimizer sin grid search.

# Demo del desastre
from sklearn.metrics import accuracy_score

alphas_random = [0.001, 0.01, 0.05, 0.1]

for \alpha in alphas_random:
    tree = DecisionTreeClassifier(ccp_alpha=\alpha, random_state=42)
    tree.fit(X_train, y_train)
    
    train_acc = accuracy_score(y_train, tree.predict(X_train))
    test_acc = accuracy_score(y_test, tree.predict(X_test))
    
    print(f"α={\alpha:5.3f} | Train: {train_acc:.3f} | Test: {test_acc:.3f}")
    # Vas a ver variabilidad enorme sin saber cuál es mejor

La solución:

from __future__ import annotations
import logging
from typing import Optional
import numpy as np
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import cross_val_score
from sklearn.base import BaseEstimator

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)


class OptimalCCP\alphaFinder:
    """
    Encuentra el valor óptimo de ccp_alpha mediante validación cruzada.
    
    Implementa el algoritmo de Cost-Complexity Pruning siguiendo la metodología
    CART, evaluando la secuencia de alphas efectivos generada por sklearn.
    """
    
    def __init__(
        self,
        estimator: Optional[BaseEstimator] = None,
        cv: int = 5,
        scoring: str = 'accuracy',
        random_state: Optional[int] = None
    ) -> None:
        """
        Args:
            estimator: Clasificador base (default: DecisionTreeClassifier).
            cv: Número de folds para cross-validation.
            scoring: Métrica de evaluación.
            random_state: Seed para reproducibilidad.
        """
        self.estimator = estimator or DecisionTreeClassifier(random_state=random_state)
        self.cv = cv
        self.scoring = scoring
        self.random_state = random_state
        
        self.ccp_alphas_: Optional[np.ndarray] = None
        self.optimal_alpha_: Optional[float] = None
        self.cv_scores_: Optional[np.ndarray] = None
    
    def find_optimal_alpha(
        self,
        X: np.ndarray,
        y: np.ndarray
    ) -> tuple[float, np.ndarray, np.ndarray]:
        """
        Encuentra el \alpha óptimo evaluando toda la secuencia.
        
        Args:
            X: Features de entrenamiento (n_samples, n_features).
            y: Target de entrenamiento (n_samples,).
            
        Returns:
            Tuple con (optimal_alpha, alphas_sequence, cv_scores).
            
        Raises:
            ValueError: Si X o y están vacíos o tienen dimensiones inconsistentes.
        """
        if len(X) == 0 or len(y) == 0:
            raise ValueError("X e y no pueden estar vacíos")
        
        if len(X) != len(y):
            raise ValueError(f"Dimensiones inconsistentes: X={len(X)}, y={len(y)}")
        
        logger.info(f"Iniciando búsqueda de \alpha óptimo con CV={self.cv}")
        
        # Generar secuencia de alphas usando el método de sklearn
        base_tree = DecisionTreeClassifier(random_state=self.random_state)
        path = base_tree.cost_complexity_pruning_path(X, y)
        ccp_alphas = path.ccp_alphas[:-1]  # Excluir el último (árbol trivial)
        
        logger.info(f"Evaluando {len(ccp_alphas)} valores de \alpha")
        
        # Evaluar cada \alpha con cross-validation
        cv_scores = []
        for \alpha in ccp_alphas:
            clf = DecisionTreeClassifier(
                ccp_alpha=\alpha,
                random_state=self.random_state
            )
            scores = cross_val_score(
                clf, X, y,
                cv=self.cv,
                scoring=self.scoring
            )
            cv_scores.append(scores.mean())
        
        cv_scores = np.array(cv_scores)
        
        # Seleccionar \alpha óptimo (máximo score CV)
        optimal_idx = np.argmax(cv_scores)
        optimal_alpha = ccp_alphas[optimal_idx]
        
        # Guardar resultados
        self.ccp_alphas_ = ccp_alphas
        self.optimal_alpha_ = optimal_alpha
        self.cv_scores_ = cv_scores
        
        logger.info(
            f"\alpha óptimo: {optimal_alpha:.6f} "
            f"(Score CV: {cv_scores[optimal_idx]:.4f})"
        )
        
        return optimal_alpha, ccp_alphas, cv_scores


# ✓ USO CORRECTO
finder = OptimalCCP\alphaFinder(cv=5, random_state=42)
optimal_alpha, alphas, scores = finder.find_optimal_alpha(X_train, y_train)

# Entrenar con el \alpha óptimo
final_tree = DecisionTreeClassifier(ccp_alpha=optimal_alpha, random_state=42)
final_tree.fit(X_train, y_train)

logger.info(f"Accuracy en test: {accuracy_score(y_test, final_tree.predict(X_test)):.4f}")

Error 2: Confundir Pre-Pruning con Post-Pruning

El problema:

# ❌ PROHIBIDO: Creer que max_depth es lo mismo que ccp_alpha
tree_pre = DecisionTreeClassifier(max_depth=5, random_state=42)
tree_pre.fit(X_train, y_train)

# "Ya lo podé con max_depth, ¿para qué necesito ccp_alpha?" 🤡

Por qué falla:

max_depth, min_samples_split, y min_samples_leaf son pre-pruning: detienen el crecimiento DURANTE la construcción del árbol basándose en reglas locales. ccp_alpha es post-pruning: deja que el árbol crezca completo y luego lo poda basándose en el costo-complejidad GLOBAL.

La diferencia matemática:

Pre-Pruning (greedy local):

  • En cada nodo, decide: "¿Este split reduce impureza suficiente?"
  • Puede detener el crecimiento prematuramente y perder splits valiosos que vienen después

Post-Pruning (globally optimal):

  • Crece el árbol completo minimizando impureza local
  • Luego evalúa cada sub-árbol: "¿El beneficio en error justifica la complejidad?"
  • Minimiza $R_\alpha(T) = R(T) + \alpha |T|$ globalmente
from sklearn.datasets import make_classification
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import accuracy_score

# Data con relaciones no-lineales complejas
X, y = make_classification(
    n_samples=500,
    n_features=10,
    n_informative=8,
    n_redundant=2,
    random_state=42
)

# Pre-pruning arbitrario
tree_pre = DecisionTreeClassifier(max_depth=5, random_state=42)
tree_pre.fit(X_train, y_train)

# Post-pruning óptimo
finder = OptimalCCP\alphaFinder(cv=5, random_state=42)
optimal_alpha, _, _ = finder.find_optimal_alpha(X_train, y_train)
tree_post = DecisionTreeClassifier(ccp_alpha=optimal_alpha, random_state=42)
tree_post.fit(X_train, y_train)

# Comparar
print(f"Pre-pruning (max_depth=5):")
print(f"  Hojas: {tree_pre.get_n_leaves()}")
print(f"  Test Acc: {accuracy_score(y_test, tree_pre.predict(X_test)):.4f}")

print(f"\nPost-pruning (\alpha={optimal_alpha:.6f}):")
print(f"  Hojas: {tree_post.get_n_leaves()}")
print(f"  Test Acc: {accuracy_score(y_test, tree_post.predict(X_test)):.4f}")

La solución:

# ✓ CORRECTO: Combinar ambos estratégicamente
# Pre-pruning conservador (límites amplios)
# + Post-pruning óptimo (ccp_alpha validado)

tree_hybrid = DecisionTreeClassifier(
    max_depth=15,              # Pre-pruning conservador
    min_samples_split=10,      # Evita splits en datos muy pequeños
    min_samples_leaf=5,        # Evita hojas con pocos samples
    ccp_alpha=optimal_alpha,   # Post-pruning óptimo

Error 3: No Visualizar la Curva de \alpha vs Performance

El problema:

# ❌ PROHIBIDO: Confiar ciegamente en el \alpha óptimo sin validar la estabilidad
optimal_alpha = 0.005  # del finder
tree = DecisionTreeClassifier(ccp_alpha=optimal_alpha, random_state=42)
tree.fit(X_train, y_train)

# "Mi \alpha óptimo es 0.005, fin de la historia" 🤡
# ¿Pero qué pasa con alphas cercanos? ¿Es un óptimo estable o un pico ruidoso?

Por qué falla:

El "óptimo" de validación cruzada puede ser RUIDOSO, especialmente con pocos folds o datasets pequeños. Si $\alpha = 0.005$ da 0.8543 y $\alpha = 0.006$ da 0.8541, esa diferencia es ruido, no señal. Sin visualizar la curva completa, no sabés si estás en un plateau estable o en un pico aleatorio.

Matemáticamente, el problema es que:

$$\alpha^* = \underset{\alpha}{\arg\max} \mathbb{E}[\text{Score}(\alpha)]$$

pero en práctica solo tenemos un estimador ruidoso del score:

$$\hat{\alpha}^* = \underset{\alpha}{\arg\max} \frac{1}{K}\sum_{k=1}^K \text{Score}_k(\alpha)$$

# Demo del problema
import matplotlib.pyplot as plt

finder = OptimalCCP\alphaFinder(cv=3, random_state=42)  # Solo 3 folds (ruidoso)
optimal_alpha, alphas, scores = finder.find_optimal_alpha(X_train, y_train)

# El óptimo puede ser un artefacto del ruido
optimal_idx = np.argmax(scores)
print(f"\alpha óptimo: {alphas[optimal_idx]:.6f}")
print(f"Score óptimo: {scores[optimal_idx]:.4f}")

# ¿Qué pasa con los vecinos?
for i in range(max(0, optimal_idx-2), min(len(alphas), optimal_idx+3)):
    print(f"  α={alphas[i]:.6f} → Score={scores[i]:.4f}")
    # Probablemente veas scores muy similares (ruido)

La solución:

# ✓ CORRECTO: Visualizar y elegir con criterio
import matplotlib.pyplot as plt
from typing import Tuple

def plot_alpha_selection(
    alphas: np.ndarray,
    cv_scores: np.ndarray,
    optimal_alpha: float
) -> Tuple[plt.Figure, plt.Axes]:
    """
    Visualiza la curva de \alpha vs score de validación cruzada.
    
    Args:
        alphas: Secuencia de valores de \alpha evaluados.
        cv_scores: Scores de CV correspondientes.
        optimal_alpha: Valor óptimo seleccionado.
        
    Returns:
        Tuple con (figura, ejes) de matplotlib.
    """
    fig, ax = plt.subplots(figsize=(10, 6))
    
    ax.plot(alphas, cv_scores, marker='o', linewidth=2, markersize=6)
    
    # Marcar el óptimo
    optimal_idx = np.argmax(cv_scores)
    ax.axvline(optimal_alpha, color='r', linestyle='--', 
               label=f'Óptimo: α={optimal_alpha:.6f}')
    ax.plot(optimal_alpha, cv_scores[optimal_idx], 
            'r*', markersize=15, label=f'Score={cv_scores[optimal_idx]:.4f}')
    
    # Identificar plateau (scores dentro del 1% del óptimo)
    threshold = 0.99 * cv_scores[optimal_idx]
    plateau_mask = cv_scores >= threshold
    plateau_alphas = alphas[plateau_mask]
    
    if len(plateau_alphas) > 1:
        ax.axvspan(plateau_alphas.min(), plateau_alphas.max(), 
                   \alpha=0.2, color='green', 
                   label=f'Plateau (>{threshold:.3f})')
    
    ax.set_xlabel('ccp_alpha', fontsize=12)
    ax.set_ylabel('CV Score', fontsize=12)
    ax.set_title('\alpha vs Cross-Validation Score', fontsize=14)
    ax.set_xscale('log')
    ax.grid(True, \alpha=0.3)
    ax.legend()
    
    return fig, ax


# Uso
finder = OptimalCCP\alphaFinder(cv=5, random_state=42)
optimal_alpha, alphas, scores = finder.find_optimal_alpha(X_train, y_train)

fig, ax = plot_alpha_selection(alphas, scores, optimal_alpha)
plt.tight_layout()
plt.savefig('/tmp/\alpha_selection.png', dpi=300, bbox_inches='tight')
plt.show()

# Ahora podés decidir con criterio:
# ¿Es un pico aislado o un plateau estable?
# Si hay plateau, podés elegir \alpha más grande (más simple) sin perder performance

Error 4: Aplicar Pruning Sin Separar Train/Test Correctamente

El problema:

# ❌ PROHIBIDO: Buscar \alpha óptimo usando TODO el dataset
from sklearn.datasets import load_iris

X, y = load_iris(return_X_y=True)

# DESASTRE: Fit del path en todo el dataset
clf = DecisionTreeClassifier(random_state=42)
path = clf.cost_complexity_pruning_path(X, y)  # ⚠️ DATA LEAKAGE
ccp_alphas = path.ccp_alphas

# Evaluar con cross-validation TAMBIÉN en todo el dataset
cv_scores = []
for \alpha in ccp_alphas:
    tree = DecisionTreeClassifier(ccp_alpha=\alpha, random_state=42)
    scores = cross_val_score(tree, X, y, cv=5)  # ⚠️ LEAKAGE
    cv_scores.append(scores.mean())

optimal_alpha = ccp_alphas[np.argmax(cv_scores)]
final_tree = DecisionTreeClassifier(ccp_alpha=optimal_alpha, random_state=42)
final_tree.fit(X, y)  # ⚠️ TRIPLE LEAKAGE

# "Mi modelo tiene 100% accuracy" 🤡
# Obvio, lo entrenaste en TODO incluyendo lo que debería ser test

Por qué falla:

Estás cometiendo DATA LEAKAGE en tres niveles:

  1. La secuencia de alphas se calculó viendo TODO el dataset
  2. La validación cruzada para elegir alpha usó TODO el dataset
  3. El modelo final se entrenó en TODO el dataset

No tenés ninguna estimación honesta de performance en datos NO VISTOS. Tu "alpha óptimo" está optimizado para el dataset completo, no para generalizar.

El flujo correcto sigue el principio sagrado de ML:

$$\text{Data} \xrightarrow{\text{split}} \begin{cases} \text{Train} \to \text{find } \alpha^* \to \text{fit model} \\ \text{Test} \to \text{evaluate ONCE} \end{cases}$$

La solución:

# ✓ CORRECTO: Split primero, SIEMPRE
from sklearn.model_selection import train_test_split

# 1. SPLIT PRIMERO (sagrado)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, 
    test_size=0.2, 
    stratify=y,  # Mantiene distribución de clases
    random_state=42
)

# 4. Evaluar UNA SOLA VEZ en test (datos vírgenes)

🎯 Resumen Ejecutivo

Pruning (poda) no es "poner max_depth=5 y listo". Es resolver el problema fundamental de balance entre accuracy y complejidad mediante la minimización de:

$$R_\alpha(T) = R(T) + \alpha |T|$$

Mediante el algoritmo de Cost-Complexity Pruning:

$$\alpha^* = \underset{\alpha_i}{\arg\max} \text{Score}_{\text{CV}}(\alpha_i)$$

Con todas las implicaciones que eso conlleva:

  • Data Leakage Prevention: Split ANTES de buscar alpha, nunca al revés
  • Algoritmo Óptimo: Post-pruning (ccp_alpha) es globalmente óptimo, pre-pruning (max_depth) es greedy local
  • Validación Cruzada: El alpha óptimo NO se adivina, se encuentra empíricamente evaluando la secuencia completa
  • Visualización Crítica: Un alpha "óptimo" ruidoso puede ser peor que un alpha subóptimo estable

De una, hágale. Con esta base de Cost-Complexity Pruning + código production-ready, estás listo para podar árboles que generalicen en vez de memorizar ruido.

¡Uy parcero, ya con esto podés mandar a la mierda el overfitting! 🌳✂️
