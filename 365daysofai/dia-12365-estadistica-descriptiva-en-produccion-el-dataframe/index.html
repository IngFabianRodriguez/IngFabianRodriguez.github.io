Por qué este tema te va a salvar el pellejo:

Hermano, si alguna vez has tenido que explicarle a un PM o a un stakeholder "cómo se ve la data" sin que se te duerman en la reunión, este artículo es para vos. La estadística descriptiva es esa vaina que te salva cuando necesitas entender tus datos ANTES de meterles un modelo de ML que después no sabes por qué no converge.

Y ojo: pandas.DataFrame.describe() parece inofensivo, pero mal usado te puede dejar con conclusiones más falsas que billete de $3. Acá te voy a enseñar cómo usarlo en producción, con typing estricto, manejo de edge cases y optimizaciones que importan cuando tenés millones de filas.

1. ¿Qué Carajos es la Estadística Descriptiva? (Y Por Qué Debería Importarte)

La Analogía del Tinto en la Oficina:

Imaginate que en tu oficina hay una cafetera. Querés saber "cómo toma tinto el equipo". Podés hacer dos cosas:

  1. Estadística Descriptiva: Contar cuántos tintos se toman al día (media), quién toma más/menos (min/max), qué tan disperso está el consumo (desviación estándar).
  2. Estadística Inferencial: Predecir cuántos tintos se van a tomar el próximo mes basándote en los datos históricos.

La estadística descriptiva es el "qué pasó", no el "qué va a pasar". Es tu primera línea de defensa contra datasets que parecen limpios pero están llenos de outliers, NaNs o distribuciones sesgadas que te van a hacer llorar en producción.

2. Fundamentos Matemáticos (Sin Esta Base, Estás Adivinando)

2.1 Medidas de Tendencia Central

La media (promedio) es la más famosa, pero también la más mentirosa:

$$\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i$$

Problema: Si tenés un outlier gigante (ej: un salario de $1,000,000 en un equipo donde todos ganan $50,000), la media te queda en $140,000 y no representa a nadie.

Solución: La mediana (percentil 50):

$$\text{Mediana} = \begin{cases} x_{\frac{n+1}{2}} & \text{si } n \text{ es impar} \\ \frac{x_{\frac{n}{2}} + x_{\frac{n}{2}+1}}{2} & \text{si } n \text{ es par} \end{cases}$$

La mediana no le para bolas a los outliers. Si el CEO gana $5M, la mediana sigue siendo $50,000.

2.2 Medidas de Dispersión

La desviación estándar te dice qué tan "regados" están tus datos:

$$\sigma = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (x_i - \bar{x})^2}$$

Interpretación:

  • $\sigma$ pequeña → Los datos están apiñaditos cerca de la media.
  • $\sigma$ grande → Los datos están más dispersos que un equipo de frontend discutiendo React vs Vue.

Varianza es simplemente $sigma^2$, útil para cálculos matemáticos pero menos intuitiva.

2.3 Cuartiles y el IQR (Rango Intercuartílico)

Los cuartiles dividen tu dataset en 4 partes iguales:

  • Q1 (25%): El 25% de los datos están por debajo de este valor.
  • Q2 (50%): Es la mediana.
  • Q3 (75%): El 75% de los datos están por debajo.

El IQR es la diferencia entre Q3 y Q1:

$$\text{IQR} = Q3 - Q1$$

Para qué sirve: Detectar outliers. La regla clásica:

$$\text{Outlier si: } x < Q1 - 1.5 \times \text{IQR} \quad \text{o} \quad x > Q3 + 1.5 \times \text{IQR}$$

@dataclass(frozen=True, slots=True)  # slots para optimizar memoria
class DescriptiveStats:
    """
    Contenedor inmutable para estadísticas descriptivas.
    
    frozen=True previene mutaciones accidentales.
    slots=True reduce overhead de memoria (~40% en datasets grandes).
    """
    count: float
    mean: float
    std: float
    min: float
    q25: float
    median: float
    q75: float
    max: float
    iqr: float
    skewness: float
    kurtosis: float
    
    def is_normal_distribution(self, threshold: float = 2.0) -> bool:
        """
        Heurística rápida para detectar distribuciones no normales.
        
        Regla de oro:
        - |skewness| < 1: distribución simétrica
        - |kurtosis| < 3: colas similares a normal
        """
        return abs(self.skewness) < 1.0 and abs(self.kurtosis - 3.0) < threshold
    
    def has_outliers(self) -> bool:
        """
        Detecta presencia de outliers usando IQR.
        
        No identifica cuáles son, solo si existen.
        Para análisis detallado, usar OutlierDetector.
        """
        lower_bound = self.q25 - 1.5 * self.iqr
        upper_bound = self.q75 + 1.5 * self.iqr
        return self.min < lower_bound or self.max > upper_bound


class DataFrameDescriber(Generic[T]):
    """
    Wrapper production-ready para estadística descriptiva.
    
    Por qué esta clase existe:
    1. pandas.describe() es limitado (no incluye IQR, skewness, kurtosis).
    2. No maneja bien columnas categóricas mezcladas.
    3. No ofrece control sobre percentiles personalizados.
    4. No tiene validaciones robustas.
    """
    
    __slots__ = ('_df', '_numeric_only', '_percentiles', '_include_advanced')
    
    def __init__(
        self,
        df: T,
        numeric_only: bool = True,
        percentiles: list[float] | None = None,
        include_advanced: bool = True
    ) -> None:
        """
        Args:
            df: DataFrame a analizar
            numeric_only: Si True, ignora columnas no numéricas
            percentiles: Percentiles personalizados (ej: [0.1, 0.5, 0.9])
            include_advanced: Incluir skewness/kurtosis (costoso en datasets grandes)
        """
        if df.empty:
            raise ValueError("DataFrame vacío. No se puede calcular estadísticas.")
        
        self._df = df
        self._numeric_only = numeric_only
        self._percentiles = percentiles or [0.25, 0.5, 0.75]
        self._include_advanced = include_advanced
        
        # Validar percentiles
        if not all(0 <= p <= 1 for p in self._percentiles):
            raise ValueError("Percentiles deben estar en [0, 1].")
        
        
logger.info

            f"DataFrameDescriber inicializado: "
            f"{len(df)} filas, {len(df.columns)} columnas"
        )

        """
        Calcula estadísticas descriptivas para una columna específica.
        
        Esta función es O(n log n) por el sort implícito en quantile().
        Para datasets >10M filas, considerar sampling estratificado.
        """
        if column not in self._df.columns:
            raise KeyError(f"Columna '{column}' no existe en el DataFrame.")
        
        series = self._df[column]
        
        # Validar tipo numérico
        if not 
pd.api.types.is
_numeric_dtype(series):
            raise TypeError(
                f"Columna '{column}' no es numérica. "
                f"Tipo detectado: {series.dtype}"
            )
        
        # Eliminar NaNs y advertir
        original_len = len(series)
        series_clean = series.dropna()
        nan_count = original_len - len(series_clean)
        
        if nan_count > 0:
            logger.warning(
                f"Columna '{column}': {nan_count} NaNs eliminados "
                f"({100*nan_count/original_len:.2f}%)"
            )
        
        if len(series_clean) == 0:
            raise ValueError(f"Columna '{column}' solo contiene NaNs.")
        
        # Cálculos básicos (optimizados por NumPy bajo el hood)
        count = float(len(series_clean))
        mean = float(series_clean.mean())
        std = float(series_clean.std(ddof=1))  # ddof=1 para sample std
        min_val = float(series_clean.min())
        max_val = float(series_clean.max())
        
        # Cuartiles (usa algoritmo 'linear' por defecto, más rápido que 'midpoint')
        q25 = float(series_clean.quantile(0.25))
        median = float(series_clean.quantile(0.50))
        q75 = float(series_clean.quantile(0.75))
        iqr = q75 - q25
        
        # Métricas avanzadas (costosas, solo si se solicitan)
        if self._include_advanced:
            with warnings.catch_warnings():
                warnings.simplefilter("ignore")  # scipy puede advertir sobre samples pequeños
                skewness = float(series_clean.skew())
                kurtosis = float(series_clean.kurtosis())
        else:
            skewness = 0.0
            kurtosis = 0.0
        
        return DescriptiveStats(
            count=count,
            mean=mean,
            std=std,
            min=min_val,
            q25=q25,
            median=median,
            q75=q75,
            max=max_val,
            iqr=iqr,
            skewness=skewness,
            kurtosis=kurtosis
        )

        Complejidad: O(n * m * log n) donde n=filas, m=columnas numéricas.
        Para optimizar: usar Dask o Polars si n > 100M.
        """
        if self._numeric_only:
            numeric_cols = self._
df.select
_dtypes(include=[np.number]).columns
        else:
            numeric_cols = self._df.columns
        
        if len(numeric_cols) == 0:
            raise ValueError("No hay columnas numéricas en el DataFrame.")
        
        results: dict[str, DescriptiveStats] = {}
        
        for col in numeric_cols:

                results[col] = self.describe_column(col)

                'max': stats.max,
                'IQR': stats.iqr,
                'skewness': stats.skewness,
                'kurtosis': stats.kurtosis,

4.2 Detector de Outliers Avanzado

from enum import Enum
from typing import Callable

class OutlierMethod(Enum):
    """
    Métodos disponibles para detección de outliers.
    
    IQR: Rápido, robusto, funciona bien con distribuciones asimétricas.
    Z_SCORE: Asume normalidad, sensible a outliers extremos.
    MAD: Median Absolute Deviation, más robusto que Z-score.
    """
    IQR = "iqr"
    Z_SCORE = "z_score"
    MAD = "mad"


class OutlierDetector:
    """
    Detecta y maneja outliers con múltiples estrategias.
    
    Por qué no usar solo pandas:
    - pandas no ofrece MAD out-of-the-box.
    - No permite estrategias de handling flexibles.
    - No logea las decisiones tomadas.
    """
    
    __slots__ = ('_method', '_threshold', '_logger')
    
    def __init__(
        self,
        method: OutlierMethod = OutlierMethod.IQR,
        threshold: float = 1.5
    ) -> None:
        """
        Args:
            method: Algoritmo de detección
            threshold: Factor multiplicador (1.5 para IQR, 3.0 para Z-score)
        """
        self._method = method
        self._threshold = threshold
        self._logger = logging.getLogger(f"{__name__}.{self.__class__.__name__}")
    
    def _detect_iqr(self, series: pd.Series) -> pd.Series:
        """
        Método IQR clásico.
        
        Ventaja: No asume distribución normal.
        Desventaja: Puede ser demasiado agresivo con threshold=1.5.
        """
        q1 = series.quantile(0.25)
        q3 = series.quantile(0.75)
        iqr = q3 - q1
        
        lower_bound = q1 - self._threshold * iqr
        upper_bound = q3 + self._threshold * iqr
        
        return (series < lower_bound) | (series > upper_bound)
    
    def _detect_zscore(self, series: pd.Series) -> pd.Series:
        """
        Z-score method.
        
        z = (x - μ) / σ
        
        Outlier si |z| > threshold (típicamente 3).
        
        PELIGRO: Si tus datos no son normales, esto no sirve.
        """
        mean = series.mean()
        std = series.std()
        
        if std == 0:
            self._logger.warning("Desviación estándar = 0, todos los valores son iguales.")
            return pd.Series([False] * len(series), index=series.index)
        
        z_scores = np.abs((series - mean) / std)
        return z_scores > self._threshold
    
    def _detect_mad(self, series: pd.Series) -> pd.Series:
        """
        Median Absolute Deviation (MAD).
        
        MAD = median(|x_i - median(x)|)
        
        Modified Z-score: M_i = 0.6745 * (x_i - median) / MAD
        
        Más robusto que Z-score porque usa mediana en lugar de media.
        El factor 0.6745 hace que MAD sea comparable con std en distribución normal.
        """
        median = series.median()
        mad = np.median(np.abs(series - median))
        
        if mad == 0:
            self._logger.warning("MAD = 0, todos los valores están en la mediana.")
            return pd.Series([False] * len(series), index=series.index)
        
        modified_z_scores = 0.6745 * np.abs(series - median) / mad
        return modified_z_scores > self._threshold
    
    def detect(self, series: pd.Series) -> pd.Series:
        """
        Detecta outliers usando el método configurado.
        
        Returns:
            Series booleana donde True indica outlier.
        """
        series_clean = series.dropna()
        
        if len(series_clean) < 4:
            self._logger.warning(
                "Muestra muy pequeña (<4 valores), "
                "detección de outliers no confiable."
            )
            return pd.Series([False] * len(series), index=series.index)
        
        if self._method == OutlierMethod.IQR:
            outliers = self._detect_iqr(series_clean)
        elif self._method == OutlierMethod.Z_SCORE:
            outliers = self._detect_zscore(series_clean)
        elif self._method == OutlierMethod.MAD:
            outliers = self._detect_mad(series_clean)
        else:
            raise ValueError(f"Método no soportado: {self._method}")
        
        # Re-indexar para incluir NaNs originales
        outliers = outliers.reindex(series.index, fill_value=False)
        
        outlier_count = outliers.sum()
        outlier_pct = 100 * outlier_count / len(series)
        
        self._
logger.info
(
            f"Outliers detectados: {outlier_count} ({outlier_pct:.2f}%) "
            f"usando {self._method.value}"
        )
        
        return outliers
    
    def handle_outliers(
        self,
        series: pd.Series,
        strategy: Literal["remove", "clip", "winsorize"] = "clip"
    ) -> pd.Series:
        """
        Maneja outliers detectados según estrategia.
        
        Estrategias:
        - remove: Elimina outliers (peligroso, cambia tamaño del dataset).
        - clip: Reemplaza outliers con límites (min/max válidos).
        - winsorize: Reemplaza outliers con percentiles.
        """
        outliers_mask = self.detect(series)
        
        if strategy == "remove":
            self._logger.warning("Estrategia 'remove' cambia el tamaño del dataset.")
            return series[~outliers_mask]
        
        elif strategy == "clip":
            # Calcular límites basados en el método
            if self._method == OutlierMethod.IQR:
                q1 = series.quantile(0.25)
                q3 = series.quantile(0.75)
                iqr = q3 - q1
                lower = q1 - self._threshold * iqr
                upper = q3 + self._threshold * iqr
            else:
                # Para Z-score/MAD, usar percentiles conservadores
                lower = series.quantile(0.01)
                upper = series.quantile(0.99)
            
            return series.clip(lower=lower, upper=upper)
        
        elif strategy == "winsorize":
            # Winsorización: reemplazar con percentiles 5 y 95
            lower = series.quantile(0.05)
            upper = series.quantile(0.95)
            
            series_winsorized = series.copy()
            series_winsorized[series < lower] = lower
            series_winsorized[series > upper] = upper
            
            return series_winsorized
        
        else:
            raise ValueError(f"Estrategia no válida: {strategy}")

5.2 Sampling Estratificado para EDA

def stratified_sample(
    df: pd.DataFrame,
    column: str,
    n_samples: int = 10_000,
    random_state: int = 42
) -> pd.DataFrame:
    """
    Toma una muestra estratificada para EDA rápido.
    
    En datasets >100M filas, analizar el 100% es innecesario.
    Una muestra de 10K filas bien balanceada te da 95% de la info.
    """
    if len(df) <= n_samples:
        return df
    
    # Dividir en cuartiles
    df['_quartile'] = pd.qcut(df[column], q=4, labels=False, duplicates='drop')
    
    # Samplear proporcionalmente de cada cuartil
    sampled = df.groupby('_quartile', group_keys=False).apply(
        lambda x: x.sample(n=n_samples // 4, random_state=random_state)
    )
    
    return sampled.drop(columns=['_quartile'])

6. Conclusión: Cómo Esto te Salva de Quedarte Embalado

Lo que aprendiste hoy:

  1. Estadística Descriptiva ≠ Solo Promedios: Media, mediana, desviación estándar, IQR, skewness, kurtosis — cada una cuenta una parte diferente de la historia.
  2. pandas.describe() es Insuficiente: No detecta outliers, no te dice si la distribución es normal, y no te da control sobre percentiles.
  3. Outliers son el Enemy #1: Un solo valor raro puede arruinar tu normalización, tu modelo y tu credibilidad en la próxima reunión con stakeholders.
  4. Typing + Error Handling = Código que No Explota a las 3 AM: En producción, los datos nunca vienen limpios. Si no manejás NaNs, tipos incorrectos y edge cases, vas a tener un mal día.

Pro-tip colombiano: Si estás en una reunión y alguien dice "los datos se ven bien", preguntale: "¿Checaste la distribución de los outliers por IQR?". Verás cómo se le baja el rollo rapidito. 😏

"Primero calculo estadísticas descriptivas robustas (mediana, IQR, skewness). Luego detecto outliers usando MAD porque es más robusto que Z-score. Finalmente, valido normalidad con skewness/kurtosis para decidir si necesito transformaciones como log o Box-Cox. Todo esto lo logeo y lo exporto para auditoría."
