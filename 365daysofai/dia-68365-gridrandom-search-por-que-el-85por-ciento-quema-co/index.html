¿Qué más, parcero? Llegamos al Día 68 y hoy toca hablar de algo que todo el mundo usa mal: la búsqueda de hiperparámetros. Y no, no es "solo probar configuraciones hasta que funcione". Dejá el café, acomodate, porque hoy aprendemos a hacer esto bien.

🎯 ¿GridSearch, RandomSearch u Optuna — En Serio Cuál Uso?

Mirá, la búsqueda de hiperparámetros es como cuando vas a un restaurante nuevo: podés leer el menú completo de cabo a rabo (Grid Search), abrir páginas al azar y pedir lo que caiga (Random Search), o pedirle al mesero que te recomiende con base en lo que ya probaste (Bayesian/Optuna). Las tres son válidas, pero en contextos muy distintos.

Definición técnica (sin rodeos):

Dado un modelo $f_\theta$ con hiperparámetros $\theta \in \Theta$, buscamos:

$$\theta^* = \arg\min_{\theta \in \Theta} \mathcal{L}_{CV}(f_\theta,\, \mathcal{D})$$

donde $\mathcal{L}_{CV}$ es la pérdida estimada por cross-validation sobre el dataset $\mathcal{D}$.

Estrategias principales:

  • Grid Search: Enumera el producto cartesiano de $\Theta$ completo. Garantiza encontrar el óptimo dentro de la grilla. Explota exponencialmente con la dimensión.
  • Random Search: Muestrea $n$ puntos aleatorios de $\Theta$. Sorprendentemente eficiente cuando pocos hiperparámetros son realmente influyentes.
  • Bayesian Optimization (Optuna): Construye un modelo sustituto (surrogate) de $\mathcal{L}_{CV}$ y usa acquisition functions para decidir qué evaluar a continuación.

Confundirlos o aplicarlos sin criterio es lo que convierte un proyecto de ML en una factura de cómputo gigante con resultados mediocres.

📐 El Espacio de Búsqueda y la Maldición de la Dimensionalidad

Este es el corazón del problema. Mucha gente trata $\Theta$ como algo plano, pero tiene estructura.

$$|\Theta_{grid}| = \prod_{i=1}^{d} |G_i|$$

donde $d$ es el número de hiperparámetros y $|G_i|$ es el número de valores candidatos para el parámetro $i$.

"Esta ecuación parece inocente pero tiene TODO el poder para destruir tu presupuesto de cómputo. Vamos a desmenuzarla símbolo por símbolo porque acá es donde la mayoría se lleva el chasco."

Descomposición Completa: Entendiendo Cada Símbolo

$|\Theta_{grid}|$ (Cardinalidad del espacio) — Cuántas evaluaciones exactas necesitas

|Θ_grid| = número total de combinaciones a evaluar
         = el presupuesto mínimo de evaluaciones de Grid Search

Analogía: Es el número de mesas en un restaurante que tenés que visitar si querés probar todos los platos de todas las combinaciones posibles de menú.

En Machine Learning:

  • Para un RandomForestClassifier con 3 parámetros y 5 valores cada uno: $5^3=125$ evaluaciones.
  • Con 6 parámetros y 10 valores: $10^6=1{,}000{,}000$ evaluaciones. Buena suerte.

Diferencia CRÍTICA: Esta es la evaluación por fold. Con cv=5, multiplicá todo por 5.

# 3 hiperparámetros, 5 valores cada uno, 5-fold CV
d = 3
values_per_param = 5
cv_folds = 5
total_fits = (values_per_param ** d) * cv_folds
print(f"Fits totales: {total_fits}")  # 625

$d$ (Dimensión del espacio) — El enemigo silencioso

d = número de hiperparámetros que decidiste incluir
  = la fuente principal de la explosión combinatoria

Analogía: Cada hiperparámetro que agregás a tu grilla es como agregar un eje en un cubo. Pasás de un segmento (1D) a un cuadrado (2D) a un cubo (3D) y así. El volumen crece brutal.

En Machine Learning:

  • Para $d=2$: el espacio es una tabla (manejable).
  • Para $d=5$: un hipercubo de 5 dimensiones donde Random Search supera a Grid Search por márgenes significativos.

Dimensionalidad importa: El paper de Bergstra & Bengio (2012) demostró empíricamente que Random Search con el mismo presupuesto de evaluaciones supera a Grid Search cuando $d \geq 3$ y pocos parámetros son verdaderamente influyentes.

📐 La Intuición de Random Search: Por Qué Funciona Mejor de lo que Parece

$$P(\text{encontrar } \theta \text{ mejor que umbral } \tau) = 1 - (1-p)^n$$

donde $p$ es la fracción del espacio mejor que $\tau$ y $n$ es el número de trials.

"Esta es la probabilidad de que al menos uno de tus $n$ puntos aleatorios caiga en una región buena."

Por qué esto importa en la práctica

Con $p=0.05$ (queremos estar en el top 5% del espacio) y $n=60$ trials:

$$P = 1 - (1 - 0.05)^{60} = 1 - 0.95^{60} \approx 1 - 0.0462 \approx 0.954$$

Con solo 60 evaluaciones tenés 95% de probabilidad de encontrar una configuración en el top 5%. Grid Search necesitaría evaluar el espacio completo para garantizar lo mismo.

import numpy as np

def prob_finding_good_region(p: float, n_trials: int) -> float:
    """
    Probabilidad de que al menos un trial caiga en región top-p.

    Args:
        p: Fracción del espacio considerada 'buena'.
        n_trials: Número de evaluaciones aleatorias.

    Returns:
        Probabilidad en [0, 1].
    """
    return 1.0 - (1.0 - p) ** n_trials

for n in [10, 30, 60, 100]:
    prob = prob_finding_good_region(p=0.05, n_trials=n)
    print(f"n={n:3d} trials → P(top 5%) = {prob:.3f}")

📐 Bayesian Optimization: El Surrogate Model de Optuna

Optuna usa Tree-structured Parzen Estimators (TPE). La idea central es modelar:

$$p(\theta | y) = \begin{cases} \ell(\theta) & \text{si } y < y^* \\ g(\theta) & \text{si } y \geq y^* \end{cases}$$

donde $y^*$ es el cuantil que separa configuraciones "buenas" de "malas", $\ell(\theta)$ es la densidad en el grupo bueno y $g(\theta)$ en el malo. La acquisition function EI (Expected Improvement) busca:

$$\text{EI}(\theta) \propto \frac{\ell(\theta)}{g(\theta)}$$

Maximizar esto significa buscar regiones donde el candidato tiene alta probabilidad de ser bueno ($\ell$ alto) y baja probabilidad de ser malo ($g $ bajo). Esa vaina es elegante de verdad.

📐 Implementación Production-Ready: GridSearchCV + Optuna

A continuación el código que realmente usarías en un proyecto serio. Nada de notebooks de juguete.

from __future__ import annotations

import logging
from dataclasses import dataclass, field
from typing import Any

import numpy as np
import optuna
import pandas as pd
from sklearn.datasets import make_classification
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import GridSearchCV, StratifiedKFold, cross_val_score
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler

logger = logging.getLogger(__name__)

optuna.logging.set_verbosity(optuna.logging.WARNING)


@dataclass
class HyperparamSearchConfig:
    """Configuración centralizada para búsqueda de hiperparámetros.

    Attributes:
        n_cv_folds: Número de folds para cross-validation.
        scoring: Métrica de evaluación compatible con sklearn.
        random_state: Semilla para reproducibilidad.
        n_optuna_trials: Número de trials para Optuna.
        n_jobs: Paralelismo para GridSearchCV.
    """

    n_cv_folds: int = 5
    scoring: str = "roc_auc"
    random_state: int = 42
    n_optuna_trials: int = 50
    n_jobs: int = -1

    grid_param_space: dict[str, list[Any]] = field(
        default_factory=lambda: {
            "classifier__n_estimators": [50, 100, 200],
            "classifier__max_depth": [3, 5, None],
            "classifier__min_samples_split": [2, 5],
        }
    )


def build_pipeline(random_state: int) -> Pipeline:
    """Construye un pipeline sklearn estándar con preprocesamiento y clasificador.

    Args:
        random_state: Semilla para el clasificador.

    Returns:
        Pipeline con StandardScaler y RandomForestClassifier.
    """
    return Pipeline(
        [
            ("scaler", StandardScaler()),
            (
                "classifier",
                RandomForestClassifier(random_state=random_state, n_jobs=-1),
            ),
        ]
    )


def run_grid_search(
    X: np.ndarray,
    y: np.ndarray,
    config: HyperparamSearchConfig,
) -> tuple[Pipeline, pd.DataFrame]:
    """Ejecuta Grid Search con cross-validation estratificada.

    Args:
        X: Features de entrenamiento. Shape (n_samples, n_features).
        y: Target binario o multiclase. Shape (n_samples,).
        config: Configuración de búsqueda.

    Returns:
        Tupla (mejor_pipeline, dataframe_de_resultados).

    Raises:
        ValueError: Si X e y tienen longitudes incompatibles.
    """
    if len(X) != len(y):
        raise ValueError(
            f"X tiene {len(X)} muestras pero y tiene {len(y)}. Deben coincidir."
        )

    logger.info(
        "Iniciando Grid Search. Combinaciones totales: %d × %d folds = %d fits.",
        np.prod([len(v) for v in config.grid_param_space.values()]),
        config.n_cv_folds,
        np.prod([len(v) for v in config.grid_param_space.values()])
        * config.n_cv_folds,
    )

    cv = StratifiedKFold(
        n_splits=config.n_cv_folds,
        shuffle=True,
        random_state=config.random_state,
    )

    pipeline = build_pipeline(config.random_state)

    gs = GridSearchCV(
        estimator=pipeline,
        param_grid=config.grid_param_space,
        cv=cv,
        scoring=config.scoring,
        n_jobs=config.n_jobs,
        refit=True,
        return_train_score=True,
        verbose=0,
    )

    gs.fit(X, y)

    results_df = (
        pd.DataFrame(gs.cv_results_)
        .sort_values("rank_test_score")
        .reset_index(drop=True)
    )

    logger.info(
        "Grid Search completo. Mejor %s: %.4f con params: %s",
        config.scoring,
        gs.best_score_,
        gs.best_params_,
    )

    return gs.best_estimator_, results_df


def run_optuna_search(
    X: np.ndarray,
    y: np.ndarray,
    config: HyperparamSearchConfig,
) -> tuple[Pipeline, optuna.Study]:
    """Ejecuta búsqueda bayesiana con Optuna (TPE sampler).

    Args:
        X: Features de entrenamiento. Shape (n_samples, n_features).
        y: Target. Shape (n_samples,).
        config: Configuración de búsqueda.

    Returns:
        Tupla (mejor_pipeline, estudio_optuna).

    Raises:
        ValueError: Si X e y tienen longitudes incompatibles.
    """
    if len(X) != len(y):
        raise ValueError(
            f"X tiene {len(X)} muestras pero y tiene {len(y)}. Deben coincidir."
        )

    cv = StratifiedKFold(
        n_splits=config.n_cv_folds,
        shuffle=True,
        random_state=config.random_state,
    )

    def objective(trial: optuna.Trial) -> float:
        params = {
            "classifier__n_estimators": trial.suggest_int(
                "n_estimators", 50, 300, step=50
            ),
            "classifier__max_depth": trial.suggest_int("max_depth", 2, 15),
            "classifier__min_samples_split": trial.suggest_int(
                "min_samples_split", 2, 20
            ),
            "classifier__max_features": trial.suggest_categorical(
                "max_features", ["sqrt", "log2", 0.5]
            ),
        }
        pipeline = build_pipeline(config.random_state)
        pipeline.set_params(**params)

        scores = cross_val_score(
            pipeline, X, y, cv=cv, scoring=config.scoring, n_jobs=config.n_jobs
        )
        return float(scores.mean())

    study = optuna.create_study(
        direction="maximize",
        sampler=optuna.samplers.TPESampler(seed=config.random_state),
        pruner=optuna.pruners.MedianPruner(n_startup_trials=10),
    )

    logger.info("Iniciando Optuna con %d trials.", config.n_optuna_trials)

    study.optimize(objective, n_trials=config.n_optuna_trials, show_progress_bar=False)

    best_params = {
        f"classifier__{k}": v for k, v in study.best_params.items()
    }
    best_pipeline = build_pipeline(config.random_state)
    best_pipeline.set_params(**best_params)
    best_pipeline.fit(X, y)

    logger.info(
        "Optuna completo. Mejor %s: %.4f con params: %s",
        config.scoring,
        study.best_value,
        study.best_params,
    )

    return best_pipeline, study


if __name__ == "__main__":
    logging.basicConfig(level=logging.INFO, format="%(levelname)s | %(message)s")

    X, y = make_classification(
        n_samples=1_000,
        n_features=20,
        n_informative=10,
        random_state=42,
    )

    config = HyperparamSearchConfig(n_optuna_trials=40)

    best_gs_model, gs_results = run_grid_search(X, y, config)
    logger.info(
        "Top 3 Grid Search:\n%s",
        gs_results[["params", "mean_test_score"]].head(3).to_string(),
    )

    best_optuna_model, study = run_optuna_search(X, y, config)
    logger.info("Mejor trial Optuna: %s", study.best_trial.params)

🚨 Errores Comunes (y Cómo No Cagarla)

Error 1: Hacer fit del scaler ANTES del split en el tuning

El problema:

from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import GridSearchCV, train_test_split
from sklearn.ensemble import RandomForestClassifier

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)  # ❌ Leak: el scaler vio TODO X, incluye validación

gs = GridSearchCV(RandomForestClassifier(), param_grid, cv=5)
gs.fit(X_scaled, y)  # ❌ Los folds de validación ya están contaminados

Por qué falla:

El scaler calcula $\mu$ y $\sigma$ sobre todo $X$. Cuando CV divide los datos, los folds de validación ya tienen información del fold de entrenamiento embebida en la normalización. Esto infla artificialmente las métricas.

$$\hat{\mu}_{leaky} = \frac{1}{n} \sum_{i=1}^{n} x_i \quad \text{(incluye validación)}$$

$$\hat{\mu}_{correcto} = \frac{1}{n_{train}} \sum_{i \in train} x_i \quad \text{(solo entrenamiento)}$$

import numpy as np
np.random.seed(42)
X_toy = np.array([[1.0], [2.0], [3.0], [100.0]])  # 100.0 es "futuro"/validación
mu_leaky   = X_toy.mean()       # 26.5
mu_correct = X_toy[:3].mean()   # 2.0
print(f"mu_leaky={mu_leaky:.1f} vs mu_correct={mu_correct:.1f}")

La solución:

from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import GridSearchCV, StratifiedKFold

pipeline = Pipeline([            # ✓ El scaler es parte del pipeline
    ("scaler", StandardScaler()),
    ("clf", RandomForestClassifier()),
])

param_grid = {
    "clf__n_estimators": [50, 100, 200],
    "clf__max_depth": [3, 5, None],
}

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
gs = GridSearchCV(pipeline, param_grid, cv=cv, scoring="roc_auc", refit=True)
gs.fit(X, y)  # ✓ Scaler se fitea SOLO en el fold de train, en cada fold

Error 2: Evaluar con los mismos datos del tuning

El problema:

gs = GridSearchCV(pipeline, param_grid, cv=5)
gs.fit(X, y)                       # ❌ Búsqueda sobre TODO el dataset
print(f"Score: {gs.best_score_}")  # ❌ Sesgado optimistamente

Por qué falla:

best_score_ en GridSearchCV es el promedio CV de la mejor configuración. Si usaste todo el dataset para tunear, no tenés una estimación honesta del rendimiento en datos nuevos. El proceso de selección de la mejor configuración es en sí mismo una fuente de sobreajuste al conjunto de validación.

$$\mathbb{E}[\text{score}_{best}] \geq \mathbb{E}[\text{score}_{true}]$$

La desigualdad se vuelve más pronunciada cuanto más grande es el espacio de búsqueda.

La solución:

from sklearn.model_selection import train_test_split, GridSearchCV, StratifiedKFold

X_train, X_test, y_train, y_test = train_test_split(  # ✓ Hold-out externo
    X, y, test_size=0.2, stratify=y, random_state=42
)

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
gs = GridSearchCV(pipeline, param_grid, cv=cv, scoring="roc_auc", refit=True)
gs.fit(X_train, y_train)              # ✓ Tuning solo en train

final_score = gs.score(X_test, y_test)  # ✓ Evaluación en datos nunca vistos
print(f"Score holdout: {final_score:.4f}")

Error 3: Usar Grid Search con espacios continuos grandes

El problema:

param_grid = {
    "clf__C":      [0.001, 0.01, 0.1, 1, 10, 100, 1000],  # ❌ Espacio denso
    "clf__gamma":  [0.0001, 0.001, 0.01, 0.1, 1, 10],     # ❌
    "clf__kernel": ["rbf", "poly", "sigmoid"],              # ❌
    "clf__degree": [2, 3, 4, 5],                           # ❌
}
# 7 × 6 × 3 × 4 = 504 combinaciones × 5 folds = 2,520 fits
# Con SVM en dataset mediano: potencialmente horas

Por qué falla:

Grid Search en espacios continuos grandes es ineficiente porque muestrea uniformemente, ignorando que la función de pérdida rara vez es plana. La mayoría de las regiones del espacio no valen la pena explorar y estás pagando cómputo para confirmar lo que ya sabías.

La solución:

import optuna
from sklearn.svm import SVC
from sklearn.model_selection import cross_val_score, StratifiedKFold

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

def objective(trial: optuna.Trial) -> float:           # ✓ Búsqueda adaptativa
    kernel = trial.suggest_categorical("kernel", ["rbf", "poly"])
    C      = trial.suggest_float("C", 1e-3, 1e3, log=True)  # ✓ log-scale
    gamma  = trial.suggest_float("gamma", 1e-4, 1e1, log=True)
    degree = trial.suggest_int("degree", 2, 5) if kernel == "poly" else 3

    clf = Pipeline([
        ("scaler", StandardScaler()),
        ("svm", SVC(C=C, kernel=kernel, gamma=gamma, degree=degree)),
    ])
    scores = cross_val_score(clf, X_train, y_train, cv=cv, scoring="roc_auc")
    return scores.mean()

study = optuna.create_study(
    direction="maximize",
    sampler=optuna.samplers.TPESampler(seed=42),
)
study.optimize(objective, n_trials=60)  # ✓ 60 trials adaptativos vs 2,520 exhaustivos

Error 4: Ignorar refit=True y su implicación en el modelo final

El problema:

gs = GridSearchCV(pipeline, param_grid, cv=5, refit=False)  # ❌ refit=False
gs.fit(X_train, y_train)
gs.predict(X_test)  # ❌ AttributeError: no best_estimator_

Por qué falla:

Con refit=False, GridSearchCV identifica los mejores hiperparámetros pero NO re-entrena el modelo final sobre todo X_train. El modelo en producción debe estar entrenado con toda la data de entrenamiento, no solo con $\frac{k-1}{k}$ fracciones de ella.

La solución:

gs = GridSearchCV(
    pipeline, param_grid, cv=5,
    scoring="roc_auc",
    refit=True,             # ✓ Re-entrena sobre TODO X_train con los mejores params
    return_train_score=True,  # ✓ Útil para detectar overfitting en tuning
)
gs.fit(X_train, y_train)

production_model = gs.best_estimator_  # ✓ Listo para deployment
y_pred = production_model.predict(X_test)

🎯 Resumen Ejecutivo

Grid/Random Search no es "probar parámetros hasta que algo funcione". Es resolver:

$$\theta^* = \arg\min_{\theta \in \Theta} \mathcal{L}_{CV}(f_\theta,\, \mathcal{D}_{train})$$

Mediante una estrategia apropiada al tamaño y estructura de $\Theta$:

$$|\Theta_{grid}| = \prod_{i=1}^{d}|G_i| \quad \text{vs} \quad n_{random} \ll |\Theta_{grid}| \quad \text{vs} \quad \text{EI}(\theta) \propto \frac{\ell(\theta)}{g(\theta)}$$

Con todas las implicaciones que eso conlleva:

  • Siempre usar Pipeline: El preprocesamiento dentro del pipeline es la diferencia entre métricas reales y métricas mentirosas por data leakage.
  • Hold-out externo obligatorio: El best_score_ de GridSearchCV es optimista. Necesitás un test set que nunca tocó el proceso de tuning para estimar rendimiento real.
  • Grid Search tiene un techo: Más de 3 hiperparámetros continuos o grillas densas → Random Search o Optuna. La matemática de probabilidad lo respalda.

De una, hágale. Con Grid Search bien aplicado para espacios pequeños, Random Search para dimensiones medias y Optuna para búsquedas serias con presupuesto limitado, tenés la triada completa para tunear modelos en producción sin quemar ni cómputo ni credibilidad.
