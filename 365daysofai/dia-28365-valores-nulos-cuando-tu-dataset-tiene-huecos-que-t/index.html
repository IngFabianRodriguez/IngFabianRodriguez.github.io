Día 28: Valores Nulos - Cuando tu Dataset Tiene Huecos que Te Dañan el Modelo

Parcero, hablemos de algo que todos hemos sufrido: abres tu CSV, corres tu modelo y… ¡pum! ValueError: Input contains NaN. Esa vaina te destroza el día. Los valores nulos son como ese compañero que nunca responde el Slack: no sabés si está ocupado, si renunció o si simplemente te está ignorando. Y ese "no saber" puede hacer que tu modelo se vuelva loco.

Hoy vamos a resolver esto con pandas.fillna() y SimpleImputer de sklearn, pero no con tutoriales de juguete. Vamos con código production-ready que no te deje tirado cuando lleguen 10 millones de filas.

🔍 El Problema Real: Por Qué los Nulos Te Explotan Todo

Analogía de Oficina

Imaginá que estás haciendo un reporte de salarios del equipo. Tres personas no pusieron su sueldo en el Excel. ¿Qué hacés?

  • Ignorarlos: Perdés información valiosa (y posiblemente sesgas el promedio).
  • Poner ceros: Tu promedio salarial se va al piso y RH se enloquece.
  • Usar el promedio: Razonable, pero si uno de los que no respondió es el VP, metiste la pata.

En ML es lo mismo. Los valores nulos pueden venir por:

  • Fallas en sensores (IoT, mediciones médicas)
  • Usuarios que no llenan campos (formularios web)
  • Errores de extracción (ETL que se cayó a medias)

Y cada caso necesita una estrategia diferente.

📐 Fundamentos Matemáticos: Estrategias de Imputación

1. Imputación por Media

La estrategia más básica: reemplazás los nulos con el promedio de la columna.

Fórmula:

$$\bar{x}_{\text{imputed}} = \frac{1}{n} \sum_{i=1}^{n} x_i \quad \text{donde } x_i \neq \text{NaN}$$

Explicación paso a paso:

  • $\bar{x}_{\text{imputed}}$: El valor que vamos a usar para llenar los nulos
  • $n$: Cantidad de valores no nulos en la columna
  • $\sum_{i=1}^{n} x_i$: Suma de todos los valores válidos
  • Dividimos la suma entre $n$ para obtener el promedio

Ventaja: Mantiene la media de la distribución.

Desventaja: Reduce la varianza (todos los nulos quedan con el mismo valor).

2. Imputación por Mediana

Mejor para distribuciones con outliers (salarios, precios de vivienda).

Fórmula:

$$\text{mediana}(X) = \begin{cases} x_{\frac{n+1}{2}} & \text{si } n \text{ es impar} \\\\ \frac{x_{\frac{n}{2}} + x_{\frac{n}{2}+1}}{2} & \text{si } n \text{ es par} \end{cases}$$

Explicación paso a paso:

  • Ordenamos todos los valores válidos de menor a mayor
  • Si hay cantidad impar de valores, tomamos el del medio
  • Si hay cantidad par, promediamos los dos centrales
  • $x_{\frac{n}{2}}$: El valor en la posición central (después de ordenar)

Ventaja: Robusta ante outliers (si un CEO gana 50 veces más, no sesga el resultado).

Desventaja: No captura la forma real de la distribución.

3. Imputación por Moda (Categorías)

Para variables categóricas (ej: ciudad, tipo de producto).

Fórmula:

$$\text{moda}(X) = \arg\max_{x_i} \left( \text{count}(x_i) \right)$$

Explicación paso a paso:

  • $\text{count}(x_i)$: Cuántas veces aparece cada categoría
  • $\arg\max$: Nos da la categoría que aparece más veces
  • Es como preguntar: "¿cuál es la respuesta más común?"

Ejemplo: Si 60% de tus usuarios son de Bogotá, los nulos en "ciudad" se llenan con "Bogotá".

4. Forward Fill / Backward Fill

Para series temporales (stock prices, temperatura).

Forward Fill:

$$x_t = \begin{cases} x_t & \text{si } x_t \neq \text{NaN} \\\\ x_{t-1} & \text{si } x_t = \text{NaN} \end{cases}$$

Explicación paso a paso:

  • $x_t$: Valor en el tiempo $t$
  • Si el valor actual es nulo, tomamos el valor anterior ($x_{t-1}$)
  • Es como decir: "si no hay nueva info, asumo que sigue igual"

Ventaja: Respeta la secuencia temporal.

Desventaja: Propaga el último valor conocido (puede no ser realista).

class ImputationError(Exception):
    """Excepción custom para errores en imputación.
    
    Por qué esto existe: En producción necesitás diferenciar entre
    errores de datos (que podés manejar) y errores de código (que son bugs).
    """
    pass


@dataclass
class ImputationReport:
    """Reporte detallado de la imputación realizada.
    
    Esto te salva en auditorías: podés mostrar exactamente
    qué hiciste con los datos faltantes.
    """
    __slots__ = ('total_nulls', 'columns_affected', 'strategy_used', 
                 'fill_values', 'null_percentage')
    
    total_nulls: int
    columns_affected: list[str]
    strategy_used: str
    fill_values: dict[str, Union[float, str]]
    null_percentage: float
    
    def __str__(self) -> str:
        return (
            f"Imputación completada:\n"
            f"  - Nulos encontrados: {self.total_nulls} ({self.null_percentage:.2f}%)\n"
            f"  - Columnas afectadas: {', '.join(self.columns_affected)}\n"
            f"  - Estrategia: {self.strategy_used}"
        )

Por qué esto te evita dolores de cabeza:

  • En producción, cuando algo falla a las 3 AM, el log te dice exactamente qué pasó
  • El @dataclass con __slots__ optimiza memoria (crítico con millones de filas)
  • El reporte te permite hacer trazabilidad completa

Paso 2: Imputador Básico con Pandas

from typing import Literal

StrategyType = Literal['mean', 'median', 'mode', 'constant', 'ffill', 'bfill']


class PandasImputer:
    """Wrapper robusto sobre pandas.fillna() con validaciones.
    
    Decisión de arquitectura: Separamos la lógica de pandas de sklearn
    porque pandas es más rápido para operaciones in-place en DataFrames.
    """
    
    def __init__(
        self, 
        strategy: StrategyType = 'mean',
        fill_value: Optional[Union[float, str]] = None
    ) -> None:
        """
        Args:
            strategy: Método de imputación
            fill_value: Valor específico si strategy='constant'
        
        Raises:
            ValueError: Si la combinación de parámetros es inválida
        """
        self.strategy = strategy
        self.fill_value = fill_value
        self._fill_values: dict[str, Union[float, str]] = {}
        
        # Validación temprana (fail fast)
        if strategy == 'constant' and fill_value is None:
            raise ValueError(
                "Parcero, si usás strategy='constant', "
                "tenés que pasar un fill_value"
            )
        
        logger.info(f"Imputer inicializado con estrategia: {strategy}")
    
    def fit(self, df: pd.DataFrame) -> 'PandasImputer':
        """Calcula los valores de relleno basándose en datos de entrenamiento.
        
        Por qué fit() separado: En producción, calculás las estadísticas
        en train y las aplicás en test/producción sin recalcular
        (para evitar data leakage).
        """
        if not isinstance(df, pd.DataFrame):
            raise ImputationError(
                f"Esperaba un DataFrame, recibí {type(df).__name__}"
            )
        
        numeric_cols = df.select_dtypes(include=[np.number]).columns
        
        if len(numeric_cols) == 0:
            logger.warning("No se encontraron columnas numéricas para imputar")
            return self
        
        # Calculamos valores según estrategia
        if self.strategy == 'mean':
            self._fill_values = df[numeric_cols].mean().to_dict()
        elif self.strategy == 'median':
            self._fill_values = df[numeric_cols].median().to_dict()
        elif self.strategy == 'mode':
            # Moda puede retornar múltiples valores, tomamos el primero
            self._fill_values = {
                col: df[col].mode()[0] if not df[col].mode().empty else 0
                for col in numeric_cols
            }
        elif self.strategy == 'constant':
            self._fill_values = {col: self.fill_value for col in numeric_cols}
        
        logger.info(f"Fit completado. Valores calculados: {self._fill_values}")
        return self
    
    def transform(self, df: pd.DataFrame) -> tuple[pd.DataFrame, ImputationReport]:
        """Aplica la imputación y retorna reporte detallado.
        
        Returns:
            Tupla con (DataFrame imputado, Reporte de cambios)
        
        Raises:
            ImputationError: Si no se llamó fit() primero
        """
        if not self._fill_values and self.strategy not in ['ffill', 'bfill']:
            raise ImputationError(
                "Tenés que llamar fit() antes de transform(). "
                "Esto no es un tutorial, es código de producción."
            )
        
        df_copy = df.copy()
        total_nulls_before = df_copy.isnull().sum().sum()
        null_cols = df_copy.columns[df_copy.isnull().any()].tolist()
        
        # Aplicamos estrategia según el caso
        if self.strategy in ['ffill', 'bfill']:
            df_copy.fillna(method=self.strategy, inplace=True)
        else:
            df_copy.fillna(value=self._fill_values, inplace=True)
        
        # Generamos reporte
        null_percentage = (total_nulls_before / df_copy.size) * 100
        report = ImputationReport(
            total_nulls=int(total_nulls_before),
            columns_affected=null_cols,
            strategy_used=self.strategy,
            fill_values=self._fill_values,
            null_percentage=float(null_percentage)
        )
        
        logger.info(str(report))
        return df_copy, report
    
    def fit_transform(
        self, 
        df: pd.DataFrame
    ) -> tuple[pd.DataFrame, ImputationReport]:
        """Atajo para fit + transform (útil en notebooks, no en prod)."""
        return self.fit(df).transform(df)

Alivio del Dolor:

  • El fit() separado te protege de data leakage (no recalculás estadísticas en test)
  • El reporte automático te permite auditar cada cambio
  • Las excepciones custom te dejan debuggear rápido (no un genérico ValueError)

Paso 3: Imputador Avanzado con scikit-learn

Para estrategias más sofisticadas (KNN, iterative imputation).

from sklearn.impute import SimpleImputer, KNNImputer
from sklearn.base import BaseEstimator, TransformerMixin


class AdvancedImputer(BaseEstimator, TransformerMixin):
    """Imputador compatible con pipelines de sklearn.
    
    Heredar de BaseEstimator + TransformerMixin nos da:
    - get_params() / set_params() automáticos (para GridSearch)
    - Compatibilidad con Pipeline de sklearn
    """
    
    def __init__(
        self,
        strategy: Literal['mean', 'median', 'most_frequent', 'constant', 'knn'] = 'mean',
        fill_value: Optional[Union[float, str]] = None,
        n_neighbors: int = 5
    ) -> None:
        """
        Args:
            strategy: Estrategia de imputación
            fill_value: Valor para strategy='constant'
            n_neighbors: Vecinos para strategy='knn'
        """
        self.strategy = strategy
        self.fill_value = fill_value
        self.n_neighbors = n_neighbors
        self._imputer: Optional[Union[SimpleImputer, KNNImputer]] = None
    
    def fit(self, X: np.ndarray, y: Optional[np.ndarray] = None) -> 'AdvancedImputer':
        """Entrena el imputador.
        
        Args:
            X: Matriz de features (puede venir de un DataFrame)
            y: Target (no lo usamos, pero sklearn lo requiere para compatibilidad)
        
        Returns:
            self (para poder encadenar llamadas)
        """
        if self.strategy == 'knn':
            self._imputer = KNNImputer(n_neighbors=self.n_neighbors)
            logger.info(f"Usando KNN con {self.n_neighbors} vecinos")
        else:
            # SimpleImputer maneja mean, median, most_frequent, constant
            self._imputer = SimpleImputer(
                strategy=self.strategy,
                fill_value=self.fill_value
            )
            logger.info(f"Usando SimpleImputer con estrategia: {self.strategy}")
        
        self._imputer.fit(X)
        return self
    
    def transform(self, X: np.ndarray) -> np.ndarray:
        """Aplica la imputación a nuevos datos.
        
        Raises:
            NotFittedError: Si llamás esto antes de fit()
        """
        if self._imputer is None:
            raise NotFittedError(
                "Este imputer no está entrenado. Llamá fit() primero."
            )
        
        return self._imputer.transform(X)
    
    def get_feature_names_out(self, input_features: Optional[list[str]] = None) -> np.ndarray:
        """Requerido para compatibilidad con Pipeline de sklearn >= 1.0.
        
        Esto te salva cuando usás ColumnTransformer y necesitás
        rastrear nombres de columnas después de la transformación.
        """
        if input_features is None:
            return np.array([f"x{i}" for i in range(self._imputer.n_features_in_)])
        return np.array(input_features)

Por qué heredar de BaseEstimator:

  • Te da get_params() gratis → funciona con GridSearchCV
  • Compatible con Pipeline → podés encadenar transformaciones
  • TransformerMixin te da fit_transform() automático

Paso 4: Pipeline Completo de Imputación

Ahora armamos un flujo end-to-end que maneje categóricas y numéricas por separado.

from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from typing import Any


class ProductionImputationPipeline:
    """Pipeline enterprise-grade para imputación mixta.
    
    Decisión de arquitectura: Separamos columnas numéricas y categóricas
    porque cada una necesita estrategias diferentes. Las categóricas
    no se pueden promediar, obvio.
    """
    
    def __init__(
        self,
        numeric_strategy: Literal['mean', 'median', 'knn'] = 'median',
        categorical_strategy: Literal['most_frequent', 'constant'] = 'most_frequent',
        n_neighbors: int = 5
    ) -> None:
        self.numeric_strategy = numeric_strategy
        self.categorical_strategy = categorical_strategy
        self.n_neighbors = n_neighbors
        self._pipeline: Optional[ColumnTransformer] = None
        self._numeric_cols: list[str] = []
        self._categorical_cols: list[str] = []
    
    def fit(self, df: pd.DataFrame) -> 'ProductionImputationPipeline':
        """Detecta tipos de columnas y construye el pipeline.
        
        Por qué hacemos esto: No sabemos de antemano qué columnas
        son numéricas o categóricas. El pipeline se adapta al input.
        """
        # Detección automática de tipos
        self._numeric_cols = df.select_dtypes(
            include=[np.number]
        ).columns.tolist()
        
        self._categorical_cols = df.select_dtypes(
            include=['object', 'category']
        ).columns.tolist()
        
        if not self._numeric_cols and not self._categorical_cols:
            raise ImputationError(
                "El DataFrame no tiene columnas numéricas ni categóricas. "
                "¿Me pasaste un DataFrame vacío?"
            )
        
        # Pipeline para columnas numéricas
        numeric_transformer = Pipeline(steps=[
            ('imputer', AdvancedImputer(
                strategy=self.numeric_strategy,
                n_neighbors=self.n_neighbors
            )),
            ('scaler', StandardScaler())  # Bonus: escalado automático
        ])
        
        # Pipeline para columnas categóricas
        categorical_transformer = Pipeline(steps=[
            ('imputer', SimpleImputer(strategy=self.categorical_strategy))
        ])
        
        # ColumnTransformer aplica cada pipeline a sus columnas
        transformers = []
        if self._numeric_cols:
            transformers.append(('num', numeric_transformer, self._numeric_cols))
        if self._categorical_cols:
            transformers.append(('cat', categorical_transformer, self._categorical_cols))
        
        self._pipeline = ColumnTransformer(
            transformers=transformers,
            remainder='passthrough'  # Columnas no especificadas pasan sin cambios
        )
        
        self._pipeline.fit(df)
        logger.info(
            f"Pipeline entrenado:\n"
            f"  Numéricas ({len(self._numeric_cols)}): {self._numeric_cols}\n"
            f"  Categóricas ({len(self._categorical_cols)}): {self._categorical_cols}"
        )
        
        return self
    
    def transform(self, df: pd.DataFrame) -> pd.DataFrame:
        """Transforma el DataFrame aplicando imputación + escalado."""
        if self._pipeline is None:
            raise NotFittedError("Llamá fit() antes de transform()")
        
        # Transformamos y reconstruimos el DataFrame
        transformed = self._pipeline.transform(df)
        
        # Recuperamos nombres de columnas (ColumnTransformer los cambia)
        output_cols = self._numeric_cols + self._categorical_cols
        result = pd.DataFrame(
            transformed,
            columns=output_cols,
            index=df.index
        )
        
        logger.info(f"Transformación completada. Shape: {result.shape}")
        return result
    
    def fit_transform(self, df: pd.DataFrame) -> pd.DataFrame:
        """Atajo para fit + transform."""
        return self.fit(df).transform(df)


# === FUNCIÓN DE UTILIDAD PARA ANÁLISIS RÁPIDO ===

def analyze_missing_data(df: pd.DataFrame) -> pd.DataFrame:
    """Genera reporte estadístico de valores nulos.
    
    Returns:
        DataFrame con columnas: [Column, Missing_Count, Missing_Percentage, Dtype]
    
    Uso típico:
        >>> report = analyze_missing_data(df)
        >>> print(report[report['Missing_Percentage'] > 5.0])
    """
    missing_stats = pd.DataFrame({
        'Column': df.columns,
        'Missing_Count': df.isnull().sum().values,
        'Missing_Percentage': (df.isnull().sum() / len(df) * 100).values,
        'Dtype': df.dtypes.values
    })
    
    # Ordenamos por porcentaje de nulos (mayor a menor)
    missing_stats = missing_stats.sort_values(
        'Missing_Percentage', 
        ascending=False
    )
    
    return missing_stats[missing_stats['Missing_Count'] > 0]

Alivio del Dolor:

  • El pipeline detecta automáticamente qué columnas son numéricas/categóricas
  • Aplicás diferentes estrategias a cada tipo sin código repetido
  • Compatible con GridSearchCV → podés tunear hiperparámetros
  • El remainder='passthrough' asegura que no perdés columnas por accidente

# === SIMULACIÓN DE DATOS REALES ===

def create_medical_dataset() -> pd.DataFrame:
    """Genera dataset sintético con patrones realistas de nulos.
    
    Simula datos médicos donde:
    - Presión arterial falta en pacientes jóvenes (no se mide siempre)
    - Tipo de sangre falta por errores administrativos (random)
    - Edad nunca falta (campo obligatorio)
    """
    np.random.seed(42)
    n_samples = 1000
    
    data = {
        'patient_id': range(1, n_samples + 1),
        'age': np.random.randint(18, 90, n_samples),
        'blood_pressure': np.random.normal(120, 15, n_samples),
        'glucose': np.random.normal(100, 20, n_samples),
        'blood_type': np.random.choice(['A+', 'A-', 'B+', 'B-', 'O+', 'O-'], n_samples),
        'city': np.random.choice(['Bogotá', 'Medellín', 'Cali', 'Barranquilla'], n_samples)
    }
    
    df = pd.DataFrame(data)
    
    # Introducimos nulos con patrones realistas
    # Presión arterial: 20% de nulos, más común en jóvenes
    young_mask = df['age'] < 30
    bp_missing = np.random.choice(
        df[young_mask].index,
        size=int(0.3 * len(df[young_mask])),
        replace=False
    )
    df.loc[bp_missing, 'blood_pressure'] = np.nan
    
    # Glucosa: 10% de nulos aleatorios
    glucose_missing = np.random.choice(
        df.index,
        size=int(0.1 * len(df)),
        replace=False
    )
    df.loc[glucose_missing, 'glucose'] = np.nan
    
    # Tipo de sangre: 15% de nulos (errores admin)
    blood_type_missing = np.random.choice(
        df.index,
        size=int(0.15 * len(df)),
        replace=False
    )
    df.loc[blood_type_missing, 'blood_type'] = np.nan
    
    return df

🎯 Lecciones de Producción: Qué Me Salvó el Pellejo

1. Nunca Imputes en el Dataset Completo (Data Leakage)

# ❌ MAL - Calculás estadísticas con train + test
imputer = SimpleImputer(strategy='mean')
df_all_imputed = imputer.fit_transform(df_all)

# ✅ BIEN - Fit solo en train, transform en test
imputer = SimpleImputer(strategy='mean')
imputer.fit(X_train)
X_train_imputed = imputer.transform(X_train)
X_test_imputed = imputer.transform(X_test)  # Usa medias de TRAIN

Por qué: Si calculás la media con train + test, tu modelo está "viendo" información del test durante el entrenamiento. En producción, no tenés datos futuros para calcular medias.

2. Documenta Por Qué Elegiste Cada Estrategia

3. Monitoreá Cuántos Nulos Estás Rellenando

def monitor_imputation_impact(df_before: pd.DataFrame, df_after: pd.DataFrame) -> dict:
    """Métrica para alertar si la imputación está rara."""
    metrics = {}
    for col in df_before.columns:
        nulls_before = df_before[col].isnull().sum()
        nulls_after = df_after[col].isnull().sum()
        
        if nulls_before > 0:
            imputed_ratio = (nulls_before - nulls_after) / len(df_before)
            metrics[col] = {
                'nulls_imputed': int(nulls_before - nulls_after),
                'imputation_ratio': float(imputed_ratio)
            }
            
            # Alerta si más del 40% de una columna era nula
            if imputed_ratio > 0.4:
                logger.warning(
                    f"⚠️ Columna '{col}' tenía {imputed_ratio*100:.1f}% de nulos. "
                    f"Considerá si es confiable usar esta feature."
                )
    
    return metrics

Si una columna tiene 80% de nulos, tal vez es mejor descartarla que inventar valores.

🔥 Caso de Guerra: Cuando KNN Te Salva (o Te Hunde)

Escenario: Dataset de IoT con temperatura y humedad. Ambos están correlacionados (cuando sube temperatura, baja humedad).

# Si usás mean en cada columna por separado:
# - Temperatura nula → rellenas con promedio de temperatura
# - Humedad nula → rellenas con promedio de humedad
# ❌ Problema: Perdiste la correlación. Ahora tenés temperatura alta con humedad alta (imposible).

# Con KNN:
# - Buscas las 5 filas más parecidas (por timestamp, ubicación, etc.)
# - Imputás basándote en sus valores
# ✅ Mantenés la correlación natural entre variables

Código:

from sklearn.impute import KNNImputer

# Datos con correlación temperatura-humedad
imputer = KNNImputer(n_neighbors=5, weights='distance')
# weights='distance' da más peso a vecinos cercanos

df_imputed = imputer.fit_transform(df[['temperature', 'humidity']])

Advertencia: KNN es lento con datasets grandes. Con +1M de filas, considerá IterativeImputer o MICE.

📊 Comparación de Estrategias: Cuándo Usar Cada Una

Estrategia Cuándo Usarla Ventaja Desventaja Complejidad
Mean Distribución normal, pocos outliers Rápida, mantiene media Reduce varianza $O(n)$
Median Outliers presentes (salarios, precios) Robusta ante extremos No captura forma real $O(n \log n)$
Mode Variables categóricas Simple, interpretable Pierde diversidad $O(n)$
Forward/Backward Fill Series temporales Respeta secuencia Propaga errores $O(n)$
Constant Nulos tienen significado especial Control total Puede introducir bias $O(1)$
KNN Features correlacionadas Captura relaciones Lenta, sensible a escala $O(n^2)$

🚀 Conclusión: De Nulos a Modelos Confiables

Parcero, los valores nulos no son un "detalle técnico" que se resuelve con un df.fillna(0) y ya. Son una decisión de modelado que puede hacer que tu modelo sea brillante o una porquería.

Recordá:

  • Analiza primero: ¿Por qué hay nulos? ¿Son al azar o tienen patrón?
  • Fit/Transform separado: Evitá data leakage como si fuera un bug en producción a las 3 AM
  • Documenta tus decisiones: Tu yo del futuro (o tu reemplazo) te lo agradecerá
  • Monitoreá el impacto: Si estás rellenando 50% del dataset, algo anda mal
