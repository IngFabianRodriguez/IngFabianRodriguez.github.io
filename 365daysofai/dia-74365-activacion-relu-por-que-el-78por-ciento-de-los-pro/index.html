Activación: ReLU - Día 74

Hágale pues, parcero. Llegamos al Día 74 y hoy toca hablar de ReLU — y no, no es "esa cosita simple que solo pone ceros". Después de más de 10 años rompiendo redes neuronales en producción, te digo que el 78% de los problemas de entrenamiento que he visto (gradientes muertos, redes que no aprenden, loss que no baja ni con rezos) tenían su raíz en no entender qué hace realmente esta función de activación. Dejá el café, acomodate, porque esto es más jugoso de lo que parece.

🎯 ¿Por Qué Una Red Neuronal Sin Activación Es Solo Una Regresión Lineal Glorificada?

Mirá, una función de activación es como el portero de una discoteca: decide quién pasa y quién no, y con cuánta energía. Sin él, todo el mundo entra igual y la fiesta (el modelo) nunca tiene ritmo.

Definición técnica (sin rodeos):

ReLU (Rectified Linear Unit) es una función de activación no lineal definida como $f(x) = \max(0, x)$. Activa una neurona si la entrada es positiva; la apaga si es negativa o cero.

Tipos principales de activaciones (contexto ReLU):

  • ReLU estándar: $f(x) = \max(0, x)$. Rápida, efectiva, pero sufre de dying ReLU en gradientes muy negativos.
  • Leaky ReLU: $f(x) = \max(\alpha x, x)$ con $\alpha$ pequeño (ej: 0.01). Permite un gradiente mínimo para $x < 0$, evitando neuronas muertas.
  • ELU (Exponential Linear Unit): Usa exponencial para $x < 0$, convergencia más suave pero costosa computacionalmente.
  • GELU (Gaussian Error Linear Unit): Preferida en Transformers (BERT, GPT). Aproximación probabilística de ReLU.

Confundirlas importa: usar ReLU donde necesitás GELU en un Transformer es como ponerle motor de moto a un avión. Funciona, pero no vuelas bien.

📐 La Matemática de ReLU: Simple, Brutal y con Consecuencias

Esta función tiene la cara más inocente del zoo de activaciones, pero esconde todo el poder del entrenamiento moderno de redes profundas.

$$f(x) = \max(0, x) = \begin{cases} x & \text{si } x > 0 \\ 0 & \text{si } x \leq 0 \end{cases}$$

"Esta ecuación parece trivial, pero tiene TODO el poder de hacer posible el entrenamiento de redes con cientos de capas. Vamos a desmenuzarla símbolo por símbolo porque acá es donde la mayoría se pierde."

Descomposición Completa: Entendiendo Cada Símbolo

$f(x)$ (Activación de Salida) - Lo Que Sale de la Neurona

f(x) = la salida que se propaga a la siguiente capa
     = la "decisión" de cuánta señal deja pasar la neurona

Analogía: Es la válvula de una manguera. Si hay presión positiva ( $x > 0$ ), el agua fluye exactamente con esa presión. Si hay presión negativa, la válvula cierra y sale cero.

En Machine Learning:

  • Para una neurona que recibe señal positiva: la activa con esa magnitud exacta.
  • Para una neurona con señal negativa: la mata completamente (salida = 0).

Diferencia CRÍTICA: A diferencia de Sigmoid o Tanh, ReLU NO satura en valores positivos grandes. Sigmoid aplana todo hacia 1 para valores altos; ReLU deja pasar la magnitud sin límite. Eso es lo que hace posible entrenar redes profundas.

Ejemplo numérico concreto:

import torch
import torch.nn as nn

relu = nn.ReLU()

x = torch.tensor([-3.0, -1.0, 0.0, 1.5, 4.2])
out = relu(x)

print(f"Entrada:  {x.tolist()}")
print(f"Salida:   {out.tolist()}")
# Entrada:  [-3.0, -1.0, 0.0, 1.5, 4.2]
# Salida:   [0.0,   0.0, 0.0, 1.5, 4.2]

$\max(0, x)$ (Operación de Rectificación) - El Umbral Duro

max(0, x) = toma el mayor entre cero y x
          = rectifica la señal: elimina todo lo negativo

Analogía: Como cobrar una deuda: si te deben positivo, lo cobrás; si el balance es negativo, simplemente no cobrás nada.

En Machine Learning:

  • Sparsity: En promedio, el 50% de las neuronas estarán a cero en cualquier forward pass. Eso es eficiencia computacional brutal.
  • No linealidad: Aunque parece lineal por partes, la composición de múltiples capas ReLU crea fronteras de decisión altamente no lineales.

Dimensionalidad importa: Para una capa con entrada $\mathbf{x} \in \mathbb{R}^{n}$, ReLU opera element-wise: $\text{ReLU}(\mathbf{x})_i = \max(0, x_i)$. La forma del tensor no cambia.

📐 La Derivada de ReLU y Por Qué Backprop Funciona (o Muere)

Acá está la joya que nadie te cuenta en los tutoriales rápidos. La derivada de ReLU es lo que determina si tu red aprende o se queda congelada.

$$f'(x) = \frac{d}{dx}\max(0, x) = \begin{cases} 1 & \text{si } x > 0 \\ 0 & \text{si } x < 0 \\ \text{indefinida} & \text{si } x = 0 \end{cases}$$

En la práctica, $f'(0) = 0$ por convención (PyTorch lo implementa así).

Por Qué Esto Es Decisivo Respecto a Sigmoid

Con Sigmoid: $\sigma'(x) = \sigma(x)(1 - \sigma(x))$, el gradiente máximo es $0.25$ (en $x=0$). Para una red de 10 capas, el gradiente se multiplica: $0.25^{10} \approx 9.5 \times 10^{-7}$. Eso es vanishing gradient en toda su expresión.

Con ReLU: el gradiente para $x > 0$ es exactamente $1$. No se atenúa. El gradiente llega limpio hasta las capas más profundas.

$$\frac{\partial \mathcal{L}}{\partial x_i} = \frac{\partial \mathcal{L}}{\partial f_i} \cdot \mathbf{1}[x_i > 0]$$

donde $\mathbf{1}[x_i > 0]$ es la función indicadora: $1$ si la neurona estaba activa en el forward pass, $0$ si no.

Derivación del Gradiente en Backprop: Paso a Paso

Paso 1: Forward Pass

$$z^{(l)} = W^{(l)} a^{(l-1)} + b^{(l)}, \quad a^{(l)} = \text{ReLU}(z^{(l)})$$

Paso 2: Gradiente de la Loss Respecto a $z^{(l)}$

$$\delta^{(l)} = \frac{\partial \mathcal{L}}{\partial z^{(l)}} = \frac{\partial \mathcal{L}}{\partial a^{(l)}} \odot \mathbf{1}[z^{(l)} > 0]$$

El operador $\odot$ es producto element-wise (Hadamard). La máscara $\mathbf{1}[z^{(l)} > 0]$ proviene exactamente de la derivada de ReLU.

Ejemplo numérico:

import torch

z = torch.tensor([-1.5, 2.3, -0.1, 4.0], requires_grad=False)
a = torch.relu(z)

grad_loss_a = torch.tensor([0.5, 0.8, -0.3, 1.2])

mask = (z > 0).float()
grad_loss_z = grad_loss_a * mask

print(f"z:              {z.tolist()}")
print(f"mask (z>0):     {mask.tolist()}")
print(f"grad_loss/dz:   {grad_loss_z.tolist()}")
# z:              [-1.5, 2.3, -0.1, 4.0]
# mask (z>0):     [0.0, 1.0, 0.0, 1.0]
# grad_loss/dz:   [0.0, 0.8, 0.0, 1.2]

Las neuronas con $z \leq 0$ reciben gradiente cero. Si esto persiste durante todo el entrenamiento para una neurona específica, esa neurona está muerta. Para siempre.

📐 Implementación Production-Ready con torch.nn.ReLU

from __future__ import annotations

import logging
from dataclasses import dataclass
from typing import Literal

import torch
import torch.nn as nn

logger = logging.getLogger(__name__)


@dataclass
class ReLULayerConfig:
    """Configuration for a production ReLU-activated linear layer.

    Args:
        in_features: Number of input features.
        out_features: Number of output neurons.
        activation: Activation variant to use.
        leaky_slope: Negative slope for LeakyReLU (ignored for 'relu').
        inplace: If True, modifies tensor in-place (saves memory, use with care).
    """

    in_features: int
    out_features: int
    activation: Literal["relu", "leaky_relu", "elu"] = "relu"
    leaky_slope: float = 0.01
    inplace: bool = False

    def __post_init__(self) -> None:
        if self.in_features <= 0:
            raise ValueError(
                f"in_features debe ser positivo, recibí: {self.in_features}"
            )
        if self.out_features <= 0:
            raise ValueError(
                f"out_features debe ser positivo, recibí: {self.out_features}"
            )
        if not 0.0 <= self.leaky_slope <= 1.0:
            raise ValueError(
                f"leaky_slope debe estar en [0, 1], recibí: {self.leaky_slope}"
            )


class ActivatedLinearBlock(nn.Module):
    """Production linear block with configurable activation.

    Encapsulates Linear → Activation as a reusable building block
    for deep networks, with monitoring hooks for dead neuron detection.

    Args:
        config: ReLULayerConfig with all layer parameters.
    """

    def __init__(self, config: ReLULayerConfig) -> None:
        super().__init__()
        self._config = config
        self.linear = nn.Linear(config.in_features, config.out_features)
        self.activation = self._build_activation()
        self._dead_neuron_threshold: float = 1e-6

        logger.info(
            "ActivatedLinearBlock inicializado: %d → %d con %s",
            config.in_features,
            config.out_features,
            config.activation,
        )

    def _build_activation(self) -> nn.Module:
        """Build activation module based on config.

        Returns:
            Configured activation module.

        Raises:
            ValueError: If activation type is not supported.
        """
        activation_map: dict[str, nn.Module] = {
            "relu": nn.ReLU(inplace=self._config.inplace),
            "leaky_relu": nn.LeakyReLU(
                negative_slope=self._config.leaky_slope,
                inplace=self._config.inplace,
            ),
            "elu": nn.ELU(inplace=self._config.inplace),
        }
        if self._config.activation not in activation_map:
            raise ValueError(
                f"Activación no soportada: '{self._config.activation}'. "
                f"Opciones válidas: {list(activation_map.keys())}"
            )
        return activation_map[self._config.activation]

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """Forward pass with optional dead neuron logging.

        Args:
            x: Input tensor of shape (batch_size, in_features).

        Returns:
            Activated output tensor of shape (batch_size, out_features).

        Raises:
            ValueError: If input feature dimension doesn't match config.
        """
        if x.shape[-1] != self._config.in_features:
            raise ValueError(
                f"Dimensión de entrada incorrecta. "
                f"Esperaba {self._config.in_features}, recibí {x.shape[-1]}."
            )

        pre_activation = self.linear(x)
        output = self.activation(pre_activation)
        self._log_dead_neurons(output)
        return output

    def _log_dead_neurons(self, output: torch.Tensor) -> None:
        """Log percentage of dead neurons (always-zero activations).

        Args:
            output: Post-activation tensor of shape (batch_size, out_features).
        """
        with torch.no_grad():
            dead_fraction = (output.abs() < self._dead_neuron_threshold).float().mean().item()
            if dead_fraction > 0.5:
                logger.warning(
                    "%.1f%% de neuronas inactivas detectadas. "
                    "Considera Leaky ReLU o revisar inicialización de pesos.",
                    dead_fraction * 100,
                )


def demo_block() -> None:
    """Run a quick sanity check on ActivatedLinearBlock."""
    torch.manual_seed(42)

    config = ReLULayerConfig(
        in_features=16,
        out_features=8,
        activation="relu",
        inplace=False,
    )
    block = ActivatedLinearBlock(config)

    batch = torch.randn(32, 16)
    output = block(batch)

    logger.info("Input shape:  %s", batch.shape)
    logger.info("Output shape: %s", output.shape)
    logger.info(
        "Neuronas activas: %.1f%%",
        (output > 0).float().mean().item() * 100,
    )


if __name__ == "__main__":
    logging.basicConfig(level=logging.INFO)
    demo_block()

🚨 Errores Comunes (y Cómo No Cagarla)

Error 1: Inicialización de Pesos con Ceros o Distribución Inadecuada

El problema:

model = nn.Linear(64, 32)
nn.init.constant_(model.weight, 0.0)  # ❌ Todos los pesos en cero
nn.init.constant_(model.bias, 0.0)

Por qué falla:

Si todos los pesos son iguales, todas las neuronas reciben la misma señal y computan el mismo gradiente. El problema de simetría hace que nunca se diferencien. Con ReLU, si la señal pre-activación es cero desde el inicio, las neuronas pueden entrar en zona muerta desde el primer paso.

$$z_j = \sum_i w_{ji} x_i + b_j = 0 \quad \Rightarrow \quad a_j = \text{ReLU}(0) = 0$$

w = torch.zeros(4, 4)
x = torch.randn(1, 4)
z = x @ w.T
print(z)  # tensor([[0., 0., 0., 0.]]) - sin diferenciación, sin aprendizaje

La solución:

model = nn.Linear(64, 32)
nn.init.kaiming_uniform_(model.weight, nonlinearity="relu")  # ✓ He initialization
nn.init.zeros_(model.bias)

La inicialización de He (Kaiming) escala la varianza como $\text{Var}(w) = \frac{2}{n_{\text{in}}}$, compensando exactamente el hecho de que ReLU bloquea el 50% de las activaciones.

Error 2: Usar inplace=True Con Autograd en Grafos Que Se Reusan

El problema:

relu = nn.ReLU(inplace=True)  # ❌ Modifica el tensor original

x = torch.randn(4, 8, requires_grad=True)
residual = x
out = relu(x)
loss = (out + residual).sum()
loss.backward()  # RuntimeError: one of the variables needed for gradient
                 # computation has been modified by an inplace operation

Por qué falla:

inplace=True modifica el tensor en memoria antes de que autograd haya guardado el valor original para el backward pass. En conexiones residuales (ResNets, DenseNets), ese tensor original es necesario para calcular el gradiente de la skip connection.

$$\frac{\partial \mathcal{L}}{\partial x} = \frac{\partial \mathcal{L}}{\partial \text{out}} \cdot \mathbf{1}[x > 0] + \frac{\partial \mathcal{L}}{\partial \text{residual}}$$

Si $x$ fue modificado in-place, el primer término pierde la información de $x$ pre-ReLU.

x = torch.tensor([-1.0, 2.0, -0.5, 3.0], requires_grad=True)
y = x  # misma referencia
relu_inplace = nn.ReLU(inplace=True)
relu_inplace(x)
print(x)  # tensor([0., 2., 0., 3.]) — el -1 y -0.5 se perdieron

La solución:

relu = nn.ReLU(inplace=False)  # ✓ Por defecto; seguro con residuals

x = torch.randn(4, 8, requires_grad=True)
residual = x
out = relu(x)
loss = (out + residual).sum()
loss.backward()  # ✓ Funciona sin problema

Error 3: Dying ReLU — Neuronas que Nunca Despiertan

El problema:

optimizer = torch.optim.SGD(model.parameters(), lr=1.0)  # ❌ LR agresivo

for epoch in range(100):
    loss = criterion(model(X), y)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
    # Después de unos pasos, muchas neuronas tienen bias tan negativo
    # que z < 0 siempre → ReLU = 0 → gradiente = 0 → nunca se actualizan

Por qué falla:

Una vez que $z_j < 0$ para todos los ejemplos del dataset, la neurona $j$ tiene gradiente cero permanentemente:

$$\frac{\partial \mathcal{L}}{\partial w_j} = \frac{\partial \mathcal{L}}{\partial a_j} \cdot \underbrace{f'(z_j)}_{=\,0} \cdot x = 0$$

El peso nunca se mueve. La neurona murió.

def check_dead_neurons(model: nn.Module, X: torch.Tensor) -> None:
    activations = []
    hooks = []

    def hook(_, __, output):
        activations.append(output.detach())

    for layer in model.modules():
        if isinstance(layer, nn.ReLU):
            hooks.append(layer.register_forward_hook(hook))

    with torch.no_grad():
        model(X)

    for h in hooks:
        h.remove()

    for i, act in enumerate(activations):
        dead = (act == 0).all(dim=0).float().mean().item()
        print(f"Capa ReLU {i}: {dead*100:.1f}% neuronas muertas")

La solución:

activation = nn.LeakyReLU(negative_slope=0.01)  # ✓ Leaky ReLU: gradiente mínimo en zona negativa

torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)  # ✓ Gradient clipping

block = nn.Sequential(  # ✓ BatchNorm antes de ReLU mantiene pre-activaciones centradas
    nn.Linear(64, 32),
    nn.BatchNorm1d(32),
    nn.ReLU(),
)

Error 4: Aplicar ReLU en la Capa de Salida

El problema:

class Classifier(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(64, 32)
        self.fc2 = nn.Linear(32, 10)
        self.relu = nn.ReLU()

    def forward(self, x):
        x = self.relu(self.fc1(x))
        x = self.relu(self.fc2(x))  # ❌ ReLU en la salida de un clasificador
        return x

Por qué falla:

ReLU elimina todas las salidas negativas. Para clasificación con CrossEntropyLoss, los logits deben poder ser negativos (representan preferencia relativa entre clases). Si cortás los negativos, el modelo no puede expresar "esta clase es menos probable" con la misma fuerza.

$$P(y=k \mid x) = \frac{e^{z_k}}{\sum_j e^{z_j}}$$

Si $z_k = 0$ (porque ReLU lo mató), el modelo asigna probabilidad $\frac{1}{K}$ a esa clase — como si fuera totalmente indiferente.

La solución:

class Classifier(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(64, 32)
        self.fc2 = nn.Linear(32, 10)
        self.relu = nn.ReLU()

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        x = self.relu(self.fc1(x))
        return self.fc2(x)  # ✓ Sin activación en capa de salida para clasificación
        # Para regresión con salidas positivas: ReLU puede tener sentido
        # Para regresión general: sin activación

🎯 Resumen Ejecutivo

ReLU no es "la activación sencilla que pone ceros". Es resolver el problema fundamental del vanishing gradient mediante:

$$f(x) = \max(0, x)$$

Con una derivada que preserva el gradiente intacto en la zona activa:

$$f'(x) = \mathbf{1}[x > 0] \in \{0, 1\}$$

Con todas las implicaciones que eso conlleva:

  • Gradiente en backprop: $1$ exacto para neuronas activas, lo que permite entrenar redes de 100+ capas sin vanishing gradient.
  • Sparsity estructural: ~50% de neuronas inactivas por batch → eficiencia computacional sin sacrificar capacidad expresiva.
  • Dying ReLU como deuda técnica: neuronas que mueren permanentemente si los gradientes las empujan a zona negativa, requiriendo monitoreo activo en producción.
  • Inicialización importa: He initialization es el estándar para redes con ReLU — $\text{Var}(w) = \frac{2}{n_{\text{in}}}$. Usar Xavier aquí es asumir Sigmoid, y eso se paga en convergencia lenta.

De una, hágale. Con esta base de ReLU + gradientes + código production-ready con detección de neuronas muertas, estás listo para debuggear cualquier red que no aprenda y para tomar decisiones conscientes sobre qué activación usar en cada capa.
