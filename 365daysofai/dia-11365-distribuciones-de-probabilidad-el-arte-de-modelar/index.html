Día 11: Distribuciones de Probabilidad - El Arte de Modelar la Incertidumbre en Producción

Por qué este tema te va a salvar el pellejo

Parcero, si hay algo que me enseñaron 10 años haciendo ML en producción es esto: la incertidumbre no es un bug, es una feature. El problema es que la mayoría de ingenieros la tratan como si fuera un null que hay que esconder debajo del tapete. Error garrafal.

Ando embalado escribiendo este artículo porque acabo de revisar un modelo en producción donde alguien usó un random.random() de toda la vida para generar datos sintéticos. ¿El resultado? Un modelo que en dev tenía 94% de accuracy y en prod no pasaba del 67%. ¿La razón? No respetaron la distribución real de los datos.

Hoy vamos a hablar de distribuciones de probabilidad con numpy.random y scipy.stats, pero no desde el romanticismo académico del "mundo fascinante de las estadísticas". No. Vamos a hablar desde la trinchera: cómo generar datos que no te revienten el modelo, cómo hacer testing estadístico que valga la pena, y cómo no quedarte como un sapo cuando tu A/B test diga que "no hay diferencia significativa" y el CEO te mire con cara de "entonces para qué te pago".

1. La Cruda Realidad: Todo en ML es una Distribución

1.1 Fundamento Matemático

Una distribución de probabilidad $P(X)$ describe cómo se comporta una variable aleatoria $X$. Para variables continuas, trabajamos con la función de densidad de probabilidad (PDF):

$$f(x) \geq 0, \quad \int_{-\infty}^{\infty} f(x)dx = 1$$

Y su correspondiente función de distribución acumulativa (CDF):

$$F(x) = P(X \leq x) = \int_{-\infty}^{x} f(t)dt$$

La distribución más famosa (y abusada) es la Normal o Gaussiana:

$$f(x|\mu,\sigma) = \frac{1}{\sigma\sqrt{2\pi}} \exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$$

Donde $\mu$ es la media y $\sigma$ la desviación estándar.

Analogía del tinto: Imagina que mides cuánto demora cada persona del equipo en tomarse un tinto (café). La mayoría se demora entre 3-5 minutos (la media), pero siempre hay un man que se lo toma en 30 segundos y otro que se demora 15 minutos leyendo Slack. Esa variabilidad es $sigma$.

1.2 Por qué esto importa en producción

Cuando entrenas un modelo, estás asumiendo que tus datos vienen de una distribución $P_{train}(X,Y)$. Cuando lo pones en producción, recibes datos de $P_{prod}(X,Y)$. Si $P_{train} < P_{prod}$, tu modelo se cae como casa de naipes.

Data drift no es un concepto abstracto. Es la diferencia entre estas distribuciones:

$$D_{KL}(P_{prod} || P_{train}) = \int P_{prod}(x) \log\frac{P_{prod}(x)}{P_{train}(x)}dx$$

class DistributionProtocol(Protocol):

@dataclass(frozen=True, slots=True)

class BaseDistribution(ABC):
    """Clase base con la lógica común que TODA distribución debe tener.
    
    ¿Por qué abstracta? Porque queremos forzar a que cada distribución
    concreta implemente su propia lógica de sampling, pero compartir
    la validación, logging y manejo de semillas.
    """
    
    __slots__ = ('_config', '_rng', '_name')
    
    def __init__(self, config: DistributionConfig, name: str):
        self._config: Final = config
        self._name: Final = name
        # BitGenerator moderno. Más rápido que np.random.seed() legacy
        self._rng: np.random.Generator = np.random.default_rng(config.seed)
        
        if config.log_sampling:
            
logger.info
(
                f"Inicializada distribución '{name}' con seed={config.seed}"
            )
    
    @abstractmethod
    def _sample_impl(self, size: int) -> np.ndarray:
        """Método abstracto que cada distribución debe implementar.
        
        ¿Por qué separar _sample_impl de sample? Separación de concerns:
        - sample(): maneja validación, logging, excepciones
        - _sample_impl(): solo se preocupa por el algoritmo de sampling
        """
        pass
    
    def sample(self, size: int) -> np.ndarray:
        """Método público con toda la infraestructura de producción."""
        if size <= 0:
            raise ValueError(
                f"size debe ser positivo, recibido: {size}. "
                f"Si esto viene de user input, valida antes de llamar esta función."
            )
        
        try:
            samples = self._sample_impl(size)
            
            if self._config.validate_output:
                self._validate_samples(samples, size)
            
            if self._config.log_sampling:
                
logger.info
(
                    f"{self._name}: Generados {size} samples. "
                    f"Stats: μ={samples.mean():.4f}, σ={samples.std():.4f}"
                )
            
            return samples
            
        except Exception as e:
            logger.error(
                f"Error en {self._name}.sample(size={size}): {e}",
                exc_info=True
            )
            raise
    
    def _validate_samples(self, samples: np.ndarray, expected_size: int) -> None:
        """Validación que te salva en producción.
        
        He visto modelos caerse porque numpy.random devolvió NaN por overflow.
        Esta validación te avisa ANTES de que el NaN se propague por todo tu pipeline.
        """
        if samples.shape[0] != expected_size:
            raise ValueError(
                f"Shape inconsistente: esperado {expected_size}, "
                f"obtenido {samples.shape[0]}"
            )
        
        if np.any(np.isnan(samples)):
            raise ValueError(
                f"Samples contienen NaN. Esto es un bug crítico. "
                f"Distribución: {self._name}"
            )
        
        if np.any(np.isinf(samples)):
            raise ValueError(
                f"Samples contienen Inf. Overflow detectado en {self._name}"
            )


class NormalDistribution(BaseDistribution):
    """Distribución Normal con todas las garantías de producción.
    
    ¿Por qué una clase en lugar de una función? Encapsulación de estado:
    - Semilla reproducible
    - Configuración inmutable
    - Métodos relacionados (sample, pdf, cdf) agrupados
    """
    
    __slots__ = ('_mu', '_sigma')
    
    def __init__(
        self,
        mu: float,
        sigma: float,
        config: DistributionConfig = DistributionConfig(seed=None)
    ):
        if sigma <= 0:
            raise ValueError(
                f"sigma debe ser positivo. Recibido: {sigma}. "
                f"Una desviación estándar negativa o cero no tiene sentido matemático."
            )
        
        super().__init__(config, name=f"Normal(μ={mu}, σ={sigma})")
        self._mu: Final[float] = mu
        self._sigma: Final[float] = sigma
    
    def _sample_impl(self, size: int) -> np.ndarray:
        """Implementación del sampling usando el Box-Muller optimizado de numpy.
        
        numpy.random.normal usa el método Box-Muller (o Ziggurat en versiones nuevas),
        que es O(1) por sample. Mucho más eficiente que métodos de rechazo.
        """
        return self._rng.normal(loc=self._mu, scale=self._sigma, size=size)
    
    def pdf(self, x: np.ndarray) -> np.ndarray:
        """Función de densidad de probabilidad.
        
        Útil para: 
        - Likelihood en Bayesian inference
        - Anomaly detection (si pdf(x) < threshold, x es outlier)
        - Plotting distribuciones
        """
        variance = self._sigma ** 2
        normalization = 1.0 / np.sqrt(2 * np.pi * variance)
        exponent = -((x - self._mu) ** 2) / (2 * variance)
        return normalization * np.exp(exponent)
    
    def cdf(self, x: np.ndarray) -> np.ndarray:
        """Función de distribución acumulativa.
        
        Útil para:
        - Calcular percentiles
        - Hypothesis testing (p-values)
        - Confidence intervals
        """
        from scipy.special import erf
        z = (x - self._mu) / (self._sigma * np.sqrt(2))
        return 0.5 * (1 + erf(z))
    
    def ppf(self, q: np.ndarray) -> np.ndarray:
        """Percent Point Function (inversa de CDF).
        
        Si CDF(x) = q, entonces PPF(q) = x.
        Útil para generar samples desde quantiles específicos.
        """
        if np.any((q < 0) | (q > 1)):
            raise ValueError("Quantiles deben estar en [0,1]")
        
        from scipy.special import erfinv
        return self._mu + self._sigma * np.sqrt(2) * erfinv(2 * q - 1)


class UniformDistribution(BaseDistribution):
    """Distribución Uniforme continua U(a, b).
    
    La distribución "democrática": todos los valores tienen la misma probabilidad.
    Útil para inicialización de pesos en redes neuronales (Xavier/He initialization).
    """
    
    __slots__ = ('_low', '_high')
    
    def __init__(
        self,
        low: float,
        high: float,
        config: DistributionConfig = DistributionConfig(seed=None)
    ):
        if low >= high:
            raise ValueError(
                f"low debe ser menor que high. Recibido: low={low}, high={high}"
            )
        
        super().__init__(config, name=f"Uniform({low}, {high})")
        self._low: Final[float] = low
        self._high: Final[float] = high
    
    def _sample_impl(self, size: int) -> np.ndarray:
        return self._rng.uniform(low=self._low, high=self._high, size=size)
    
    def pdf(self, x: np.ndarray) -> np.ndarray:
        """PDF de la uniforme: 1/(b-a) si x ∈ [a,b], 0 otherwise."""
        result = np.zeros_like(x, dtype=np.float64)
        mask = (x >= self._low) & (x <= self._high)
        result[mask] = 1.0 / (self._high - self._low)
        return result
    
    def cdf(self, x: np.ndarray) -> np.ndarray:
        """CDF de la uniforme: (x-a)/(b-a) si x ∈ [a,b]."""
        result = np.zeros_like(x, dtype=np.float64)
        
        # Antes de a: CDF = 0
        # Después de b: CDF = 1
        # Entre a y b: lineal
        mask_before = x < self._low
        mask_after = x > self._high
        mask_middle = ~mask_before & ~mask_after
        
        result[mask_after] = 1.0
        result[mask_middle] = (x[mask_middle] - self._low) / (self._high - self._low)
        
        return result

    Escenario real: Tienes un modelo de regresión pero no puedes usar
    datos de producción en dev por GDPR. Necesitas datos sintéticos que
    sigan la misma distribución.
    
    Args:
        n_samples: Número de samples a generar
        feature_dist: Distribución para las features
        noise_scale: Escala del ruido gaussiano
        seed: Semilla para reproducibilidad
    
    Returns:
        (X, y) donde X son las features e y es el target
    """
    
logger.info
(f"Generando {n_samples} samples sintéticos")
    
    # Features desde la distribución especificada
    X = feature_dist.sample(n_samples)
    
    # Target: relación no lineal + ruido
    # y = sin(X) + ε, donde ε ~ N(0, noise_scale)
    noise_dist = NormalDistribution(
        mu=0.0,
        sigma=noise_scale,

    y = np.sin(X) + noise

(
        f"Dataset generado. X stats: μ={X.mean():.4f}, σ={X.std():.4f}. "
        f"y stats: μ={y.mean():.4f}, σ={y.std():.4f}"
    )
    
    return X.reshape(-1, 1), y

from scipy import stats
from typing import NamedTuple, List
from enum import Enum

class TestResult(NamedTuple):
    """Resultado de un test estadístico.
    
    ¿Por qué NamedTuple? 
    - Inmutable (thread-safe)
    - Type hints nativos
    - Menos memoria que dataclass
    - Unpacking fácil: statistic, pvalue = result
    """
    statistic: float
    pvalue: float
    test_name: str
    conclusion: str
    details: dict


class SignificanceLevel(Enum):
    """Niveles de significancia estándar.
    
    Enum para evitar magic numbers en el código.
    En producción, esto previene bugs tipo: "¿era 0.05 o 0.005?"
    """
    VERY_STRICT = 0.001  # 99.9% confianza
    STRICT = 0.01        # 99% confianza
    STANDARD = 0.05      # 95% confianza
    RELAXED = 0.10       # 90% confianza


class StatisticalTester:
    """Suite de tests estadísticos production-ready.
    
    En producción, NO basta con calcular un p-value.
    Necesitas:
    - Validar assumptions (normalidad, homocedasticidad)
    - Interpretar resultados
    - Loggear todo para auditorías
    """
    
    __slots__ = ('_alpha', '_logger')
    
    def __init__(self, alpha: SignificanceLevel = SignificanceLevel.STANDARD):
        self._alpha: Final[float] = alpha.value
        self._logger = logging.getLogger(self.__class__.__name__)
    
    def normality_test(
        self,
        data: np.ndarray,
        test_type: str = 'shapiro'
    ) -> TestResult:
        """Test de normalidad con Shapiro-Wilk o Kolmogorov-Smirnov.
        
        ¿Cuándo usar cada uno?
        - Shapiro-Wilk: n < 5000, más potente
        - Kolmogorov-Smirnov: n > 5000, más rápido
        
        Hypothesis Testing 101:
        H₀: Los datos vienen de una distribución normal
        H₁: Los datos NO vienen de una distribución normal
        
        Si p-value < α, rechazamos H₀ (los datos no son normales)
        """
        if len(data) < 3:
            raise ValueError(
                "Se necesitan al menos 3 samples para test de normalidad. "
                f"Recibido: {len(data)}"
            )
        
        if test_type == 'shapiro':
            if len(data) > 5000:
                self._logger.warning(
                    f"Shapiro-Wilk con {len(data)} samples puede ser lento. "
                    f"Considera usar 'kstest'"
                )
            statistic, pvalue = stats.shapiro(data)
            test_name = "Shapiro-Wilk"
        
        elif test_type == 'kstest':
            # KS test contra distribución normal con parámetros estimados
            mean, std = data.mean(), data.std(ddof=1)
            statistic, pvalue = stats.kstest(
                data,
                lambda x: stats.norm.cdf(x, loc=mean, scale=std)
            )
            test_name = "Kolmogorov-Smirnov"
        
        else:
            raise ValueError(f"Test desconocido: {test_type}")
        
        conclusion = (
            "Los datos NO son normales" if pvalue < self._alpha
            else "No hay evidencia suficiente contra normalidad"
        )
        
        details = {
            'n_samples': len(data),
            'data_mean': float(data.mean()),
            'data_std': float(data.std(ddof=1)),
            'alpha': self._alpha
        }
        
        self._
logger.info
(
            f"{test_name}: statistic={statistic:.6f}, "
            f"p-value={pvalue:.6f}, conclusion='{conclusion}'"
        )
        
        return TestResult(
            statistic=statistic,
            pvalue=pvalue,
            test_name=test_name,
            conclusion=conclusion,
            details=details
        )
    
    def two_sample_ttest(
        self,
        sample_a: np.ndarray,
        sample_b: np.ndarray,
        equal_var: bool = False
    ) -> TestResult:
        """T-test para comparar medias de dos muestras.
        
        Escenario clásico: A/B testing.
        - sample_a: grupo control
        - sample_b: grupo tratamiento
        
        Pregunta: ¿El tratamiento tuvo efecto significativo?
        
        H₀: μ_a = μ_b (no hay diferencia)
        H₁: μ_a ≠ μ_b (hay diferencia)
        
        Args:
            equal_var: Si True, asume varianzas iguales (Student's t-test).
                      Si False, usa Welch's t-test (más robusto).
        """
        if len(sample_a) < 2 or len(sample_b) < 2:
            raise ValueError("Cada muestra debe tener al menos 2 observaciones")
        
        # Primero, verificar normalidad (assumption del t-test)
        norm_a = self.normality_test(sample_a)
        norm_b = self.normality_test(sample_b)
        
        if norm_a.pvalue < self._alpha or norm_b.pvalue < self._alpha:
            self._logger.warning(
                "Una o ambas muestras no son normales. "
                "T-test puede no ser apropiado. Considera Mann-Whitney U test."
            )
        
        # Realizar t-test
        statistic, pvalue = stats.ttest_ind(
            sample_a,
            sample_b,
            equal_var=equal_var
        )
        
        test_name = "Student's t-test" if equal_var else "Welch's t-test"
        
        conclusion = (
            f"Hay diferencia significativa entre las medias (p={pvalue:.4f} < {self._alpha})"
            if pvalue < self._alpha
            else f"No hay evidencia de diferencia significativa (p={pvalue:.4f} >= {self._alpha})"
        )
        
        # Effect size: Cohen's d
        pooled_std = np.sqrt(
            ((len(sample_a) - 1) * sample_a.var(ddof=1) +
             (len(sample_b) - 1) * sample_b.var(ddof=1)) /
            (len(sample_a) + len(sample_b) - 2)
        )
        cohens_d = (sample_a.mean() - sample_b.mean()) / pooled_std
        
        details = {
            'n_a': len(sample_a),
            'n_b': len(sample_b),
            'mean_a': float(sample_a.mean()),
            'mean_b': float(sample_b.mean()),
            'std_a': float(sample_a.std(ddof=1)),
            'std_b': float(sample_b.std(ddof=1)),
            'cohens_d': float(cohens_d),
            'effect_interpretation': self._interpret_cohens_d(cohens_d),
            'equal_var_assumed': equal_var,
            'alpha': self._alpha
        }

logger.info
(
            f"{test_name}: t={statistic:.4f}, p={pvalue:.6f}, "
            f"Cohen's d={cohens_d:.4f}, conclusion='{conclusion}'"
        )
        
        return TestResult(
            statistic=statistic,
            pvalue=pvalue,
            test_name=test_name,
            conclusion=conclusion,
            details=details
        )
    
    @staticmethod
    def _interpret_cohens_d(d: float) -> str:
        """Interpretación de Cohen's d según Cohen (1988).
        
        ¿Por qué importa effect size?
        Puedes tener p-value < 0.05 (significancia estadística)
        pero effect size tiny (irrelevancia práctica).
        
        Ejemplo: "El grupo B convirtió 0.01% más que el grupo A,
        p=0.03". Estadísticamente significativo, pero ¿le importa al CEO?
        """
        abs_d = abs(d)
        if abs_d < 0.2:
            return "negligible"
        elif abs_d < 0.5:
            return "small"
        elif abs_d < 0.8:
            return "medium"
        else:
            return "large"
    
    def confidence_interval(
        self,
        data: np.ndarray,
        confidence: float = 0.95
    ) -> Tuple[float, float]:
        """Intervalo de confianza para la media.
        
        Interpretación correcta:
        "Si repetimos el experimento infinitas veces, el 95% de los
        intervalos calculados contendrán la media poblacional verdadera."
        
        NO significa: "Hay 95% de probabilidad de que μ esté en este intervalo"
        (frecuentista vs bayesiano).
        """
        if confidence <= 0 or confidence >= 1:
            raise ValueError(f"Confidence debe estar en (0,1), recibido: {confidence}")
        
        mean = data.mean()
        sem = stats.sem(data)  # Standard Error of the Mean
        
        # Degrees of freedom
        df = len(data) - 1
        
        # Intervalo usando distribución t de Student
        ci = stats.t.interval(
            confidence,
            df=df,
            loc=mean,
            scale=sem
        )
        
        self._
logger.info
(
            f"Intervalo de confianza {confidence*100}%: "
            f"[{ci[0]:.4f}, {ci[1]:.4f}], mean={mean:.4f}"
        )
        
        return ci

logger.info
("\n--- Hypothesis Test ---")
    if norm_control.pvalue < alpha.value or norm_treatment.pvalue < alpha.value:
        logger.warning("Usando Mann-Whitney U test (datos no normales)")
        statistic, pvalue = stats.mannwhitneyu(control, treatment, alternative='two-sided')
        test_result = TestResult(
            statistic=statistic,
            pvalue=pvalue,
            test_name="Mann-Whitney U",
            conclusion="Ver interpretación",
            details={}
        )
    else:
        # Levene's test para homocedasticidad
        _, levene_p = stats.levene(control, treatment)
        equal_var = levene_p > alpha.value
        
        
logger.info
(
            f"Levene's test: p={levene_p:.4f}, "
            f"equal_var={equal_var}"
        )
        
        test_result = tester.two_sample_ttest(
            control,
            treatment,
            equal_var=equal_var
        )
    
    # 3. Intervalos de confianza
    
logger.info
("\n--- Confidence Intervals ---")
    ci_control = tester.confidence_interval(control, confidence=0.95)
    ci_treatment = tester.confidence_interval(treatment, confidence=0.95)
    
    # 4. Resumen ejecutivo
    
logger.info
("\n=== RESUMEN EJECUTIVO ===")
    
logger.info
(test_result.conclusion)
    
    if test_result.pvalue < alpha.value:
        rel_change = (treatment.mean() - control.mean()) / control.mean() * 100
        
logger.info
(
            f"Cambio relativo: {rel_change:+.2f}% "
            f"({'mejora' if rel_change > 0 else 'deterioro'})"
        )
    
    return {
        'test_result': test_result,
        'normality_control': norm_control,
        'normality_treatment': norm_treatment,
        'ci_control': ci_control,
        'ci_treatment': ci_treatment,
        'relative_change_pct': (treatment.mean() - control.mean()) / control.mean() * 100
    }

Alivio del Dolor: Este código hace el A/B testing completo y correcto. No más "solo miré las medias y se veían diferentes". Ahora tienes normalidad, homocedasticidad, effect size, intervalos de confianza. Todo lo que necesitas para defender tus resultados ante el VP de producto.

4. Caso Real: Detección de Anomalías con Distribuciones

4.1 El Problema

Tienes un sistema que registra latencias de API. Necesitas detectar cuando algo anda mal automáticamente.

Approach ingenuo: if latency > 1000ms: alert()

Problema: En Black Friday, 1000ms puede ser normal. En 3am de un martes, 100ms ya es sospechoso.

Solución estadística: Modelo probabilístico adaptativo.

4.2 Implementación

from collections import deque
from datetime import datetime
from typing import Deque

class AnomalyDetector:
    """Detector de anomalías basado en distribución rolling.
    
    Approach: Mantenemos una ventana rolling de N observaciones.
    Modelamos la distribución y detectamos outliers como aquellos
    que caen fuera de k desviaciones estándar.
    
    ¿Por qué rolling? Porque las distribuciones cambian en el tiempo (non-stationarity).
    """
    
    __slots__ = (
        '_window_size',
        '_threshold_std',
        '_observations',
        '_dist',
        '_logger',
        '_alert_cooldown',
        '_last_alert_time'
    )
    
    def __init__(
        self,
        window_size: int = 1000,
        threshold_std: float = 3.0,
        alert_cooldown_seconds: int = 300
    ):
        """
        Args:
            window_size: Número de observaciones para calcular la distribución
            threshold_std: Número de desviaciones estándar para considerar anomalía
            alert_cooldown_seconds: Tiempo mínimo entre alertas (evitar spam)
        """
        if window_size < 30:
            raise ValueError("window_size debe ser >= 30 para estadística robusta")
        
        if threshold_std < 2.0:
            logger.warning(
                f"threshold_std={threshold_std} puede generar muchos false positives. "
                f"Considera >= 3.0"
            )
        
        self._window_size: Final[int] = window_size
        self._threshold_std: Final[float] = threshold_std
        self._observations: Deque[float] = deque(maxlen=window_size)
        self._dist: NormalDistribution | None = None
        self._logger = logging.getLogger(self.__class__.__name__)
        self._alert_cooldown: Final[int] = alert_cooldown_seconds
        self._last_alert_time: datetime | None = None
    
    def update(self, value: float) -> None:
        """Añadir nueva observación y actualizar distribución."""
        if not np.isfinite(value):
            raise ValueError(f"Valor inválido: {value}")
        
        self._observations.append(value)
        
        # Solo recalcular distribución cuando tenemos suficientes datos
        if len(self._observations) >= 30:
            mu = np.mean(self._observations)
            sigma = np.std(self._observations, ddof=1)
            
            # Prevenir sigma=0 (todos los valores iguales)
            if sigma < 1e-10:
                sigma = 1e-10
            
            self._dist = NormalDistribution(
                mu=mu,
                sigma=sigma,
                config=DistributionConfig(seed=None, validate_output=False)
            )
    
    def is_anomaly(self, value: float) -> Tuple[bool, dict]:
        """Detectar si un valor es anómalo.
        
        Returns:
            (is_anomaly, details) donde details contiene:
            - z_score: Cuántas sigmas está del mean
            - percentile: En qué percentil cae
            - should_alert: Si se debe enviar alerta (respetando cooldown)
        """
        if self._dist is None:
            return False, {'reason': 'insufficient_data', 'n_obs': len(self._observations)}
        
        # Z-score: cuántas desviaciones estándar
        z_score = (value - self._dist._mu) / self._dist._sigma
        
        # Percentile
        percentile = self._dist.cdf(np.array([value]))[0]
        
        is_anomaly = abs(z_score) > self._threshold_std
        
        # Decidir si alertar (respetando cooldown)
        should_alert = False
        if is_anomaly:
            now = 
datetime.now
()
            if self._last_alert_time is None:
                should_alert = True
            else:
                seconds_since_last = (now - self._last_alert_time).total_seconds()
                if seconds_since_last >= self._alert_cooldown:
                    should_alert = True
            
            if should_alert:
                self._last_alert_time = now
                self._logger.warning(
                    f"ANOMALÍA DETECTADA: value={value:.2f}, "
                    f"z_score={z_score:.2f}, percentile={percentile:.4f}"
                )
        
        return is_anomaly, {
            'z_score': float(z_score),
            'percentile': float(percentile),
            'threshold_std': self._threshold_std,
            'current_mu': float(self._dist._mu),
            'current_sigma': float(self._dist._sigma),
            'should_alert': should_alert,
            'n_observations': len(self._observations)
        }
    
    def get_thresholds(self) -> Tuple[float, float]:
        """Obtener los thresholds actuales (lower, upper)."""
        if self._dist is None:
            raise ValueError("No hay distribución calculada aún")
        
        lower = self._dist._mu - self._threshold_std * self._dist._sigma
        upper = self._dist._mu + self._threshold_std * self._dist._sigma
        
        return lower, upper

Alivio del Dolor: Este detector se adapta automáticamente a cambios en la distribución (Black Friday vs 3am). El cooldown previene alert fatigue. El z-score te da severidad cuantitativa.

5. Optimizaciones y Consideraciones de Producción

5.1 Complejidad Computacional

Operación Complejidad Notas
np.random.normal(size=n) $O(n)$ Ziggurat algorithm, muy optimizado
stats.norm.pdf(x) $O(n)$ Vectorizado, no hay loops en Python
stats.shapiro(x) $O(n^2)$ Lento para n > 5000
stats.kstest(x) $O(n \log n)$ Preferible para n grande

Lección: Para n > 5000, usa KS test en lugar de Shapiro-Wilk.

5.2 Memory Footprint

Lección: Para inferencia en producción, considera float32 si la precisión lo permite. 50% menos memoria = más throughput.

6. Conclusión: La Probabilidad es tu Amiga (Si la Respetas)

Parcero, después de este tour de force por distribuciones, aquí está el TL;DR:

  1. numpy.random es rápido pero minimalista: Úsalo para sampling, pero siempre con semillas y validación.
  2. scipy.stats es tu estadístico personal: PDF, CDF, hypothesis testing, intervalos de confianza. Todo lo que necesitas para no quedar como sapo en una reunión con el data science team.
  3. Validación, validación, validación: NaN, Inf, distribuciones incorrectas... En producción, todo falla de formas creativas. Valida SIEMPRE.
  4. Effect size > p-value: Un p-value significativo no implica relevancia práctica. Cohen's d te dice si el CEO debería importarle.
  5. Reproducibilidad no es opcional: Semillas, hashing de config, versionamiento. Si no puedes reproducir, no puedes debuggear.

La próxima vez que alguien te diga "solo genera números random", ya sabes qué responder. Y si te toca hacer un A/B test, no vas a ser el que salga con "las medias se ven diferentes" sin ningún rigor estadístico.
