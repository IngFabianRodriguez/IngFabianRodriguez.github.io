t-SNE - Día 58

¿Qué más, parcero? Llegamos al Día 58 y hoy tocamos uno de esos algoritmos que la gente usa como si fuera magia negra: t-SNE. Y no, no es "ese visualizador bonito para las presentaciones". Después de 10 años viendo equipos tomar decisiones de arquitectura basadas en clusters que t-SNE inventó solito, te juro que esto me quita el sueño. El 85% de los papers que usan t-SNE lo interpretan mal, confundiendo distancias globales con estructura local. Dejá el café, acomodate, porque hoy desmenuzamos esta vaina desde las tripas.

🎯 ¿Por Qué t-SNE No Es Solo "Hacer Puntos Bonitos"?

Mirá, t-SNE es como cuando en una reunión de 50 personas intentás reorganizar las mesas para que cada quien quede cerca de con quien más habla, pero el salón es pequeño y bidimensional. No podés respetar todas las distancias originales, así que sacrificás las relaciones lejanas para preservar las cercanas.

Definición técnica (sin rodeos):

t-SNE (t-distributed Stochastic Neighbor Embedding) es un algoritmo de reducción de dimensionalidad no lineal que minimiza la divergencia KL entre una distribución de similitudes en el espacio de alta dimensión $P$ y una distribución de similitudes en el espacio de baja dimensión $Q$.

Tipos principales:

  • t-SNE estándar: Usa toda la matriz de similitudes. Complejidad $O(N^2)$. Para datasets pequeños (< 10K muestras).
  • Barnes-Hut t-SNE: Aproximación con árboles octales/quad-trees. Complejidad $O(N log N)$. Es el que usa scikit-learn por defecto con method='barnes_hut'.

Confundirlos importa: si tenés 100K filas y usás el estándar, vas a estar esperando hasta el día 365 de este challenge.

📐 La Distribución de Similitudes en Alta Dimensión

El corazón de t-SNE empieza construyendo similitudes entre pares de puntos en el espacio original. Dado un punto $x_i$, la similitud condicional hacia $x_j$ se define como una Gaussiana centrada en $x_i$:

$$p_{j|i} = \frac{\exp\!\left(-\|x_i - x_j\|^2 \,/\, 2\sigma_i^2\right)}{\displaystyle\sum_{k \neq i} \exp\!\left(-\|x_i - x_k\|^2 \,/\, 2\sigma_i^2\right)}$$

Esta ecuación parece simple pero tiene TODO el poder de decidir qué vecinos importan. Vamos a desmenuzarla símbolo por símbolo porque acá es donde la mayoría se pierde.

Descomposición Completa: Entendiendo Cada Símbolo

$p_{j|i}$ (Similitud Condicional) - La Probabilidad de Ser Vecino

p_{j|i} = probabilidad de que x_i "elija" a x_j como vecino
        = qué tan probable es que sean amigos en el espacio original

Analogía: Es como la probabilidad de que en esa reunión de 50 personas, la persona $i$ vaya a hablar con la persona $j$ en el break.

En Machine Learning:

  • Para puntos muy cercanos: $p_{j|i}$ se acerca a 1.
  • Para puntos lejanos: $p_{j|i}$ se acerca a 0 exponencialmente.

Diferencia CRÍTICA: Es asimétrica. $p_{j|i} \neq p_{i|j}$. Por eso t-SNE simetriza: $p_{ij} = \frac{p_{j|i} + p_{i|j}}{2N}$.

$\sigma_i^2$ (Varianza del Kernel Gaussiano) - El Ancho del Vecindario

σ_i = radio del "vecindario" alrededor del punto i
    = no es fijo: se calibra por punto mediante búsqueda binaria

Analogía: La persona extrovertida tiene un $\sigma_i$ grande (considera vecinos lejanos); la introvertida lo tiene pequeño.

En Machine Learning: $\sigma_i$ se elige para que la perplejidad de la distribución condicional sea igual al parámetro perplexity que vos configurás. Esa vaina es crítica.

Dimensionalidad importa: $\|x_i - x_j\|^2$ se calcula en el espacio original de $D$ dimensiones. Si $D = 784$ (MNIST), esta distancia euclidiana puede volverse poco informativa (maldición de la dimensionalidad).

Ejemplo numérico concreto:

import numpy as np

# Tres puntos en 2D (espacio original)
x = np.array([[0.0, 0.0], [1.0, 0.0], [10.0, 0.0]])

sigma_i = 1.0  # Fija para el ejemplo

# Distancias al cuadrado desde x[0]
dists_sq = np.sum((x[0] - x) ** 2, axis=1)
print("Distancias²:", dists_sq)  # [0, 1, 100]

# Numeradores del kernel gaussiano (excluyendo i=0)
numerators = np.exp(-dists_sq[1:] / (2 * sigma_i**2))
print("Numeradores:", numerators)  # [0.6065, ~0.0]

# Similitudes condicionales
p_cond = numerators / numerators.sum()
print("p(j|i=0):", p_cond)  # [~1.0, ~0.0] → x[1] es el vecino casi único

📐 La Distribución t-Student en Baja Dimensión y la Función de Costo

En el espacio de baja dimensión (típicamente 2D o 3D), t-SNE NO usa una Gaussiana. Usa una distribución t-Student con 1 grado de libertad (también llamada distribución de Cauchy):

$$q_{ij} = \frac{\left(1 + \|y_i - y_j\|^2\right)^{-1}}{\displaystyle\sum_{k \neq l}\left(1 + \|y_k - y_l\|^2\right)^{-1}}$$

Y la función de costo que minimiza es la Divergencia KL entre $P$ y $Q$:

$$\mathcal{L} = KL(P \| Q) = \sum_{i \neq j} p_{ij} \log \frac{p_{ij}}{q_{ij}}$$

¿Por Qué t-Student y No Gaussiana en Baja Dimensión?

Esa es LA pregunta. La t-Student tiene colas más pesadas que la Gaussiana. Esto significa que puntos moderadamente distantes en alta dimensión pueden estar bastante separados en 2D sin generar una penalización enorme. El resultado: los clusters quedan bien separados visualmente y los puntos dentro del cluster quedan comprimidos juntos.

Gradiente: Cómo Aprende t-SNE

La optimización ocurre por gradient descent. El gradiente de $\mathcal{L}$ respecto a $y_i$ es:

$$\frac{\partial \mathcal{L}}{\partial y_i} = 4 \sum_{j} (p_{ij} - q_{ij})(y_i - y_j)\left(1 + \|y_i - y_j\|^2\right)^{-1}$$

Lo que dice esta ecuación: si $p_{ij} > q_{ij}$ (los puntos están más separados en 2D de lo que deberían), el gradiente atrae $y_i$ hacia $y_j$. Si $p_{ij} < q_{ij}$, los repele. Es un sistema de fuerzas atractivas y repulsivas.

Implementación Production-Ready

from __future__ import annotations

import logging
from dataclasses import dataclass
from typing import Optional

import numpy as np
from sklearn.manifold import TSNE
from sklearn.preprocessing import StandardScaler

logger = logging.getLogger(__name__)


@dataclass
class TSNEConfig:
    """Configuración para reducción de dimensionalidad con t-SNE.

    Attributes:
        n_components: Dimensiones del espacio de salida (2 o 3).
        perplexity: Número efectivo de vecinos. Debe ser < n_samples / 3.
        learning_rate: Tasa de aprendizaje. 'auto' = max(N/early_exaggeration/4, 50).
        n_iter: Iteraciones máximas de optimización.
        random_state: Semilla para reproducibilidad.
        n_jobs: Paralelismo. -1 usa todos los cores.
        scale_input: Si True, aplica StandardScaler antes del embedding.
    """

    n_components: int = 2
    perplexity: float = 30.0
    learning_rate: float | str = "auto"
    n_iter: int = 1000
    random_state: int = 42
    n_jobs: int = -1
    scale_input: bool = True


@dataclass
class TSNEResult:
    """Resultado del embedding t-SNE.

    Attributes:
        embedding: Array (n_samples, n_components) con las coordenadas reducidas.
        kl_divergence: Valor final de la divergencia KL (menor es mejor).
        n_iter_actual: Iteraciones ejecutadas antes de converger.
    """

    embedding: np.ndarray
    kl_divergence: float
    n_iter_actual: int


class TSNEReducer:
    """Reductor de dimensionalidad basado en t-SNE con preprocesamiento integrado.

    Example:
        >>> import numpy as np
        >>> X = np.random.default_rng(42).standard_normal((200, 50))
        >>> reducer = TSNEReducer(TSNEConfig(perplexity=20))
        >>> result = reducer.fit_transform(X)
        >>> result.embedding.shape
        (200, 2)
    """

    def __init__(self, config: Optional[TSNEConfig] = None) -> None:
        """Inicializa el reductor con la configuración dada.

        Args:
            config: Configuración t-SNE. Si None, usa defaults.
        """
        self._config = config or TSNEConfig()
        self._scaler: Optional[StandardScaler] = None
        self._tsne: Optional[TSNE] = None

    def _validate_input(self, X: np.ndarray) -> None:
        """Valida dimensiones y perplejidad respecto al dataset.

        Args:
            X: Matriz de entrada (n_samples, n_features).

        Raises:
            ValueError: Si X no es 2D o la perplejidad es inválida.
        """
        if X.ndim != 2:
            raise ValueError(
                f"X debe ser 2D. Recibido: {X.ndim}D con shape {X.shape}."
            )

        n_samples = X.shape[0]
        max_perplexity = (n_samples - 1) / 3.0

        if self._config.perplexity >= max_perplexity:
            raise ValueError(
                f"perplexity={self._config.perplexity} debe ser < {max_perplexity:.1f} "
                f"(n_samples={n_samples} / 3). Reducí perplexity o aumentá el dataset."
            )

        if self._config.n_components not in (2, 3):
            raise ValueError(
                f"n_components debe ser 2 o 3 para visualización. "
                f"Recibido: {self._config.n_components}."
            )

    def fit_transform(self, X: np.ndarray) -> TSNEResult:
        """Escala (opcional) y aplica t-SNE al input.

        Args:
            X: Matriz (n_samples, n_features) con los datos en alta dimensión.

        Returns:
            TSNEResult con embedding, KL divergence e iteraciones.

        Raises:
            ValueError: Si los parámetros o el input son inválidos.
        """
        self._validate_input(X)

        X_processed = X.copy().astype(np.float64)

        if self._config.scale_input:
            logger.info("Aplicando StandardScaler antes de t-SNE.")
            self._scaler = StandardScaler()
            X_processed = self._scaler.fit_transform(X_processed)

        self._tsne = TSNE(
            n_components=self._config.n_components,
            perplexity=self._config.perplexity,
            learning_rate=self._config.learning_rate,
            n_iter=self._config.n_iter,
            random_state=self._config.random_state,
            n_jobs=self._config.n_jobs,
        )

        logger.info(
            "Iniciando t-SNE: n_samples=%d, n_features=%d, perplexity=%.1f",
            X_processed.shape[0],
            X_processed.shape[1],
            self._config.perplexity,
        )

        embedding = self._tsne.fit_transform(X_processed)

        result = TSNEResult(
            embedding=embedding,
            kl_divergence=float(self._tsne.kl_divergence_),
            n_iter_actual=int(self._tsne.n_iter_),
        )

        logger.info(
            "t-SNE completado. KL divergence=%.4f, iteraciones=%d",
            result.kl_divergence,
            result.n_iter_actual,
        )

        return result

🚨 Errores Comunes (y Cómo No Cagarla)

Error 1: Interpretar Distancias Globales Como Reales

El problema:

from sklearn.manifold import TSNE
import numpy as np

X = np.random.default_rng(0).standard_normal((500, 50))
embedding = TSNE(random_state=42).fit_transform(X)

# ❌ Medir distancia entre centroides de clusters en el embedding
centroid_a = embedding[:250].mean(axis=0)
centroid_b = embedding[250:].mean(axis=0)
dist_global = np.linalg.norm(centroid_a - centroid_b)
print(f"Distancia entre grupos: {dist_global:.2f}")  # ❌ Este número NO significa nada

Por qué falla:

t-SNE preserva estructura local, no global. La función de costo penaliza fuertemente separar vecinos cercanos, pero básicamente ignora la relación entre clusters distantes. La distancia entre clusters en el embedding es un artefacto de la inicialización y la perplejidad, no una propiedad del dato real.

$$KL(P \| Q) = \sum_{i \neq j} p_{ij} \log \frac{p_{ij}}{q_{ij}} \quad \Rightarrow \quad p_{ij} \approx 0 \text{ para pares lejanos, sin penalización real}$$

La solución:

# ✓ Usar t-SNE solo para visualizar estructura local/clusters
# ✓ Para distancias globales o similaridad entre clusters, usar PCA o UMAP
from sklearn.decomposition import PCA

pca = PCA(n_components=2, random_state=42)
embedding_pca = pca.fit_transform(X)  # ✓ Distancias globales sí son interpretables
explained = pca.explained_variance_ratio_.sum()
print(f"Varianza explicada: {explained:.2%}")

Error 2: Perplejidad Mal Calibrada

El problema:

from sklearn.manifold import TSNE
import numpy as np

X = np.random.default_rng(0).standard_normal((100, 20))

# ❌ Perplejidad mayor o igual a n_samples / 3
tsne_bad = TSNE(perplexity=50, random_state=42)  # 50 >= 100/3 ≈ 33.3
embedding = tsne_bad.fit_transform(X)  # Puede converger mal o no converger

Por qué falla:

La perplejidad define el número efectivo de vecinos que considera cada punto. Matemáticamente, calibra $\sigma_i$ mediante búsqueda binaria para satisfacer:

$$\text{Perp}(P_i) = 2^{H(P_i)}, \quad H(P_i) = -\sum_j p_{j|i} \log_2 p_{j|i}$$

Si perplexity supera $N/3$, cada punto intenta ser "vecino de todos" y el embedding colapsa en una nube sin estructura.

import numpy as np
n = 10
# Distribución uniforme (perplexity máxima posible)
p_uniform = np.ones(n - 1) / (n - 1)
H = -np.sum(p_uniform * np.log2(p_uniform + 1e-10))
print(f"Entropía máxima: {H:.3f} bits → Perp = {2**H:.1f}")  # ≈ n-1

La solución:

from sklearn.manifold import TSNE
import numpy as np

def safe_perplexity(n_samples: int, desired: float = 30.0) -> float:
    """Calcula perplejidad segura dado el tamaño del dataset."""
    max_safe = (n_samples - 1) / 3.0 - 1.0
    return min(desired, max_safe)

X = np.random.default_rng(0).standard_normal((100, 20))
perp = safe_perplexity(n_samples=100, desired=30.0)  # ✓ Retorna ~32.3 → usa 30
tsne = TSNE(perplexity=perp, random_state=42)
embedding = tsne.fit_transform(X)

Error 3: Usar t-SNE Para Features de un Modelo Downstream

El problema:

from sklearn.manifold import TSNE
from sklearn.ensemble import RandomForestClassifier
import numpy as np

X_train = np.random.default_rng(0).standard_normal((800, 50))
y_train = (X_train[:, 0] > 0).astype(int)
X_test = np.random.default_rng(1).standard_normal((200, 50))

# ❌ Usar embedding t-SNE como features para clasificación
tsne = TSNE(random_state=42)
X_train_emb = tsne.fit_transform(X_train)
# X_test_emb = tsne.transform(X_test)  ❌ TSNE no tiene .transform()
# Y aunque lo tuviera, el embedding cambia con cada run

Por qué falla:

t-SNE es transductivo: no existe función de proyección aprendida. sklearn.manifold.TSNE no implementa .transform() porque para incrustar un nuevo punto habría que re-optimizar toda la distribución $Q$. Además, el embedding no es determinista en estructura global entre corridas diferentes.

$$y_i^* = \arg\min_{y_i} KL(P \| Q) \quad \text{(optimización global, no función cerrada)}$$

La solución:

# ✓ Para features de modelos: PCA, UMAP (tiene .transform()), o Autoencoders
from sklearn.decomposition import PCA
from sklearn.ensemble import RandomForestClassifier
import numpy as np

X_train = np.random.default_rng(0).standard_normal((800, 50))
y_train = (X_train[:, 0] > 0).astype(int)
X_test = np.random.default_rng(1).standard_normal((200, 50))

# ✓ PCA sí tiene .transform() y es determinista
pca = PCA(n_components=10, random_state=42)
X_train_red = pca.fit_transform(X_train)   # ✓ fit en train
X_test_red = pca.transform(X_test)         # ✓ transform en test

clf = RandomForestClassifier(random_state=42)
clf.fit(X_train_red, y_train)
print(f"Accuracy: {clf.score(X_test_red, (X_test[:, 0] > 0).astype(int)):.3f}")

Error 4: No Escalar Antes de t-SNE

El problema:

import pandas as pd
from sklearn.manifold import TSNE

# Features con escalas completamente distintas
df = pd.DataFrame({
    "edad": [25, 30, 45],
    "salario": [3000000, 8000000, 15000000],  # rango: millones ❌
    "hijos": [0, 2, 3],
})

# ❌ Sin escalar: salario domina todas las distancias euclidianas
tsne = TSNE(random_state=42, perplexity=1.5)
embedding = tsne.fit_transform(df.values)

Por qué falla:

Las distancias euclidianas $\|x_i - x_j\|^2$ en alta dimensión son dominadas por la feature con mayor varianza. El kernel gaussiano se calibra sobre esas distancias sesgadas: salario domina y edad/hijos desaparecen.

La solución:

from sklearn.preprocessing import StandardScaler
from sklearn.manifold import TSNE
import numpy as np

X_raw = np.array([[25, 3_000_000, 0],
                  [30, 8_000_000, 2],
                  [45, 15_000_000, 3]], dtype=float)

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_raw)  # ✓ Media 0, std 1 por feature

tsne = TSNE(random_state=42, perplexity=1.5)
embedding = tsne.fit_transform(X_scaled)
print("Embedding escalado:\n", embedding.round(2))

🎯 Resumen Ejecutivo

t-SNE no es "el visualizador bonito". Es resolver la minimización:

$$\mathcal{L} = KL(P \| Q) = \sum_{i \neq j} p_{ij} \log \frac{p_{ij}}{q_{ij}}$$

Donde $P$ usa kernels Gaussianos en alta dimensión y $Q$ usa distribución t-Student en baja dimensión, optimizado por:

$$\frac{\partial \mathcal{L}}{\partial y_i} = 4 \sum_{j}(p_{ij} - q_{ij})(y_i - y_j)\!\left(1 + \|y_i - y_j\|^2\right)^{-1}$$

Con todas las implicaciones que eso conlleva:

  • Solo preserva estructura local: Las distancias entre clusters NO son interpretables. Para comparaciones globales, usá PCA o UMAP.
  • Sin .transform(): t-SNE es transductivo. Si necesitás proyectar nuevos puntos, UMAP o PCA son la opción correcta.
  • Perplejidad es un hiperparámetro real: Calibrala entre 5 y (n_samples - 1) / 3. Probá múltiples valores y comparás los KL divergences finales.
  • Escalá siempre: StandardScaler antes de t-SNE no es opcional en datos reales con features heterogéneas.
