¿Qué es un DataFrame y Por Qué Debería Importarte?

Un DataFrame es una estructura tabular bidimensional (filas × columnas) con índices etiquetados. Piensa en esto como una tabla de Excel pero con esteroides y sin crashearse cuando tienes 10 millones de registros.

La Matemática Detrás (Que Nadie Te Cuenta)

Un DataFrame es esencialmente una colección de vectores columna que comparten un índice:

$$D=[c1c2⋯cn]∈Rm×n$$

Donde:

  • $m$ = número de filas (observaciones)
  • $n$ = número de columnas (features)
  • $\vec{c}_j$ = columna $j$ con tipo de dato potencialmente distinto

¿Por qué esto importa? Porque internamente pandas optimiza operaciones vectorizadas con NumPy. Si haces un loop sobre filas, estás botando rendimiento como si sacaras café por el desagüe. La complejidad de operaciones vectorizadas es $O(n)$ vs $O(n \cdot k)$ de loops anidados.

2. Transformaciones Vectorizadas (El Arte de NO Usar Loops)

class DataFrameTransformer:
    """
    Pipeline de transformaciones optimizadas para DataFrames
    
    Principio: Todas las operaciones DEBEN ser vectorizadas
    Complejidad: O(n) vs O(n²) de loops
    """
    __slots__ = ()
    
    @staticmethod
    def compute_rolling_features(
        df: pd.DataFrame,
        value_col: str,
        window_sizes: list[int],
        groupby_col: Optional[str] = None
    ) -> pd.DataFrame:
        """
        Calcula features de ventana deslizante
        
        Matemática:
        Para ventana de tamaño w:
        $$\mu_w(t) = \frac{1}{w}\sum_{i=t-w+1}^{t} x_i$$
        $$\sigma_w(t) = \sqrt{\frac{1}{w}\sum_{i=t-w+1}^{t} (x_i - \mu_w(t))^2}$$
        
        Por qué .rolling() y no loops:
        - Implementado en Cython (C-level performance)
        - Manejo eficiente de memoria con ventanas
        - Automáticamente maneja edge cases (inicio de serie)
        """
        result = df.copy()
        
        for window in window_sizes:
            # Agrupar si se especificó (ej: por user_id)
            if groupby_col:
                grouped = result.groupby(groupby_col)[value_col]
            else:
                grouped = result[value_col]
            
            # Rolling mean
            result[f'{value_col}_rolling_mean_{window}'] = (
                grouped.rolling(window=window, min_periods=1).mean()
            )
            
            # Rolling std
            result[f'{value_col}_rolling_std_{window}'] = (
                grouped.rolling(window=window, min_periods=1).std()
            )
            
            # Rolling max/min (para detectar outliers)
            result[f'{value_col}_rolling_max_{window}'] = (
                grouped.rolling(window=window, min_periods=1).max()
            )
            
        
logger.info
(f"Generadas {len(window_sizes) * 3} features rolling")
        return result
    
    @staticmethod
    def encode_categorical_with_frequency(
        df: pd.DataFrame,
        cat_col: str,
        min_frequency: float = 0.01
    ) -> pd.DataFrame:
        """
        Encoding categórico basado en frecuencia (mejor que One-Hot en alta cardinalidad)
        
        Matemática:
        $$f(c) = \frac{\text{count}(c)}{N}$$
        
        Donde c es la categoría y N el total de observaciones.
        
        Por qué esto vs One-Hot:
        - One-Hot explota en dimensionalidad con >100 categorías
        - Frequency encoding preserva orden de importancia
        - Evita sparse matrices gigantes que matan la memoria
        """
        result = df.copy()
        
        # Calcular frecuencias
        freq_map = df[cat_col].value_counts(normalize=True)
        
        # Agrupar categorías raras
        rare_threshold = min_frequency
        freq_map = freq_map.where(freq_map >= rare_threshold, other='_RARE_')
        
        # Aplicar encoding
        result[f'{cat_col}_freq'] = df[cat_col].map(
            freq_
map.to
_dict()
        ).fillna(0.0)  # Categorías no vistas = 0
        
        
logger.info
(
            f"Encoded '{cat_col}': {len(freq_map)} categorías únicas "
            f"({(freq_map == '_RARE_').sum()} agrupadas como raras)"
        )
        
        return result
    
    @staticmethod
    def handle_missing_data(
        df: pd.DataFrame,
        strategy: Literal['mean', 'median', 'mode', 'ffill', 'drop'] = 'median',
        threshold: float = 0.5
    ) -> pd.DataFrame:
        """
        Manejo robusto de missing data
        
        Decisión de arquitectura:
        - threshold: Elimina columnas con >50% nulls (no son salvables)
        - median vs mean: median es robusta a outliers
        - ffill: Forward fill para series temporales
        
        Complejidad: O(n·m) donde n=filas, m=columnas
        """
        result = df.copy()
        
        # 1. Eliminar columnas con demasiados nulls
        null_pct = result.isnull().mean()
        cols_to_drop = null_pct[null_pct > threshold].index
        
        if len(cols_to_drop) > 0:
            logger.warning(
                f"Eliminando {len(cols_to_drop)} columnas con >{threshold*100}% nulls: "
                f"{list(cols_to_drop)}"
            )
            result = result.drop(columns=cols_to_drop)
        
        # 2. Imputar según estrategia
        numeric_cols = 
result.select
_dtypes(include=[np.number]).columns
        
        if strategy == 'mean':
            result[numeric_cols] = result[numeric_cols].fillna(
                result[numeric_cols].mean()
            )
        elif strategy == 'median':
            result[numeric_cols] = result[numeric_cols].fillna(
                result[numeric_cols].median()
            )
        elif strategy == 'ffill':
            result = result.fillna(method='ffill').fillna(method='bfill')
        elif strategy == 'drop':
            result = result.dropna()
        
        
logger.info
(f"Missing data manejada con estrategia '{strategy}'")
        return result

class DataFrameAnalyzer:
    """Análisis exploratorio production-grade"""
    __slots__ = ()
    
    @staticmethod
    def generate_quality_report(df: pd.DataFrame) -> DataQualityReport:
        """
        Genera reporte completo de calidad
        
        Complejidad: O(n·m) - inevitable para análisis completo
        Optimización: Usa métodos nativos de pandas (escritos en C)
        """
        # Conteo de nulls (vectorizado)
        missing_cells = df.isnull().sum().sum()
        total_cells = df.shape[0] * df.shape[1]
        
        # Memoria (incluye overhead de índice)
        memory_usage = df.memory_usage(deep=True).sum() / 1024**2
        
        # Tipos de columnas
        numeric_cols = len(
df.select
_dtypes(include=[np.number]).columns)
        categorical_cols = len(
df.select
_dtypes(include=['object', 'category']).columns)
        datetime_cols = len(
df.select
_dtypes(include=['datetime64']).columns)
        
        return DataQualityReport(
            total_rows=len(df),
            total_cols=len(df.columns),
            missing_cells=missing_cells,
            missing_percentage=(missing_cells / total_cells * 100) if total_cells > 0 else 0.0,
            duplicate_rows=df.duplicated().sum(),
            memory_usage_mb=memory_usage,
            numeric_cols=numeric_cols,
            categorical_cols=categorical_cols,
            datetime_cols=datetime_cols
        )
    
    @staticmethod
    def detect_outliers_iqr(
        df: pd.DataFrame,
        column: str,
        multiplier: float = 1.5
    ) -> pd.Series:
        """
        Detección de outliers usando IQR (Interquartile Range)
        
        Matemática:
        $$\text{IQR} = Q_3 - Q_1$$
        $$\text{Lower Bound} = Q_1 - k \cdot \text{IQR}$$
        $$\text{Upper Bound} = Q_3 + k \cdot \text{IQR}$$
        
        Donde k=1.5 es estándar (Tukey's fences)
        
        Por qué IQR vs Z-score:
        - IQR es robusto a distribuciones no-normales
        - Z-score asume normalidad (raramente se cumple en producción)
        - IQR no se afecta por outliers extremos
        """
        Q1 = df[column].quantile(0.25)
        Q3 = df[column].quantile(0.75)
        IQR = Q3 - Q1
        
        lower_bound = Q1 - multiplier * IQR
        upper_bound = Q3 + multiplier * IQR
        
        is_outlier = (df[column] < lower_bound) | (df[column] > upper_bound)
        
        
logger.info
(
            f"Outliers en '{column}': {is_outlier.sum()} "
            f"({is_outlier.sum()/len(df)*100:.2f}%)"
        )
        
        return is_outlier

from typing import Callable

class MLDataPipeline:

    - Logging en cada stage para debugging
    """
    __slots__ = ('_transformers', '_name')
    
    def __init__(self, name: str = "default_pipeline") -> None:
        self._transformers: list[Callable[[pd.DataFrame], pd.DataFrame]] = []
        self._name = name
    
    def add_step(

Complejidad Computacional: La Verdad Que Nadie Te Dice

Operación Complejidad Notas
df[col] $O(1)$ Acceso por columna (hash lookup)
df.loc[idx] $O(\log n)$ Binary search si índice ordenado
df.iterrows() $O(n \cdot m)$ EVITAR - Loop Python puro
df.apply() con función custom $O(n)$ Pero con overhead de Python
Operaciones vectorizadas (+, *, etc) $O(n)$ Implementado en C (NumPy)
df.groupby().agg() $O(n \log n)$ Sort interno para agrupación
df.merge() $O(n + m)$ Hash join (mejor caso)
df.sort_values() $O(n \log n)$ Timsort (Python)

La regla de oro: Si escribiste un for loop recorriendo filas, lo estás haciendo mal en el 95% de los casos.

Optimizaciones de Memoria: Cuando Tienes 50GB de CSV

def load_large_csv_chunked(
    path: Path,
    chunksize: int = 100_000,
    filter_func: Optional[Callable[[pd.DataFrame], pd.DataFrame]] = None
) -> pd.DataFrame:
    """
    Carga CSVs gigantes por chunks para evitar OOM
    
    Matemática del problema:
    Si un DataFrame de n filas usa M bytes de RAM:
    $$\text{RAM}_{\text{required}} = n \cdot m \cdot \text{bytes\_per\_cell}$$
    
    Chunking reduce peak memory a:
    $$\text{RAM}_{\text{peak}} = c \cdot m \cdot \text{bytes\_per\_cell}$$
    
    Donde c es el tamaño del chunk (c << n)
    """
    chunks: list[pd.DataFrame] = []
    
    
logger.info
(f"Cargando CSV en chunks de {chunksize:,} filas")
    
    for i, chunk in enumerate(
pd.read
_csv(path, chunksize=chunksize), 1):
        # Filtrar chunk si se especificó
        if filter_func:
            chunk = filter_func(chunk)
        
        # Optimizar tipos (reduce 50-70% memoria)
        chunk = optimize_dtypes(chunk)
        
        chunks.append(chunk)
        
        if i % 10 == 0:
            
logger.info
(f"Procesados {i * chunksize:,} filas")
    
    # Concatenar (pandas optimiza internamente)
    result = pd.concat(chunks, ignore_index=True)
    
logger.info
(f"Total cargado: {len(result):,} filas")
    
    return result

def optimize_dtypes(df: pd.DataFrame) -> pd.DataFrame:
    """
    Reduce uso de memoria optimizando dtypes
    
    Ahorro típico: 50-70% de RAM
    
    Ejemplos:
    - int64 → int32 si max_value < 2^31
    - float64 → float32 (suficiente para ML)
    - object → category si cardinalidad < 50%
    """
    result = df.copy()
    
    # Optimizar enteros
    for col in 
result.select
_dtypes(include=['int64']).columns:
        col_min = result[col].min()
        col_max = result[col].max()
        
        if col_min >= 0:
            if col_max < 2**8:
                result[col] = result[col].astype('uint8')
            elif col_max < 2**16:
                result[col] = result[col].astype('uint16')
            elif col_max < 2**32:
                result[col] = result[col].astype('uint32')
        else:
            if col_min > -2**7 and col_max < 2**7:
                result[col] = result[col].astype('int8')
            elif col_min > -2**15 and col_max < 2**15:
                result[col] = result[col].astype('int16')
            elif col_min > -2**31 and col_max < 2**31:
                result[col] = result[col].astype('int32')
    
    # Optimizar floats
    for col in 
result.select
_dtypes(include=['float64']).columns:
        result[col] = result[col].astype('float32')
    
    # Optimizar categóricas
    for col in 
result.select
_dtypes(include=['object']).columns:
        num_unique = result[col].nunique()
        num_total = len(result[col])
        
        if num_unique / num_total < 0.5:  # <50% valores únicos
            result[col] = result[col].astype('category')
    
    return result

  • 20% del tiempo se gasta entrenando modelos bonitos

Si no dominas pandas, estás intentando correr antes de caminar. Los DataFrames son tu navaja suiza para:

  1. ETL robusto con validación y logging
  2. Feature engineering vectorizado (nada de loops)
  3. Análisis exploratorio que te salva de entrenar con basura
  4. Optimización de memoria para datasets industriales

La diferencia entre un junior y un senior no es saber más algoritmos de ML, es dominar el data wrangling production-ready. Cuando tu pipeline de datos no falla a las 3AM, cuando procesas 10M de filas en minutos en vez de horas, cuando tus features tienen sentido matemático... ahí es donde generas valor real.

Hágale, practique estas técnicas en un proyecto real. Y la próxima vez que vea un for row in df.iterrows(), acuérdese de este artículo y refactoricelo con operaciones vectorizadas. Su servidor de producción se lo agradecerá. 🚀
