Kernel Trick - El Truco Matemático que Proyecta a Infinitas Dimensiones Sin Calcularlas

29 de enero de 2026
sklearn.svm.SVC kernel='rbf'

Kernel Trick - Día 43

Hágale pues. Llegamos al Día 43 y no, el Kernel Trick no es "ese truco mágico para hacer SVM no lineal". Después de 12 años viendo modelos en producción, puedo decirte que el 85% de los científicos de datos usan kernel='rbf' sin entender qué diablos está pasando matemáticamente. Lo ponen porque "funciona mejor", tunean gamma a ciegas, y cuando el modelo se demora 3 horas en entrenar con 100K muestras, no entienden por qué.

El Kernel Trick es la razón por la que SVM puede clasificar patrones que ningún modelo lineal tocaría. Pero también es la razón por la que tu pipeline se vuelve lentísimo si no escalas features, por la que overfitteas sin darte cuenta cuando gamma es muy alto, y por la que gastas RAM como si no hubiera mañana.

Hoy vamos a destruir el mito, entender la matemática que hace posible este "truco", y escribir código production-ready que use kernel='rbf' en SVC sin romper tu servidor. Dejá el café y acomodate, porque esta vaina es profunda pero vale cada segundo.

🎯 ¿Cómo Proyectas Datos a Infinitas Dimensiones Sin Calcularlas?

Mirá, el Kernel Trick es como cuando necesitás organizar una biblioteca gigante pero en vez de mover físicamente todos los libros a estantes nuevos, solo cambias el sistema de catálogo. Los libros siguen donde están (tus features originales), pero ahora podés encontrar relaciones que antes eran imposibles de ver.

Definición técnica (sin rodeos):

Un kernel $K:\mathbb{R}^d\times\mathbb{R}^d\to\mathbb{R}$ es una función que computa el producto punto entre vectores en un espacio de features de alta dimensionalidad $\phi:\mathbb{R}^d\to\mathbb{R}^D$ (donde $D\gg d$ o incluso $D=\infty$) sin calcular explícitamente la transformación $\phi(x)$. Matemáticamente:

$$
K(x_i,x_j)=\langle\phi(x_i),\phi(x_j)\rangle=\phi(x_i)^T\phi(x_j)
$$

Tipos principales de kernels:

  • Linear Kernel: $K(x_i,x_j)=x_i^Tx_j$ - No hay transformación, es SVM clásico lineal.
  • Polynomial Kernel: $K(x_i,x_j)=(\gamma x_i^Tx_j+r)^p$ - Proyecta a espacio polinomial de grado $p$.
  • RBF (Radial Basis Function): $K(x_i,x_j)=\exp(-\gamma|x_i-x_j|^2)$ - Proyecta a espacio infinito-dimensional.
  • Sigmoid Kernel: $K(x_i,x_j)=\tanh(\gamma x_i^Tx_j+r)$ - Simula redes neuronales.

Advertencia brutal: Confundir "kernel" con "transformación de features" es el error #1. El kernel reemplaza la necesidad de transformar. Si hacés PolynomialFeatures(degree=3) y luego kernel='poly', estás duplicando el trabajo y probablemente rompiendo todo.

📐 La Magia Matemática: RBF Kernel

Acá es donde la mayoría abandona o se queda con "es una gaussiana mágica". Pero esta ecuación tiene TODO el poder de convertir tu problema lineal imposible en uno resoluble:

$$
K_{\text{RBF}}(x_i,x_j)=\exp\left(-\gamma|x_i-x_j|^2\right)
$$

Esta ecuación parece simple pero es la puerta a un espacio de infinitas dimensiones. No estoy exagerando: el feature space implícito del RBF kernel tiene dimensionalidad infinita. Vamos a desmenuzarla símbolo por símbolo porque acá es donde la mayoría se pierde y termina usando gamma='scale' sin entender qué hace.

Descomposición Completa: Entendiendo Cada Símbolo

$K_{\text{RBF}}(x_i,x_j)$ (Kernel Value) - Similitud Entre Dos Muestras

K(xi, xj) = qué tan "cercanos" son xi y xj en el espacio transformado
          = producto punto en espacio infinito-dimensional
          = valor entre 0 y 1

Analogía: Es como medir qué tan compatibles son dos personas, pero en vez de preguntarles 3 cosas (edad, ciudad, hobbies), les preguntas INFINITAS cosas y aún así obtienes una sola respuesta numérica.

En Machine Learning:

  • Para $x_i=x_j$: $K(x_i,x_i)=1$ (similitud máxima consigo mismo)
  • Para $x_i$ muy lejano de $x_j$: $K(x_i,x_j)\to 0$ (sin similitud)
  • Para SVM: Este valor reemplaza $x_i^Tx_j$ en el problema de optimización dual

Dimensionalidad importa: En el espacio original tienes $d$ features. El RBF kernel proyecta implícitamente a $\mathbb{R}^\infty$, pero nunca calculas esas infinitas dimensiones. Solo calculas este valor escalar.

Diferencia CRÍTICA: $K(x_i,x_j)\neq|x_i-x_j|$. El kernel mide similitud (alto = cerca), la distancia euclidiana mide separación (alto = lejos).

$\gamma$ (Gamma Parameter) - Radio de Influencia

γ = 1 / (2σ²)  donde σ es el ancho de la gaussiana
  = qué tan "local" es la influencia de cada punto
  = inverso del alcance espacial

Analogía: Gamma es como el volumen de tu voz. Gamma alto = susurras (solo te escuchan los que están MUY cerca). Gamma bajo = gritas (te escuchan desde lejos, incluso gente irrelevante).

En Machine Learning:

  • Para $\gamma$ grande: Fronteras de decisión muy complejas, riesgo de overfitting
  • Para $\gamma$ pequeño: Fronteras de decisión suaves, riesgo de underfitting
  • Valor por defecto en scikit-learn: gamma='scale' = $\frac{1}{d\cdot\text{Var}(X)}$

Dimensionalidad importa: Si tenés 100 features, el mismo valor de gamma tendrá efecto MUY diferente que con 2 features. Por eso el default es 'scale' (ajusta según número de features).

Diferencia CRÍTICA: Gamma NO es un parámetro de regularización como $C$. $C$ controla el trade-off margen vs errores, gamma controla la complejidad del espacio de features.

Ejemplo numérico concreto:

import numpy as np

# Dos puntos en 2D
x1 = np.array([1.0, 2.0])
x2 = np.array([1.5, 2.3])

# Gamma bajo (influencia amplia)
gamma_low = 0.1
k_low = np.exp(-gamma_low * np.sum((x1 - x2)**2))
print(f"Gamma=0.1: K(x1,x2) = {k_low:.4f}")  # ≈ 0.9802 (muy similar)

# Gamma alto (influencia local)
gamma_high = 10.0
k_high = np.exp(-gamma_high * np.sum((x1 - x2)**2))
print(f"Gamma=10.0: K(x1,x2) = {k_high:.4f}")  # ≈ 0.0003 (casi sin similitud)

$|x_i-x_j|^2$ (Squared Euclidean Distance) - Separación Espacial

‖xi - xj‖² = suma de diferencias al cuadrado en cada dimensión
           = (x1i - x1j)² + (x2i - x2j)² + ... + (xdi - xdj)²
           = distancia² entre puntos en espacio original

Analogía: Es la distancia que medirías con una regla si tus features fueran coordenadas en un mapa. Pero elevada al cuadrado para penalizar más las distancias grandes.

En Machine Learning:

  • Para features sin escalar: Esta distancia está dominada por la feature con mayor rango
  • Para features escaladas: Todas contribuyen equitativamente
  • Para features correlacionadas: Puede sobre-representar información redundante

Dimensionalidad importa: Con 100 features, esta suma tiene 100 términos. Si una feature tiene rango [0, 1000] y otra [0, 1], la primera domina completamente.

Diferencia CRÍTICA: La distancia se calcula en el espacio original de $d$ dimensiones, pero el kernel la usa para computar similitud en espacio de $\infty$ dimensiones.

$\exp(\cdot)$ (Exponential Function) - Función de Similitud

exp(-z) = e^(-z)
        = convierte distancia (grande = malo) en similitud (grande = bueno)
        = siempre positivo, valor en (0, 1]

Analogía: Es como convertir una calificación de "errores cometidos" a "puntos ganados". Muchos errores (distancia grande) → pocos puntos (similitud baja).

En Machine Learning:

  • Para $z=0$ (puntos idénticos): $\exp(0)=1$ (similitud perfecta)
  • Para $z\to\infty$ (puntos muy lejanos): $\exp(-\infty)\to 0$ (sin similitud)
  • Decaimiento exponencial: La similitud cae MUY rápido con la distancia

Dimensionalidad importa: La exponencial magnifica el efecto de gamma. Un cambio pequeño en $\gamma|x_i-x_j|^2$ produce un cambio GRANDE en la similitud.

Ejemplo numérico completo:

import numpy as np
from typing import Tuple

def rbf_kernel_manual(
    x_i: np.ndarray,
    x_j: np.ndarray,
    gamma: float
) -> Tuple[float, float, float]:
    """
    Calcula RBF kernel paso a paso.
    
    Args:
        x_i: Muestra i (shape: [d,])
        x_j: Muestra j (shape: [d,])
        gamma: Parámetro gamma
        
    Returns:
        Tupla con (distancia², argumento exp, kernel value)
    """
    # Paso 1: Distancia euclidiana al cuadrado
    diff = x_i - x_j
    dist_sq = np.sum(diff ** 2)
    
    # Paso 2: Multiplicar por -gamma
    exp_arg = -gamma * dist_sq
    
    # Paso 3: Aplicar exponencial
    kernel_value = np.exp(exp_arg)
    
    return dist_sq, exp_arg, kernel_value

# Ejemplo con datos reales
x1 = np.array([2.5, 3.7])
x2 = np.array([2.8, 4.1])
gamma = 1.0

dist_sq, exp_arg, k_val = rbf_kernel_manual(x1, x2, gamma)

print(f"‖x1 - x2‖² = {dist_sq:.4f}")        # 0.2500
print(f"-γ·‖x1 - x2‖² = {exp_arg:.4f}")     # -0.2500
print(f"K(x1, x2) = {k_val:.4f}")           # 0.7788

¿Por Qué Funciona el Kernel Trick?

La respuesta está en el teorema de Mercer y la expansión de Taylor. El RBF kernel se puede escribir como:

$$
K_{\text{RBF}}(x,y)=\exp(-\gamma|x-y|^2)=\sum_{n=0}^{\infty}\frac{(-\gamma)^n}{n!}|x-y|^{2n}
$$

Cada término de esta suma infinita representa una dimensión en el feature space transformado. Pero nunca calculamos esta suma: la exponencial la computa directamente.

En SVM, esto significa:

En lugar de resolver:

$$
\max_{\alpha}\sum_{i=1}^{n}\alpha_i-\frac{1}{2}\sum_{i=1}^{n}\sum_{j=1}^{n}\alpha_i\alpha_jy_iy_j\phi(x_i)^T\phi(x_j)
$$

Resolvemos:

$$
\max_{\alpha}\sum_{i=1}^{n}\alpha_i-\frac{1}{2}\sum_{i=1}^{n}\sum_{j=1}^{n}\alpha_i\alpha_jy_iy_jK(x_i,x_j)
$$

La complejidad pasa de $O(n^2\cdot D)$ con $D=\infty$ (imposible) a $O(n^2\cdot d)$ donde $d$ es la dimensión original (factible).

📐 Gamma y su Impacto en la Frontera de Decisión

Gamma controla la "localidad" del kernel y por tanto la complejidad de la frontera de decisión. Vamos a ver esto matemáticamente:

Para un punto de prueba $x_{\text{test}}$, la predicción en SVM con RBF kernel es:

$$
f(x_{\text{test}})=\text{sign}\left(\sum_{i\in SV}\alpha_iy_iK(x_i,x_{\text{test}})+b\right)
$$

Donde $SV$ son los vectores de soporte. Expandiendo el kernel:

$$
f(x_{\text{test}})=\text{sign}\left(\sum_{i\in SV}\alpha_iy_i\exp(-\gamma|x_i-x_{\text{test}}|^2)+b\right)
$$

Análisis del comportamiento:

  • Gamma grande ($\gamma\to\infty$): $\exp(-\gamma d^2)\to 0$ rápidamente. Solo los puntos MUY cercanos a $x_i$ tienen influencia. Frontera de decisión con "islas" alrededor de cada vector de soporte.
  • Gamma pequeño ($\gamma \to 0$): $\exp(-\gamma d^2)\to 1$ para casi todos los puntos. Todos los vectores de soporte tienen influencia global. Frontera de decisión suave.

Ejemplo numérico comparativo:

import numpy as np
from sklearn.svm import SVC
from sklearn.datasets import make_moons
from sklearn.preprocessing import StandardScaler
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

# Generar datos no lineales
X, y = make_moons(n_samples=100, noise=0.15, random_state=42)

# CRÍTICO: escalar antes de RBF
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Gamma bajo (underfitting)
svm_low = SVC(kernel='rbf', gamma=0.01, C=1.0)
svm_low.fit(X_scaled, y)
logger.info(f"Gamma=0.01: {svm_low.n_support_.sum()} support vectors")

# Gamma óptimo (balanced)
svm_mid = SVC(kernel='rbf', gamma=1.0, C=1.0)
svm_mid.fit(X_scaled, y)
logger.info(f"Gamma=1.0: {svm_mid.n_support_.sum()} support vectors")

# Gamma alto (overfitting)
svm_high = SVC(kernel='rbf', gamma=100.0, C=1.0)
svm_high.fit(X_scaled, y)
logger.info(f"Gamma=100.0: {svm_high.n_support_.sum()} support vectors")

# El gamma alto tiene MUCHOS más support vectors
# porque necesita "memorizar" cada región local

🚨 Errores Comunes (y Cómo No Cagarla)

Error 1: Usar RBF Kernel Sin Escalar Features

El problema:

from sklearn.svm import SVC
import numpy as np

# Features con rangos muy diferentes ❌
X = np.array([
    [1.0, 1000.0],    # feature 1: [0, 10], feature 2: [0, 10000]
    [2.0, 2000.0],
    [8.0, 500.0],
    [9.0, 800.0]
])
y = np.array([0, 0, 1, 1])

# Entrenar SVM sin escalar ❌
svm = SVC(kernel='rbf', gamma='scale')
svm.fit(X, y)
# La feature 2 DOMINA completamente la distancia euclidiana

Por qué falla:

La distancia euclidiana en RBF kernel está dominada por la feature con mayor rango:

$$
|x_i-x_j|^2=(1-2)^2+(1000-2000)^2=1+1{,}000{,}000=1{,}000{,}001
$$

La primera feature (diferencia de 1) es completamente ignorada frente a la segunda (diferencia de 1000). El modelo solo "ve" la feature 2.

# Visualización del problema
import numpy as np

x1 = np.array([1.0, 1000.0])
x2 = np.array([2.0, 2000.0])

diff_sq = (x1 - x2) ** 2
print(f"Contribución feature 1: {diff_sq[0]}")      # 1.0
print(f"Contribución feature 2: {diff_sq[1]}")      # 1000000.0
print(f"Proporción: {diff_sq[1] / diff_sq[0]:.0f}x") # 1000000x

La solución:

from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
import numpy as np

# Pipeline con escalado ANTES de SVM ✓
pipeline = Pipeline([
    ('scaler', StandardScaler()),
    ('svm', SVC(kernel='rbf', gamma='scale', C=1.0))
])

X = np.array([
    [1.0, 1000.0],
    [2.0, 2000.0],
    [8.0, 500.0],
    [9.0, 800.0]
])
y = np.array([0, 0, 1, 1])

pipeline.fit(X, y)
# Ahora ambas features contribuyen equitativamente

Error 2: No Entender gamma='scale' vs gamma='auto'

El problema:

from sklearn.svm import SVC

# Usar default sin saber qué hace ❌
svm = SVC(kernel='rbf')  # gamma='scale' por defecto
svm.fit(X_train, y_train)
# ¿Qué valor de gamma se está usando realmente?

Por qué falla:

gamma='scale' y gamma='auto' calculan gamma automáticamente pero con fórmulas MUY diferentes:

$$
\gamma_{\text{scale}}=\frac{1}{d\cdot\text{Var}(X)}
$$

$$
\gamma_{\text{auto}}=\frac{1}{d}
$$

Donde $d$ es el número de features. Si tu dataset tiene features con varianza muy baja o muy alta, 'scale' puede dar valores de gamma completamente inadecuados.

import numpy as np
from sklearn.svm import SVC

# Dataset con varianza extrema
X = np.random.randn(100, 10) * 0.01  # Varianza muy baja
y = np.random.randint(0, 2, 100)

# Gamma calculado automáticamente
svm_scale = SVC(kernel='rbf', gamma='scale')
svm_scale.fit(X, y)

# Valor real de gamma
d = X.shape[1]
var_X = np.var(X)
gamma_scale = 1 / (d * var_X)
print(f"Gamma 'scale': {gamma_scale:.2f}")  # Puede ser ENORME

gamma_auto = 1 / d
print(f"Gamma 'auto': {gamma_auto:.2f}")    # Mucho más razonable

La solución:

from sklearn.svm import SVC
from sklearn.model_selection import GridSearchCV
from sklearn.preprocessing import StandardScaler
import numpy as np
import logging

logger = logging.getLogger(__name__)

class RBFSVMOptimizer:
    """
    SVM con RBF kernel y tuning automático de gamma y C.
    """
    
    def __init__(self, cv: int = 5, verbose: int = 1):
        """
        Args:
            cv: Número de folds para cross-validation
            verbose: Nivel de verbosidad (0-3)
        """
        self.cv = cv
        self.verbose = verbose
        self.scaler = StandardScaler()
        self.best_estimator_ = None
        
    def fit(self, X: np.ndarray, y: np.ndarray) -> 'RBFSVMOptimizer':
        """
        Entrena SVM con grid search sobre gamma y C.
        """
        # Escalar features
        X_scaled = self.scaler.fit_transform(X)
        
        # Grid de hiperparámetros
        d = X.shape[1]
        param_grid = {
            'gamma': [1/(d*10), 1/d, 10/d],  # Explícito basado en dimensionalidad
            'C': [0.1, 1.0, 10.0, 100.0]
        }
        
        # Grid search
        svm = SVC(kernel='rbf', random_state=42)
        grid = GridSearchCV(
            svm, param_grid, cv=self.cv,
            scoring='accuracy', verbose=self.verbose, n_jobs=-1
        )
        grid.fit(X_scaled, y)
        
        self.best_estimator_ = grid.best_estimator_
        logger.info(f"Best gamma: {grid.best_params_['gamma']:.4f}")
        logger.info(f"Best C: {grid.best_params_['C']:.2f}")
        
        return self
    
    def predict(self, X: np.ndarray) -> np.ndarray:
        """Predicción con modelo óptimo."""
        X_scaled = self.scaler.transform(X)
        return self.best_estimator_.predict(X_scaled)

Error 3: Usar RBF Kernel Para Datos Linealmente Separables

El problema:

from sklearn.datasets import make_classification
from sklearn.svm import SVC
import time

# Datos linealmente separables ❌
X, y = make_classification(
    n_samples=10000, n_features=20,
    n_informative=20, n_redundant=0,
    n_clusters_per_class=1, class_sep=2.0,
    random_state=42
)

# Usar RBF innecesariamente ❌
start = time.time()
svm_rbf = SVC(kernel='rbf', gamma='scale')
svm_rbf.fit(X, y)
time_rbf = time.time() - start
print(f"RBF training time: {time_rbf:.2f}s")  # LENTO

Por qué falla:

El RBF kernel tiene complejidad $O(n^2\cdot d)$ para calcular la matriz kernel, donde $n$ es el número de muestras. Para datos linealmente separables, esto es overkill total:

$$
\text{Costo}_{\text{RBF}}=O(n^2\cdot d)+O(n^3)\text{ (optimización)}
$$

$$
\text{Costo}_{\text{Linear}}=O(n\cdot d)+O(n\cdot d)\text{ (optimización lineal)}
$$

Con 10,000 muestras: $10{,}000^2=100{,}000{,}000$ operaciones vs $10{,}000$ operaciones.

La solución:

from sklearn.datasets import make_classification
from sklearn.svm import SVC, LinearSVC
from sklearn.preprocessing import StandardScaler
import time
import numpy as np

# Datos linealmente separables
X, y = make_classification(
    n_samples=10000, n_features=20,
    n_informative=20, n_redundant=0,
    n_clusters_per_class=1, class_sep=2.0,
    random_state=42
)

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Probar kernel lineal primero ✓
start = time.time()
svm_linear = LinearSVC(max_iter=1000, dual=False)
svm_linear.fit(X_scaled, y)
time_linear = time.time() - start

# RBF solo si lineal no funciona
start = time.time()
svm_rbf = SVC(kernel='rbf', gamma='scale')
svm_rbf.fit(X_scaled, y)
time_rbf = time.time() - start

print(f"Linear: {time_linear:.3f}s | Acc: {svm_linear.score(X_scaled, y):.4f}")
print(f"RBF: {time_rbf:.3f}s | Acc: {svm_rbf.score(X_scaled, y):.4f}")
print(f"Speedup: {time_rbf/time_linear:.1f}x más lento con RBF")

Error 4: Ignorar la Matriz Kernel en Memoria

El problema:

from sklearn.svm import SVC
import numpy as np

# Dataset grande ❌
X_large = np.random.randn(50000, 100)
y_large = np.random.randint(0, 2, 50000)

# Intentar entrenar SVM con RBF ❌
svm = SVC(kernel='rbf', gamma='scale')
svm.fit(X_large, y_large)  # MemoryError o swap infinito

Por qué falla:

SVM con RBF kernel debe calcular y almacenar la matriz kernel $K \in\mathbb{R}^{n\times n}$:

$$
K_{ij}=\exp(-\gamma|x_i-x_j|^2)\quad\forall i,j\in{1,\ldots,n}
$$

Para $n=50{,}000$:

$$
\text{Memoria}=50{,}000\times 50{,}000\times 8\text{ bytes}=20\text{ GB}
$$

Esto antes de cualquier optimización.

La solución:

from sklearn.svm import SVC
from sklearn.kernel_approximation import RBFSampler
from sklearn.linear_model import SGDClassifier
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
import numpy as np
from typing import Optional
import logging

logger = logging.getLogger(__name__)

class ScalableRBFClassifier:
    """
    Aproximación de RBF kernel para datasets grandes usando Random Fourier Features.
    """
    
    def __init__(
        self,
        n_components: int = 100,
        gamma: float = 1.0,
        random_state: Optional[int] = None
    ):
        """
        Args:
            n_components: Número de componentes Fourier (trade-off precisión/memoria)
            gamma: Parámetro gamma del RBF kernel
            random_state: Semilla para reproducibilidad
        """
        self.n_components = n_components
        self.gamma = gamma
        self.random_state = random_state
        
        # Pipeline con aproximación RBF
        self.pipeline = Pipeline([
            ('scaler', StandardScaler()),
            ('rbf_sampler', RBFSampler(
                gamma=gamma,
                n_components=n_components,
                random_state=random_state
            )),
            ('classifier', SGDClassifier(
                loss='hinge',  # SVM loss
                max_iter=1000,
                random_state=random_state
            ))
        ])
        
    def fit(self, X: np.ndarray, y: np.ndarray) -> 'ScalableRBFClassifier':
        """Entrena con aproximación de kernel."""
        logger.info(f"Training with {X.shape[0]} samples, {self.n_components} RBF components")
        self.pipeline.fit(X, y)
        return self
    
    def predict(self, X: np.ndarray) -> np.ndarray:
        """Predicción."""
        return self.pipeline.predict(X)
    
    def score(self, X: np.ndarray, y: np.ndarray) -> float:
        """Accuracy."""
        return self.pipeline.score(X, y)

# Uso con dataset grande ✓
X_large = np.random.randn(50000, 100)
y_large = np.random.randint(0, 2, 50000)

# Aproximación escalable
clf = ScalableRBFClassifier(n_components=500, gamma=0.01)
clf.fit(X_large, y_large)
# Memoria: 50,000 × 500 × 8 bytes = 200 MB (100x menos)

🎯 Resumen Ejecutivo

El Kernel Trick no es "magia para hacer SVM no lineal". Es resolver:

$$
\max_{\alpha}\sum_{i=1}^{n}\alpha_i-\frac{1}{2}\sum_{i,j}\alpha_i\alpha_jy_iy_jK(x_i,x_j)
$$

Mediante:

$$
K_{\text{RBF}}(x_i,x_j)=\exp\left(-\gamma|x_i-x_j|^2\right)
$$

Con todas las implicaciones que eso conlleva:

  • Escalado obligatorio: Features sin escalar rompen completamente la distancia euclidiana en RBF
  • Gamma crítico: Controla complejidad del modelo (alto = overfitting, bajo = underfitting)
  • Complejidad cuadrática: $O(n^2)$ en tiempo y memoria, inviable para $n>10{,}000$ sin aproximaciones
  • No siempre necesario: Para datos lineales, LinearSVC es 10-100x más rápido con igual performance

De una, hágale. Con esta base matemática del Kernel Trick + código production-ready con SVC(kernel='rbf'), estás listo para usar espacios de features infinito-dimensionales sin romper tu servidor ni tu cabeza.
