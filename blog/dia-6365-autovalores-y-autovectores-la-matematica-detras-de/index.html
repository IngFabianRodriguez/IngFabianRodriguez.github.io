🎯 1. Introducción: Por Qué Esta Vaina Importa

Hace años, cuando estaba optimizando un sistema de recomendaciones que andaba más lento que una fila del banco, un senior me dijo: "Parcero, el problema no es el código. Es que no entendés la matemática que estás usando". Tenía razón.

Los autovalores y autovectores no son teoría abstracta. Son la base de:

  • PageRank (el algoritmo que hizo billonario a Larry Page)
  • PCA (reducción de dimensionalidad en ML)
  • Sistemas dinámicos (predecir si tu arquitectura de microservicios va a colapsar)
  • Análisis de estabilidad (cadenas de Markov en recomendaciones)

Analogía de Oficina:

Imaginá que tu empresa es una matriz. Cada empleado (vector) tiene una "dirección natural" en la que trabaja mejor. Cuando aplicás una transformación (reorganización, nuevo proceso), los autovectores son esos empleados que mantienen su dirección pero cambian su productividad (autovalor).

Si un autovalor es > 1, ese empleado se vuelve más productivo. Si es < 1, se quema. Si es = 1, sigue igual. PageRank es encontrar al empleado más influyente de una red.

📐 2. Fundamentos Matemáticos (Con Rigor, Sin Humo)

2.1 Definición Formal

Sea $A \in \mathbb{R}^{n \times n}$ una matriz cuadrada. Un autovalor $\lambda$ y su correspondiente autovector $\mathbf{v} \neq \mathbf{0}$ satisfacen:

$$A\mathbf{v} = \lambda \mathbf{v}$$

Interpretación geométrica: La transformación lineal $A$ escala el vector $\mathbf{v}$ por un factor $lambda$, pero no cambia su dirección.

2.2 Ecuación Característica

Para encontrar $lambda$, resolvemos:

$$\det(A - \lambda I) = 0$$

Esto genera un polinomio característico de grado $n$:

$$p(\lambda) = \det(A - \lambda I) = (-1)^n \lambda^n + c_{n-1}\lambda^{n-1} + \cdots + c_0$$

Problema computacional: Resolver este polinomio directamente es $O(n^3)$ y numéricamente inestable para $n > 10$.

2.3 Teorema de Perron-Frobenius (El Corazón de PageRank)

Teorema: Si $A$ es una matriz estocástica (filas suman 1) y primitiva (existe $k$ tal que $A^k > 0$), entonces:

  1. Existe un único autovalor dominante $\lambda_1 = 1$
  2. Su autovector correspondiente $\mathbf{v}_1$ tiene entradas positivas
  3. Todos los demás autovalores satisfacen $|\lambda_i| < 1$

Consecuencia para PageRank: El autovector dominante representa la distribución estacionaria de probabilidad en la red (el ranking de páginas).

2.4 Descomposición Espectral

Si $A$ es diagonalizable, podemos escribir:

$$A = V \Lambda V^{-1}$$

donde:

  • $V = [\mathbf{v}_1, \mathbf{v}_2, \ldots, \mathbf{v}_n]$ (matriz de autovectores)
  • $\Lambda = \text{diag}(\lambda_1, \lambda_2, \ldots, \lambda_n)$ (autovalores en la diagonal)

Aplicación práctica: Para calcular $A^k$ (necesario en cadenas de Markov):

$$A^k = V \Lambda^k V^{-1}$$

Complejidad: $O(n^3)$ una vez, luego $O(n^2)$ por cada potencia.

🌐 3. Caso de Uso Real: PageRank de Google

3.1 El Problema Original

En 1998, Larry Page y Sergey Brin tenían un problema: ¿cómo rankear billones de páginas web?

Su insight: una página es importante si:

  1. Muchas páginas apuntan a ella (popularidad)
  2. Esas páginas también son importantes (recursividad)

3.2 Formulación Matemática

Sea $G = (V, E)$ un grafo dirigido (páginas = nodos, links = aristas). Definimos la matriz de transición $M$:

$$M_{ij} = \begin{cases} \frac{1}{L(j)} & \text{si existe link de } j \to i \\ 0 & \text{en otro caso} \end{cases}$$

donde $L(j)$ = número de links salientes de la página $j$.

El vector de PageRank $\mathbf{r}$ satisface:

$$\mathbf{r} = M \mathbf{r}$$

Esto es exactamente la definición de autovector con $lambda = 1$!

3.3 El Problema del "Dead End"

Si una página no tiene links salientes (sumidero), $M$ no es estocástica. Solución de Google:

$$M' = \alpha M + (1 - \alpha) \frac{\mathbf{e} \mathbf{e}^T}{n}$$

donde:

  • $\alpha = 0.85$ (damping factor, probabilidad de seguir un link)
  • $\mathbf{e}$ = vector de unos
  • $(1 - \alpha)$ = probabilidad de "teletransportarse" a cualquier página

Ahora $M'$ es estocástica y primitiva → Perron-Frobenius aplica.

            self._validate_matrix(matrix)
        
        self._

(f"Computing eigenvalues for {matrix.shape} matrix")
        start_time = time.perf_counter()
        
        try:
            # numpy.linalg.eig usa LAPACK (dgeev para matrices reales)
            # Internamente: reducción a Hessenberg + algoritmo QR
            eigenvalues, eigenvectors = np.linalg.eig(matrix)
            
        except np.linalg.LinAlgError as e:
            self._logger.error(f"Eigendecomposition failed: {e}")
            raise
        
        elapsed = time.perf_counter() - start_time
        self._
logger.info
(f"Computation completed in {elapsed:.4f}s")
        
        return EigenResult(
            eigenvalues=eigenvalues,
            eigenvectors=eigenvectors,
            method="numpy.linalg.eig",
            computation_time=elapsed,
            iterations=None  # numpy no expone iteraciones internas
        )
    
    def _validate_matrix(self, matrix: FloatArray) -> None:
        """Validación estricta de la matriz.
        
        Complejidad: O(1) para shape, O(n²) para NaN/Inf check
        """
        if matrix.ndim != 2:
            raise ValueError(f"Expected 2D matrix, got {matrix.ndim}D")
        
        n, m = matrix.shape
        if n != m:
            raise ValueError(f"Matrix must be square, got shape {matrix.shape}")
        
        if not np.isfinite(matrix).all():
            raise ValueError("Matrix contains NaN or Inf values")
        
        self._logger.debug(f"Matrix validation passed: shape={matrix.shape}")

4.3 Implementación: SciPy (Más Robusto)

class ScipyEigenSolver:
    """Solver usando scipy.linalg.eig (más control numérico).
    
    Diferencias con NumPy:
    - Mejor manejo de matrices casi-singulares
    - Permite seleccionar subconjunto de autovalores
    - Control de balanceo/permutación para estabilidad
    """
    
    __slots__ = ('_logger', '_check_finite')
    
    def __init__(self, check_finite: bool = True) -> None:
        self._logger = logging.getLogger(f"{__name__}.ScipyEigenSolver")
        self._check_finite = check_finite
    
    def compute(
        self,
        matrix: FloatArray,
        left: bool = False,  # Calcular autovectores izquierdos
        right: bool = True,  # Calcular autovectores derechos
        overwrite: bool = False  # Permite modificar input (más rápido)
    ) -> EigenResult:
        """Calcula autovalores con scipy (más opciones que numpy).
        
        Args:
            matrix: Matriz de entrada
            left: Si True, retorna autovectores izquierdos (A.T @ v)
            right: Si True, retorna autovectores derechos (A @ v)
            overwrite: Si True, permite modificar matrix (ahorra memoria)
            
        Returns:
            EigenResult con metadata extendida
        """
        self._
logger.info
(
            f"SciPy eigendecomposition: shape={matrix.shape}, "
            f"left={left}, right={right}"
        )
        
        start_time = time.perf_counter()
        
        # scipy.linalg.eig tiene más control que numpy:
        # - check_finite: validación explícita de NaN/Inf
        # - overwrite_a: permite destruir input (performance crítica)
        result = sp_linalg.eig(
            matrix,
            left=left,
            right=right,
            overwrite_a=overwrite,
            check_finite=self._check_finite
        )
        
        elapsed = time.perf_counter() - start_time
        
        # scipy.linalg.eig retorna tupla variable según left/right
        eigenvalues = result[0]
        eigenvectors = result[1] if right else result[2] if left else None
        
        return EigenResult(
            eigenvalues=eigenvalues,
            eigenvectors=eigenvectors,
            method="scipy.linalg.eig",
            computation_time=elapsed,
            iterations=None
        )

4.4 Implementación: Método de las Potencias (Educational + Eficiente)

@dataclass(slots=True)
class PowerIterationConfig:
    """Configuración para el método de las potencias."""
    max_iterations: int = 1000
    tolerance: float = 1e-10
    normalize: bool = True  # Normalizar en cada iteración (estabilidad)


class PowerIterationSolver:
    """Implementación del método de las potencias para autovalor dominante.
    
    Complejidad: O(k·n²) donde k = número de iteraciones
    
    Ventaja sobre métodos directos:
    - Solo necesita el autovalor dominante (PageRank, cadenas de Markov)
    - Más eficiente para matrices sparse (k << n típicamente)
    - Memoria constante O(n) vs O(n²)
    
    Teoría: Si λ₁ > |λ₂| ≥ |λ₃| ≥ ... (autovalor dominante único),
    entonces para casi cualquier v₀:
    
        lim[k→∞] (Aᵏ v₀) / ||Aᵏ v₀|| = v₁  (autovector dominante)
    
    Tasa de convergencia: O((λ₂/λ₁)ᵏ) - depende del gap espectral
    """
    
    __slots__ = ('_config', '_logger')
    
    def __init__(self, config: PowerIterationConfig | None = None) -> None:
        self._config = config or PowerIterationConfig()
        self._logger = logging.getLogger(f"{__name__}.PowerIterationSolver")
    
    def compute_dominant(
        self,
        matrix: FloatArray,
        initial_vector: FloatArray | None = None
    ) -> EigenResult:
        """Calcula el autovalor/vector dominante iterativamente.
        
        Args:
            matrix: Matriz cuadrada A
            initial_vector: Vector inicial v₀ (aleatorio si None)
            
        Returns:
            EigenResult con autovalor dominante (λ₁) y autovector (v₁)
            
        Raises:
            ValueError: Si la matriz no converge (gap espectral insuficiente)
        """
        n = matrix.shape[0]
        
        # Inicialización: vector aleatorio normalizado
        if initial_vector is None:
            # Usar seed para reproducibilidad en tests
            rng = np.random.default_rng(seed=42)
            v = rng.random(n)
        else:
            v = initial_vector.copy()
        
        if self._config.normalize:
            v = v / np.linalg.norm(v)
        
        self._
logger.info
(
            f"Power iteration starting: max_iter={self._config.max_iterations}"
        )
        
        start_time = time.perf_counter()
        eigenvalue_prev = 0.0
        
        for iteration in range(self._config.max_iterations):
            # Paso de potencia: v_{k+1} = A @ v_k
            v_new = matrix @ v
            
            # Estimación del autovalor: Rayleigh quotient
            # λ ≈ (v^T A v) / (v^T v)
            eigenvalue = v.T @ v_new
            
            # Normalización (previene overflow para λ > 1)
            if self._config.normalize:
                norm = np.linalg.norm(v_new)
                if norm < 1e-12:  # Protección contra vector cero
                    raise ValueError(
                        f"Power iteration collapsed at iter {iteration}"
                    )
                v_new = v_new / norm
            
            # Criterio de convergencia: |λ_k - λ_{k-1}| < tol
            if abs(eigenvalue - eigenvalue_prev) < self._config.tolerance:
                self._
logger.info
(
                    f"Converged at iteration {iteration}: λ={eigenvalue:.10f}"
                )
                break
            
            v = v_new
            eigenvalue_prev = eigenvalue
            
            if iteration % 100 == 0:
                self._logger.debug(
                    f"Iter {iteration}: λ={eigenvalue:.10f}, "
                    f"Δλ={abs(eigenvalue - eigenvalue_prev):.2e}"
                )
        else:
            # Loop terminó sin break (no convergió)
            self._logger.warning(
                f"Did not converge after {self._config.max_iterations} iterations"
            )
        
        elapsed = time.perf_counter() - start_time
        
        # Retornar solo el autovalor dominante
        # (extender a array para mantener consistencia de tipo)
        return EigenResult(
            eigenvalues=np.array([eigenvalue], dtype=np.complex128),
            eigenvectors=v.reshape(-1, 1),  # Columna vector
            method="power_iteration",
            computation_time=elapsed,
            iterations=iteration + 1
        )

📐 Análisis Matemático de Convergencia:

Sea $\mathbf{v}_0 = c_1 \mathbf{v}_1 + c_2 \mathbf{v}_2 + \cdots + c_n \mathbf{v}_n$ (expansión en autovectores).

Después de $k$ iteraciones:

$$A^k \mathbf{v}_0 = c_1 \lambda_1^k \mathbf{v}_1 + c_2 \lambda_2^k \mathbf{v}_2 + \cdots + c_n \lambda_n^k \mathbf{v}_n$$

Si $|lambda_1| > |lambda_2| geq cdots geq |lambda_n|$, entonces:

$$\frac{A^k \mathbf{v}_0}{\|A^k \mathbf{v}_0\|} \approx \frac{\lambda_1^k \mathbf{v}_1}{\|\lambda_1^k \mathbf{v}_1\|} = \pm \mathbf{v}_1$$

Error después de $k$ iteraciones:

$$\epsilon_k \leq C \left|\frac{\lambda_2}{\lambda_1}\right|^k$$

donde $C$ depende de $c_1, c_2$ (componentes del vector inicial).

Implicación práctica: Si $\lambda_1 / \lambda_2 = 2$ (gap moderado), necesitamos ~10 iteraciones para error $< 0.1%$%.

💡 7. Alivio del Dolor: ¿Qué Problemas Resuelve Esto?

Problema 1: "Mi sistema de recomendaciones converge lento"

Síntoma: Tu algoritmo tarda 10 segundos en rankear 1000 items.

Causa: Estás usando numpy.linalg.eig en matrices grandes (complejidad $O(n^3)$).

Solución con este código:

# ❌ ANTES (lento)
eigenvalues, eigenvectors = np.linalg.eig(large_matrix)  # 10s

# ✅ DESPUÉS (rápido)
solver = PowerIterationSolver()
result = solver.compute_dominant(large_matrix)  # 1.2s (8x más rápido)

Problema 2: "No entiendo por qué mi cadena de Markov no converge"

Síntoma: Tu simulación da resultados inconsistentes.

Causa: La matriz no es primitiva (autovalor dominante no único).

Solución con este código:

# Validar que la matriz sea estocástica primitiva
eigenvalues = np.linalg.eigvals(transition_matrix)
eigenvalues_sorted = np.sort(np.abs(eigenvalues))[::-1]

if eigenvalues_sorted[0] != 1.0:

Problema 3: "¿Cuándo usar numpy vs scipy vs implementación custom?"

Regla de decisión:
