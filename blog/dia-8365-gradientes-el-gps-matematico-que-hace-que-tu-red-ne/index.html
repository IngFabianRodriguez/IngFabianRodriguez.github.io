Día 8: Gradientes - El GPS Matemático que Hace que tu Red Neuronal No se Pierda en el Espacio

Introducción: Cuando el Cálculo se Vuelve tu Mejor Amigo

Mirá, parce, hoy vamos a hablar de algo que literalmente mueve todo el machine learning moderno: los gradientes. Y no, no es solo "la derivada" que te enseñaron en cálculo y que seguro pensaste que nunca ibas a usar.

El gradiente es el GPS matemático que le dice a tu modelo "ey, para ese lado NO, que te estrellás". Es la brújula que convierte una red neuronal random (que literalmente tira outputs al azar) en una máquina que predice con precisión.

Y acá viene lo brutal: si no entendés gradientes a profundidad, estás programando redes neuronales a ciegas. Estás poniendo hiperparámetros como si fueras brujo echando sal, sin saber por qué tu loss explota o por qué tu modelo no converge ni con rezos.

Hoy vamos a romper ese gradiente hasta los huesos. Con matemáticas, con PyTorch, y con código que podés poner en producción mañana mismo.

1. ¿Qué Carajos es un Gradiente? (Y Por Qué Debería Importarte)

La Analogía del Cerro

Imaginá que estás perdido en un cerro de niebla espesa. No ves nada. Tu objetivo: bajar al pueblo que está en el valle.

¿Qué hacés?

Palpás el suelo con el pie. Si sentís que el terreno baja hacia la derecha, caminás para allá. Si baja hacia adelante, vas hacia adelante. Esa inclinación que sentís es el gradiente.

En machine learning:

  • El cerro = Tu función de pérdida (loss function)
  • Tu posición = Los pesos actuales de tu red
  • El valle = El mínimo de la función (donde el error es mínimo)
  • El gradiente = La dirección de máxima pendiente

La Matemática Seria

El gradiente de una función $f: \mathbb{R}^n \to \mathbb{R}$ es el vector de todas sus derivadas parciales:

$$\nabla f(\mathbf{w}) = \begin{bmatrix} \frac{\partial f}{\partial w_1} \\ \frac{\partial f}{\partial w_2} \\ \vdots \\ \frac{\partial f}{\partial w_n} \end{bmatrix}$$

¿Qué nos dice este vector?

  1. Dirección: Hacia dónde crece más rápido la función
  2. Magnitud: Qué tan empinada es esa subida

Y la jugada maestra del Gradient Descent:

$$\mathbf{w}_{new} = \mathbf{w}_{old} - \eta \nabla f(\mathbf{w}_{old})$$

Donde $\eta$ es el learning rate (cuánto nos movemos en cada paso).

¿Por qué restamos? Porque queremos bajar el cerro (minimizar el error), no subirlo. El gradiente apunta hacia arriba, entonces vamos en sentido contrario.

2. PyTorch Autograd: El Motor que Calcula Gradientes por Vos

Acá viene la magia negra moderna: vos NO tenés que calcular esas derivadas parciales a mano. PyTorch lo hace automáticamente usando una técnica llamada diferenciación automática (autograd).

¿Cómo Funciona Autograd?

PyTorch construye un grafo computacional dinámico. Cada operación que hacés (suma, multiplicación, activación) queda registrada como un nodo en ese grafo.

Input → Linear → ReLU → Linear → Softmax → Loss

Cuando llamás .backward(), PyTorch recorre ese grafo de atrás para adelante (backpropagation) aplicando la regla de la cadena para calcular todos los gradientes.

La Regla de la Cadena (Chain Rule)

Si tenés una composición de funciones:

$$y = f(g(h(x)))$$

La derivada es:

$$\frac{dy}{dx} = \frac{df}{dg} \cdot \frac{dg}{dh} \cdot \frac{dh}{dx}$$

Eso es backpropagation. No es magia, es cálculo de primeros semestres aplicado de forma recursiva.

3. torch.autograd.grad: El Bisturí Quirúrgico de los Gradientes

Ahora sí, manos a la obra. PyTorch tiene dos formas principales de obtener gradientes:

  1. .backward(): Acumula gradientes en el atributo .grad de los tensores
  2. torch.autograd.grad(): Te da los gradientes como valores de retorno (más control, más flexible)

Hoy nos enfocamos en torch.autograd.grad porque es production-grade cuando necesitás:

  • Calcular gradientes de orden superior (derivadas de derivadas)
  • Gradientes con respecto a inputs no-parámetros
  • Múltiples salidas respecto a múltiples entradas
  • Control fino sobre qué se retiene en memoria

class GradientMode(Enum):
    """Modos de cálculo de gradiente.
    
    ¿Por qué esto existe? Porque en producción a veces necesitás:
    - FIRST_ORDER: Solo el gradiente básico (∂L/∂w)
    - SECOND_ORDER: Hessiana o gradientes de gradientes (para optimizadores avanzados)
    - JACOBIAN: Matriz jacobiana completa (para análisis de sensibilidad)
    """
    FIRST_ORDER = "first_order"
    SECOND_ORDER = "second_order" 
    JACOBIAN = "jacobian"


@dataclass(frozen=True, slots=True)  # slots=True → optimización de memoria
class GradientResult:
    """Resultado del cálculo de gradiente.
    
    ¿Por qué frozen? Porque los gradientes son inmutables una vez calculados.
    Si cambiás el gradiente después, perdés la trazabilidad de de dónde vino.
    
    ¿Por qué slots? Porque vamos a crear MILLONES de estos objetos en training.
    Slots reduce el uso de memoria ~40% vs __dict__.
    """
    gradients: Tuple[Tensor, ...]
    inputs: Tuple[Tensor, ...]
    output: Tensor
    mode: GradientMode
    
    def __post_init__(self):
        """Validación post-inicialización."""
        if len(self.gradients) != len(self.inputs):
            raise ValueError(
                f"Gradientes ({len(self.gradients)}) y inputs ({len(self.inputs)}) "
                f"deben tener la misma cantidad"
            )

4.2. El Motor de Gradientes

class GradientEngine:
    """Motor de cálculo de gradientes con autograd.
    
    Decisiones de arquitectura:
    1. No heredamos de nn.Module porque NO somos un modelo, somos una utilidad
    2. Usamos __slots__ para reducir overhead de memoria
    3. Todo método que puede fallar lanza excepciones específicas (no genéricas)
    """
    
    __slots__ = ('_retain_graph', '_create_graph', '_allow_unused')
    
    def __init__(
        self,
        retain_graph: bool = False,
        create_graph: bool = False,
        allow_unused: bool = False
    ):
        """
        Args:
            retain_graph: Si True, mantiene el grafo computacional después de .backward()
                         ¿Cuándo usarlo? Cuando necesitás calcular múltiples gradientes
                         sobre el mismo grafo (ej. multi-task learning)
                         
            create_graph: Si True, el gradiente mismo es diferenciable (para 2nd order)
                         ¿Cuándo usarlo? Hessiana, Newton's method, meta-learning
                         WARNING: Consume MUCHA memoria
                         
            allow_unused: Si True, no lanza error si algún input no contribuye al output
                         ¿Cuándo usarlo? Arquitecturas complejas con skip connections
        """
        self._retain_graph = retain_graph
        self._create_graph = create_graph
        self._allow_unused = allow_unused
        
        
logger.info
(
            f"GradientEngine inicializado: retain_graph={retain_graph}, "
            f"create_graph={create_graph}, allow_unused={allow_unused}"
        )
    
    def compute_gradient(
        self,
        func: DifferentiableFunction,
        inputs: Tuple[Tensor, ...],
        mode: GradientMode = GradientMode.FIRST_ORDER
    ) -> GradientResult:
        """Calcula gradientes usando torch.autograd.grad.
        
        ¿Por qué este método en vez de solo llamar autograd.grad directamente?
        1. Validación de inputs (evita crashes crípticos)
        2. Logging automático (debugging en producción)
        3. Manejo de errores específico (mensajes claros)
        4. Wrapper para modos avanzados (Hessiana, Jacobiana)
        
        Args:
            func: Función diferenciable a evaluar
            inputs: Tupla de tensores de entrada (deben tener requires_grad=True)
            mode: Modo de cálculo de gradiente
            
        Returns:
            GradientResult con gradientes, inputs, output y modo
            
        Raises:
            ValueError: Si los inputs no requieren gradientes
            RuntimeError: Si el cálculo de gradiente falla
        """
        # Validación de inputs
        for i, inp in enumerate(inputs):
            if not isinstance(inp, Tensor):
                raise TypeError(
                    f"Input {i} debe ser Tensor, recibido {type(inp).__name__}"
                )
            if not inp.requires_grad:
                raise ValueError(
                    f"Input {i} debe tener requires_grad=True para calcular gradientes. "
                    f"¿Olvidaste llamar .requires_grad_(True)?"
                )
        
        logger.debug(f"Calculando gradientes para {len(inputs)} inputs en modo {mode.value}")
        
        try:
            # Forward pass
            output = func(*inputs)
            
            if not isinstance(output, Tensor):
                raise TypeError(
                    f"La función debe retornar un Tensor, retornó {type(output).__name__}"
                )
            
            if output.numel() != 1:
                logger.warning(
                    f"Output tiene {output.numel()} elementos. "
                    f"torch.autograd.grad requiere escalar. Aplicando .sum()"
                )
                output = output.sum()
            
            # Cálculo de gradientes según el modo
            if mode == GradientMode.FIRST_ORDER:
                gradients = self._compute_first_order(output, inputs)
            elif mode == GradientMode.SECOND_ORDER:
                gradients = self._compute_second_order(output, inputs)
            elif mode == GradientMode.JACOBIAN:
                gradients = self._compute_jacobian(output, inputs)
            else:
                raise ValueError(f"Modo {mode} no soportado")
            
            
logger.info
(
                f"Gradientes calculados exitosamente: "
                f"{len(gradients)} tensores, modo={mode.value}"
            )
            
            return GradientResult(
                gradients=gradients,
                inputs=inputs,
                output=output,
                mode=mode
            )
            
        except RuntimeError as e:
            logger.error(f"Error en cálculo de gradientes: {str(e)}")
            raise RuntimeError(
                f"Falló el cálculo de gradientes. Error original: {str(e)}. "
                f"¿El grafo está roto? ¿Hay operaciones no diferenciables?"
            ) from e
    
    def _compute_first_order(
        self,
        output: Tensor,
        inputs: Tuple[Tensor, ...]
    ) -> Tuple[Tensor, ...]:
        """Calcula gradientes de primer orden (∂output/∂inputs).
        
        Esta es la operación más común: dado un loss, ¿cómo cambian los pesos?
        
        Complejidad: O(n) donde n = número de operaciones en el grafo
        Memoria: O(n) para almacenar el grafo (se libera si retain_graph=False)
        """
        grads = torch.autograd.grad(
            outputs=output,
            inputs=inputs,
            grad_outputs=None,  # Vector de 1s para escalares
            retain_graph=self._retain_graph,
            create_graph=self._create_graph,
            allow_unused=self._allow_unused
        )
        
        # torch.autograd.grad puede retornar None si allow_unused=True
        return tuple(g if g is not None else torch.zeros_like(inp) 
                    for g, inp in zip(grads, inputs))
    
    def _compute_second_order(
        self,
        output: Tensor,
        inputs: Tuple[Tensor, ...]
    ) -> Tuple[Tensor, ...]:
        """Calcula gradientes de segundo orden (∂²output/∂inputs²).
        
        ¿Cuándo lo usás? 
        - Newton's method (optimización de segundo orden)
        - Análisis de curvatura del loss landscape
        - Meta-learning (MAML)
        
        WARNING: Esto es CARO. Complejidad O(n²), memoria O(n²).
        Solo usalo si realmente lo necesitás.
        """
        # Primer gradiente
        first_grads = self._compute_first_order(output, inputs)
        
        # Segundo gradiente (gradiente del gradiente)
        second_grads = []
        for first_grad in first_grads:
            if first_grad.numel() != 1:
                first_grad = first_grad.sum()  # Reducir a escalar
            
            second_grad = torch.autograd.grad(
                outputs=first_grad,
                inputs=inputs,
                retain_graph=True,
                create_graph=False,  # No necesitamos tercer orden (aún)
                allow_unused=self._allow_unused
            )
            second_grads.append(second_grad)
        
        return tuple(second_grads)
    
    def _compute_jacobian(
        self,
        output: Tensor,
        inputs: Tuple[Tensor, ...]
    ) -> Tuple[Tensor, ...]:
        """Calcula la matriz Jacobiana completa.
        
        La Jacobiana es la generalización del gradiente para funciones vectoriales:
        
        Si f: R^n → R^m, entonces J ∈ R^(m×n):
        
        J[i,j] = ∂f_i/∂x_j
        
        ¿Cuándo lo usás?
        - Análisis de sensibilidad (cómo un cambio en input afecta TODO el output)
        - Inversión de redes generativas
        - Neural ODEs
        """
        # Implementación usando torch.autograd.functional.jacobian
        from torch.autograd.functional import jacobian
        
        def wrapper(*args):
            return func(*args)
        
        jac = jacobian(wrapper, inputs, create_graph=self._create_graph)
        
        return tuple(jac) if isinstance(jac, tuple) else (jac,)

4.3. Funciones de Utilidad para Análisis de Gradientes

class GradientAnalyzer:
    """Herramientas para analizar propiedades de gradientes.
    
    ¿Por qué esto es crítico en producción?
    - Vanishing gradients: El gradiente se vuelve microscópico (red no aprende)
    - Exploding gradients: El gradiente explota (NaNs, colapso del training)
    - Dead neurons: Neuronas con gradiente permanentemente cero
    """
    
    __slots__ = ()
    
    @staticmethod
    def gradient_norm(gradients: Tuple[Tensor, ...]) -> Tensor:
        """Calcula la norma L2 total de todos los gradientes.
        
        ¿Por qué la norma L2?
        ||∇f|| = sqrt(∑ᵢ (∂f/∂wᵢ)²)
        
        Te dice la "magnitud total" del gradiente. Si es muy pequeña (<1e-7),
        tenés vanishing gradients. Si es muy grande (>100), tenés exploding gradients.
        """
        total_norm = torch.tensor(0.0, dtype=torch.float32)
        for grad in gradients:
            if grad is not None:
                total_norm += 
grad.data
.norm(2).item() ** 2
        
        return torch.sqrt(total_norm)
    
    @staticmethod
    def gradient_stats(gradients: Tuple[Tensor, ...]) -> dict[str, float]:
        """Estadísticas descriptivas de gradientes.
        
        ¿Qué buscamos?
        - mean ≈ 0: Los gradientes deben estar centrados
        - std < 1: No queremos varianza gigante (inestabilidad)
        - max/min: Outliers que pueden causar problemas
        """
        all_grads = 
torch.cat
([g.flatten() for g in gradients if g is not None])
        
        return {
            'mean': all_grads.mean().item(),
            'std': all_grads.std().item(),
            'min': all_grads.min().item(),
            'max': all_grads.max().item(),
            'median': all_grads.median().item(),
            'norm_l1': all_grads.abs().sum().item(),
            'norm_l2': all_grads.norm(2).item(),
            'norm_linf': all_grads.abs().max().item()
        }
    
    @staticmethod
    def check_gradient_health(
        gradients: Tuple[Tensor, ...],
        threshold_vanishing: float = 1e-7,
        threshold_exploding: float = 100.0
    ) -> dict[str, bool]:
        """Chequeo de salud de gradientes.
        
        Returns:
            Dict con flags de problemas detectados
        """
        norm = GradientAnalyzer.gradient_norm(gradients)
        stats = GradientAnalyzer.gradient_stats(gradients)
        
        return {
            'vanishing': norm < threshold_vanishing,
            'exploding': norm > threshold_exploding,
            'has_nan': torch.isnan(torch.tensor([g.sum() for g in gradients if g is not None])).any().item(),
            'has_inf': torch.isinf(torch.tensor([g.sum() for g in gradients if g is not None])).any().item(),
            'healthy': (
                norm >= threshold_vanishing and 
                norm <= threshold_exploding and
                not torch.isnan(torch.tensor([g.sum() for g in gradients if g is not None])).any() and
                not torch.isinf(torch.tensor([g.sum() for g in gradients if g is not None])).any()
            )
        }

5.1. Caso 1: Gradiente de una Función Simple

def ejemplo_basico():
    """Ejemplo: calcular ∂(x²+y²)/∂x y ∂(x²+y²)/∂y"""
    
    # Definir función
    def quadratic_sum(x: Tensor, y: Tensor) -> Tensor:
        return x**2 + y**2
    
    # Crear inputs
    x = torch.tensor([3.0], requires_grad=True)
    y = torch.tensor([4.0], requires_grad=True)
    
    # Calcular gradientes
    engine = GradientEngine()
    result = engine.compute_gradient(
        func=quadratic_sum,
        inputs=(x, y),
        mode=GradientMode.FIRST_ORDER
    )
    
    # Analizar
    print(f"f(x,y) = {result.output.item()}")  # 25.0
    print(f"∂f/∂x = {result.gradients[0].item()}")  # 6.0 (2*3)
    print(f"∂f/∂y = {result.gradients[1].item()}")  # 8.0 (2*4)
    
    # Verificación matemática
    # f(x,y) = x² + y²
    # ∂f/∂x = 2x = 2(3) = 6 ✓
    # ∂f/∂y = 2y = 2(4) = 8 ✓


if __name__ == "__main__":
    ejemplo_basico()

import torch.nn as nn

def ejemplo_red_neuronal():
    """Calcular gradientes de un perceptrón simple."""
    
    # Red simple: 2 inputs → 3 hidden → 1 output
    class SimpleNet(nn.Module):
        def __init__(self):
            super().__init__()
            self.fc1 = nn.Linear(2, 3)
            self.fc2 = nn.Linear(3, 1)
        
        def forward(self, x: Tensor) -> Tensor:
            x = torch.relu(self.fc1(x))
            return self.fc2(x)
    
    model = SimpleNet()
    
    # Input
    x = torch.tensor([[1.0, 2.0]], requires_grad=True)
    
    # Forward
    output = model(x)
    
    # Gradiente respecto al input (no a los pesos)
    engine = GradientEngine()
    
    def forward_wrapper(inp: Tensor) -> Tensor:
        return model(inp)
    
    result = engine.compute_gradient(
        func=forward_wrapper,
        inputs=(x,),
        mode=GradientMode.FIRST_ORDER
    )
    
    print(f"Output: {result.output.item():.4f}")
    print(f"Gradiente ∂output/∂input: {result.gradients[0]}")
    
    # Esto te dice: "si cambio el input en dirección del gradiente,
    # ¿cuánto cambia el output?"
    # Útil para: adversarial examples, saliency maps, input optimization


if __name__ == "__main__":
    ejemplo_red_neuronal()

5.3. Caso 3: Gradientes de Segundo Orden (Hessiana)

def ejemplo_segundo_orden():
    """Calcular la diagonal de la Hessiana para análisis de curvatura."""
    
    def rosenbrock(x: Tensor, y: Tensor) -> Tensor:
        """Función de Rosenbrock: f(x,y) = (1-x)² + 100(y-x²)²
        
        Es un benchmark clásico de optimización porque tiene un valle
        muy estrecho y curvado (difícil de optimizar).
        """
        return (1 - x)**2 + 100 * (y - x**2)**2
    
    x = torch.tensor([0.5], requires_grad=True)
    y = torch.tensor([0.5], requires_grad=True)
    
    engine = GradientEngine(create_graph=True)  # ¡create_graph=True para 2do orden!
    
    # Primer gradiente
    result_1st = engine.compute_gradient(
        func=rosenbrock,
        inputs=(x, y),
        mode=GradientMode.FIRST_ORDER
    )
    
    print(f"Función: {result_1st.output.item():.4f}")
    print(f"∂f/∂x: {result_1st.gradients[0].item():.4f}")
    print(f"∂f/∂y: {result_1st.gradients[1].item():.4f}")
    
    # Segundo gradiente (Hessiana diagonal)
    # ∂²f/∂x²
    grad_x = result_1st.gradients[0]
    hessian_xx = torch.autograd.grad(
        grad_x, x, retain_graph=True, create_graph=False
    )[0]
    
    # ∂²f/∂y²
    grad_y = result_1st.gradients[1]
    hessian_yy = torch.autograd.grad(
        grad_y, y, retain_graph=True, create_graph=False
    )[0]
    
    print(f"∂²f/∂x²: {hessian_xx.item():.4f}")
    print(f"∂²f/∂y²: {hessian_yy.item():.4f}")
    
    # ¿Qué nos dice la Hessiana?
    # Si H > 0: Tenemos un mínimo local (curvatura hacia arriba)
    # Si H < 0: Tenemos un máximo local (curvatura hacia abajo)
    # Si H ≈ 0: Región plana (saddle point)


if __name__ == "__main__":
    ejemplo_segundo_orden()

6.1. Problema: Gradientes que Explotan

def debug_exploding_gradients():
    """Simular y detectar exploding gradients."""
    
    # Red PROFUNDA sin normalización (receta para el desastre)
    class DeepUnstableNet(nn.Module):
        def __init__(self, layers: int = 50):
            super().__init__()
            self.layers = nn.ModuleList([
                nn.Linear(100, 100) for _ in range(layers)
            ])
        
        def forward(self, x: Tensor) -> Tensor:
            for layer in self.layers:
                x = layer(x)  # Sin activación, sin normalización
                # Los gradientes se MULTIPLICAN en cada capa
                # Si los pesos son >1, esto explota exponencialmente
            return x.sum()
    
    model = DeepUnstableNet(layers=50)
    x = torch.randn(1, 100, requires_grad=True)
    
    # Forward + Backward
    output = model(x)
    output.backward()
    
    # Analizar gradientes
    all_grads = [p.grad for p in model.parameters() if p.grad is not None]
    
    analyzer = GradientAnalyzer()
    health = analyzer.check_gradient_health(tuple(all_grads))
    stats = analyzer.gradient_stats(tuple(all_grads))
    
    print(f"Salud del gradiente: {health}")
    print(f"Estadísticas: {stats}")
    
    # Soluciones si detectás exploding:
    # 1. Gradient clipping: torch.nn.utils.clip_grad_norm_()
    # 2. Batch normalization: nn.BatchNorm1d()
    # 3. Residual connections: x = x + layer(x)
    # 4. Learning rate más bajo


if __name__ == "__main__":
    debug_exploding_gradients()

6.2. Problema: Vanishing Gradients

def debug_vanishing_gradients():
    """Detectar vanishing gradients con sigmoides."""
    
    class VanishingNet(nn.Module):
        def __init__(self):
            super().__init__()
            self.layers = nn.ModuleList([
                nn.Linear(100, 100) for _ in range(20)
            ])
        
        def forward(self, x: Tensor) -> Tensor:
            for layer in self.layers:
                x = torch.sigmoid(layer(x))  # Sigmoid aplasta valores a (0,1)
                # Su derivada: σ'(x) = σ(x)(1-σ(x)) ≤ 0.25
                # Después de 20 capas: 0.25^20 ≈ 10^-12 (MUERTO)
            return x.sum()
    
    model = VanishingNet()
    x = torch.randn(1, 100, requires_grad=True)
    
    output = model(x)
    output.backward()
    
    # Ver gradiente de la PRIMERA capa (más alejada del loss)
    first_layer_grad = model.layers[0].weight.grad
    
    analyzer = GradientAnalyzer()
    norm = analyzer.gradient_norm((first_layer_grad,))

7.1. Gradient Checkpointing (Trading Compute por Memoria)

from torch.utils.checkpoint import checkpoint

def gradient_checkpointing_example():
    """Usar gradient checkpointing para redes GIGANTES.
    
    ¿El problema? Backprop necesita almacenar TODAS las activaciones
    del forward pass para calcular gradientes. En una red profunda,
    esto consume TONELADAS de memoria.
    
    ¿La solución? Gradient checkpointing:
    - En forward: Solo guardamos activaciones cada N capas
    - En backward: Recalculamos las activaciones intermedias on-the-fly
    
    Trade-off: +20-30% de tiempo de cómputo, -50% de uso de memoria
    """
    
    class HugeLayer(nn.Module):
        def __init__(self):
            super().__init__()
            self.linear = nn.Linear(1000, 1000)
        
        def forward(self, x: Tensor) -> Tensor:
            return torch.relu(self.linear(x))
    
    class CheckpointedNet(nn.Module):
        def __init__(self, num_layers: int = 100):
            super().__init__()
            self.layers = nn.ModuleList([
                HugeLayer() for _ in range(num_layers)
            ])
        
        def forward(self, x: Tensor) -> Tensor:
            # Sin checkpointing (uso normal):
            # for layer in self.layers:
            #     x = layer(x)
            
            # Con checkpointing:
            for layer in self.layers:
                x = checkpoint(layer, x, use_reentrant=False)
                # use_reentrant=False → método moderno, más seguro
            
            return x.sum()
    
    model = CheckpointedNet(num_layers=100)
    x = torch.randn(32, 1000)  # Batch de 32
    
    output = model(x)
    output.backward()
    
    print("Gradientes calculados con checkpointing")
    print("Memoria ahorrada: ~50% vs método estándar")


if __name__ == "__main__":
    gradient_checkpointing_example()

from torch.cuda.amp import autocast, GradScaler

def mixed_precision_example():
    """Mixed precision: Usar FP16 en vez de FP32.
    
    ¿Por qué?
    - FP16 usa 50% menos memoria
    - FP16 es 2-3x más rápido en GPUs modernas (Tensor Cores)
    
    ¿El problema? FP16 tiene menor precisión (puede causar underflow)
    
    ¿La solución? Automatic Mixed Precision (AMP):
    - Forward/backward en FP16
    - Acumulación de gradientes en FP32
    - Gradient scaling para evitar underflow
    """
    
    model = nn.Linear(1000, 1000).cuda()
    optimizer = torch.optim.Adam(model.parameters())
    scaler = GradScaler()  # Maneja el scaling automáticamente
    
    x = torch.randn(32, 1000).cuda()
    y = torch.randn(32, 1000).cuda()
    
    # Training loop
    for epoch in range(5):
        
optimizer.zero
_grad()
        
        # Forward en precisión mixta
        with autocast():
            output = model(x)
            loss = nn.functional.mse_loss(output, y)
        
        # Backward con scaling
        scaler.scale(loss).backward()  # loss.backward() pero con scaling
        
        # Update de pesos
        scaler.step(optimizer)  # optimizer.step() pero con unscaling
        scaler.update()  # Ajusta el scale factor
        
        print(f"Epoch {epoch}, Loss: {loss.item():.4f}")
    
    print("✓ Training con FP16 completado")
    print("Speedup esperado: 2-3x en GPUs modernas")
