Día 51/365: Aprendizaje No Supervisado - La Optimización Sin Etiquetas que el 85% Confunde con "Le Tiré K-Means y Salió Algo"

Hágale pues, llegamos al Día 51 y no, no es "cuando no tenés etiquetas y le tirás K-Means a ver qué sale". Después de 12 años viendo equipos tratando de encontrar "insights ocultos" en los datos sin entender lo que están haciendo, te digo una cosa: el 85% de los proyectos de aprendizaje no supervisado fallan no porque el algoritmo sea malo, sino porque nadie definió qué carajos significa "éxito" cuando no hay ground truth.

El dolor real del aprendizaje no supervisado no es la ausencia de etiquetas. Es que te obliga a confrontar algo incómodo: tenés que definir qué estructura buscás ANTES de buscarla. Y eso, parcero, es filosofía + matemática + ingeniería, todo al mismo tiempo.

Dejá el café, acomodate, y preparate porque vamos a desmenuzar qué significa realmente "aprender sin supervisión", cuándo sirve, cuándo es una pérdida de tiempo, y cómo implementarlo sin que se te caiga en producción.

🎯 ¿Qué Carajos Estás Optimizando Si No Hay Etiquetas?

Mirá, el aprendizaje no supervisado es como cuando te dan un almacén desordenado y te dicen "organizá esto como te parezca lógico". No te dicen CÓMO organizarlo (eso sería supervisado), pero tampoco significa que podés hacer cualquier cosa. Tenés que definir un criterio de "buena organización" y optimizarlo.

Definición técnica (sin rodeos):

El aprendizaje no supervisado busca descubrir la estructura subyacente en un conjunto de datos $\mathcal{X}=\{\mathbf{x}_1,\mathbf{x}_2,\ldots,\mathbf{x}_n\}$ donde $\mathbf{x}_i\in\mathbb{R}^d$, sin etiquetas $y_i$. El objetivo es encontrar una función $f:\mathcal{X}\rightarrow\mathcal{Z}$ que maximice/minimice alguna medida de estructura $\mathcal{L}(\mathcal{X},f)$.

Tipos principales:

  • Clustering: Agrupar datos similares. Optimizás cohesión intra-cluster y separación inter-cluster. Ejemplo: K-Means minimiza $\sum_{k=1}^{K}\sum_{\mathbf{x}\in C_k}|\mathbf{x}-\boldsymbol{mu}_k|^2$.
  • Reducción de Dimensionalidad: Proyectar a espacio de menor dimensión preservando estructura. PCA maximiza varianza explicada, t-SNE preserva vecindarios locales.
  • Detección de Anomalías: Identificar puntos que no siguen el patrón general. Optimizás likelihood bajo una distribución asumida o densidad kernel.
  • Reglas de Asociación: Descubrir patrones frecuentes (ej: market basket analysis). Optimizás métricas como soporte, confianza, lift.

La confusión mortal: La gente piensa que "no supervisado" significa "sin objetivo". FALSO. El objetivo está implícito en la función que optimizás. Si optimizás distancia euclidiana en K-Means, asumís clusters esféricos. Si optimizás densidad en DBSCAN, asumís clusters de forma arbitraria. Cambiar el algoritmo es cambiar la definición de "estructura".

📐 La Función Objetivo Oculta

Acá es donde el 90% se pierde. Todo algoritmo no supervisado está optimizando ALGO, pero ese algo no viene de etiquetas sino de una hipótesis sobre la estructura de los datos.

$$\mathcal{L}(\mathcal{X},f)=\text{medida de estructura}(\mathcal{X},f(\mathcal{X}))$$

Esta ecuación parece abstracta pero es LA CLAVE de todo. Cuando elegís un algoritmo no supervisado, estás eligiendo:

  1. Qué es "estructura" (clusters, manifolds, densidad, independencia estadística)
  2. Cómo medirla (distancia, varianza, información mutua, likelihood)
  3. Cómo optimizarla (EM, gradient descent, spectral methods)

Vamos a desmenuzar esto con el ejemplo más común: clustering.

Descomposición Completa: El Caso de Clustering (K-Means)

La función objetivo de K-Means es:

$$J=\sum_{k=1}^{K}\sum_{\mathbf{x}_i\in C_k}\|\mathbf{x}_i-\boldsymbol{\mu}_k\|^2$$

$J$ (Función de Costo) - Within-Cluster Sum of Squares

J = suma de varianzas intra-cluster
  = qué tan "apretados" están los clusters
  = inversamente proporcional a qué tan "buenos" son

Analogía: Es como medir qué tan cerca viven los empleados de su oficina asignada. Mientras más cerca, mejor la asignación (menor costo de transporte).

En Machine Learning:

  • Para datos bien separados: $J$ será bajo porque los clusters naturales coinciden con la asignación
  • Para datos sin estructura: $J$ será alto sin importar cuántos clusters uses
  • Para K muy alto: $J\rightarrow 0$ (overfitting, cada punto es su propio cluster)

Dimensionalidad importa: $J$ crece con $d$ (dimensiones) porque las distancias se inflan. Por eso siempre se normaliza o se reduce dimensionalidad primero.

Diferencia CRÍTICA: K-Means optimiza varianza dentro de clusters, NO separación entre clusters. Por eso falla con clusters no esféricos (elipses, lunas, etc.).

$K$ (Número de Clusters) - El Hiperparámetro del Millón

K = cuántos grupos querés
  = lo que NO te da el algoritmo
  = lo que tenés que validar con métodos externos

Analogía: Es como decidir cuántas categorías de productos necesitás en tu tienda. Muy pocas y todo se mezcla, demasiadas y perdés el overview.

En Machine Learning:

  • Para datos con clusters claros: existe un $K$ óptimo donde el "codo" del gráfico $J$ vs $K$ se dobla
  • Para datos sin estructura: no hay codo, $J$ decrece linealmente con $K$
  • Para validación: se usa Silhouette Score, Davies-Bouldin Index, Gap Statistic

Ejemplo numérico concreto:

import numpy as np
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
import matplotlib.pyplot as plt

# Datos sintéticos con 3 clusters claros
np.random.seed(42)
cluster1 = np.random.randn(50, 2) + [0, 0]
cluster2 = np.random.randn(50, 2) + [5, 5]
cluster3 = np.random.randn(50, 2) + [0, 5]
X = np.vstack([cluster1, cluster2, cluster3])

# Probar diferentes K
inertias = []
silhouettes = []
K_range = range(2, 11)

for k in K_range:
    kmeans = KMeans(n_clusters=k, random_state=42, n_init=10)
    labels = kmeans.fit_predict(X)
    inertias.append(kmeans.inertia_)  # Esto es J
    silhouettes.append(silhouette_score(X, labels))

print(f"K=3 (óptimo esperado):")
print(f"  Inertia (J): {inertias[1]:.2f}")
print(f"  Silhouette: {silhouettes[1]:.3f}")

Output típico:

K=3 (óptimo esperado):
  Inertia (J): 245.67
  Silhouette: 0.612

$\boldsymbol{\mu}_k$ (Centroide del Cluster k) - El Representante

μₖ = promedio de todos los puntos en Cₖ
   = el "centro de masa" del cluster
   = lo que se actualiza en cada iteración

Analogía: Es el gerente de cada departamento. Los empleados (puntos) se asignan al gerente más cercano, y el gerente se mueve al "centro" de su equipo.

En Machine Learning:

  • Para datos numéricos: $\boldsymbol{\mu}_k=\frac{1}{|C_k|}\sum_{\mathbf{x}_i\in C_k}\mathbf{x}_i$
  • Para datos categóricos: K-Modes usa modas en lugar de medias
  • Para datos mixtos: K-Prototypes combina distancia euclidiana + matching

Dimensionalidad importa: $\boldsymbol{mu}_k\in\mathbb{R}^d$, así que en 1000 dimensiones tenés 1000 valores por centroide.

$\|\mathbf{x}_i-\boldsymbol{\mu}_k\|^2$ (Distancia Cuadrática) - La Métrica

‖xᵢ - μₖ‖² = distancia euclidiana al cuadrado
            = suma de diferencias cuadráticas por dimensión
            = lo que se minimiza para cada punto

Analogía: Es qué tan lejos vive cada empleado de la oficina. Se eleva al cuadrado para penalizar más las distancias grandes.

En Machine Learning:

  • Para features en escala similar: funciona bien
  • Para features en escalas diferentes: dominan las features con mayor rango (SIEMPRE normalizar)
  • Para datos no euclidianos: usar cosine distance (text), Manhattan (sparse), etc.

Diferencia CRÍTICA: La distancia euclidiana asume que todas las dimensiones son igual de importantes y que las features son independientes. Si tenés correlación entre features, PCA primero.

📐 ¿Por Qué Sin Etiquetas Sigue Siendo Optimización?

La pregunta del millón: si no hay $y$ para medir error, ¿cómo sabés si estás "aprendiendo"?

La respuesta está en que cambias el objetivo de predicción a reconstrucción/estructura:

Supervisado:

$$\min_{\theta}\mathbb{E}_{(\mathbf{x},y)\sim\mathcal{D}}[\mathcal{L}(f_\theta(\mathbf{x}),y)]$$

Minimizás el error entre predicción $f_\theta(\mathbf{x})$ y etiqueta real $y$.

No Supervisado:

$$\min_{\theta}\mathbb{E}_{\mathbf{x}\sim\mathcal{D}}[\mathcal{L}_{\text{struct}}(\mathbf{x},f_\theta(\mathbf{x}))]$$

Minimizás alguna medida de "qué tan bien capturás la estructura", donde $\mathcal{L}_{\text{struct}}$ depende del problema:

  • Autoencoders: $\mathcal{L}_{\text{struct}}=\|\mathbf{x}-\text{decoder}(\text{encoder}(\mathbf{x}))\|^2$ (error de reconstrucción)
  • K-Means: $\mathcal{L}_{\text{struct}}=\sum_k\sum_{\mathbf{x}\in C_k}\|\mathbf{x}-\boldsymbol{\mu}_k\|^2$ (compactness)
  • PCA: $\mathcal{L}_{\text{struct}}=-\sum_{i=1}^{k}\lambda_i$ donde $\lambda_i$ son eigenvalues (varianza capturada)

Ejemplo: Autoencoder para Detección de Anomalías

from __future__ import annotations

import logging
from typing import Tuple
import numpy as np
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)


class SimpleAutoencoder(nn.Module):
    """Autoencoder simple para detección de anomalías.
    
    El objetivo es minimizar el error de reconstrucción:
    L = E[||x - decoder(encoder(x))||²]
    
    Puntos con alto error de reconstrucción son anomalías.
    """
    
    def __init__(self, input_dim: int, latent_dim: int):
        """
        Args:
            input_dim: Dimensión de entrada
            latent_dim: Dimensión del espacio latente (bottleneck)
        """
        super().__init__()
        
        if latent_dim >= input_dim:
            raise ValueError(
                f"latent_dim ({latent_dim}) debe ser < input_dim ({input_dim})"
            )
        
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, input_dim // 2),
            nn.ReLU(),
            nn.Linear(input_dim // 2, latent_dim)
        )
        
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, input_dim // 2),
            nn.ReLU(),
            nn.Linear(input_dim // 2, input_dim)
        )
        
        logger.info(
            f"Autoencoder creado: {input_dim} → {latent_dim} → {input_dim}"
        )
    
    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """Forward pass: encode → decode."""
        encoded = self.encoder(x)
        decoded = self.decoder(encoded)
        return decoded
    
    def reconstruction_error(self, x: torch.Tensor) -> np.ndarray:
        """Calcula error de reconstrucción por punto.
        
        Returns:
            Array con ||x - x̂||² para cada punto
        """
        self.eval()
        with torch.no_grad():
            x_reconstructed = self.forward(x)
            errors = torch.mean((x - x_reconstructed) ** 2, dim=1)
        return errors.numpy()


def train_autoencoder(
    model: SimpleAutoencoder,
    X_train: np.ndarray,
    epochs: int = 50,
    batch_size: int = 32,
    learning_rate: float = 0.001
) -> list[float]:
    """Entrena el autoencoder minimizando error de reconstrucción."""
    
    dataset = TensorDataset(
        torch.FloatTensor(X_train),
        torch.FloatTensor(X_train)  # Target es el mismo input
    )
    dataloader = DataLoader(dataset, batch_size=batch_size, shuffle=True)
    
    optimizer = torch.optim.Adam(model.parameters(), lr=learning_rate)
    criterion = nn.MSELoss()
    
    losses = []
    
    for epoch in range(epochs):
        model.train()
        epoch_loss = 0.0
        
        for batch_x, batch_target in dataloader:
            optimizer.zero_grad()
            
            reconstructed = model(batch_x)
            loss = criterion(reconstructed, batch_target)
            
            loss.backward()
            optimizer.step()
            
            epoch_loss += loss.item()
        
        avg_loss = epoch_loss / len(dataloader)
        losses.append(avg_loss)
        
        if (epoch + 1) % 10 == 0:
            logger.info(f"Epoch {epoch + 1}/{epochs}, Loss: {avg_loss:.4f}")
    
    return losses


def detect_anomalies(
    model: SimpleAutoencoder,
    X: np.ndarray,
    threshold_percentile: float = 95.0
) -> Tuple[np.ndarray, float]:
    """Detecta anomalías usando error de reconstrucción.
    
    Args:
        model: Autoencoder entrenado
        X: Datos a evaluar
        threshold_percentile: Percentil para threshold (95 = top 5% son anomalías)
    
    Returns:
        (is_anomaly, threshold)
    """
    if not 0 < threshold_percentile < 100:
        raise ValueError("threshold_percentile debe estar en (0, 100)")
    
    errors = model.reconstruction_error(torch.FloatTensor(X))
    threshold = np.percentile(errors, threshold_percentile)
    
    is_anomaly = errors > threshold
    
    logger.info(
        f"Detectadas {is_anomaly.sum()} anomalías "
        f"({100 * is_anomaly.mean():.1f}% de {len(X)} puntos)"
    )
    
    return is_anomaly, threshold


# Ejemplo de uso
if __name__ == "__main__":
    np.random.seed(42)
    torch.manual_seed(42)
    
    # Datos normales + algunas anomalías
    normal_data = np.random.randn(1000, 20)
    anomalies = np.random.randn(50, 20) * 5  # Escala mayor = anomalías
    X = np.vstack([normal_data, anomalies])
    
    # Crear y entrenar
    model = SimpleAutoencoder(input_dim=20, latent_dim=5)
    losses = train_autoencoder(model, normal_data, epochs=50)
    
    # Detectar anomalías
    is_anomaly, threshold = detect_anomalies(model, X, threshold_percentile=95)
    
    # Validar
    true_anomalies = np.concatenate([
        np.zeros(1000, dtype=bool),
        np.ones(50, dtype=bool)
    ])
    
    detected_anomalies_idx = np.where(is_anomaly)[0]
    true_positives = np.sum(detected_anomalies_idx >= 1000)
    
    logger.info(f"Verdaderos positivos: {true_positives}/50 anomalías reales")
    logger.info(f"Threshold: {threshold:.4f}")

🚨 Errores Comunes (y Cómo No Cagarla)

Error 1: Asumir Que "No Supervisado" Significa "Sin Validación"

El problema:

# ❌ MALO: Clustering sin validar NADA
from sklearn.cluster import KMeans

kmeans = KMeans(n_clusters=5, random_state=42)
labels = kmeans.fit_predict(X)

print(f"Listo, tengo {len(set(labels))} clusters!")
# Y ahora qué? Son buenos? Malos? Quién sabe

Por qué falla:

Aunque no tengás etiquetas ground truth, SÍ tenés métricas internas:

$$\text{Silhouette}=\frac{b-a}{\max(a,b)}$$

Donde $a$ es distancia promedio intra-cluster y $b$ es distancia promedio al cluster más cercano. Rango: $[-1,1]$, donde 1 es perfecto.

import numpy as np
from sklearn.metrics import silhouette_score, davies_bouldin_score

# Si no validás, esto podría pasar
np.random.seed(123)
X_random = np.random.randn(100, 2)  # Datos SIN estructura real

kmeans_random = KMeans(n_clusters=5, random_state=42)
labels_random = kmeans_random.fit_predict(X_random)

print(f"Silhouette en datos random: {silhouette_score(X_random, labels_random):.3f}")
# Output típico: ~0.2 (muy bajo, indica que no hay clusters reales)

La solución:

from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score, davies_bouldin_score, calinski_harabasz_score
import logging

logger = logging.getLogger(__name__)

def validate_clustering(
    X: np.ndarray,
    labels: np.ndarray,
    algorithm_name: str = "KMeans"
) -> dict[str, float]:
    """Valida resultados de clustering con múltiples métricas.
    
    Métricas internas (no requieren ground truth):
    - Silhouette: [-1, 1], más alto es mejor (cohesión vs separación)
    - Davies-Bouldin: [0, ∞), más bajo es mejor (ratio intra/inter cluster)
    - Calinski-Harabasz: [0, ∞), más alto es mejor (variance ratio)
    """
    n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
    
    if n_clusters < 2:
        raise ValueError("Se requieren al menos 2 clusters para validación")
    
    metrics = {
        "silhouette": silhouette_score(X, labels),
        "davies_bouldin": davies_bouldin_score(X, labels),
        "calinski_harabasz": calinski_harabasz_score(X, labels)
    }
    
    logger.info(f"Validación {algorithm_name} con {n_clusters} clusters:")
    logger.info(f"  Silhouette: {metrics['silhouette']:.3f} (target: >0.5)")
    logger.info(f"  Davies-Bouldin: {metrics['davies_bouldin']:.3f} (target: <1.0)")
    logger.info(f"  Calinski-Harabasz: {metrics['calinski_harabasz']:.1f} (target: alto)")
    
    return metrics

# Uso correcto
kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
labels = kmeans.fit_predict(X)
metrics = validate_clustering(X, labels, "KMeans")

if metrics["silhouette"] < 0.3:
    logger.warning("⚠️  Silhouette bajo: considera cambiar K o usar otro algoritmo")

Error 2: No Normalizar Datos Antes de Clustering

El problema:

# ❌ MALO: Features en escalas diferentes
import pandas as pd

df = pd.DataFrame({
    'salario': [30000, 45000, 50000, 80000, 120000],  # Rango: 90K
    'edad': [25, 30, 28, 35, 45]                      # Rango: 20
})

kmeans = KMeans(n_clusters=2)
labels = kmeans.fit_predict(df.values)
# El salario dominará COMPLETAMENTE porque es 1000x mayor

Por qué falla:

K-Means usa distancia euclidiana:

$$d(\mathbf{x}_i,\boldsymbol{\mu}_k)=\sqrt{\sum_{j=1}^{d}(x_{ij}-\mu_{kj})^2}$$

Si $x_1$ (salario) varía en decenas de miles y $x_2$ (edad) varía en decenas, entonces:

# Punto 1: salario=30000, edad=25
# Punto 2: salario=31000, edad=45
diferencia_salario = (31000 - 30000) ** 2  # 1,000,000
diferencia_edad = (45 - 25) ** 2            # 400

# La edad es IRRELEVANTE (400 vs 1M)

La solución:

from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
import numpy as np

def clustering_with_scaling(
    X: np.ndarray,
    n_clusters: int,
    method: str = "standard"
) -> tuple[np.ndarray, StandardScaler, KMeans]:
    """Clustering con normalización apropiada.
    
    Args:
        X: Datos (n_samples, n_features)
        n_clusters: Número de clusters
        method: "standard" (z-score) o "minmax" (0-1)
    
    Returns:
        (labels, scaler, kmeans)
    """
    if method == "standard":
        scaler = StandardScaler()
    elif method == "minmax":
        from sklearn.preprocessing import MinMaxScaler
        scaler = MinMaxScaler()
    else:
        raise ValueError(f"method debe ser 'standard' o 'minmax', no '{method}'")
    
    # CRÍTICO: fit solo en train, transform en train/test
    X_scaled = scaler.fit_transform(X)
    
    kmeans = KMeans(n_clusters=n_clusters, random_state=42, n_init=10)
    labels = kmeans.fit_predict(X_scaled)
    
    logger.info(f"Datos normalizados con {method} scaling")
    logger.info(f"  Media por feature: {X_scaled.mean(axis=0)}")
    logger.info(f"  Std por feature: {X_scaled.std(axis=0)}")
    
    return labels, scaler, kmeans

# Uso en producción
labels, scaler, kmeans = clustering_with_scaling(X, n_clusters=3)

# Para nuevos datos
X_new = np.array([[35000, 30]])
X_new_scaled = scaler.transform(X_new)  # ✓ Usar mismo scaler
label_new = kmeans.predict(X_new_scaled)

Error 3: Usar Clustering Cuando Deberías Reducir Dimensionalidad Primero

El problema:

# ❌ MALO: Clustering en 1000 dimensiones
from sklearn.datasets import make_classification

X_high_dim, _ = make_classification(
    n_samples=500,
    n_features=1000,  # ← 1000 dimensiones
    n_informative=10,
    n_redundant=990,
    random_state=42
)

kmeans = KMeans(n_clusters=5)
labels = kmeans.fit_predict(X_high_dim)
# La "maldición de la dimensionalidad" arruinó todo

Por qué falla:

En alta dimensionalidad, las distancias pierden significado. Para dos puntos random $\mathbf{x},\mathbf{y}\in\mathbb{R}^d$:

$$\lim_{d\to\infty}\frac{\max_id(\mathbf{x},\mathbf{y}_i)-\min_id(\mathbf{x},\mathbf{y}_i)}{\min_id(\mathbf{x},\mathbf{y}_i)}\to 0$$

Es decir, todos los puntos quedan igualmente lejos (concentración de distancias).

# Demo de concentración
np.random.seed(42)

for d in [2, 10, 100, 1000]:
    X = np.random.randn(100, d)
    centroid = X[0]
    
    distances = np.linalg.norm(X - centroid, axis=1)
    ratio = (distances.max() - distances.min()) / distances.min()
    
    print(f"d={d:4d}: ratio max/min distance = {ratio:.4f}")

# Output esperado:
# d=   2: ratio max/min distance = 2.1453
# d=  10: ratio max/min distance = 0.8234
# d= 100: ratio max/min distance = 0.2891
# d=1000: ratio max/min distance = 0.0956  ← Todo está igual de lejos

La solución:

from sklearn.decomposition import PCA
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler
import numpy as np
import logging

logger = logging.getLogger(__name__)


def clustering_with_pca(
    X: np.ndarray,
    n_clusters: int,
    variance_threshold: float = 0.95
) -> tuple[np.ndarray, PCA, KMeans]:
    """Pipeline correcto: Scale → PCA → Clustering.
    
    Args:
        X: Datos originales (n_samples, n_features)
        n_clusters: Número de clusters
        variance_threshold: Varianza a retener (ej: 0.95 = 95%)
    
    Returns:
        (labels, pca, kmeans)
    """
    if not 0 < variance_threshold <= 1:
        raise ValueError("variance_threshold debe estar en (0, 1]")
    
    # Paso 1: Normalizar
    scaler = StandardScaler()
    X_scaled = scaler.fit_transform(X)
    
    # Paso 2: Reducir dimensionalidad
    pca = PCA(n_components=variance_threshold, random_state=42)
    X_reduced = pca.fit_transform(X_scaled)
    
    original_dims = X.shape[1]
    reduced_dims = X_reduced.shape[1]
    variance_explained = pca.explained_variance_ratio_.sum()
    
    logger.info(f"PCA redujo {original_dims} → {reduced_dims} dims")
    logger.info(f"Varianza retenida: {variance_explained:.2%}")
    
    # Paso 3: Clustering en espacio reducido
    kmeans = KMeans(n_clusters=n_clusters, random_state=42, n_init=10)
    labels = kmeans.fit_predict(X_reduced)
    
    return labels, pca, kmeans


# Comparación
X_high, _ = make_classification(n_samples=500, n_features=1000, n_informative=10, random_state=42)

# SIN PCA
labels_bad = KMeans(n_clusters=5).fit_predict(X_high)
silhouette_bad = silhouette_score(X_high, labels_bad)

# CON PCA
labels_good, pca, kmeans = clustering_with_pca(X_high, n_clusters=5, variance_threshold=0.95)
X_reduced = pca.transform(StandardScaler().fit_transform(X_high))
silhouette_good = silhouette_score(X_reduced, labels_good)

print(f"Silhouette SIN PCA: {silhouette_bad:.3f}")
print(f"Silhouette CON PCA: {silhouette_good:.3f}")
# Típicamente: 0.15 vs 0.45 (3x mejor)

🎯 Resumen Ejecutivo

El aprendizaje no supervisado no es "cuando no tenés etiquetas". Es optimizar una medida de estructura sobre $\mathcal{X}$:

$$\min_{\theta}\mathbb{E}_{\mathbf{x}\sim\mathcal{D}}[\mathcal{L}_{\text{struct}}(\mathbf{x},f_\theta(\mathbf{x}))]$$

Mediante algoritmos que maximizan/minimizan:

$$\begin{aligned} \text{Clustering}&:\min\sum_k\sum_{\mathbf{x}\in C_k}\|\mathbf{x}-\boldsymbol{\mu}_k\|^2\\ \text{PCA}&:\max\sum_{i=1}^{k}\lambda_i\\ \text{Autoencoder}&:\min\|\mathbf{x}-\text{decoder}(\text{encoder}(\mathbf{x}))\|^2 \end{aligned}$$

Con todas las implicaciones que eso conlleva:

  • Elegir algoritmo es elegir hipótesis: K-Means asume clusters esféricos, DBSCAN asume densidad, PCA asume linealidad
  • Validación sin ground truth es posible: Silhouette, Davies-Bouldin, Gap Statistic, error de reconstrucción
  • Preprocesamiento es CRÍTICO: normalización, reducción de dimensionalidad, manejo de outliers
  • Interpretabilidad es difícil: tenés que validar con expertos de dominio que los clusters/componentes tienen sentido
