Día 31: Aprendizaje Supervisado - La Base Matemática Que Nadie Te Explica (Y Por Qué Tu Modelo Falla en Producción)

Hágale pues, llegamos al Día 31 y vamos a hablar del corazón de Machine Learning: el aprendizaje supervisado. Y no, no es "ese tema básico que todo el mundo conoce". Es el tema que todos creen dominar hasta que les toca llevar un modelo a producción y se dan cuenta que andaban embalados con los fundamentos.

Después de +10 años viendo modelos explotar en runtime, te digo una vaina: el 80% de los bugs en ML vienen de no entender las bases matemáticas del supervised learning. Entonces dejá el café y acomodate, que esto va largo pero cada minuto vale oro.

🎯 ¿Qué Chimba Es Esto del Aprendizaje Supervisado?

Mirá, el aprendizaje supervisado es como cuando le enseñás a un pelado nuevo en el equipo. Le mostrás ejemplos etiquetados: "Este bug es crítico, este es menor, este ni lo mirés". Después de ver suficientes ejemplos, el pelado empieza a clasificar bugs nuevos solo. Eso mismo hace un modelo supervisado.

Definición técnica (sin rodeos):

El aprendizaje supervisado es un paradigma donde entrenamos un modelo $f_\theta$ usando un dataset $\mathcal{D} = \{(x_i, y_i)\}_{i=1}^{N}$ donde cada ejemplo $x_i$ tiene su etiqueta $y_i$ correspondiente. El objetivo es encontrar los parámetros $\theta^*$ que minimicen una función de pérdida.

Tipos principales:

  • Regresión: Predecir valores continuos (ej. precio de una casa, temperatura)
  • Clasificación: Predecir categorías discretas (ej. spam/no-spam, gato/perro)

La diferencia no es cosmética. Regresión y clasificación usan funciones de pérdida completamente diferentes, y confundirlas es una receta para el desastre en producción.

📐 Fórmula Central: El Problema de Optimización

Todo supervised learning se reduce a resolver este problema matemático:

$$\theta^* = \arg\min_{\theta} \mathcal{L}(\theta) = \arg\min_{\theta} \frac{1}{N} \sum_{i=1}^{N} \ell(f_\theta(x_i), y_i) + \lambda R(\theta)$$

Esta ecuación parece simple pero tiene TODO el poder del ML moderno. Vamos a desmenuzarla símbolo por símbolo porque acá es donde la mayoría se pierde.

Descomposición Completa: Entendiendo Cada Símbolo

$\theta^*$ (Theta estrella) - El Santo Grial

θ* = los parámetros óptimos que estamos buscando

Analogía: Imaginá que estás buscando la mejor configuración de un aire acondicionado para tu oficina. $\theta^*$ es esa temperatura perfecta donde nadie se queja de frío ni calor.

En Machine Learning:

  • Para una red neuronal: $\theta$ incluye TODOS los pesos y sesgos de todas las capas
  • Para regresión lineal: $\theta = [w_1, w_2, ..., w_n, b]$ (pesos + bias)
  • Es un vector (o tensor) con todos los parámetros aprendibles del modelo

Dimensionalidad importa: Si tenés 1000 features y 1 output en regresión lineal, $\theta \in \mathbb{R}^{1001}$ (1000 pesos + 1 bias). Para una red con 3 capas de 100 neuronas cada una, estás hablando de decenas de miles de parámetros.

$\arg\min_{\theta}$ (Argumento del mínimo) - Lo Que Buscamos

argmin = "dame los VALORES de θ que minimizan lo que sigue"

Diferencia CRÍTICA que confunde a muchos:

  • $\min_{\theta} f(\theta)$ = el VALOR mínimo (un número escalar)
  • $\arg\min_{\theta} f(\theta)$ = los PARÁMETROS que logran ese mínimo (un vector)

Ejemplo numérico concreto:

f(θ) = (θ - 3)²

min f(θ) = 0           # El valor mínimo de la función
argmin f(θ) = 3        # El valor de θ que logra ese mínimo

# En código:
theta_values = np.linspace(0, 6, 100)
losses = (theta_values - 3)**2

print(f"Valor mínimo: {losses.min()}")        # 0
print(f"Argmin: {theta_values[losses.argmin()]}")  # 3

En nuestro caso, buscamos qué configuración de parámetros $\theta$ hace que el error total sea el más pequeño posible.

$\mathcal{L}(\theta)$ (La Función de Pérdida Total) - Tu Objetivo

L(θ) = qué tan mal le va a tu modelo con los parámetros θ

Objetivo fundamental: Queremos $\mathcal{L}(\theta)$ lo más pequeño posible. Mientras más pequeño, mejor predice tu modelo.

Esta función se descompone en DOS partes fundamentales que balancea el entrenamiento:

PARTE 1: Error Empírico (Qué Tan Bien Fitteás Los Datos)

$$\frac{1}{N} \sum_{i=1}^{N} \ell(f_\theta(x_i), y_i)$$

Vamos término por término porque cada uno importa:

$N$ - Tamaño del Dataset

N = número total de ejemplos en tu dataset

Si tenés 10,000 registros de clientes, $N = 10000$. Si tenés 50 ejemplos, $N = 50$. Este número afecta directamente la confiabilidad de tu modelo.

$\sum_{i=1}^{N}$ - Suma Sobre Todos los Ejemplos

Σ = sumá desde i=1 hasta i=N

Esto significa: "para cada ejemplo en tu dataset, calculá algo y sumá todos los resultados".

Visualización:

# Si N = 4
Σ(i=1 to 4) error_i = error₁ + error₂ + error₃ + error₄

# En código:
total_error = 0
for i in range(N):
    total_error += error[i]

$x_i$ e $y_i$ - Features y Labels

xᵢ = las features (características) del ejemplo i-ésimo
yᵢ = la etiqueta verdadera (ground truth) del ejemplo i-ésimo

Ejemplo concreto de dataset de casas:

# Ejemplo 1
x₁ = [120m², 3 habitaciones, zona norte, 5 años]  # Features
y₁ = 250_000_000  # Precio real en pesos

# Ejemplo 2
x₂ = [80m², 2 habitaciones, zona sur, 10 años]
y₂ = 180_000_000

# Tu dataset completo:
# D = {(x₁, y₁), (x₂, y₂), ..., (xₙ, yₙ)}

Cada $x_i$ es un vector de features, cada $y_i$ es el valor que querés predecir.

$f_\theta(x_i)$ - La Predicción de Tu Modelo

f_θ(xᵢ) = lo que tu modelo PREDICE para el ejemplo i
         = ŷᵢ (y-hat, predicción)

Esta es tu función de predicción. Su forma depende del tipo de modelo:

Regresión lineal:

$$f_\theta(x) = \theta^T x = w_1x_1 + w_2x_2 + ... + w_nx_n + b$$

Ejemplo:

x = [120, 3, 1, 5]  # [m², habitaciones, zona_codificada, años]
θ = [2000, 30_000_000, 50_000_000, -1_000_000, 100_000_000]
     # [peso_m², peso_hab, peso_zona, peso_años, bias]

f_θ(x) = 2000(120) + 30M(3) + 50M(1) + (-1M)(5) + 100M
       = 240k + 90M + 50M - 5M + 100M
       = 235.24M pesos

Red neuronal (conceptual):

$$f_\theta(x) = \text{softmax}(W_3 \cdot \sigma(W_2 \cdot \sigma(W_1 x + b_1) + b_2) + b_3)$$

Donde $\sigma$ es una función de activación (ReLU, sigmoid, etc.).

Punto clave: $f_\theta$ DEPENDE de $\theta$. Cambiás los parámetros, cambian las predicciones. Todo el entrenamiento es ajustar $\theta$ para que $f_\theta(x_i)$ se parezca a $y_i$.

$\ell(f_\theta(x_i), y_i)$ - Error Individual

ℓ = el error del ejemplo i
  = qué tan lejos está tu predicción de la realidad
  = qué tan equivocado estás en este ejemplo específico

Usualmente escribimos $\hat{y}_i = f_\theta(x_i)$ (predicción), entonces:

$$\ell(\hat{y}_i, y_i) = \text{alguna medida de distancia entre predicción y realidad}$$

Para REGRESIÓN (valores continuos) - Mean Squared Error:

$$\ell(\hat{y}_i, y_i) = (\hat{y}_i - y_i)^2$$

¿Por qué elevar al cuadrado? Hay TRES razones fundamentales:

  1. Simetría (errores positivos y negativos son igual de malos):
subestimar 5 millones → (ŷ - y) = -5M → error² = 25M²
sobrestimar 5 millones → (ŷ - y) = +5M → error² = 25M²
# Sin el cuadrado, se cancelarían en el promedio
  1. Penaliza errores grandes desproporcionadamente:
error pequeño: (1M)² = 1M²
error mediano: (5M)² = 25M²    ← 5× peor pero penaliza 25×
error grande:  (10M)² = 100M²  ← 10× peor pero penaliza 100×

# Esto obliga al modelo a esforzarse MÁS en corregir outliers
  1. Diferenciable en todos lados (crucial para gradient descent):
# Derivada de x²
d/dx(x²) = 2x  # Suave, continua, fácil de optimizar

# Comparar con valor absoluto |x|
d/dx|x| = undefined en x=0  # Quiebre, problemas numéricos

Ejemplo numérico:

y = 200M  # Precio real
ŷ = 180M  # Predicción

error = (180M - 200M)² = (-20M)² = 400M²
# Nota: 400M² no son pesos, son "pesos al cuadrado"
# Es una unidad artificial que usamos para optimizar

Para CLASIFICACIÓN (categorías) - Cross-Entropy:

$$\ell(\hat{y}_i, y_i) = -[y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]$$

(Esto lo desglosamos a fondo más adelante)

$\frac{1}{N}$ - Promediando el Error

1/N = promedio del error sobre todos los ejemplos

¿Por qué necesitamos promediar?

Sin el promedio, no podés comparar modelos entrenados en datasets de diferente tamaño:

# Dataset A
N_A = 100 ejemplos
error_total_A = 5000
# ¿Es bueno o malo? No sabés.

# Dataset B  
N_B = 10000 ejemplos
error_total_B = 50000
# ¿Es peor que A? Tampoco sabés.

# CON promedio:
error_promedio_A = 5000/100 = 50
error_promedio_B = 50000/10000 = 50
# Ahora sí: ambos tienen el mismo error promedio por ejemplo

El $\frac{1}{N}$ normaliza el error independientemente del tamaño del dataset. Esto es crucial cuando hacés validación cruzada o comparás modelos.

Entonces, juntando toda la PARTE 1:

$$\frac{1}{N} \sum_{i=1}^{N} \ell(f_\theta(x_i), y_i) = \text{Error promedio de tu modelo en el dataset de entrenamiento}$$

En español:

"Calculá el error de cada ejemplo, sumá todos esos errores, y sacá el promedio."

Este término solo mide qué tan bien tu modelo "memoriza" los datos de entrenamiento. Pero memorizar no es suficiente...

PARTE 2: Regularización (Evitando Overfitting)

$$+ \lambda R(\theta)$$

Esta parte PREVIENE que tu modelo sea demasiado complejo y memorice en vez de generalizar.

$R(\theta)$ - El Término de Regularización

R(θ) = penalización por complejidad del modelo
     = qué tan "complicados" son tus parámetros

Los dos tipos más comunes:

L2 Regularization (Ridge) - Prefiere Pesos Pequeños:

$$R(\theta) = \|\theta\|_2^2 = \sum_{j=1}^{d} \theta_j^2 = \theta_1^2 + \theta_2^2 + ... + \theta_d^2$$

Donde:

  • $d$ = número de parámetros (dimensionalidad de $\theta$)
  • $\|\theta\|_2$ = norma euclidiana (longitud del vector $\theta$)

Intuición geométrica:

En 2D, $\|\theta\|_2 = \sqrt{\theta_1^2 + \theta_2^2}$ es la longitud del vector $\theta$ desde el origen.

θ₂ ↑
   |    /
   |   /|θ|
   |  /
   |────→ θ₁
   
|θ| = √(θ₁² + θ₂²)

Al minimizar $\|\theta\|_2^2$, empujás todos los parámetros hacia cero (pero no a exactamente cero).

Ejemplo de preferencia:

Modelo A: θ = [100, 0, 0]    → R(θ) = 10000
Modelo B: θ = [10, 10, 10]   → R(θ) = 300

# L2 prefiere B: pesos distribuidos vs. un peso gigante
# Esto previene que una sola feature domine

¿Por qué funciona?

Pesos grandes amplifican el ruido en los datos. Si $w_1 = 1000$ y tu feature tiene ruido de ±0.01, el impacto es ±10 en la predicción. Con $w_1 = 10$, el impacto es solo ±0.1.

L1 Regularization (Lasso) - Hace Feature Selection:

$$R(\theta) = \|\theta\|_1 = \sum_{j=1}^{d} |\theta_j| = |\theta_1| + |\theta_2| + ... + |\theta_d|$$

Diferencia clave con L2:

Modelo A: θ = [100, 0, 0]    → R_L1(θ) = 100,  R_L2(θ) = 10000
Modelo B: θ = [10, 10, 10]   → R_L1(θ) = 30,   R_L2(θ) = 300

# L1 también prefiere B, pero por razón diferente
# L1 literalmente ELIMINA features (las pone en 0)
# L2 solo las hace pequeñas

Efecto de L1 en gradiente:

$$\frac{\partial}{\partial \theta_j} |\theta_j| = \text{sign}(\theta_j) = \begin{cases} +1 & \text{si } \theta_j > 0 \\ -1 & \text{si } \theta_j < 0 \end{cases}$$

El gradiente es CONSTANTE (±1), no proporcional a $\theta_j$. Esto empuja con fuerza constante hacia cero, causando que muchos parámetros lleguen exactamente a 0.

$\lambda$ (Lambda) - El Hiperparámetro Más Importante

λ = qué tan importante es la simplicidad vs. el ajuste a los datos
  = el "perilla" que controla el trade-off

Rango típico: $\lambda \in [0.0001, 10]$ (depende de la escala de tus datos)

Efectos de diferentes valores:

$\lambda = 0$ (sin regularización):

L(θ) = error_promedio + 0 × R(θ) = error_promedio

# Solo minimizás error en training
# RIESGO: Overfitting total

$\lambda$ pequeño (0.001):

L(θ) = error_promedio + 0.001 × R(θ)

# Regularización suave
# Poco control sobre complejidad

$\lambda$ mediano (0.1):

L(θ) = error_promedio + 0.1 × R(θ)

# Balance típico para producción

$\lambda$ grande (10):

L(θ) = error_promedio + 10 × R(θ)

# Fuerte preferencia por simplicidad
# RIESGO: Underfitting (modelo demasiado simple)

Visualización del trade-off:

λ = 0:      Modelo complejo que memoriza
            ╱╲╱╲╱╲╱╲  (curva loca)
            Training error: 0.01 ✓
            Test error: 2.50 ✗ (overfitting)

λ = 0.01:   Balance saludable
            ╱‾‾╲___╱  (curva suave)
            Training error: 0.10
            Test error: 0.12 ✓ (generaliza)

λ = 100:    Modelo demasiado simple
            ─────────  (casi línea recta)
            Training error: 1.50 ✗
            Test error: 1.48 (underfitting)

¿Cómo elegir $\lambda$**?**

Cross-validation:

lambdas = [0.0001, 0.001, 0.01, 0.1, 1.0, 10.0]

for λ in lambdas:
    model = train(data, lambda=λ)
    val_error = evaluate(model, validation_set)
    
# Elegí el λ con menor val_error

El Signo "+" - Multi-Objective Optimization

+ = estás sumando DOS objetivos en conflicto

No podés optimizar ambos perfectamente. Siempre hay un trade-off:

Objetivo 1: Minimizar $\frac{1}{N} \sum \ell$ (ajustarte bien a los datos)

Objetivo 2: Minimizar $R(\theta)$ (mantener el modelo simple)

Si optimizás SOLO el objetivo 1 ($\lambda = 0$): Overfitting

Si optimizás SOLO el objetivo 2 ($\lambda \to \infty$): Underfitting

El "+" los combina en un solo objetivo balanceado.

La Ecuación Completa Junta

$$\theta^* = \arg\min_{\theta} \underbrace{\frac{1}{N} \sum_{i=1}^{N} \ell(f_\theta(x_i), y_i)}_{\text{Qué tan bien fitteás los datos}} + \underbrace{\lambda R(\theta)}_{\text{Qué tan complejo es el modelo}}$$

En español plano:

"Encontrá los parámetros $\theta^*$ que minimicen TANTO el error promedio en tu dataset COMO la complejidad del modelo, balanceando ambos objetivos con el parámetro $\lambda$."

Ejemplo numérico completo:

# Setup
N = 1000  # ejemplos
d = 5     # features
λ = 0.1   # regularización

# Calcular loss total
error_promedio = 0
for i in range(N):
    y_pred = f_θ(x[i])
    error_promedio += (y_pred - y[i])**2
error_promedio /= N  # Divide por N

regularizacion = sum(θ[j]**2 for j in range(d))

L_total = error_promedio + λ * regularizacion

# Ejemplo con números
# error_promedio = 0.25
# regularizacion = 100 (parámetros grandes)
# L_total = 0.25 + 0.1 × 100 = 0.25 + 10 = 10.25

# Si reducís regularizacion a 10:
# L_total = 0.25 + 0.1 × 10 = 0.25 + 1 = 1.25 ✓ mejor!

📐 Gradient Descent: Cómo Encontramos $\theta^*$

Ahora sabemos QUÉ queremos minimizar. Pero, ¿CÓMO lo minimizamos? Enter: Gradient Descent.

$$\theta^{(t+1)} = \theta^{(t)} - \eta \nabla_\theta \mathcal{L}(\theta^{(t)})$$

Esta es probablemente L ecuación más importante en todo ML. Entenderla a fondo es la diferencia entre ser un usuario de sklearn y ser un ML engineer que puede debuggear modelos en producción.

Descomposición Completa

$\theta^{(t)}$ - Parámetros en la Iteración t

θ⁽ᵗ⁾ = los valores de los parámetros en el paso t del entrenamiento

El superíndice $(t)$ indica la iteración/época del entrenamiento:

Timeline de entrenamiento:

θ⁽⁰⁾ = [0.5, -0.3, 0.8]   # Inicialización aleatoria
θ⁽¹⁾ = [0.48, -0.25, 0.75] # Después de 1 paso
θ⁽²⁾ = [0.45, -0.20, 0.70] # Después de 2 pasos
...
θ⁽¹⁰⁰⁰⁾ = [1.23, 0.87, -0.45] # Después de 1000 pasos

# Esperamos que θ⁽¹⁰⁰⁰⁾ ≈ θ* (óptimo)

Cada iteración, los parámetros se actualizan. El proceso iterativo continúa hasta convergencia.

$\theta^{(t+1)}$ - Parámetros Actualizados

θ⁽ᵗ⁺¹⁾ = los nuevos valores de los parámetros
        = el resultado de aplicar una actualización

Si estás en la iteración 5, entonces:

  • $\theta^{(5)}$ = parámetros actuales
  • $\theta^{(6)}$ = parámetros después de la actualización

$\eta$ (Eta) - Learning Rate

η = qué tan grande es cada paso de actualización
  = la "velocidad" de aprendizaje
  = el hiperparámetro más sensible

Valores típicos: $\eta \in [0.0001, 0.1]$ (depende del problema)

Analogía perfecta: Estás bajando una montaña en la niebla buscando el valle más bajo (el mínimo de la loss function).

$\eta$ grande (ej. 0.5):

Ventaja: Das pasos grandes → llegas rápido
Riesgo: Podés saltar por encima del mínimo

Visualización:

Loss
  ↑         ╱╲
  |        ╱  ╲
  |    1→ ╱    ╲ ←2
  |      ╱  ★   ╲  (saltás el mínimo ★)
  └──────────────→ θ

$\eta$ pequeño (ej. 0.001):

Ventaja: Pasos cuidadosos, no te pasás
Riesgo: MUY lento, pueden fallar optimización

$\eta$ perfecto (ej. 0.01):

Balance: Converge en tiempo razonable sin divergir

Ejemplo numérico detallado:

# Función simple: L(θ) = (θ - 5)²
# Gradiente: dL/dθ = 2(θ - 5)

θ⁽⁰⁾ = 10.0  # Inicio
gradiente = 2(10 - 5) = 10  # Dirección de crecimiento

# Con η = 0.1 (razonable)
θ⁽¹⁾ = 10.0 - 0.1 × 10 = 9.0
θ⁽²⁾ = 9.0 - 0.1 × 2(9-5) = 9.0 - 0.8 = 8.2
θ⁽³⁾ = 8.2 - 0.1 × 2(8.2-5) = 8.2 - 0.64 = 7.56
# ... converge suavemente hacia 5

# Con η = 1.0 (muy grande)
θ⁽¹⁾ = 10.0 - 1.0 × 10 = 0.0  # Te pasaste!
θ⁽²⁾ = 0.0 - 1.0 × 2(0-5) = 0.0 - (-10) = 10.0  # Volviste!
# ... oscila sin converger

# Con η = 0.001 (muy pequeño)
θ⁽¹⁾ = 10.0 - 0.001 × 10 = 9.99  # Avance minúsculo
# Tomaría 5000+ iteraciones llegar a 5

Señales de problemas con $\eta$:

# η muy alto
Loss en epoch 1: 1.5
Loss en epoch 2: 3.2  # ¡Aumenta!
Loss en epoch 3: NaN  # Explosión

# η muy bajo
Loss en epoch 1: 2.5
Loss en epoch 100: 2.49  # Avance microscópico
Loss en epoch 1000: 2.30  # Aún muy lejos del óptimo

$\nabla_\theta$ (Nabla) - El Operador Gradiente

∇_θ = "derivada parcial respecto a CADA componente de θ"
     = vector de todas las derivadas parciales

Si $\theta = [\theta_1, \theta_2, \theta_3]$, el gradiente es:

$$\nabla_\theta \mathcal{L} = \begin{bmatrix} \frac{\partial \mathcal{L}}{\partial \theta_1} \\ \frac{\partial \mathcal{L}}{\partial \theta_2} \\ \frac{\partial \mathcal{L}}{\partial \theta_3} \end{bmatrix}$$

Cada componente responde una pregunta:

  • $\frac{\partial \mathcal{L}}{\partial \theta_1}$: "Si aumento $\theta_1$ un poquito (mantener otros fijos), ¿cuánto aumenta $\mathcal{L}$?"
  • $\frac{\partial \mathcal{L}}{\partial \theta_2}$: "Si aumento $\theta_2$ un poquito, ¿cuánto aumenta $\mathcal{L}$?"
  • Y así...

Ejemplo concreto:

L(θ₁, θ₂) = θ₁² + 2θ₂²  # Función de loss

# Calcular gradiente
∂L/∂θ₁ = 2θ₁
∂L/∂θ₂ = 4θ₂

# En punto θ = [3, 1]
∇L = [2×3, 4×1] = [6, 4]

# Interpretación:
# - Si aumento θ₁, L aumenta 6 veces más rápido
# - Si aumento θ₂, L aumenta 4 veces más rápido
# - Ambos gradientes positivos → ambos deben DISMINUIR

Intuición geométrica fundamental:

El gradiente $\nabla_\theta \mathcal{L}$ apunta en la dirección de MAYOR CRECIMIENTO de $\mathcal{L}$.

∇L
      ↑  (dirección de mayor aumento)
      |
    ╱ | ╲
  ╱   |   ╲
╱     ★     ╲  (mínimo)
─────────────

Por eso hacemos "menos gradiente" ($-\nabla$): queremos ir en la dirección de MENOR loss, no mayor.

El Signo "-" (Menos)

- = movete en dirección OPUESTA al gradiente
  = bajá por la pendiente hacia el valle

Lógica:

  • Gradiente positivo → loss AUMENTA si aumentás $\theta$ → REDUCÍ $\theta$
  • Gradiente negativo → loss DISMINUYE si aumentás $\theta$ → AUMENTÁ $\theta$

Ejemplo con ambos casos:

# Caso 1: Gradiente positivo
θ_actual = 8.0
∇L = +5.0  # Loss aumenta hacia la derecha
θ_nuevo = 8.0 - η(+5.0) = 8.0 - 0.1(5) = 7.5  # Movete izquierda

# Caso 2: Gradiente negativo  
θ_actual = 2.0
∇L = -3.0  # Loss disminuye hacia la derecha
θ_nuevo = 2.0 - η(-3.0) = 2.0 + 0.1(3) = 2.3  # Movete derecha

Ejemplo Completo Paso a Paso

Vamos a minimizar $\mathcal{L}(\theta) = (\theta - 3)^2$ usando gradient descent.

Objetivo: Encontrar $\theta^* = 3$ (el mínimo obviamente es 3)

Gradiente:

$$\frac{d\mathcal{L}}{d\theta} = 2(\theta - 3)$$

Simulación con $\eta = 0.5$**, empezando desde** $\theta^{(0)} = 10$:

# Iteración 0 → 1
θ⁽⁰⁾ = 10.0
∇L = 2(10 - 3) = 14
θ⁽¹⁾ = 10.0 - 0.5 × 14 = 10.0 - 7.0 = 3.0 ✓

# ¡Boom! Llegamos en un paso

Eso fue suerte con $\eta$ perfecto. Probemos desde más lejos con $\eta$ menos perfecto:

# η = 0.2, empezando en θ⁽⁰⁾ = 0

Iteración 1:
θ⁽⁰⁾ = 0.0
∇L = 2(0 - 3) = -6
θ⁽¹⁾ = 0.0 - 0.2(-6) = 0.0 + 1.2 = 1.2
L(1.2) = (1.2-3)² = 3.24

Iteración 2:
θ⁽¹⁾ = 1.2
∇L = 2(1.2 - 3) = -3.6
θ⁽²⁾ = 1.2 - 0.2(-3.6) = 1.2 + 0.72 = 1.92
L(1.92) = (1.92-3)² = 1.1664

Iteración 3:
θ⁽²⁾ = 1.92
∇L = 2(1.92 - 3) = -2.16
θ⁽³⁾ = 1.92 + 0.432 = 2.352
L(2.352) = 0.4199

Iteración 4:
θ⁽³⁾ = 2.352
∇L = 2(2.352 - 3) = -1.296
θ⁽⁴⁾ = 2.352 + 0.2592 = 2.6112
L(2.6112) = 0.1512

# Después de ~10 iteraciones llegarías muy cerca de 3

Observá cómo:

  1. El loss SIEMPRE baja (monotónicamente)
  2. Los pasos se hacen más pequeños cerca del mínimo (el gradiente se achica)
  3. Convergencia gradual hacia $\theta^* = 3$

📐 El Gradiente Completo (Con Regularización)

$$\nabla_\theta \mathcal{L} = \frac{1}{N} \sum_{i=1}^{N} \nabla_\theta \ell(f_\theta(x_i), y_i) + \lambda \nabla_\theta R(\theta)$$

Esta fórmula muestra cómo calcular el gradiente que usamos en el update de gradient descent. Tiene dos partes (como la loss original):

PARTE 1: Gradiente del Error Empírico

$$\frac{1}{N} \sum_{i=1}^{N} \nabla_\theta \ell(f_\theta(x_i), y_i)$$

En palabras:

1. Para cada ejemplo (xᵢ, yᵢ) en tu dataset:
   a. Calculá la predicción: ŷᵢ = f_θ(xᵢ)
   b. Calculá el error: ℓ(ŷᵢ, yᵢ)
   c. Calculá el gradiente de ese error: ∇_θ ℓ
2. Promediá todos esos gradientes

Algoritmo paso a paso:

gradientes_acumulados = zeros_like(θ)

for i in range(N):
    # Forward pass
    y_pred = f_θ(x[i])
    
    # Calcular gradiente del ejemplo i
    grad_i = calcular_gradiente(θ, x[i], y[i], y_pred)
    
    # Acumular
    gradientes_acumulados += grad_i

# Promediar
gradiente_final = gradientes_acumulados / N

Usando chain rule (fundamental en backpropagation):

$$\nabla_\theta \ell(f_\theta(x_i), y_i) = \frac{\partial \ell}{\partial f_\theta} \cdot \nabla_\theta f_\theta(x_i)$$

Esto dice:

Gradiente total = (cómo cambia el error con la predicción)
                  × (cómo cambia la predicción con los parámetros)

Ejemplo para MSE en regresión lineal:

Si $f_\theta(x) = \theta^T x$ y $\ell = (f_\theta(x) - y)^2$:

$$\frac{\partial \ell}{\partial f_\theta} = 2(f_\theta(x) - y)$$

$$\nabla_\theta f_\theta(x) = x$$

Entonces:

$$\nabla_\theta \ell = 2(f_\theta(x) - y) \cdot x$$

PARTE 2: Gradiente de Regularización

$$\lambda \nabla_\theta R(\theta)$$

Para L2 ($R(\theta) = \sum_j \theta_j^2$):

$$\nabla_\theta R(\theta) = 2\theta = \begin{bmatrix} 2\theta_1 \\ 2\theta_2 \\ \vdots \\ 2\theta_d \end{bmatrix}$$

Derivación de un componente:

$$\frac{\partial}{\partial \theta_j} \sum_{k=1}^{d} \theta_k^2 = \frac{\partial}{\partial \theta_j}(\theta_1^2 + \theta_2^2 + ... + \theta_j^2 + ... + \theta_d^2)$$

Solo el término $\theta_j^2$ depende de $\theta_j$:

$$= \frac{\partial}{\partial \theta_j} \theta_j^2 = 2\theta_j$$

Interpretación:

El gradiente L2 simplemente empuja TODOS los parámetros hacia cero con fuerza proporcional a su magnitud actual.

# Ejemplo
θ = [10, 5, -3]
∇R = 2θ = [20, 10, -6]

# En el update:
θ_nuevo = θ - η(∇L_data + λ∇R)
        = θ - η∇L_data - η(2λ)θ
        = (1 - 2ηλ)θ - η∇L_data

# El factor (1 - 2ηλ) < 1 causa "weight decay"
# Los pesos se encogen un poco en cada iteración

Para L1 ($R(\theta) = \sum_j |\theta_j|$):

$$\nabla_\theta R(\theta) = \text{sign}(\theta)$$

Donde:

$$\text{sign}(\theta_j) = \begin{cases} +1 & \text{si } \theta_j > 0 \\ -1 & \text{si } \theta_j < 0 \\ \text{undefined} & \text{si } \theta_j = 0 \end{cases}$$

En práctica usamos 0 cuando $\theta_j = 0$.

Diferencia crítica con L2:

# L2
θ = 100 → ∇R = 200 (fuerza proporcional)
θ = 10  → ∇R = 20  (fuerza más débil)
θ = 1   → ∇R = 2   (muy débil cerca de 0)

# L1  
θ = 100 → ∇R = 1  (fuerza constante)
θ = 10  → ∇R = 1  (misma fuerza)
θ = 1   → ∇R = 1  (misma fuerza hasta cruzar 0)

# L1 empuja con fuerza constante → lleva parámetros a EXACTAMENTE 0
# L2 empuja con fuerza decreciente → solo los hace pequeños

📐 Gradiente de MSE para Regresión Lineal (Derivación Completa)

Para regresión lineal con MSE y regularización L2:

$$\nabla_\theta \mathcal{L} = \frac{2}{N} \sum_{i=1}^{N} (f_\theta(x_i) - y_i) x_i + 2\lambda\theta$$

Vamos a derivar esto desde cero.

Setup Completo

  • Modelo: $f_\theta(x) = \theta^T x = \sum_{j=1}^{d} \theta_j x_j$ (regresión lineal)
  • Loss individual: $\ell_i = (f_\theta(x_i) - y_i)^2$
  • Regularización: $R(\theta) = \sum_{j=1}^{d} \theta_j^2$

Loss total:

$$\mathcal{L}(\theta) = \frac{1}{N} \sum_{i=1}^{N} (f_\theta(x_i) - y_i)^2 + \lambda \sum_{j=1}^{d} \theta_j^2$$

Expandiendo $f_\theta(x_i) = \theta^T x_i$:

$$= \frac{1}{N} \sum_{i=1}^{N} (\theta^T x_i - y_i)^2 + \lambda \sum_{j=1}^{d} \theta_j^2$$

Derivación Respecto a UN Parámetro

Primero derivamos respecto a $\theta_k$ (un parámetro específico), luego generalizamos.

Paso 1: Derivar el término de error

$$\frac{\partial}{\partial \theta_k} \left[ \frac{1}{N} \sum_{i=1}^{N} (\theta^T x_i - y_i)^2 \right]$$

Aplicamos chain rule:

$$= \frac{1}{N} \sum_{i=1}^{N} \frac{\partial}{\partial \theta_k} (\theta^T x_i - y_i)^2$$

La derivada de $(u)^2$ es $2u \cdot \frac{\partial u}{\partial \theta_k}$:

$$= \frac{1}{N} \sum_{i=1}^{N} 2(\theta^T x_i - y_i) \cdot \frac{\partial}{\partial \theta_k}(\theta^T x_i - y_i)$$

Ahora, $\theta^T x_i = \theta_1 x_{i1} + \theta_2 x_{i2} + ... + \theta_k x_{ik} + ... + \theta_d x_{id}$

Cuando derivás respecto a $\theta_k$, solo sobrevive el término con $\theta_k$:

$$\frac{\partial}{\partial \theta_k}(\theta^T x_i) = x_{ik}$$

(Todos los demás $\theta_j$ con $j \neq k$ son tratados como constantes)

El término $-y_i$ es constante respecto a $\theta_k$, entonces:

$$\frac{\partial}{\partial \theta_k}(\theta^T x_i - y_i) = x_{ik}$$

Sustituyendo:

$$= \frac{2}{N} \sum_{i=1}^{N} (\theta^T x_i - y_i) x_{ik}$$

Paso 2: Derivar el término de regularización

$$\frac{\partial}{\partial \theta_k} \left[ \lambda \sum_{j=1}^{d} \theta_j^2 \right]$$

La suma $\sum_{j=1}^{d} \theta_j^2 = \theta_1^2 + \theta_2^2 + ... + \theta_k^2 + ... + \theta_d^2$

Solo $\theta_k^2$ depende de $\theta_k$:

$$= \lambda \frac{\partial}{\partial \theta_k} \theta_k^2 = \lambda \cdot 2\theta_k = 2\lambda\theta_k$$

Paso 3: Sumar ambos términos

$$\frac{\partial \mathcal{L}}{\partial \theta_k} = \frac{2}{N} \sum_{i=1}^{N} (\theta^T x_i - y_i) x_{ik} + 2\lambda\theta_k$$

Vectorizando Para Todos los Parámetros

Para obtener el gradiente completo de todos los parámetros simultáneamente:

$$\nabla_\theta \mathcal{L} = \begin{bmatrix} \frac{\partial \mathcal{L}}{\partial \theta_1} \\ \vdots \\ \frac{\partial \mathcal{L}}{\partial \theta_d} \end{bmatrix} = \frac{2}{N} \sum_{i=1}^{N} (\theta^T x_i - y_i) x_i + 2\lambda\theta$$

En forma matricial (mucho más eficiente para implementar):

Si definimos:

  • $X$ = matriz de features (N × d), donde la fila $i$ es $x_i^T$
  • $y$ = vector de labels (N × 1)
  • $\theta$ = vector de parámetros (d × 1)

Entonces:

$$X\theta = \begin{bmatrix} x_1^T \theta \\ x_2^T \theta \\ \vdots \\ x_N^T \theta \end{bmatrix} = \begin{bmatrix} f_\theta(x_1) \\ f_\theta(x_2) \\ \vdots \\ f_\theta(x_N) \end{bmatrix}$$

El gradiente se vuelve:

$$\nabla_\theta \mathcal{L} = \frac{2}{N} X^T(X\theta - y) + 2\lambda\theta$$

Verificación dimensional:

  • $X$: N × d
  • $X\theta$: N × 1 (predicciones)
  • $X\theta - y$: N × 1 (errores)
  • $X^T$: d × N
  • $X^T(X\theta - y)$: d × 1 ✓ (correcto, misma dimensión que $\theta$)

Interpretación Intuitiva

$$\underbrace{(\theta^T x_i - y_i)}_{\text{error en ejemplo i}} \times \underbrace{x_i}_{\text{features del ejemplo}}$$

¿Por qué multiplicar por $x_i$**?**

Mirá este ejemplo concreto:

# Predicción de precio de casa
xᵢ = [120m², 3 habitaciones, zona=1, 5 años]
yᵢ = 200M (precio real)
ŷᵢ = f_θ(xᵢ) = 180M (predicción)

error = 180M - 200M = -20M (subestimamos)

# Gradiente para cada peso:
∂L/∂θ₁ (m²) = -20M × 120 = -2400M·m²
∂L/∂θ₂ (hab) = -20M × 3 = -60M
∂L/∂θ₃ (zona) = -20M × 1 = -20M
∂L/∂θ₄ (años) = -20M × 5 = -100M

Interpretación:

  1. Todos los gradientes son negativos → todos los pesos deben AUMENTAR
  2. El área (120) tiene el gradiente más grande → su peso será el que más cambie
  3. Features con valores grandes causan updates grandes

Por eso SIEMPRE normalizás features en producción:

# Sin normalizar
x = [1000, 0.01]  # Escalas muy diferentes
gradientes = error × x = [-20 × 1000, -20 × 0.01] = [-20000, -0.2]

# Con learning rate fijo η = 0.01:
θ₁_nuevo = θ₁ - 0.01(-20000) = θ₁ + 200  # ¡Salto gigante!
θ₂_nuevo = θ₂ - 0.01(-0.2) = θ₂ + 0.002  # Microscópico

# Con normalizar primero:
x_norm = [0, 1]  # Después de normalizar
gradientes = [-20 × 0, -20 × 1] = [0, -20]
# Ambos parámetros actualizan a escala similar ✓

📐 Mean Squared Error (MSE) - Desglose Profundo

$$\text{MSE} = \frac{1}{N} \sum_{i=1}^{N} (y_i - \hat{y}_i)^2$$

Ya vimos esto antes, pero profundicemos más en el "por qué".

Componentes

$(y_i - \hat{y}_i)$ - El Residuo

Residuo = diferencia entre realidad y predicción
        = puede ser positivo o negativo

y = 100  # Valor real
ŷ = 95   # Predicción

residuo = y - ŷ = 100 - 95 = +5  # Subestimaste

$(y_i - \hat{y}_i)^2$ - Residuo al Cuadrado

Las 3 razones por las que elevamos al cuadrado:

1. Simetría (errores en ambas direcciones son igualmente malos):

subestimar 5 → residuo = +5 → error² = 25
sobrestimar 5 → residuo = -5 → error² = 25

# Sin el cuadrado:
promedio([+5, -5, +3, -3]) = 0  # ¡Parece perfecto pero no lo es!

# Con el cuadrado:
promedio([25, 25, 9, 9]) = 17  # Correctamente penaliza

2. Penalización no lineal de errores grandes:

Error Error² Ratio
1 1 1×
2 4 4×
5 25 25×
10 100 100×

Un error 10× peor se penaliza 100× más. Esto hace que el modelo se ESFUERCE más en corregir outliers.

Gráfica visual:

Error²
  ↑
  |         ╱
  |      ╱
  |   ╱
  | ╱
  └──────→ Error
  
Crece cuadráticamente

3. Propiedades matemáticas suaves:

$$\frac{d}{dx}x^2 = 2x$$

  • Diferenciable en todos lados (incluido x=0)
  • Derivada simple y eficiente de calcular
  • No tiene quiebres como $|x|$

Comparar con Mean Absolute Error:

$$\frac{d}{dx}|x| = \begin{cases} +1 & x > 0 \\ -1 & x < 0 \\ \text{undefined} & x = 0 \end{cases}$$

El quiebre en 0 complica la optimización.

Alternativas y Cuándo Usarlas

Mean Absolute Error (MAE):

$$\text{MAE} = \frac{1}{N} \sum_{i=1}^{N} |y_i - \hat{y}_i|$$

Diferencia clave:

# Dataset con outlier
predicciones = [10, 11, 12, 100]  # El 100 es outlier
ground_truth = [10, 11, 12, 13]
errores = [0, 0, 0, 87]

MSE = (0² + 0² + 0² + 87²) / 4 = 7569/4 = 1892.25
MAE = (0 + 0 + 0 + 87) / 4 = 87/4 = 21.75

# MSE castiga el outlier 1892.25 vs 21.75
# MAE es ~87× más robusto a outliers

Cuándo usar MSE:

  • Outliers son IMPORTANTES (ej. detectar fraude, predicciones financieras)
  • Querés penalizar fuertemente grandes errores
  • Datos siguen distribución Normal

Cuándo usar MAE:

  • Outliers son RUIDO que querés ignorar
  • Todas las magnitudes de error importan igual
  • Datos con distribución heavy-tailed

Huber Loss (combina ambos):

$$L_\delta = \begin{cases} \frac{1}{2}(y - \hat{y})^2 & \text{si } |y - \hat{y}| \leq \delta \\ \delta|y - \hat{y}| - \frac{1}{2}\delta^2 & \text{si } |y - \hat{y}| > \delta \end{cases}$$

Se comporta como MSE para errores pequeños, como MAE para errores grandes. ¡Lo mejor de ambos mundos!

📐 Cross-Entropy Loss - Desglose Profundo

$$\text{CE} = -\frac{1}{N} \sum_{i=1}^{N} [y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]$$

Esta es la función de pérdida estándar para clasificación binaria donde:

  • $y_i \in \{0, 1\}$ (etiqueta verdadera)
  • $\hat{y}_i \in (0, 1)$ (probabilidad predicha de clase 1)

La Fórmula Tiene DOS Casos

Caso 1: Ejemplo Positivo ($y_i = 1$)

Cuando $y_i = 1$:

$$\text{Loss}_i = -\log(\hat{y}_i)$$

Porque:

  • Primer término: $y_i \log(\hat{y}_i) = 1 \cdot \log(\hat{y}_i)$ ✓
  • Segundo término: $(1-y_i)\log(1-\hat{y}_i) = 0 \cdot \log(...) = 0$ (se cancela)

Tabla de comportamiento:

$\hat{y}_i$ $-\log(\hat{y}_i)$ Interpretación
0.99 0.01 Muy confiado y correcto ✓✓
0.90 0.11 Confiado y correcto ✓
0.70 0.36 Moderadamente confiado
0.50 0.69 Incierto
0.30 1.20 Incorrecto
0.10 2.30 Muy incorrecto ✗
0.01 4.61 Catastrófico ✗✗

Gráfica:

Loss
  ↑ 10 |     ╱
   8 |    ╱
   6 |   ╱
   4 |  ╱
   2 | ╱
   0 |╱___________
     0   0.5    1.0  ŷ
     
Cuando ŷ → 0 (muy equivocado), loss → ∞
Cuando ŷ → 1 (muy correcto), loss → 0

Caso 2: Ejemplo Negativo ($y_i = 0$)

Cuando $y_i = 0$:

$$\text{Loss}_i = -\log(1 - \hat{y}_i)$$

Porque:

  • Primer término: $y_i \log(\hat{y}_i) = 0$ (se cancela)
  • Segundo término: $(1-y_i)\log(1-\hat{y}_i) = 1 \cdot \log(1-\hat{y}_i)$ ✓

Tabla de comportamiento:

$\hat{y}_i$ $1-\hat{y}_i$ $-\log(1-\hat{y}_i)$ Interpretación
0.01 0.99 0.01 Muy confiado y correcto ✓✓
0.10 0.90 0.11 Confiado y correcto ✓
0.30 0.70 0.36 Moderadamente confiado
0.50 0.50 0.69 Incierto
0.70 0.30 1.20 Incorrecto
0.90 0.10 2.30 Muy incorrecto ✗
0.99 0.01 4.61 Catastrófico ✗✗

¿Por Qué Funciona Cross-Entropy?

Viene de teoría de información. La cross-entropy entre dos distribuciones $p$ (verdadera) y $q$ (predicha) es:

$$H(p, q) = -\sum_x p(x) \log q(x)$$

En clasificación binaria:

  • $p$ = distribución verdadera: probabilidad 1 en la clase correcta, 0 en las demás
  • $q = \hat{y}$ = tu distribución predicha

Para un ejemplo con $y=1$:

$$p = [0, 1]$$

(0% clase 0, 100% clase 1)

$$q = [1-\hat{y}, \hat{y}]$$

(tu predicción)

$$H(p,q) = -[0 \cdot \log(1-\hat{y}) + 1 \cdot \log(\hat{y})] = -\log(\hat{y})$$

¡Exactamente nuestra fórmula!

Minimizar cross-entropy = hacer que tu distribución predicha $q$ se parezca a la verdadera $p$**.**

Gradiente de Cross-Entropy

$$\frac{\partial \text{CE}}{\partial \hat{y}_i} = -\frac{y_i}{\hat{y}_i} + \frac{1-y_i}{1-\hat{y}_i}$$

Derivación:

$$\frac{\partial}{\partial \hat{y}} [-y \log(\hat{y}) - (1-y)\log(1-\hat{y})]$$

$$= -y \cdot \frac{1}{\hat{y}} - (1-y) \cdot \frac{-1}{1-\hat{y}}$$

$$= -\frac{y}{\hat{y}} + \frac{1-y}{1-\hat{y}}$$

Para $y=1$:

$$\frac{\partial \text{CE}}{\partial \hat{y}} = -\frac{1}{\hat{y}}$$

Ejemplos:

ŷ = 0.1 → gradiente = -10  # Fuerte señal de aumentar ŷ
ŷ = 0.5 → gradiente = -2   # Señal moderada
ŷ = 0.9 → gradiente = -1.11 # Señal débil (ya estás cerca)

Para $y=0$:

$$\frac{\partial \text{CE}}{\partial \hat{y}} = \frac{1}{1-\hat{y}}$$

Ejemplos:

ŷ = 0.9 → gradiente = +10   # Fuerte señal de disminuir ŷ
ŷ = 0.5 → gradiente = +2    # Señal moderada  
ŷ = 0.1 → gradiente = +1.11  # Señal débil

Combinando con Sigmoid (Magia Matemática)

En clasificación binaria usualmente:

$$\hat{y} = \sigma(z) = \frac{1}{1 + e^{-z}}$$

Donde $z = \theta^T x$ (el logit, salida pre-activación).

Propiedad mágica: La derivada de cross-entropy + sigmoid se SIMPLIFICA:

$$\frac{\partial \text{CE}}{\partial z} = \hat{y} - y$$

¡El gradiente es simplemente el residuo! Super simple y eficiente.

Derivación completa:

Por chain rule:

$$\frac{\partial \text{CE}}{\partial z} = \frac{\partial \text{CE}}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial z}$$

Sabemos que para sigmoid:

$$\frac{\partial \sigma}{\partial z} = \sigma(z)(1-\sigma(z)) = \hat{y}(1-\hat{y})$$

Para $y=1$:

$$\frac{\partial \text{CE}}{\partial z} = -\frac{1}{\hat{y}} \cdot \hat{y}(1-\hat{y}) = -(1-\hat{y}) = \hat{y} - 1$$

Para $y=0$:

$$\frac{\partial \text{CE}}{\partial z} = \frac{1}{1-\hat{y}} \cdot \hat{y}(1-\hat{y}) = \hat{y}$$

En ambos casos: $\hat{y} - y$ ✓

Por esto cross-entropy + sigmoid es TAN popular: gradiente súper simple y numéricamente estable.

🎯 Resumen Ejecutivo

El aprendizaje supervisado no es "predecir Y dado X". Es resolver:

$$\min_\theta \underbrace{\mathbb{E}_{(x,y) \sim \mathcal{D}}[\ell(f_\theta(x), y)]}_{\text{Ajuste a los datos}} + \underbrace{\lambda R(\theta)}_{\text{Regularización}}$$

Mediante:

$$\theta^{(t+1)} = \theta^{(t)} - \eta \nabla_\theta \mathcal{L}(\theta^{(t)})$$

Con todas las implicaciones que eso conlleva:

  • Normalización: Previene gradientes desbalanceados
  • Regularización: Previene overfitting
  • Learning rate: Controla convergencia vs divergencia
  • Batch size: Balancea memoria y generalización

🚨 Errores Comunes (y Cómo No Cagarla)

Error 1: No Normalizar Features

El problema:

# Esto EXPLOTA en producción
X = np.array([[1000, 0.01], [2000, 0.02]])  # Escalas MUY diferentes
model.fit(X, y, learning_rate=0.1)
# Loss → NaN en ~3 epochs

Por qué falla:

Gradientes sin normalizar:

$$\nabla_{\theta_1} \mathcal{L} = \frac{2}{N} \sum (error) \times 1000$$

$$\nabla_{\theta_2} \mathcal{L} = \frac{2}{N} \sum (error) \times 0.01$$

Con el mismo $\eta$:

θ₁ -= 0.1 × (error × 1000)  # Salto gigante!
θ₂ -= 0.1 × (error × 0.01)  # Microscópico

$\theta_1$ diverge mientras $\theta_2$ se estanca.

La solución:

# SIEMPRE normalizar
mean = X_train.mean(axis=0)
std = X_train.std(axis=0)
X_normalized = (X - mean) / (std + 1e-8)

Error 2: Learning Rate Incorrecto

Señales de problemas:

# η muy alto
Epoch 0: loss = 1.5
Epoch 1: loss = 8.3  # ¡Aumenta!
Epoch 2: loss = NaN  # Explosión total

# η muy bajo  
Epoch 0: loss = 2.5
Epoch 100: loss = 2.49  # Microscópico avance
Epoch 1000: loss = 2.30  # Aún muy lejos

Regla empírica:

  1. Empezá con $\eta = 0.01$
  2. Si loss aumenta → reducí 10× ($\eta = 0.001$)
  3. Si converge muy lento → aumentá 3× ($\eta = 0.03$)
  4. Grid search: $[0.001, 0.003, 0.01, 0.03, 0.1]$

Error 3: Overfitting Sin Darte Cuenta

Señal clásica:

train_loss = 0.01  # ¡Excelente!
val_loss = 2.50    # WTF?

# Gap gigante = overfitting

Soluciones:

  1. Aumentar $\lambda$ (regularización): $0 \to 0.01 \to 0.1$
  2. Más datos de entrenamiento
  3. Early stopping cuando val_loss empieza a subir
  4. Reducir complejidad del modelo

Error 4: No Validar Inputs

Bug clásico que vi mil veces:

# Usuario pasa features sin normalizar
X_prod = get_features_from_api()  # Escala diferente!
prediction = model.predict(X_prod)  # Predicción BASURA

Siempre guardá las estadísticas de normalización:

@dataclass
class PreprocessingStats:
    mean: NDArrayFloat
    std: NDArrayFloat
    
    def normalize(self, X: NDArrayFloat) -> NDArrayFloat:
        return (X - self.mean) / (self.std + 1e-8)

# Guardar con el modelo
stats = PreprocessingStats(mean=..., std=...)
save_model(model, stats)

# Cargar en inference
model, stats = load_model()
X_normalized = stats.normalize(X_new)
pred = model.predict(X_normalized)
