🎯 ¿Por Qué Sigmoide NO es lo Mismo que Softmax?

Mirá, Sigmoide es como cuando tenés UN interruptor y querés saber "¿prendido o apagado?". Softmax es como cuando tenés VARIOS botones y solo podés presionar UNO, pero necesitás calcular cuál tiene más probabilidad de ser presionado.

Definición técnica (sin rodeos):

  • Sigmoide (Logística): Función $\sigma: \mathbb{R} \to (0,1)$ que mapea cualquier número real a un valor entre 0 y 1, interpretable como probabilidad de clase positiva.

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

  • Softmax: Función $\text{softmax}: \mathbb{R}^K \to \Delta^{K-1}$ que transforma un vector de $K$ logits en una distribución de probabilidad válida sobre $K$ clases (suma = 1).

$$\text{softmax}(z)_i = \frac{e^{z_i}}{\sum_{j=1}^K e^{z_j}} \quad \text{para } i = 1, \ldots, K$$

Casos de uso principales:

  • Sigmoide: Clasificación binaria (1 output), multi-label (múltiples outputs independientes), cualquier tarea donde las clases NO son mutuamente excluyentes.
  • Softmax: Clasificación multiclase (K outputs), donde las clases SON mutuamente excluyentes (una imagen es gato O perro O caballo, no dos a la vez).

ADVERTENCIA CRÍTICA: Si usás Sigmoide en multiclase, perdés la restricción de que las probabilidades sumen 1. Si usás Softmax en multi-label, forzás exclusividad donde no existe. Esto no solo infla tus métricas, te hace tomar decisiones de negocio equivocadas.

📐 La Función Sigmoide: El Interruptor Suave

Arrancamos con la joya. La función sigmoide (o logística) es la base de la regresión logística y el fundamento de muchísimas arquitecturas de redes neuronales.

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

Esta ecuación parece simple pero tiene TODO el poder de convertir cualquier número real (positivo gigante, negativo gigante, lo que sea) en una probabilidad válida entre 0 y 1. Vamos a desmenuzarla símbolo por símbolo porque acá es donde la mayoría se pierde.

Descomposición Completa: Entendiendo Cada Símbolo

$z$ (Logit / Score) - El Input Crudo

z = entrada sin restricciones
  = output de tu modelo antes de activación
  = combinación lineal de features

Analogía: El puntaje crudo de un examen antes de convertirlo a escala de 0-100.

En Machine Learning:

  • Para regresión logística: $z = w^T x + b$
  • Para una neurona: $z = \sum_{i} w_i x_i + b$

Rango: $z \in (-\infty, +\infty)$. No tiene límites, puede ser cualquier número real.

Diferencia CRÍTICA: $z$ NO es una probabilidad. Es un "score" o "logit" que necesita transformación.

Ejemplo numérico concreto:

import numpy as np
from scipy.special import expit

# Logits típicos
z_examples = np.array([-10.0, -2.0, 0.0, 2.0, 10.0])
print(f"Logits (z): {z_examples}")
# Output: Logits (z): [-10.  -2.   0.   2.  10.]

$e$ (Número de Euler) - La Base Exponencial

e ≈ 2.71828...
  = constante matemática fundamental
  = base del logaritmo natural

Analogía: El ritmo natural de crecimiento en la naturaleza (interés compuesto continuo, decaimiento radiactivo, etc.).

En Machine Learning:

  • Garantiza que $\sigma(z)$ sea suave y diferenciable en todos lados
  • Su derivada tiene propiedades hermosas: $\frac{d}{dz}e^z = e^z$

Por qué $e$ y no otra base: Porque $\frac{d\sigma}{dz} = \sigma(z)(1 - \sigma(z))$, una propiedad que simplifica ENORMEMENTE el backpropagation.

$e^{-z}$ (Exponencial Negativa) - El Numerador Invertido

e^{-z} = 1 / e^z
       = cantidad que decrece cuando z crece

Comportamiento clave:

  • Si $z \to +\infty$, entonces $e^{-z} \to 0$
  • Si $z \to -\infty$, entonces $e^{-z} \to +\infty$

En Machine Learning:

  • Valores altos de $z$ (clase positiva) → $e^{-z}$ pequeño → $\sigma(z) \approx 1$
  • Valores bajos de $z$ (clase negativa) → $e^{-z}$ grande → $\sigma(z) \approx 0$

Problema de implementación naive:

# ❌ ESTO PUEDE EXPLOTAR
z_big = 1000.0
exp_neg_z = np.exp(-z_big)  # Underflow a 0
denominator = 1 + exp_neg_z
sigmoid_naive = 1 / denominator
print(f"Sigmoid naive: {sigmoid_naive}")  # 1.0 (perdiste precisión)

$1 + e^{-z}$ (Denominador) - El Normalizador

1 + e^{-z} = factor de normalización
           = garantiza que el output esté en (0, 1)
           = nunca es cero (siempre ≥ 1)

Dimensionalidad importa: Este es un escalar. Si $z$ es un vector, aplicas sigmoide elemento por elemento (broadcasting).

Propiedad de simetría:

$$\sigma(-z) = 1 - \sigma(z)$$

Esto significa que la sigmoide es simétrica respecto al punto $(0, 0.5)$.

Ejemplo numérico completo:

from scipy.special import expit

z = np.array([-10.0, -2.0, 0.0, 2.0, 10.0])
sigmoid_values = expit(z)  # ✓ Implementación estable de scipy

print(f"z: {z}")
print(f"σ(z): {sigmoid_values}")
print(f"σ(-z): {expit(-z)}")
print(f"1 - σ(z): {1 - sigmoid_values}")
print(f"Simetría OK: {np.allclose(expit(-z), 1 - sigmoid_values)}")

# Output:
# z: [-10.  -2.   0.   2.  10.]
# σ(z): [4.53978687e-05 1.19202922e-01 5.00000000e-01 8.80797078e-01 9.99954602e-01]
# σ(-z): [9.99954602e-01 8.80797078e-01 5.00000000e-01 1.19202922e-01 4.53978687e-05]
# 1 - σ(z): [9.99954602e-01 8.80797078e-01 5.00000000e-01 1.19202922e-01 4.53978687e-05]
# Simetría OK: True

Gradiente de Sigmoide: Por Qué Es Tan Importante

El gradiente de la sigmoide es fundamental para el entrenamiento con backpropagation:

$$\frac{d\sigma}{dz} = \sigma(z) \cdot (1 - \sigma(z))$$

¿Por qué es hermoso?: El gradiente se puede calcular usando SOLO el output de la sigmoide. No necesitás recalcular exponenciales.

Implicación práctica: En forward pass guardás $\sigma(z)$, en backward pass calculás el gradiente como $\sigma \cdot (1 - \sigma)$. Eficiente y elegante.

Problema del vanishing gradient:

z_range = np.linspace(-10, 10, 100)
sigmoid_vals = expit(z_range)
gradient_vals = sigmoid_vals * (1 - sigmoid_vals)

print(f"Gradiente en z=-10: {expit(-10) * (1 - expit(-10)):.10f}")  # ≈ 0.0000454
print(f"Gradiente en z=0: {expit(0) * (1 - expit(0)):.10f}")      # 0.25
print(f"Gradiente en z=10: {expit(10) * (1 - expit(10)):.10f}")   # ≈ 0.0000454

El gradiente máximo es 0.25 (en $z=0$) y se desvanece rápidamente en los extremos. Por eso las redes profundas con sigmoide en todas las capas sufren vanishing gradient.

📐 La Función Softmax: Competencia Entre Clases

Ahora la función softmax, la generalización multiclase de la sigmoide. Acá el juego cambia: no es "cada salida independiente", sino "todas las salidas compiten por probabilidad total = 1".

$$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i}}{\sum_{j=1}^K e^{z_j}} \quad \text{para } i = 1, \ldots, K$$

Esta ecuación convierte un vector de $K$ logits en una distribución de probabilidad válida. Vamos símbolo por símbolo.

Descomposición Completa: Entendiendo Cada Símbolo

$\mathbf{z}$ (Vector de Logits) - Scores Para Todas las Clases

z = [z_1, z_2, ..., z_K]
  = vector de scores sin restricciones
  = outputs de tu red antes de softmax

Analogía: Los puntajes de K candidatos en una elección antes de convertirlos a porcentajes de votos.

En Machine Learning:

  • Para clasificación 3-clases: $\mathbf{z} = [z_{\text{gato}}, z_{\text{perro}}, z_{\text{caballo}}]$
  • Cada $z_i \in (-\infty, +\infty)$, sin restricciones

Dimensionalidad importa: $\mathbf{z} \in \mathbb{R}^K$. Es un vector, no un escalar.

Ejemplo numérico concreto:

from scipy.special import softmax

# Logits para 4 clases
z = np.array([2.0, 1.0, 0.1, 3.0])
print(f"Logits: {z}")
# Output: Logits: [2.  1.  0.1 3. ]

$e^{z_i}$ (Exponencial de Logit Individual) - Score Exponenciado

e^{z_i} = exponencial del score de la clase i
        = "peso" no normalizado de la clase i

Por qué exponencial:

  • Convierte diferencias aditivas en diferencias multiplicativas
  • Hace que diferencias grandes de logits se traduzcan en diferencias ENORMES de probabilidades
  • Garantiza que todos los valores sean positivos (requisito para probabilidades)

Comportamiento clave:

  • Si $z_i = z_j + 2$, entonces $e^{z_i} = e^2 \cdot e^{z_j} \approx 7.39 \cdot e^{z_j}$
  • Pequeñas diferencias de logits → grandes diferencias de probabilidad

Ejemplo:

z = np.array([2.0, 1.0, 0.1, 3.0])
exp_z = np.exp(z)
print(f"e^z: {exp_z}")
print(f"Ratios: clase_4/clase_1 = {exp_z[3]/exp_z[0]:.2f}")
# Output: e^z: [ 7.3890561   2.71828183  1.10517092 20.08553692]
# Ratios: clase_4/clase_1 = 2.72

$\sum_{j=1}^K e^{z_j}$ (Suma de Exponenciales) - Función de Partición

sum_exp = Σ e^{z_j}
        = normalización global
        = "función de partición" en física estadística

Analogía: La suma total de votos para calcular porcentajes.

En Machine Learning:

  • Garantiza que $\sum_{i=1}^K \text{softmax}(\mathbf{z})_i = 1$
  • Es la misma para todas las clases (se calcula una vez)

Problema de implementación naive:

# ❌ ESTO EXPLOTA CON OVERFLOW
z_big = np.array([1000.0, 1001.0, 999.0])
exp_z = np.exp(z_big)  # inf, inf, inf
sum_exp = np.sum(exp_z)  # inf
softmax_naive = exp_z / sum_exp  # nan, nan, nan
print(f"Softmax naive: {softmax_naive}")  # [nan nan nan]

La solución (trick de estabilidad numérica):

$$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i - \max(\mathbf{z})}}{\sum_{j=1}^K e^{z_j - \max(\mathbf{z})}}$$

Restar $\max(\mathbf{z})$ NO cambia el resultado (es invariante por traslación) pero previene overflow.

$\frac{e^{z_i}}{\sum_{j=1}^K e^{z_j}}$ (Probabilidad Normalizada) - El Output Final

softmax(z)_i = probabilidad de la clase i
             = valor entre 0 y 1
             = suma a 1 con todas las otras clases

Propiedades matemáticas:

  1. $\text{softmax}(\mathbf{z})_i \in (0, 1)$ para todo $i$
  2. $\sum_{i=1}^K \text{softmax}(\mathbf{z})_i = 1$ (distribución válida)
  3. $\arg\max_i \text{softmax}(\mathbf{z})_i = \arg\max_i z_i$ (preserva orden)

Ejemplo numérico completo:

from scipy.special import softmax

z = np.array([2.0, 1.0, 0.1, 3.0])
probs = softmax(z)

print(f"Logits: {z}")
print(f"Softmax probs: {probs}")
print(f"Suma: {np.sum(probs):.10f}")
print(f"Clase predicha: {np.argmax(probs)}")

# Output:
# Logits: [2.  1.  0.1 3. ]
# Softmax probs: [0.23632777 0.08714432 0.03530352 0.64122439]
# Suma: 1.0000000000
# Clase predicha: 3

Gradiente de Softmax: La Matriz Jacobiana

El gradiente de softmax es más complejo porque cada output depende de TODOS los logits:

$$\frac{\partial \text{softmax}(\mathbf{z})_i}{\partial z_j} = \begin{cases} \text{softmax}(\mathbf{z})_i \cdot (1 - \text{softmax}(\mathbf{z})_i) & \text{si } i = j \\ -\text{softmax}(\mathbf{z})_i \cdot \text{softmax}(\mathbf{z})_j & \text{si } i \neq j \end{cases}$$

Implicación práctica: En backpropagation con cross-entropy, el gradiente combinado de softmax + cross-entropy se simplifica a $\hat{y} - y$ (diferencia entre predicción y ground truth). Por eso siempre se usan juntas.

📐 Relación Entre Sigmoide y Softmax

Acá viene la conexión que muchos no ven: la sigmoide es un caso especial de softmax para $K=2$.

Para clasificación binaria, podés usar:

Opción 1: Sigmoide (1 output)

$$P(y=1|\mathbf{x}) = \sigma(z) = \frac{1}{1 + e^{-z}}$$

$$P(y=0|\mathbf{x}) = 1 - \sigma(z)$$

Opción 2: Softmax (2 outputs)

$$\mathbf{z} = [z_0, z_1]$$

$$P(y=0|\mathbf{x}) = \frac{e^{z_0}}{e^{z_0} + e^{z_1}}$$

$$P(y=1|\mathbf{x}) = \frac{e^{z_1}}{e^{z_0} + e^{z_1}}$$

Demostración de equivalencia: Si $z = z_1 - z_0$, entonces:

$$\frac{e^{z_1}}{e^{z_0} + e^{z_1}} = \frac{e^{z_1}}{e^{z_0} + e^{z_1}} \cdot \frac{e^{-z_0}}{e^{-z_0}} = \frac{e^{z_1 - z_0}}{1 + e^{z_1 - z_0}} = \frac{e^z}{1 + e^z} = \frac{1}{1 + e^{-z}} = \sigma(z)$$

Verificación numérica:

# Sigmoide con 1 output
z_diff = 2.0
prob_sigmoid = expit(z_diff)

# Softmax con 2 outputs
z_softmax = np.array([0.0, 2.0])  # z_1 - z_0 = 2.0
prob_softmax = softmax(z_softmax)

print(f"Sigmoide: P(y=1) = {prob_sigmoid:.6f}")
print(f"Softmax: P(y=1) = {prob_softmax[1]:.6f}")
print(f"Equivalentes: {np.isclose(prob_sigmoid, prob_softmax[1])}")

# Output:
# Sigmoide: P(y=1) = 0.880797
# Softmax: P(y=1) = 0.880797
# Equivalentes: True

En la práctica: Usá sigmoide para binaria (más eficiente, 1 parámetro menos). Usá softmax para multiclase.

🚨 Errores Comunes (y Cómo No Cagarla)

Error 1: Usar Sigmoide en Multiclase Mutuamente Excluyente

El problema:

# ❌ MAL: Usar sigmoid para 3 clases mutuamente excluyentes
import numpy as np
from scipy.special import expit

logits = np.array([2.0, 1.0, 3.0])  # Gato, Perro, Caballo
probs_wrong = expit(logits)  # Aplica sigmoid elemento por elemento

print(f"Logits: {logits}")
print(f"'Probabilidades' con sigmoid: {probs_wrong}")
print(f"Suma: {np.sum(probs_wrong):.4f}")  # ❌ NO suma 1

# Output:
# Logits: [2. 1. 3.]
# 'Probabilidades' con sigmoid: [0.88079708 0.73105858 0.95257413]
# Suma: 2.5644  # ❌ Esto NO es una distribución de probabilidad válida

Por qué falla:

Sigmoide trata cada logit de forma independiente. No hay restricción de que sumen 1. Esto significa:

$$P(\text{gato}) + P(\text{perro}) + P(\text{caballo}) = 0.88 + 0.73 + 0.95 = 2.56 \neq 1$$

En un problema multiclase mutuamente excluyente (una imagen ES gato O perro O caballo, no puede ser dos a la vez), las probabilidades DEBEN sumar 1. Si usás sigmoide:

  • Perdés interpretabilidad: ¿qué significa que la suma sea 2.56?
  • No podés comparar probabilidades entre clases directamente
  • Las métricas como accuracy siguen "funcionando" (usás argmax), pero tus probabilidades calibradas están MAL

La solución:

# ✓ BIEN: Usar softmax para multiclase mutuamente excluyente
from scipy.special import softmax

logits = np.array([2.0, 1.0, 3.0])
probs_correct = softmax(logits)

print(f"Logits: {logits}")
print(f"Probabilidades con softmax: {probs_correct}")
print(f"Suma: {np.sum(probs_correct):.10f}")  # ✓ Suma exactamente 1

# Output:
# Logits: [2. 1. 3.]
# Probabilidades con softmax: [0.23632777 0.08714432 0.64122439]
# Suma: 1.0000000000  # ✓ Distribución válida

Error 2: Usar Softmax en Multi-Label (Clases No Excluyentes)

El problema:

# ❌ MAL: Usar softmax para multi-label
# Escenario: imagen puede tener {cielo, árbol, persona} simultáneamente

logits = np.array([2.5, 1.8, -0.5])  # [cielo, árbol, persona]
probs_wrong = softmax(logits)

print(f"Logits: {logits}")
print(f"'Probabilidades' con softmax: {probs_wrong}")
print(f"Predicción (threshold=0.5): {probs_wrong > 0.5}")  # ❌ Solo 1 clase activa

# Output:
# Logits: [ 2.5  1.8 -0.5]
# 'Probabilidades' con softmax: [0.58974948 0.29402137 0.11622915]
# Predicción (threshold=0.5): [ True False False]  # ❌ Forzó exclusividad

Por qué falla:

Softmax fuerza competencia entre clases. Si una probabilidad sube, las otras TIENEN que bajar para mantener suma=1. En multi-label:

  • Una imagen puede tener cielo=Sí (0.9), árbol=Sí (0.85), persona=No (0.1) simultáneamente
  • Con softmax, si cielo tiene alta probabilidad (0.59), automáticamente "roba" probabilidad de árbol (0.29)

La restricción $\sum P_i = 1$ NO tiene sentido cuando las clases no son mutuamente excluyentes.

La solución:

# ✓ BIEN: Usar sigmoid independiente para cada clase en multi-label
from scipy.special import expit

logits = np.array([2.5, 1.8, -0.5])  # [cielo, árbol, persona]
probs_correct = expit(logits)  # Sigmoid independiente

print(f"Logits: {logits}")
print(f"Probabilidades con sigmoid: {probs_correct}")
print(f"Suma: {np.sum(probs_correct):.4f}")  # No tiene que sumar 1
print(f"Predicción (threshold=0.5): {probs_correct > 0.5}")

# Output:
# Logits: [ 2.5  1.8 -0.5]
# Probabilidades con sigmoid: [0.92414182 0.85814894 0.37754067]
# Suma: 2.1598  # ✓ No hay restricción de suma
# Predicción (threshold=0.5): [ True  True False]  # ✓ Múltiples activas OK

Implementación production-ready para multi-label:

from __future__ import annotations
import numpy as np
from scipy.special import expit
from typing import Dict
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class MultiLabelClassifier:
    def __init__(self, class_names: list[str], threshold: float = 0.5) -> None:
        if not class_names:
            raise ValueError("class_names no puede estar vacío")
        if not 0 < threshold < 1:
            raise ValueError(f"threshold debe estar en (0,1), got {threshold}")
        
        self.class_names = class_names
        self.threshold = threshold
        logger.info(f"MultiLabelClassifier inicializado con {len(class_names)} clases")
    
    def predict_proba(self, logits: np.ndarray) -> np.ndarray:
        if logits.shape[-1] != len(self.class_names):
            raise ValueError(
                f"Última dimensión de logits ({logits.shape[-1]}) "
                f"debe coincidir con número de clases ({len(self.class_names)})"
            )
        
        probs = expit(logits)
        logger.debug(f"Probabilidades calculadas: shape={probs.shape}")
        return probs
    
    def predict(self, logits: np.ndarray) -> Dict[str, np.ndarray]:
        probs = self.predict_proba(logits)
        predictions = probs > self.threshold
        
        return {
            'probabilities': probs,
            'predictions': predictions
        }

# Ejemplo de uso
if __name__ == "__main__":
    classifier = MultiLabelClassifier(
        class_names=['cielo', 'árbol', 'persona'],
        threshold=0.5
    )
    
    logits = np.array([2.5, 1.8, -0.5])
    results = classifier.predict(logits)
    
    for name, prob, pred in zip(
        classifier.class_names,
        results['probabilities'],
        results['predictions']
    ):
        print(f"{name}: P={prob:.3f}, Activa={pred}")

Error 3: Overflow/Underflow al Implementar Softmax Naive

El problema:

# ❌ MAL: Implementación naive sin estabilidad numérica
def softmax_naive(z: np.ndarray) -> np.ndarray:
    exp_z = np.exp(z)
    return exp_z / np.sum(exp_z)

# Esto explota:
logits_big = np.array([1000.0, 1001.0, 999.0])
probs = softmax_naive(logits_big)
print(f"Softmax naive: {probs}")  # [nan nan nan]

# Esto también falla (underflow):
logits_small = np.array([-1000.0, -1001.0, -999.0])
probs = softmax_naive(logits_small)
print(f"Softmax naive: {probs}")  # [nan nan nan]

Por qué falla:

Cuando $z_i$ es muy grande (ej: 1000), entonces $e^{z_i} = e^{1000} \approx 10^{434}$, que excede el máximo float64 ( $\approx 10^{308}$ ). El resultado es inf.

Cuando $z_i$ es muy negativo (ej: -1000), entonces $e^{z_i} = e^{-1000} \approx 10^{-434}$, que es menor que el mínimo float64 representable. El resultado es 0.0 (underflow).

En ambos casos, $\frac{\inf}{\inf}$ o $\frac{0}{0}$ da nan.

La matemática de la solución:

Softmax es invariante por traslación:

$$\text{softmax}(\mathbf{z})_i = \text{softmax}(\mathbf{z} + c)_i \quad \text{para cualquier constante } c$$

Demostración:

$$\frac{e^{z_i + c}}{\sum_j e^{z_j + c}} = \frac{e^{z_i} \cdot e^c}{\sum_j (e^{z_j} \cdot e^c)} = \frac{e^{z_i} \cdot e^c}{e^c \cdot \sum_j e^{z_j}} = \frac{e^{z_i}}{\sum_j e^{z_j}}$$

Elegimos $c = -\max(\mathbf{z})$ para que el valor máximo de $\mathbf{z} + c$ sea 0:

$$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i - \max(\mathbf{z})}}{\sum_{j=1}^K e^{z_j - \max(\mathbf{z})}}$$

Ahora:

  • El exponente más grande es 0, entonces $e^0 = 1$ (no overflow)
  • Los otros exponentes son negativos, sus exponenciales están en $(0, 1]$ (no underflow crítico)

La solución (production-ready):

# ✓ BIEN: Implementación estable numéricamente
def softmax_stable(z: np.ndarray) -> np.ndarray:
    if z.size == 0:
        raise ValueError("z no puede estar vacío")
    
    z_shifted = z - np.max(z)
    exp_z = np.exp(z_shifted)
    return exp_z / np.sum(exp_z)

# Ahora funciona:
logits_big = np.array([1000.0, 1001.0, 999.0])
probs = softmax_stable(logits_big)
print(f"Softmax estable: {probs}")  # [0.268 0.731 0.0988]  ✓

# También funciona con valores pequeños:
logits_small = np.array([-1000.0, -1001.0, -999.0])
probs = softmax_stable(logits_small)
print(f"Softmax estable: {probs}")  # [0.268 0.0988 0.731]  ✓

Error 4: No Entender la "Temperatura" en Softmax

El problema:

# Softmax sin temperatura (T=1)
logits = np.array([1.0, 2.0, 3.0])
probs_normal = softmax(logits)

print(f"Logits: {logits}")
print(f"Probs (T=1): {probs_normal}")
print(f"Entropía: {-np.sum(probs_normal * np.log(probs_normal + 1e-10)):.4f}")

# Output:
# Logits: [1. 2. 3.]
# Probs (T=1): [0.09003057 0.24472847 0.66524096]
# Entropía: 0.8997  # Distribución concentrada en clase 3

Concepto de temperatura:

Podés controlar qué tan "confiada" es la distribución usando temperatura $T$:

$$\text{softmax}_T(\mathbf{z})_i = \frac{e^{z_i / T}}{\sum_{j=1}^K e^{z_j / T}}$$

  • $T \to 0$: Distribución "fría" → Casi toda la probabilidad en la clase con max logit (determinística)
  • $T = 1$: Softmax estándar
  • $T \to \infty$: Distribución "caliente" → Todas las clases con probabilidad uniforme (máxima entropía)

Ejemplo:

def softmax_temperature(z: np.ndarray, T: float = 1.0) -> np.ndarray:
    if T <= 0:
        raise ValueError(f"Temperatura debe ser positiva, got {T}")
    
    z_scaled = z / T
    z_shifted = z_scaled - np.max(z_scaled)
    exp_z = np.exp(z_shifted)
    return exp_z / np.sum(exp_z)

logits = np.array([1.0, 2.0, 3.0])

# Baja temperatura (T=0.5): más confiada
probs_cold = softmax_temperature(logits, T=0.5)
entropy_cold = -np.sum(probs_cold * np.log(probs_cold + 1e-10))

# Normal (T=1)
probs_normal = softmax_temperature(logits, T=1.0)
entropy_normal = -np.sum(probs_normal * np.log(probs_normal + 1e-10))

# Alta temperatura (T=2.0): más uniforme
probs_hot = softmax_temperature(logits, T=2.0)
entropy_hot = -np.sum(probs_hot * np.log(probs_hot + 1e-10))

print(f"T=0.5 (fría): {probs_cold}, entropía={entropy_cold:.4f}")
print(f"T=1.0 (normal): {probs_normal}, entropía={entropy_normal:.4f}")
print(f"T=2.0 (caliente): {probs_hot}, entropía={entropy_hot:.4f}")

# Output:
# T=0.5 (fría): [0.0348 0.1381 0.8271], entropía=0.5775
# T=1.0 (normal): [0.0900 0.2447 0.6652], entropía=0.8997
# T=2.0 (caliente): [0.1863 0.3072 0.5065], entropía=1.0372

Uso práctico: Knowledge distillation usa temperatura alta ( $T=5$ o $T=10$ ) para que el modelo "profesor" genere probabilidades más suaves (más información en las clases incorrectas) que ayudan al modelo "estudiante" a aprender mejor.

🎯 Resumen Ejecutivo

Sigmoide y Softmax no son "esas funciones bonitas para clasificación". Son transformaciones matemáticas fundamentales que convierten logits sin restricciones en probabilidades válidas, con reglas de uso ESTRICTAS según el tipo de problema.

Función Sigmoide (clasificación binaria / multi-label):

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

Mediante la transformación exponencial que mapea $\mathbb{R} \to (0,1)$ con gradiente $\sigma(z)(1 - \sigma(z))$.

Función Softmax (clasificación multiclase mutuamente excluyente):

$$\text{softmax}(\mathbf{z})_i = \frac{e^{z_i}}{\sum_{j=1}^K e^{z_j}}$$

Mediante normalización exponencial con estabilidad numérica $max(\mathbf{z})$.

Con todas las implicaciones que eso conlleva:

  • Sigmoide ≠ Softmax: No son intercambiables. Sigmoide para independencia entre outputs, Softmax para competencia.
  • Multi-label ≠ Multiclase: Multi-label usa sigmoid independiente (suma puede ser ≠ 1), Multiclase usa softmax (suma = 1 obligatorio).
  • Estabilidad numérica: Usar scipy.special.expit y scipy.special.softmax o implementar el trick $z - \max(z)$ para evitar overflow.
  • Temperatura: Controla "confianza" de softmax, crítica en knowledge distillation.

De una, hágale. Con esta base de Sigmoide/Softmax + código production-ready con scipy.special.expit y scipy.special.softmax, estás listo para implementar clasificación binaria, multiclase y multi-label sin romper producción por overflow o usar la función incorrecta.

No es solo "poner números bonitos". Es entender la matemática, reconocer cuándo usar cada una, y no cagarla con overflow silencioso que te infla las métricas.
