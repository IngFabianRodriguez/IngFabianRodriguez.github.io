ðŊ ÂŋPara QuÃĐ Sirve Realmente el AUC si Ya TenÃĐs Accuracy?

MirÃĄ, el ROC y el AUC son como el velocÃ­metro versus el odÃģmetro de tu carro. La accuracy te dice "hoy anduviste rÃĄpido". El AUC te dice "ÂŋquÃĐ tan bien discrimina tu modelo en TODOS los umbrales posibles?". Son cosas completamente distintas, y confundirlas te cuesta clientes, plata y reputaciÃģn.

DefiniciÃģn tÃĐcnica (sin rodeos):

El ROC (Receiver Operating Characteristic) es una curva que representa el trade-off entre la Tasa de Verdaderos Positivos $TPR$ y la Tasa de Falsos Positivos $FPR$ al variar el umbral de decisiÃģn $\tau \in [0, 1]$.

El AUC (Area Under the Curve) es el ÃĄrea bajo esa curva:

$$AUC = \int_0^1 TPR(\tau)\, d\tau$$

Dos conceptos que vas a necesitar entender juntos:

  • ROC Curve: la curva visual que mapea $FPR$ vs $TPR$ para cada umbral. Te dice cÃģmo se comporta el modelo en todo el espectro de sensibilidad/especificidad.
  • AUC Score: el escalar entre 0 y 1 que resume esa curva. Un clasificador perfecto tiene $AUC=1.0$. Un clasificador aleatorio tiene $AUC=0.5$.

La confusiÃģn mortal es asumir que AUC alto significa modelo bueno. En datasets desbalanceados con clases raras, el AUC puede estar en 0.92 mientras el modelo ignora completamente la clase positiva. Esa vaina se llama ilusiÃģn de rendimiento y es el bug mÃĄs comÃšn que veo en equipos de ML.

ð TPR y FPR: Los Dos Ejes del Universo ROC

Antes de entender la curva, hay que entender sus componentes. Todo lo demÃĄs se deriva de aquÃ­.

$$TPR(\tau) = \frac{TP(\tau)}{TP(\tau) + FN(\tau)}, \quad FPR(\tau) = \frac{FP(\tau)}{FP(\tau) + TN(\tau)}$$

"Esta ecuaciÃģn parece trivial pero tiene TODO el poder del anÃĄlisis de clasificadores. Vamos a desmenuzarla sÃ­mbolo por sÃ­mbolo porque acÃĄ es donde la mayorÃ­a se pierde."

DescomposiciÃģn Completa: Entendiendo Cada SÃ­mbolo

$TPR$ (True Positive Rate / Sensibilidad / Recall) â PropÃģsito Simple

TPR = ÂŋDe todos los positivos reales, cuÃĄntos detectÃĐ?
    = Recall = Sensibilidad

AnalogÃ­a: Es el detector de incendios. Si hay 100 incendios y tu alarma suena 90 veces, tu TPR es 0.90. No importa cuÃĄntas veces suene en falso, eso lo mide otra cosa.

En Machine Learning:

  • TPR alto â modelo que NO se pierde positivos reales (crÃ­tico en diagnÃģstico mÃĐdico, fraude)
  • TPR bajo â modelo que deja pasar positivos (falsos negativos caros)

Diferencia CRÃTICA: $TPR$ solo considera los positivos reales. Los negativos no aparecen acÃĄ.

$FPR$ (False Positive Rate / Tasa de Alarmas Falsas) â PropÃģsito Simple

FPR = ÂŋDe todos los negativos reales, cuÃĄntos clasificÃĐ mal como positivos?
    = 1 - Especificidad

AnalogÃ­a: La alarma de tu carro que suena cuando pasa el bus. De 1000 buses que pasan (negativos reales), si tu alarma suena 50 veces, $FPR=0.05$.

En Machine Learning:

  • FPR bajo â pocas falsas alarmas (crÃ­tico cuando el costo de revisar un falso positivo es alto)
  • FPR alto â modelo ruidoso, manda demasiadas alertas que no son reales

Dimensionalidad importa: Ambos son proporciones $\in [0,1]$. El plot ROC es el espacio $[0,1] \times [0,1]$.

$\tau$ (Umbral de decisiÃģn) â PropÃģsito Simple

Ï = el corte sobre la probabilidad predicha para decir "esto es positivo"
  = por defecto 0.5 en sklearn, pero casi nunca es el Ãģptimo

AnalogÃ­a: El punto del termÃģmetro donde decidÃ­s "ya tengo fiebre". A 37.5Â°C Âŋes fiebre o no? Depende del contexto. La curva ROC explora todos los puntos del termÃģmetro a la vez.

ð El AUC Como Probabilidad de Ranking â La InterpretaciÃģn que Cambia Todo

AcÃĄ estÃĄ la joya que los libros de texto omiten. El AUC tiene una interpretaciÃģn probabilÃ­stica que es mucho mÃĄs poderosa que "ÃĄrea bajo la curva":

$$AUC = P(\hat{p}(x^{+}) > \hat{p}(x^{-}))$$

Donde $x^{+}$ es una muestra positiva aleatoria y $x^{-}$ es una negativa aleatoria.

"Esta ecuaciÃģn cambia cÃģmo evaluÃĄs tus modelos para siempre. El AUC es la probabilidad de que tu modelo le asigne un score MÃS ALTO a un positivo que a un negativo elegidos al azar."

DescomposiciÃģn: La EstadÃ­stica de Wilcoxon-Mann-Whitney

Esta interpretaciÃģn viene de la estadÃ­stica no paramÃĐtrica. El AUC es equivalente al estadÃ­stico U de Mann-Whitney:

$$AUC = \frac{\sum_{i:\, y_i=1} \sum_{j:\, y_j=0} \mathbf{1}[\hat{p}_i > \hat{p}_j]}{n^{+} \cdot n^{-}}$$

$n^{+}$ y $n^{-}$ (Conteos de clase) â PropÃģsito Simple

nâš = nÃšmero de muestras positivas en el conjunto de evaluaciÃģn
nâŧ = nÃšmero de muestras negativas en el conjunto de evaluaciÃģn

Por quÃĐ importa: el denominador $n^{+} \cdot n^{-}$ es el nÃšmero total de pares posibles positivo-negativo. El numerador cuenta cuÃĄntos de esos pares el modelo rankea correctamente.

Ejemplo numÃĐrico completo:

from __future__ import annotations

import numpy as np


def auc_manual(y_true: np.ndarray, y_scores: np.ndarray) -> float:
    """
    Calcula AUC usando la interpretaciÃģn de Mann-Whitney.

    Args:
        y_true: Etiquetas binarias (0 o 1).
        y_scores: Probabilidades predichas para la clase positiva.

    Returns:
        AUC score entre 0.0 y 1.0.

    Raises:
        ValueError: Si y_true no contiene ambas clases.
    """
    pos_mask = y_true == 1
    neg_mask = y_true == 0

    if not pos_mask.any() or not neg_mask.any():
        raise ValueError("y_true debe contener muestras de ambas clases (0 y 1).")

    pos_scores = y_scores[pos_mask]
    neg_scores = y_scores[neg_mask]

    n_pos = len(pos_scores)
    n_neg = len(neg_scores)

    concordant = sum(
        1
        for p in pos_scores
        for n in neg_scores
        if p > n
    )
    ties = sum(
        0.5
        for p in pos_scores
        for n in neg_scores
        if p == n
    )

    return (concordant + ties) / (n_pos * n_neg)


y_true = np.array([0, 0, 1, 1, 0, 1, 0, 1, 1, 0])
y_scores = np.array([0.1, 0.4, 0.35, 0.8, 0.2, 0.9, 0.05, 0.7, 0.6, 0.45])

auc_manual_result = auc_manual(y_true, y_scores)
print(f"AUC manual (Mann-Whitney): {auc_manual_result:.4f}")

ð ImplementaciÃģn Production-Ready: ROC/AUC con EvaluaciÃģn Completa

from __future__ import annotations

import logging
from dataclasses import dataclass, field

import numpy as np
from sklearn.metrics import (
    roc_auc_score,
    roc_curve,
    average_precision_score,
)

logger = logging.getLogger(__name__)


@dataclass
class ROCEvaluator:
    """
    Evaluador de curvas ROC y AUC para clasificadores binarios.

    Centraliza el cÃĄlculo de mÃĐtricas de ranking (AUC-ROC, AUC-PR)
    y selecciÃģn Ãģptima de umbral usando el criterio de Youden.

    Attributes:
        pos_label: Etiqueta de la clase positiva.
        drop_intermediate: Si True, simplifica la curva ROC eliminando
            puntos redundantes.
    """

    pos_label: int = 1
    drop_intermediate: bool = True
    _fpr: np.ndarray = field(default_factory=lambda: np.array([]))
    _tpr: np.ndarray = field(default_factory=lambda: np.array([]))
    _thresholds: np.ndarray = field(default_factory=lambda: np.array([]))
    _auc: float = field(default=0.0)
    _fitted: bool = field(default=False, repr=False)

    def fit(
        self,
        y_true: np.ndarray,
        y_scores: np.ndarray,
    ) -> ROCEvaluator:
        """
        Calcula la curva ROC y el AUC para las predicciones dadas.

        Args:
            y_true: Etiquetas binarias reales. Shape (n_samples,).
            y_scores: Scores o probabilidades para la clase positiva.
                Shape (n_samples,).

        Returns:
            Self para encadenamiento de mÃĐtodos.

        Raises:
            ValueError: Si los arrays tienen shapes incompatibles o
                y_true no contiene ambas clases.
        """
        if y_true.shape != y_scores.shape:
            raise ValueError(
                f"Shape mismatch: y_true {y_true.shape} != y_scores {y_scores.shape}."
            )

        unique_labels = np.unique(y_true)
        if len(unique_labels) < 2:
            raise ValueError(
                f"y_true debe contener al menos 2 clases. Encontrado: {unique_labels}."
            )

        logger.info(
            "Calculando ROC. n_samples=%d, positivos=%d (%.1f%%)",
            len(y_true),
            y_true.sum(),
            100 * y_true.mean(),
        )

        self._fpr, self._tpr, self._thresholds = roc_curve(
            y_true,
            y_scores,
            pos_label=self.pos_label,
            drop_intermediate=self.drop_intermediate,
        )
        self._auc = roc_auc_score(y_true, y_scores)
        self._fitted = True

        logger.info("AUC-ROC calculado: %.4f", self._auc)
        return self

    def optimal_threshold_youden(self) -> dict[str, float]:
        """
        Determina el umbral Ãģptimo usando el Ã­ndice J de Youden.

        El Ã­ndice J maximiza la suma de Sensibilidad + Especificidad,
        lo que equivale a encontrar el punto mÃĄs alejado de la diagonal
        de azar en la curva ROC.

        Returns:
            Diccionario con 'threshold', 'tpr' y 'fpr' Ãģptimos.

        Raises:
            RuntimeError: Si fit() no fue llamado previamente.
        """
        if not self._fitted:
            raise RuntimeError("LlamÃĄ a fit() antes de calcular el umbral Ãģptimo.")

        j_scores = self._tpr - self._fpr
        optimal_idx = int(np.argmax(j_scores))

        return {
            "threshold": float(self._thresholds[optimal_idx]),
            "tpr": float(self._tpr[optimal_idx]),
            "fpr": float(self._fpr[optimal_idx]),
            "youden_j": float(j_scores[optimal_idx]),
        }

    @property
    def auc(self) -> float:
        """AUC-ROC calculado. Requiere llamar fit() primero."""
        if not self._fitted:
            raise RuntimeError("LlamÃĄ a fit() antes de acceder al AUC.")
        return self._auc

    @property
    def curve(self) -> tuple[np.ndarray, np.ndarray, np.ndarray]:
        """Retorna (fpr, tpr, thresholds) de la curva ROC."""
        if not self._fitted:
            raise RuntimeError("LlamÃĄ a fit() antes de acceder a la curva.")
        return self._fpr, self._tpr, self._thresholds


def evaluate_binary_classifier(
    y_true: np.ndarray,
    y_scores: np.ndarray,
) -> dict[str, float]:
    """
    EvaluaciÃģn completa de un clasificador binario con mÃĐtricas de ranking.

    Calcula AUC-ROC, AUC-PR (Average Precision) y el umbral Ãģptimo
    por el criterio de Youden. AUC-PR es mÃĄs informativa en datasets
    desbalanceados donde la clase positiva es la minoritaria.

    Args:
        y_true: Etiquetas binarias reales.
        y_scores: Probabilidades predichas para la clase positiva.

    Returns:
        Diccionario con mÃĐtricas: 'auc_roc', 'auc_pr',
        'optimal_threshold', 'optimal_tpr', 'optimal_fpr'.

    Raises:
        ValueError: Si los inputs son invÃĄlidos.
    """
    evaluator = ROCEvaluator()
    evaluator.fit(y_true, y_scores)

    optimal = evaluator.optimal_threshold_youden()
    auc_pr = average_precision_score(y_true, y_scores)

    logger.info("AUC-PR (Average Precision): %.4f", auc_pr)

    return {
        "auc_roc": evaluator.auc,
        "auc_pr": auc_pr,
        "optimal_threshold": optimal["threshold"],
        "optimal_tpr": optimal["tpr"],
        "optimal_fpr": optimal["fpr"],
        "youden_j": optimal["youden_j"],
    }

$$AP = \sum_n (R_n - R_{n-1}) \cdot P_n$$

from __future__ import annotations

import numpy as np
from sklearn.metrics import roc_auc_score, average_precision_score


def select_auc_metric(
    y_true: np.ndarray,
    y_scores: np.ndarray,
    imbalance_threshold: float = 0.1,
) -> dict[str, float]:
    """
    Selecciona la mÃĐtrica AUC apropiada segÃšn el balance del dataset.

    Args:
        y_true: Etiquetas binarias reales.
        y_scores: Probabilidades predichas para la clase positiva.
        imbalance_threshold: ProporciÃģn mÃ­nima de positivos para usar AUC-ROC.

    Returns:
        Diccionario con ambas mÃĐtricas y la recomendada.

    Raises:
        ValueError: Si y_true estÃĄ vacÃ­o o no tiene ambas clases.
    """
    if len(y_true) == 0:
        raise ValueError("y_true no puede estar vacÃ­o.")

    pos_ratio = float(y_true.mean())
    auc_roc = roc_auc_score(y_true, y_scores)
    auc_pr = average_precision_score(y_true, y_scores)

    recommended = "AUC-PR" if pos_ratio < imbalance_threshold else "AUC-ROC"

    return {
        "auc_roc": auc_roc,
        "auc_pr": auc_pr,
        "pos_ratio": pos_ratio,
        "recommended_metric": recommended,
    }

Error 2: Usar el Umbral Default de 0.5 en ProducciÃģn

El problema:

Por quÃĐ falla:

El umbral $\tau=0.5$ solo es Ãģptimo cuando el costo de FP y FN es idÃĐntico y las clases estÃĄn balanceadas. En cualquier otro caso, estÃĄs dejando dinero o seguridad sobre la mesa. La curva ROC existe exactamente para explorar todos los umbrales posibles.

$$J_{Youden} = TPR(\tau) - FPR(\tau) = \arg\max_{\tau} \{Sensibilidad + Especificidad - 1\}$$

La soluciÃģn:

from __future__ import annotations

import numpy as np
from sklearn.metrics import roc_curve


def find_optimal_threshold(
    y_true: np.ndarray,
    y_scores: np.ndarray,
    cost_fn: float = 1.0,
    cost_fp: float = 1.0,
) -> float:
    """
    Encuentra el umbral Ãģptimo ponderado por el costo asimÃĐtrico de errores.

    Para costos simÃĐtricos (cost_fn=cost_fp=1.0), equivale al Ã­ndice J de Youden.
    Para fraude, usar cost_fn=10.0, cost_fp=1.0 penaliza mÃĄs los falsos negativos.

    Args:
        y_true: Etiquetas binarias reales.
        y_scores: Probabilidades predichas para la clase positiva.
        cost_fn: Costo relativo de un Falso Negativo.
        cost_fp: Costo relativo de un Falso Positivo.

    Returns:
        Umbral Ãģptimo como float.

    Raises:
        ValueError: Si los costos no son positivos.
    """
    if cost_fn <= 0 or cost_fp <= 0:
        raise ValueError("Los costos deben ser positivos.")

    fpr, tpr, thresholds = roc_curve(y_true, y_scores)
    cost_scores = cost_fn * tpr - cost_fp * fpr
    return float(thresholds[np.argmax(cost_scores)])

El problema:

Por quÃĐ falla:

La soluciÃģn:

from __future__ import annotations

import logging
import numpy as np
from sklearn.metrics import roc_auc_score

logger = logging.getLogger(__name__)


def evaluate_auc_split(
    model,
    X_train: np.ndarray,
    y_train: np.ndarray,
    X_test: np.ndarray,
    y_test: np.ndarray,
    overfit_threshold: float = 0.05,
) -> dict[str, float]:
    """
    EvalÃša AUC en train y test, detectando overfitting automÃĄticamente.

    Args:
        model: Clasificador entrenado con mÃĐtodo predict_proba.
        X_train: Features de entrenamiento.
        y_train: Etiquetas de entrenamiento.
        X_test: Features de evaluaciÃģn.
        y_test: Etiquetas de evaluaciÃģn.
        overfit_threshold: Delta AUC mÃĄximo aceptable antes de alertar.

    Returns:
        Diccionario con auc_train, auc_test y delta_auc.

    Raises:
        ValueError: Si el modelo no tiene predict_proba.
    """
    if not hasattr(model, "predict_proba"):
        raise ValueError("El modelo debe implementar predict_proba.")

    auc_train = roc_auc_score(
        y_train, model.predict_proba(X_train)[:, 1]
    )
    auc_test = roc_auc_score(
        y_test, model.predict_proba(X_test)[:, 1]
    )
    delta = auc_train - auc_test

    if delta > overfit_threshold:
        logger.warning(
            "Posible overfitting detectado. ÎAUC=%.4f (train=%.4f, test=%.4f)",
            delta, auc_train, auc_test,
        )

    return {"auc_train": auc_train, "auc_test": auc_test, "delta_auc": delta}
