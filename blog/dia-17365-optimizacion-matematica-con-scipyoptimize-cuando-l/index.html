Día 17: Optimización Matemática con scipy.optimize - Cuando la Intuición No Alcanza para Encontrar el Mínimo

Parcero, hoy vamos a hablar de algo que separa a los que "prueban valores a ver qué pasa" de los ingenieros que saben optimizar de verdad: optimización matemática. Y no, no es meter un for con 10,000 iteraciones rezando encontrar el mejor valor. Es usar las matemáticas para que el computador haga el trabajo pesado de forma inteligente.

Si alguna vez te has quedado embalado tratando de ajustar hiperparámetros manualmente, o peor, usando grid search cuando tienes 15 dimensiones, este artículo es para vos. Vamos a usar scipy.optimize para resolver problemas de optimización de forma profesional.

¿Qué Diablos es la Optimización Matemática?

Imaginate que tenés que encontrar el punto más bajo de una montaña, pero estás vendado. Podés caminar en cualquier dirección y sentir si vas subiendo o bajando. La optimización matemática es básicamente eso: encontrar el mínimo (o máximo) de una función sin probar todos los puntos posibles.

Analogía de oficina: Es como cuando tenés que asignar tareas a tu equipo minimizando el tiempo total. No vas a probar las 5,040 combinaciones posibles de 7 personas en 7 tareas. Usás un algoritmo que te lleva al óptimo de forma inteligente.

Fundamento Matemático

Queremos encontrar:

$x^* = \arg\min_{x \in \mathbb{R}^n} f(x)$

Donde:

  • $f: \mathbb{R}^n \rightarrow \mathbb{R}$ es nuestra función objetivo
  • $x^*$ es el punto óptimo que minimiza $f$

Para funciones diferenciables, la condición necesaria es:

$\nabla f(x^*) = 0$

Es decir, el gradiente (la derivada multidimensional) debe ser cero en el óptimo. Pero ojo, no todo punto con gradiente cero es un mínimo. También puede ser un máximo o un punto silla.

La condición suficiente para un mínimo local requiere que la matriz Hessiana sea definida positiva:

$\nabla^2 f(x^*) \succ 0$

Tipos de Optimización que Te Van a Salvar la Vida

1. Optimización Sin Restricciones

Es el caso más simple: encontrar el mínimo de $f(x)$ sin límites en X.

Ejemplo real: Ajustar los pesos de una red neuronal para minimizar el error.

2. Optimización con Restricciones (Constrained Optimization)

Ahora sí se pone interesante. Queremos:

$\min_{x} f(x) \quad \text{sujeto a} \quad g_i(x) \leq 0, \ h_j(x) = 0$

Ejemplo real: Optimizar el portafolio de inversión maximizando retorno pero con restricciones de riesgo y presupuesto.

3. Optimización Convexa

Cuando $f(x)$ es convexa, cualquier mínimo local es también global. Esta es la joya de la corona porque garantiza encontrar LA solución óptima.

Una función es convexa si:

$f(\lambda x_1 + (1-\lambda)x_2) \leq \lambda f(x_1) + (1-\lambda)f(x_2) \quad \forall \lambda \in [0,1]$

Métodos de Optimización en scipy.optimize

Método 1: Nelder-Mead (Sin Derivadas)

Es como caminar en la montaña con una brújula pero sin mapa. Funciona bien para funciones no diferenciables o ruidosas.

Complejidad: Aproximadamente $O(n^2)$ evaluaciones por iteración.

Método 2: BFGS (Quasi-Newton)

Usa aproximaciones del Hessiano. Es el caballo de batalla para optimización sin restricciones.

La actualización BFGS es:

$B_{k+1} = B_k + \frac{y_k y_k^T}{y_k^T s_k} - \frac{B_k s_k s_k^T B_k}{s_k^T B_k s_k}$

Donde $s_k = x_{k+1} - x_k$$y_k = \nabla f(x_{k+1}) - \nabla f(x_k)$

$y_k = \nabla f(x_{k+1}) - \nabla f(x_k)$

Método 3: L-BFGS-B (BFGS con Límites)

Es BFGS pero con restricciones de caja (box constraints): $a \leq x \leq b$. Ideal para hiperparámetros que tienen rangos válidos.

Método 4: SLSQP (Sequential Least Squares Programming)

Para optimización con restricciones generales. Usa programación cuadrática secuencial.

class OptimizationMethod(Enum):
    """Métodos de optimización soportados."""
    NELDER_MEAD = "Nelder-Mead"  # Sin derivadas, robusto
    BFGS = "BFGS"  # Quasi-Newton, requiere gradiente

    POWELL = "Powell"  # Sin derivadas, direcciones conjugadas
    CG = "CG"  # Gradiente conjugado
    TRUST_CONSTR = "trust-constr"  # Región de confianza, más robusto

@dataclass(frozen=True)
class OptimizationConstraint:
    """
    Representa una restricción de optimización.
    
    Frozen=True para inmutabilidad (thread-safe, cacheable).
    """
    type: str  # 'ineq' o 'eq'
    fun: Callable[[Vector], float]
    jac: Optional[Callable[[Vector], Vector]] = None
    
    def __post_init__(self) -> None:
        """Validación post-inicialización."""
        if self.type not in {'ineq', 'eq'}:
            raise ValueError(f"Tipo de restricción inválido: {self.type}")


@dataclass
class OptimizationConfig:
    """
    Configuración completa de optimización.
    
    Usa __slots__ para optimización de memoria (importante con
    miles de configuraciones en grid search distribuido).
    """
    __slots__ = (
        'method', 'max_iterations', 'tolerance', 'use_gradient',
        'use_hessian', 'bounds', 'constraints', 'options'
    )
    
    method: OptimizationMethod = OptimizationMethod.L_BFGS_B
    max_iterations: int = 1000
    tolerance: float = 1e-6
    use_gradient: bool = True
    use_hessian: bool = False
    bounds: Optional[Bounds] = None
    constraints: List[OptimizationConstraint] = field(default_factory=list)
    options: Dict[str, Any] = field(default_factory=dict)
    
    def validate(self) -> None:
        """
        Valida la configuración.
        
        Por qué validación explícita: falla rápido en lugar de descubrir
        configuraciones inválidas después de 30 minutos de entrenamiento.
        """
        if self.max_iterations <= 0:
            raise ValueError("max_iterations debe ser positivo")
        
        if self.tolerance <= 0:
            raise ValueError("tolerance debe ser positiva")
        
        # Métodos que requieren gradiente
        gradient_methods = {
            OptimizationMethod.BFGS,
            OptimizationMethod.L_BFGS_B,
            
OptimizationMethod.CG

        }
        
        if self.method in gradient_methods and not self.use_gradient:
            logger.warning(
                f"{self.method.value} funciona mejor con gradiente. "
                "Considera usar_gradient=True o cambiar a Nelder-Mead"
            )


@dataclass
class OptimizationResult:
    """
    Resultado de optimización con metadatos completos.
    
    Por qué no usar directamente OptimizeResult de scipy:
    - Queremos agregar métricas custom (tiempo, # evaluaciones, convergencia)
    - Necesitamos serialización para logging distribuido
    - Validación de resultados con múltiples criterios
    """
    x: Vector  # Punto óptimo encontrado
    fun: float  # Valor de la función en x
    success: bool  # ¿Convergió?
    message: str  # Mensaje de estado
    nit: int  # Número de iteraciones
    nfev: int  # Número de evaluaciones de función
    njev: int  # Número de evaluaciones de Jacobiano
    execution_time: float  # Tiempo de ejecución en segundos
    
    # Metadatos de validación
    gradient_norm: Optional[float] = None
    constraint_violation: Optional[float] = None
    
    def is_valid(
        self,
        max_gradient_norm: float = 1e-3,
        max_constraint_violation: float = 1e-6
    ) -> bool:
        """
        Valida si el resultado es confiable.
        
        Por qué no confiar solo en 'success': scipy puede decir success=True
        pero con gradiente alto (convergencia prematura) o violando restricciones.
        """
        if not self.success:
            return False
        
        if self.gradient_norm is not None:
            if self.gradient_norm > max_gradient_norm:
                logger.warning(
                    f"Gradiente alto: {self.gradient_norm:.2e} > {max_gradient_norm:.2e}"
                )
                return False
        
        if self.constraint_violation is not None:
            if self.constraint_violation > max_constraint_violation:
                logger.warning(
                    f"Violación de restricciones: {self.constraint_violation:.2e}"
                )
                return False
        
        return True
    
    def __repr__(self) -> str:
        """Representación legible para logs."""
        return (
            f"OptimizationResult(fun={
self.fun
:.6f}, "
            f"success={self.success}, nit={self.nit}, "
            f"time={self.execution_time:.2f}s)"
        )


def timing_decorator(func: Callable[..., T]) -> Callable[..., T]:
    """
    Decorator para medir tiempo de ejecución.
    
    Por qué decorator: reutilizable, no contamina la lógica de negocio,
    fácil de activar/desactivar en diferentes ambientes.
    """
    @wraps(func)
    def wrapper(*args: Any, **kwargs: Any) -> T:
        start_time = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start_time
        logger.debug(f"{func.__name__} ejecutado en {elapsed:.4f}s")
        return result
    return wrapper


class RobustOptimizer:
    """
    Optimizador robusto con manejo de errores y validación.
    
    Arquitectura:
    - Separa configuración de ejecución (SRP)
    - Maneja múltiples métodos de optimización
    - Validación exhaustiva de resultados
    - Logging detallado para debugging en producción
    """
    
    __slots__ = ('config', '_evaluation_count', '_gradient_count')
    
    def __init__(self, config: OptimizationConfig) -> None:
        """
        Inicializa el optimizador.
        
        Por qué validar en __init__: falla rápido antes de computación costosa.
        """
        config.validate()
        self.config = config
        self._evaluation_count = 0
        self._gradient_count = 0
    
    def _wrap_objective(
        self,
        objective: ScalarFunction
    ) -> ScalarFunction:
        """
        Envuelve la función objetivo para contar evaluaciones y manejar errores.
        
        Por qué wrapper: debugging en producción. Cuando tu función no converge,
        necesitas saber cuántas veces se evaluó y dónde falló.
        """
        def wrapped(x: Vector) -> float:
            try:
                self._evaluation_count += 1
                value = objective(x)
                
                # Validación de sanidad
                if not np.isfinite(value):
                    logger.error(f"Función objetivo retornó {value} en x={x}")
                    raise ValueError("Función objetivo retornó valor no finito")
                
                if self._evaluation_count % 100 == 0:
                    logger.debug(
                        f"Evaluación #{self._evaluation_count}: f(x)={value:.6f}"
                    )
                
                return value
                
            except Exception as e:
                logger.error(f"Error evaluando función objetivo: {e}")
                raise
        
        return wrapped
    
    def _wrap_gradient(
        self,
        gradient: Optional[VectorFunction]
    ) -> Optional[VectorFunction]:
        """Envuelve el gradiente para contar evaluaciones."""
        if gradient is None:
            return None
        
        def wrapped(x: Vector) -> Vector:
            try:
                self._gradient_count += 1
                grad = gradient(x)
                
                if not np.all(np.isfinite(grad)):
                    logger.error(f"Gradiente no finito en x={x}")
                    raise ValueError("Gradiente contiene valores no finitos")
                
                return grad
                
            except Exception as e:
                logger.error(f"Error evaluando gradiente: {e}")
                raise
        
        return wrapped
    
    def _compute_constraint_violation(
        self,
        x: Vector
    ) -> float:
        """
        Calcula la violación total de restricciones.
        
        Por qué: scipy puede decir que convergió pero violando restricciones
        ligeramente. Necesitas cuantificar esto para validación.
        """
        total_violation = 0.0
        
        for constraint in self.config.constraints:
            value = 
constraint.fun
(x)
            
            if constraint.type == 'eq':
                # Restricción de igualdad: debe ser 0
                total_violation += abs(value)
            else:  # 'ineq'
                # Restricción de desigualdad: debe ser <= 0
                if value > 0:
                    total_violation += value
        
        return total_violation
    
    @timing_decorator
    def optimize(
        self,
        objective: ScalarFunction,
        x0: Vector,
        gradient: Optional[VectorFunction] = None,
        hessian: Optional[Callable[[Vector], Matrix]] = None
    ) -> OptimizationResult:
        """
        Ejecuta la optimización.
        
        Args:
            objective: Función a minimizar
            x0: Punto inicial
            gradient: Gradiente de la función (opcional)
            hessian: Hessiano de la función (opcional)
        
        Returns:
            OptimizationResult con todos los metadatos
        
        Raises:
            ValueError: Si la configuración es inválida
            RuntimeError: Si la optimización falla catastróficamente
        """
        
logger.info
(
            f"Iniciando optimización con {self.config.method.value} "
            f"desde x0={x0}"
        )
        
        # Reiniciar contadores
        self._evaluation_count = 0
        self._gradient_count = 0
        
        # Envolver funciones para monitoreo
        wrapped_objective = self._wrap_objective(objective)
        wrapped_gradient = self._wrap_gradient(gradient) if self.config.use_gradient else None
        
        # Preparar argumentos para scipy.optimize.minimize
        minimize_kwargs: Dict[str, Any] = {
            'method': self.config.method.value,
            'options': {
                'maxiter': self.config.max_iterations,
                **self.config.options
            },
            'tol': self.config.tolerance
        }
        
        # Agregar gradiente si está disponible
        if wrapped_gradient is not None:
            minimize_kwargs['jac'] = wrapped_gradient
        
        # Agregar Hessiano si está disponible y el método lo soporta
        if hessian is not None and self.config.use_hessian:
            if self.config.method in {
OptimizationMethod.TRUST
_CONSTR}:
                minimize_kwargs['hess'] = hessian
        
        # Agregar bounds si están configurados
        if self.config.bounds is not None:
            minimize_kwargs['bounds'] = self.config.bounds
        
        # Agregar restricciones si existen
        if self.config.constraints:
            constraints_list = [
                {'type': c.type, 'fun': 
c.fun
, 'jac': c.jac}
                for c in self.config.constraints
            ]
            minimize_kwargs['constraints'] = constraints_list
        
        try:
            start_time = time.perf_counter()
            
            # EJECUCIÓN DE OPTIMIZACIÓN
            scipy_result: OptimizeResult = minimize(
                wrapped_objective,
                x0,
                **minimize_kwargs
            )
            
            execution_time = time.perf_counter() - start_time
            
            # Calcular norma del gradiente si está disponible
            gradient_norm = None
            if wrapped_gradient is not None:
                try:
                    final_gradient = wrapped_gradient(scipy_result.x)
                    gradient_norm = float(np.linalg.norm(final_gradient))
                except Exception as e:
                    logger.warning(f"No se pudo calcular gradiente final: {e}")
            
            # Calcular violación de restricciones
            constraint_violation = None
            if self.config.constraints:
                constraint_violation = self._compute_constraint_violation(
                    scipy_result.x
                )
            
            # Construir resultado
            result = OptimizationResult(
                x=scipy_result.x,
                fun=float(scipy_
result.fun
),
                success=scipy_result.success,
                message=scipy_result.message,
                nit=scipy_result.nit,
                nfev=scipy_result.nfev,
                njev=getattr(scipy_result, 'njev', self._gradient_count),
                execution_time=execution_time,
                gradient_norm=gradient_norm,
                constraint_violation=constraint_violation
            )
            
            
logger.info
(f"Optimización completada: {result}")
            
            # Validación final
            if not 
result.is
_valid():
                logger.warning(
                    "Resultado marcado como exitoso pero falla validación. "
                    "Considera revisar tolerancias o cambiar método."
                )
            
            return result
            
        except Exception as e:
            logger.error(f"Error catastrófico en optimización: {e}", exc_info=True)
            raise RuntimeError(f"Optimización falló: {e}") from e

class RosenbrockFunction:
    """
    Función de Rosenbrock: el benchmark clásico de optimización.
    
    f(x, y) = (a - x)² + b(y - x²)²
    
    Mínimo global en (a, a²) con valor 0.
    Es difícil porque tiene un valle estrecho y curvo.
    """
    
    def __init__(self, a: float = 1.0, b: float = 100.0) -> None:
        self.a = a
        self.b = b
    
    def __call__(self, x: Vector) -> float:
        """Evalúa la función."""
        if len(x) != 2:
            raise ValueError("Rosenbrock requiere exactamente 2 variables")
        
        return (self.a - x[0])**2 + self.b * (x[1] - x[0]**2)**2
    
    def gradient(self, x: Vector) -> Vector:
        """Gradiente analítico."""
        if len(x) != 2:
            raise ValueError("Rosenbrock requiere exactamente 2 variables")
        
        dx = -2*(self.a - x[0]) - 4*self.b*x[0]*(x[1] - x[0]**2)
        dy = 2*self.b*(x[1] - x[0]**2)
        
        return np.array([dx, dy], dtype=np.float64)
    
    def hessian(self, x: Vector) -> Matrix:
        """Hessiano analítico."""
        if len(x) != 2:
            raise ValueError("Rosenbrock requiere exactamente 2 variables")
        
        h11 = 2 - 4*self.b*(x[1] - x[0]**2) + 8*self.b*x[0]**2
        h12 = h21 = -4*self.b*x[0]
        h22 = 2*self.b
        
        return np.array([[h11, h12], [h21, h22]], dtype=np.float64)


def optimize_portfolio(
    expected_returns: Vector,
    covariance_matrix: Matrix,
    risk_aversion: float = 2.0,
    max_weight: float = 0.3
) -> OptimizationResult:
    """
    Optimización de portafolio (Markowitz).
    
    Minimiza: w^T Σ w - λ w^T μ
    Sujeto a: Σw_i = 1, 0 <= w_i <= max_weight
    
    Donde:
    - w: pesos del portafolio
    - Σ: matriz de covarianza
    - μ: retornos esperados
    - λ: aversión al riesgo
    
    Este es un caso real que te encuentras en fintech.
    """
    n_assets = len(expected_returns)
    
    def objective(w: Vector) -> float:
        """Función objetivo: riesgo - retorno ponderado."""
        portfolio_variance = w @ covariance_matrix @ w
        portfolio_return = w @ expected_returns
        return portfolio_variance - risk_aversion * portfolio_return
    
    def gradient(w: Vector) -> Vector:
        """Gradiente de la función objetivo."""
        return 2 * covariance_matrix @ w - risk_aversion * expected_returns
    
    # Restricción: suma de pesos = 1
    def sum_constraint(w: Vector) -> float:
        return np.sum(w) - 1.0
    
    def sum_constraint_jac(w: Vector) -> Vector:
        return np.ones(n_assets, dtype=np.float64)
    
    # Configuración
    config = OptimizationConfig(
        method=OptimizationMethod.SLSQP,
        max_iterations=500,
        tolerance=1e-8,
        use_gradient=True,
        bounds=Bounds(
            lb=np.zeros(n_assets),  # No short-selling
            ub=np.full(n_assets, max_weight)  # Límite por activo
        ),
        constraints=[
            OptimizationConstraint(
                type='eq',
                fun=sum_constraint,
                jac=sum_constraint_jac
            )
        ]
    )
    
    # Punto inicial: equal weight
    x0 = np.ones(n_assets) / n_assets
    
    optimizer = RobustOptimizer(config)
    return optimizer.optimize(objective, x0, gradient=gradient)

La Matemática Detrás de los Métodos

Gradiente Descendente Básico

$x_{k+1} = x_k - \alpha_k \nabla f(x_k)$

Donde $\alpha_k$ es el learning rate (tamaño de paso).

Problema: Learning rate fijo es problemático. Muy grande → diverge, muy pequeño → lento.

Método de Newton

$x_{k+1} = x_k - [\nabla^2 f(x_k)]^{-1} \nabla f(x_k)$

Usa el Hessiano para dirección óptima. Convergencia cuadrática cerca del óptimo.

Problema: Calcular y invertir Hessiano es $O(n^3)$, prohibitivo para grandes n.

BFGS (Lo Mejor de Ambos Mundos)

Aproxima el Hessiano usando solo gradientes. Complejidad $O(n^2)$ en lugar de $O(n^3)$.

La magia está en la actualización:

$B_{k+1} = B_k + \frac{y_k y_k^T}{y_k^T s_k} - \frac{B_k s_k s_k^T B_k}{s_k^T B_k s_k}$

Esta fórmula garantiza que $B_k$ permanece positiva definida (requisito para minimización).

Consejos de Trinchera

1. Siempre normaliza tus variables

Si x[0] está en [0, 1] pero x[1] en [0, 10000], el optimizador se confunde. Normaliza todo a la misma escala.

2. Usa múltiples puntos iniciales

Optimización no convexa puede tener múltiples mínimos locales. Prueba 5-10 x0 diferentes y quédate con el mejor.

3. Provee gradiente cuando puedas

Diferenciación numérica es lenta e inexacta. Si podés calcular el gradiente analíticamente (o con autograd), hacelo.

4. BFGS para < 200 variables, L-BFGS para más

BFGS almacena matriz completa, L-BFGS solo últimas actualizaciones. Para alta dimensión, L-BFGS o CG.

5. Valida restricciones manualmente

No confíes solo en el optimizador. Después de obtener $x^*$, verifica que cumpla las restricciones.

Conclusión

La optimización matemática no es magia negra. Es simplemente usar las matemáticas para que el computador encuentre el mejor valor de forma inteligente en lugar de probar todo. Las claves:

  1. Entiende qué método usar: convexa → cualquiera, no convexa → múltiples intentos, con restricciones → SLSQP o trust-constr
  2. Provee gradiente: 10x más rápido y preciso
  3. Valida resultados: success=True no es suficiente
  4. Maneja errores: funciones mal condicionadas son comunes en producción

Con el framework que te mostré, no te vas a quedar embalado tratando de debuggear por qué tu optimizador no converge o por qué da resultados raros. Tenés logging, validación, y type safety que te atrapan errores antes de que sea tarde.
