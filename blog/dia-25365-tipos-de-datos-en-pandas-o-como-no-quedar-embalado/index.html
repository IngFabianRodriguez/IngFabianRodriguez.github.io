Día 25: Tipos de Datos en Pandas — O Cómo No Quedar Embalado con un DataFrame de 40GB

🎯 El Problema Real (Sin Humo)

Parcero, hablemos en serio: ¿alguna vez te pasó que corriste un script de pandas que aparentemente era sencillo y el proceso se quedó pegado comiendo RAM como si no hubiera mañana? ¿O peor, que una operación que debía tomar segundos se demoró minutos y no sabías por qué?

Esa vaina pasa porque nadie te dijo que pandas.dtypes es uno de los controles más críticos en Data Science.

Hoy vamos a destapar el capó de los tipos de datos en pandas. No es solo un detalle cosmético del DataFrame; es la diferencia entre un pipeline que escala y uno que te deja esperando mientras tomas el tercer tinto del día.

🔥 El Problema en Concreto

Imagínate esto: tienes un CSV con 10 millones de registros. Una columna tiene categorías como ["Bogotá", "Medellín", "Cali"]. Si pandas infiere que esa columna es object (strings individuales), cada string se guarda como un puntero en memoria. Eso significa que "Bogotá" se repite 3 millones de veces en RAM como si fuera único cada vez.

Ahora, si usas category, pandas guarda un diccionario con {"Bogotá": 0, "Medellín": 1, "Cali": 2} y solo almacena los códigos. La columna pasa de ocupar ~240MB a ~10MB. De una.

📊 Tipos de Datos en Pandas: El Arsenal

Pandas maneja tipos de datos que heredan de NumPy, pero con esteroides. Acá está el mapa mental que debes tener grabado:

1. Integers (int8, int16****, int32****, int64****, uint8****, ...)

Cada variante define el rango y el espacio en memoria:

$$\text{int}N: [-2^{N-1}, 2^{N-1} - 1], \quad \text{uint}N: [0, 2^N - 1]$$

  • int8: [-128, 127] → 1 byte
  • int64: [-9.2e18, 9.2e18] → 8 bytes

Analogía: Es como elegir entre un carro compacto (int8) y una tractomula (int64) para llevar una caja de zapatos. Si sabés que tu columna edad nunca va a superar 120, usar int64 es desperdicio.

2. Floats (float16, float32****, float64****)

Números con decimales. Más bits = más precisión, pero más memoria:

$$\text{Error relativo} \approx 2^{-p}$$

donde $p$ es el número de bits de la mantisa (23 para float32, 52 para float64).

3. Object (object)

El cajón de sastre. Es el tipo más peligroso porque pandas lo usa para todo lo que no puede categorizar: strings, listas, dicts, objetos mixtos. Cada elemento es un puntero a un objeto Python, lo que mata la performance.

4. Category (category)

El MVP para variables categóricas. Almacena:

  • Un array de códigos enteros (int8, int16, etc.)
  • Un diccionario de mapeo

$$\text{Memoria} \approx N \cdot \text{sizeof(code)} + K \cdot \text{sizeof(category)}$$

donde $N$ es el número de filas y $K$ es el número de categorías únicas.

5. Boolean (bool)

1 byte por valor. Simple y efectivo.

6. Datetime (datetime64[ns])

Timestamps con precisión de nanosegundos. Fundamental para series temporales.

🔧 Implementación Production-Ready

Vamos a construir un sistema de gestión de tipos que usarías en producción. Nada de notebooks de juguete.

from typing import Dict, List, Literal, Protocol, TypeVar, Generic
from dataclasses import dataclass
import pandas as pd
import numpy as np
import logging
from contextlib import contextmanager
import time

# Configuración de logging
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)


@dataclass(frozen=True, slots=True)
class MemoryStats:
    """
    Inmutable stats de memoria.
    __slots__ optimiza el consumo de memoria de la clase misma.
    """
    total_mb: float
    per_column_mb: Dict[str, float]
    
    def reduction_percentage(self, other: 'MemoryStats') -> float:
        """Calcula la reducción porcentual de memoria."""
        if other.total_mb == 0:
            return 0.0
        return 100 * (1 - self.total_mb / other.total_mb)


class DataFrameOptimizer:
    """
    Optimizador de tipos de datos para DataFrames.
    
    Estrategia:
    1. Detecta rangos de valores en columnas numéricas
    2. Downcasting inteligente basado en rangos reales
    3. Conversión automática a category para columnas con baja cardinalidad
    
    Justificación matemática:
    Para una columna con K valores únicos y N filas:
    - object: O(N) punteros de 8 bytes cada uno
    - category: O(N) códigos + O(K) strings únicos
    
    Si K << N, entonces category es óptimo.
    """
    
    def __init__(
        self,
        categorical_threshold: float = 0.5,
        force_categorical_threshold: int = 100
    ):
        """
        Args:
            categorical_threshold: Si % valores únicos < threshold, convierte a category
            force_categorical_threshold: Si # valores únicos < N, fuerza category
        """
        self.categorical_threshold = categorical_threshold
        self.force_categorical_threshold = force_categorical_threshold
    
    def get_memory_stats(self, df: pd.DataFrame) -> MemoryStats:
        """Calcula estadísticas de memoria del DataFrame."""
        total_mb = df.memory_usage(deep=True).sum() / 1024**2
        per_column = {
            col: df[col].memory_usage(deep=True) / 1024**2
            for col in df.columns
        }
        return MemoryStats(total_mb=total_mb, per_column_mb=per_column)
    
    def _optimize_integers(self, series: pd.Series) -> pd.Series:
        """
        Downcast inteligente de enteros.
        
        Estrategia: Encuentra el tipo más pequeño que contenga el rango.
        """
        if not pd.api.types.is_integer_dtype(series):
            return series
        
        min_val, max_val = series.min(), series.max()
        
        # Tipos unsigned si solo hay positivos
        if min_val >= 0:
            if max_val < 256:  # 2^8
                return series.astype(np.uint8)
            elif max_val < 65536:  # 2^16
                return series.astype(np.uint16)
            elif max_val < 4294967296:  # 2^32
                return series.astype(np.uint32)
        else:
            # Tipos signed
            if -128 <= min_val and max_val < 127:
                return series.astype(np.int8)
            elif -32768 <= min_val and max_val < 32767:
                return series.astype(np.int16)
            elif -2147483648 <= min_val and max_val < 2147483647:
                return series.astype(np.int32)
        
        return series
    
    def _optimize_floats(self, series: pd.Series) -> pd.Series:
        """
        Downcast de floats.
        
        CUIDADO: float32 tiene ~7 dígitos de precisión.
        Solo usar si la aplicación lo tolera.
        """
        if not pd.api.types.is_float_dtype(series):
            return series
        
        # Chequeamos si el downcast a float32 preserva los valores
        # dentro de un margen de tolerancia
        as_float32 = series.astype(np.float32)
        
        # Tolerancia relativa: 1e-6
        if np.allclose(series, as_float32, rtol=1e-6, equal_nan=True):
            return as_float32
        
        return series
    
    def _should_be_categorical(self, series: pd.Series) -> bool:
        """
        Decide si una columna debe ser categórica.
        
        Heurística:
        1. Si hay < force_categorical_threshold valores únicos → category
        2. Si % valores únicos < categorical_threshold → category
        """
        n_unique = series.nunique()
        n_total = len(series)
        
        if n_unique < self.force_categorical_threshold:
            return True
        
        if n_total > 0 and (n_unique / n_total) < self.categorical_threshold:
            return True
        
        return False
    
    def optimize(self, df: pd.DataFrame) -> pd.DataFrame:
        """
        Optimiza todos los tipos de datos del DataFrame.
        
        Returns:
            DataFrame optimizado (copia)
        """
        optimized = df.copy()
        
        for col in optimized.columns:
            series = optimized[col]
            original_dtype = series.dtype
            
            try:
                # Object columns → intentar convertir
                if series.dtype == 'object':
                    # ¿Es numérico oculto?
                    try:
                        numeric = pd.to_numeric(series, errors='coerce')
                        if numeric.notna().sum() > len(series) * 0.9:
                            # Si >90% son numéricos válidos, lo tomamos
                            optimized[col] = numeric
                            series = optimized[col]
                    except Exception:
                        pass
                    
                    # ¿Debe ser categórico?
                    if self._should_be_categorical(series):
                        optimized[col] = series.astype('category')
                        logger.info(
                            f"✓ {col}: object → category "
                            f"({series.nunique()} categorías)"
                        )
                        continue
                
                # Downcast numéricos
                if pd.api.types.is_integer_dtype(series):
                    optimized[col] = self._optimize_integers(series)
                elif pd.api.types.is_float_dtype(series):
                    optimized[col] = self._optimize_floats(series)
                
                # Log solo si hubo cambio
                if optimized[col].dtype != original_dtype:
                    logger.info(f"✓ {col}: {original_dtype} → {optimized[col].dtype}")
                    
            except Exception as e:
                logger.warning(f"✗ No se pudo optimizar {col}: {e}")
                continue
        
        return optimized


@contextmanager
def measure_memory():
    """Context manager para medir mejora en memoria."""
    import tracemalloc
    tracemalloc.start()
    
    start_time = time.perf_counter()
    yield
    
    current, peak = tracemalloc.get_traced_memory()
    elapsed = time.perf_counter() - start_time
    
    tracemalloc.stop()
    
    logger.info(f"⏱️  Tiempo: {elapsed:.2f}s | Memoria pico: {peak / 1024**2:.2f} MB")

# EJEMPLO DE USO EN PRODUCCIÓN

def demo_optimization():
    """
    Demo realista: dataset con múltiples tipos problemáticos.
    """
    # Generamos un dataset "sucio" como los que llegan en producción
    np.random.seed(42)
    n_rows = 1_000_000
    
    data = {
        # IDs que podrían ser int8 pero pandas los pone int64
        'user_id': np.random.randint(0, 200, n_rows),
        
        # Edad (nunca > 120)
        'age': np.random.randint(18, 90, n_rows),
        
        # Ciudad (cardinalidad baja, pero object)
        'city': np.random.choice(
            ['Bogotá', 'Medellín', 'Cali', 'Barranquilla'],
            n_rows
        ),
        
        # Números con decimales que cabrían en float32
        'price': np.random.uniform(10.0, 1000.0, n_rows),
        
        # Boolean disfrazado de object
        'is_active': np.random.choice(['True', 'False'], n_rows),
        
        # Alta cardinalidad (no debería ser category)
        'transaction_id': [f"TX{i:010d}" for i in range(n_rows)]
    }
    
    df = pd.DataFrame(data)
    
    # ANTES de optimizar
    logger.info("=" * 60)
    logger.info("📊 ANTES DE OPTIMIZACIÓN")
    logger.info("=" * 60)
    logger.info(f"\n{df.dtypes}\n")
    
    optimizer = DataFrameOptimizer(
        categorical_threshold=0.5,
        force_categorical_threshold=50
    )
    
    stats_before = optimizer.get_memory_stats(df)
    logger.info(f"Memoria total: {stats_before.total_mb:.2f} MB")
    
    # OPTIMIZACIÓN
    with measure_memory():
        df_optimized = optimizer.optimize(df)
    
    # DESPUÉS de optimizar
    logger.info("\n" + "=" * 60)
    logger.info("📊 DESPUÉS DE OPTIMIZACIÓN")
    logger.info("=" * 60)
    logger.info(f"\n{df_optimized.dtypes}\n")
    
    stats_after = optimizer.get_memory_stats(df_optimized)
    logger.info(f"Memoria total: {stats_after.total_mb:.2f} MB")
    
    reduction = stats_after.reduction_percentage(stats_before)
    logger.info(f"\n🎉 REDUCCIÓN: {reduction:.1f}%")
    
    return df, df_optimized


if __name__ == "__main__":
    df_original, df_optimized = demo_optimization()

🧠 Justificación Matemática: ¿Por Qué Importa?

La complejidad espacial de un DataFrame es:

$$M_{\text{total}} = \sum_{i=1}^{C} N \cdot \text{sizeof}(\text{dtype}_i)$$

donde $C$ es el número de columnas y $N$ el número de filas.

Para una columna categórica con $K$ valores únicos:

$$M_{\text{category}} = N \cdot \text{sizeof(code)} + K \cdot \overline{L}_{\text{string}}$$

Para object (strings):

$$M_{\text{object}} = N \cdot (\text{sizeof(ptr)} + \overline{L}_{\text{string}})$$

Cuando $K \ll N$ (baja cardinalidad), $M_{\text{category}} \ll M_{\text{object}}$**.**

Ejemplo:

  • $N = 10^6$ filas
  • $K = 4$ categorías ("Bogotá", "Medellín", "Cali", "Barranquilla")
  • Promedio 8 caracteres por string

object: $10^6 \cdot (8 + 8) \approx 16$ MB

category: $10^6 \cdot 1 + 4 \cdot 8 \approx 1$ MB

Reducción del 93%. De una.

🚨 Dolores Comunes y Soluciones

1. "Mi DataFrame ocupa demasiada RAM"

Causa: Columnas object cuando deberían ser category o numéricas mal dimensionadas.

Solución: Usa el optimizador de arriba. En serio, córrelo en tus pipelines.

2. "Las operaciones de groupby son lentas"

Causa: object columns en el by parameter. Pandas tiene que hashear strings completos.

Solución: Convierte a category antes del groupby:

df['city'] = df['city'].astype('category')
df.groupby('city')['sales'].sum()  # 5-10x más rápido

3. "Perdí precisión en mis cálculos"

Causa: Downcast agresivo a float32 sin validar tolerancia.

Solución: Siempre valida con np.allclose() antes de hacer downcast (como en el código).

4. "Mis CSVs se corrompen al leer/escribir"

Causa: Pandas infiere tipos mal al leer. Ej: una columna de IDs numéricos con ceros a la izquierda.

Solución: Especifica dtype explícitamente al leer:

df = pd.read_csv(
    'data.csv',
    dtype={
        'user_id': 'uint32',
        'postal_code': str,  # Evita que "00123" se vuelva 123
        'city': 'category'
    }
)

🎯 Conclusión: Alivio del Dolor

Parcero, dominar pandas.dtypes no es opcional si querés escalar. Acá está el checklist que debes aplicar siempre:

  1. ✅ Después de leer un CSV: Revisa con df.dtypes y df.memory_usage(deep=True).
  2. ✅ Antes de procesamiento pesado: Optimiza tipos. 50% de reducción de memoria es común.
  3. ✅ Variables categóricas: Si $K < 0.5 \cdot N$, usa category. Punto.
  4. ✅ Integers: No uses int64 para edades, flags, o IDs pequeños. Usa int8, int16 según el rango.
  5. ✅ Floats: Solo baja a float32 si validaste precisión. Los errores numéricos te van a hacer llorar en producción.

El código de arriba te evita que un DataFrame de 5GB se vuelva 20GB y te deje embalado esperando a que termine un .groupby().
